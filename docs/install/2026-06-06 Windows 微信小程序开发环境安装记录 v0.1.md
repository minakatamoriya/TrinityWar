# 2026-06-06 Windows 微信小程序开发环境安装记录 v0.1

## 一、适用目标

本文记录 TrinityWar 在 Windows 机器上用于微信小游戏/小程序开发的本地基础环境安装方式。

推荐组合：

- Windows 作为主开发环境。
- 微信开发者工具运行在 Windows。
- Node.js、npm、PostgreSQL、项目 dev server 运行在 Windows。
- Redis 使用 Docker Desktop 容器运行，并通过 `localhost:6379` 提供给后端 worker。
- Docker Desktop 使用 WSL2 backend。

这样可以避免 Windows 原生 Redis 维护问题，也避免把项目主路径搬到 WSL 后影响微信开发者工具读取。

## 二、已在当前机器执行的安装

当前机器已通过 `winget` 安装：

```powershell
winget install --id Microsoft.WSL --exact --accept-package-agreements --accept-source-agreements --silent
winget install --id Docker.DockerDesktop --exact --accept-package-agreements --accept-source-agreements --silent
```

安装结果：

- `Microsoft.WSL` 已安装，版本为 `2.7.3.0`。
- `Docker Desktop` 已安装，版本为 `4.76.0`。
- Docker Desktop 文件已落到 `C:\Program Files\Docker\Docker`。
- 当前 PowerShell 会话的 PATH 可能尚未刷新，可直接使用：

```powershell
& "C:\Program Files\Docker\Docker\resources\bin\docker.exe" version
```

## 三、仍需完成的 Windows 可选功能

如果运行 `wsl --status` 时提示：

```text
wsl.exe --install --no-distribution
Wsl/WSL_E_WSL_OPTIONAL_COMPONENT_REQUIRED
```

说明 Windows 可选功能还没有启用。需要管理员 PowerShell 执行：

```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
wsl --set-default-version 2
```

执行后通常需要重启 Windows。

## 四、Docker Desktop 首次启动

重启后打开 Docker Desktop。

首次启动需要确认：

- 使用 WSL 2 based engine。
- Docker Desktop 状态为 Running。
- 可以在 PowerShell 运行：

```powershell
docker version
docker info
```

如果当前 PowerShell 找不到 `docker`，先重开一个 PowerShell，或临时使用完整路径：

```powershell
& "C:\Program Files\Docker\Docker\resources\bin\docker.exe" version
```

## 五、安装 Redis 容器

Docker Desktop 正常运行后执行：

```powershell
docker run -d --name trinitywar-redis -p 6379:6379 redis:7-alpine
```

验证：

```powershell
docker exec trinitywar-redis redis-cli ping
```

预期输出：

```text
PONG
```

日常启动和停止：

```powershell
docker start trinitywar-redis
docker stop trinitywar-redis
```

## 六、项目侧使用方式

在运行 raid worker 或后端服务前设置：

```powershell
$env:REDIS_URL="redis://localhost:6379"
```

启动 worker：

```powershell
npm run worker:raid-settlement --workspace @trinitywar/game-server
```

启动补扫：

```powershell
npm run worker:raid-settlement:sweep --workspace @trinitywar/game-server
```

## 七、其他机器重复安装清单

在新 Windows 机器上按顺序执行：

1. 安装 Node.js、PostgreSQL、微信开发者工具。
2. 拉取项目并执行 `npm install`。
3. 安装 WSL：

```powershell
winget install --id Microsoft.WSL --exact --accept-package-agreements --accept-source-agreements --silent
```

4. 安装 Docker Desktop：

```powershell
winget install --id Docker.DockerDesktop --exact --accept-package-agreements --accept-source-agreements --silent
```

5. 管理员 PowerShell 启用 WSL2 必要功能：

```powershell
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
wsl --set-default-version 2
```

6. 重启 Windows。
7. 打开 Docker Desktop，等待 Running。
8. 创建 Redis 容器：

```powershell
docker run -d --name trinitywar-redis -p 6379:6379 redis:7-alpine
docker exec trinitywar-redis redis-cli ping
```

9. 项目目录设置 Redis 地址并启动 worker：

```powershell
$env:REDIS_URL="redis://localhost:6379"
npm run worker:raid-settlement --workspace @trinitywar/game-server
```
