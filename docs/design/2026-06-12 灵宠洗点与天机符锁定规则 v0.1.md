# 2026-06-12 灵宠洗点与天机符锁定规则 v0.1

## 1. 文档目的

本文整理当前灵宠洗点体系的最新规则口径，明确商业材料在不同洗法中的定位。重点描述“天机符锁定”、“灵髓 7 选 1”和“灵玉高级 3 选 1（排除式）”的设计逻辑。

本版结论：

> 天机符负责普通金币洗的词条保留；灵髓负责常规定向修补；灵玉负责高级定向修补，并支持排除当前候选组后继续抽取。

本文覆盖后续词条池扩展到 20 条及以上时的体系设计，后端实现、配置表和运营文案以本文定义为准。

## 2. 设计前提

1. 灵宠词条池后续会扩展到至少 20 条。
2. `灵髓` 和 `灵玉` 分别由稀有植物与传说植物产出。
3. 体系需要保持移动端友好，避免复杂交互。
4. 天机符不新增独立洗法，而是作为普通金币洗的锁定消耗。

## 3. 当前洗点体系

灵宠最多 5 个词条。当前体系由 3 种主动洗法和 1 个锁定保护功能构成：

| 类型 | 核心定义 | 主要消耗 | 定位 |
|---|---|---|---|
| 普通金币洗 | 已解锁词条全部随机刷新 | 金币 | 低成本起胚、快速刷词条 |
| 普通金币洗 + 锁定 | 锁定满意词条，未锁定词条随机刷新 | 金币 + 天机符 | 保住已有成果，继续优化整体 |
| 灵髓 7 选 1 | 指定一个词条位，排除自身后从剩余常规词条池中随机选一个结果 | 灵髓 | 常规定向修补，适合中期换方向 |
| 灵玉 高级 3 选 1 | 指定一个词条位，从高级候选组中选一个结果；可继续抽下一组，排除当前组候选 | 灵玉 | 高级定向修补，适合后期收尾 |

三个体系差异明显，各有特色：

- 普通金币洗：主要是靠运气碰出好结果，天机符锁定则为最后两条词条的精修准备。
- 灵髓：面向大范围单词条修补，适合中期换方向或扩大词条组合空间。
- 灵玉：面向高级定向排除洗，适合关键槽位的高阶收尾和更强控制。

设计拆分：

| 维度 | 作用 |
|---|---|
| 洗法 | 决定“洗哪里”和“给几个结果” |
| 锁定 | 决定普通金币洗中“哪些词条不许变化” |
| 材料 | 决定候选来源的品质、保底与可控度 |

## 4. 材料与洗法定位

### 4.1 天机符

- 用途：普通金币洗时锁定已满意词条。
- 产出：商业点券道具；面向中后期保护需求。
- 核心价值：购买“已有成果不被破坏”的安全感。

### 4.2 灵髓

- 用途：指定词条位的常规定向修补。
- 产出：稀有植物。
- 设计原则：7 选 1 作为体验表达，但实际规则是排除自身词条后从剩余常规词条池中随机选一个结果。
- 定义：指定一个词条位，系统从剩余的常规可替换词条池中随机生成候选结果，玩家从中选一个。
- 说明：当词条池扩展到 20 条时，灵髓实质上是“在剩余 19 条词条中选一个”，7 选 1 是交互表达而非实际词条总量。

### 4.3 灵玉

- 用途：指定词条位的高级定向修补。
- 产出：传说植物。
- 设计原则：比灵髓更高级的候选池、更强的控制能力、排除当前候选组后继续抽取。
- 定义：指定一个词条位，系统先给出 3 个高级候选；若不满意，玩家可继续消耗灵玉抽取下一组，系统会排除当前组中已展示的全部候选。
- 说明：灵玉不是简单“少一个选项”，而是“高级候选 + 排除当前组”。更适合关键收尾场景。

## 5. 天机符锁定规则

### 5.1 基础规则

1. 只有普通金币洗可以使用锁定。
2. 锁定词条不会被本次金币洗改变。
3. 未锁定词条会按普通金币洗规则随机刷新。
4. 每锁定 1 条词条，本次洗点额外消耗 1 枚天机符。
5. 单次最多锁定 3 条词条。
6. 不能锁定全部词条。
7. 未解锁词条位不能参与锁定，也不会参与刷新。
8. 锁定功能需玩家达到 50 级后开放。

低等级金币洗说明：
- 10 级首次开放普通金币洗，但此时天机符锁定功能不可用。
- 20 级继续开放普通金币洗，仍旧没有锁定功能。
- 30 级 / 40 级继续提升普通金币洗价格。
- 50 级后可解锁天机符锁定功能。
- 10 级金币洗消耗 200 金币，为后续标准价的五分之一。
- 20 级金币洗消耗 400 金币。
- 30 级金币洗消耗 600 金币。
- 40 级金币洗消耗 800 金币。
- 50 级金币洗消耗 1000 金币。

成本表：

| 锁定数量 | 额外消耗 | 实际刷新范围 |
|---:|---:|--- |
| 0 条 | 0 天机符 | 全部已解锁词条 |
| 1 条 | 1 天机符 | 除锁定词条外的已解锁词条 |
| 2 条 | 2 天机符 | 除锁定词条外的已解锁词条 |
| 3 条 | 3 天机符 | 除锁定词条外的已解锁词条 |

### 5.2 最大锁定数

本版定稿：常规金币洗最多锁定 3 条。

不开放锁 4 条的原因：

1. 锁 4 条后只刷新 1 条，会与指定洗的定义过近。
2. 最后一条词条的精修应交给指定洗，避免金币洗锁定功能压缩指定洗价值。
3. 锁 3 洗 2 保持中后期随机空间。

### 5.3 洗法定位

| 阶段 | 推荐方式 | 玩家目标 |
|---|---|---|
| 词条整体较差 | 普通金币洗 | 低成本刷出胚子 |
| 已有 1-2 条满意词条 | 金币洗 + 锁定 | 保留好词条，继续优化整体 |
| 已有 3 条满意词条 | 锁 3 洗 2 | 冲更好组合 |
| 只差最后 1 条 | 灵髓 / 灵玉 指定洗 | 单点收尾 |

## 6. 灵髓与灵玉高级洗法

### 6.1 灵髓 7 选 1

#### 设计目标

- 提供常规定向修补能力。
- 适合词条池扩展后仍保持简单、稳定的定向体验。
- 让玩家在中期或换方向时感知到定向控制。

#### 规则

1. 玩家指定一个词条位。
2. 系统排除当前词条本身后，从剩余的常规可替换词条池中随机生成一组候选结果，体验上表现为“7 个候选中的 1 个”。
3. 玩家从候选中选取一个结果，替换当前词条。
4. 候选池随词条库扩展，范围会变大；但体验仍保持“选择一个结果”的形式。

#### 设计说明

- 未来词条池达到 20 条时，灵髓实际上是“在剩余 19 条常规词条中选一个”，不再是固定 7 个。7 选 1 是交互表达而非实际词条总量。
- 灵髓候选应覆盖常规词条、稀有词条，以及与当前词条位相匹配的可替换结果。
- 该方式适合中期“我想换一个方向/修一个词条”的场景。

### 6.2 灵玉 高级 3 选 1（排除式）

#### 设计目标

- 让灵玉成为真正高级的指定洗材料。
- 提供更高阶、更可控的结果池。
- 将排除逻辑融入移动端友好的“3 选 1”体验。

#### 规则

1. 玩家指定一个词条位。
2. 系统从高级候选池中生成第一组 3 个候选结果。
3. 玩家可以直接选择一个结果进行替换。
4. 若不满意，玩家可继续消耗 1 枚灵玉抽取下一组 3 个候选。
5. 下一组候选不会包含当前组中已展示的全部 3 个结果。
6. 递进抽取时，系统保持候选池的高级性，并逐步排除已看过的组合。

#### 设计说明

- 该方案本质上是“定向选项 + 排除已展示组”。
- 与传统 3 选 1 的差异在于：玩家可以放弃一整组结果，继续向后抽；这体现了灵玉的高级控制力。
- 由于后续词条池至少 20 条，排除当前组后仍有足够空间，避免结果池过快收窄。
- 该体验在移动端仍旧友好：界面展示当前 3 个候选，按钮提示“选中 / 换一组”。

#### 关键保障

- 候选来源须来自高级池或传说池，确保灵玉结果高于灵髓。
- 新组抽取时，至少排除上一组 3 个结果，防止玩家重复看到同一组合。
- 若连续抽取多组，应保证总体候选池足够大，避免“循环翻牌”感。

### 6.3 体系比较

| 材料 | 交互形式 | 候选池 | 代表场景 |
|---|---|---|---|
| 灵髓 | 7 选 1 | 常规词条池 / 可替换池 | 中期定向修补、换方向 |
| 灵玉 | 高级 3 选 1 + 换组 | 高级/传说候选池 | 后期收尾、关键槽位进阶 |
| 天机符 | 锁定 | 不改变词条，只保护 | 保住已有优良词条 |

## 7. 与指定洗的边界

### 7.1 不冲突的定义

锁定与指定洗不冲突，因为两者解决的问题不同。

| 功能 | 本质 | 玩家心理 |
|---|---|---|
| 金币洗 + 锁定 | 保护已有成果，再继续随机 | 我已经有几条好的，别洗没 |
| 灵髓 / 灵玉 指定洗 | 只处理一个词条位，并给出可选结果 | 我就要修这一条 |

### 7.2 互斥规则

1. 使用普通金币洗时，可以勾选锁定词条。
2. 使用灵髓或灵玉时，不出现天机符锁定入口。
3. 指定洗天然只改变目标词条位，其余词条保持不变，不视为锁定。

玩家侧解释：

> 指定洗本来就只洗一条，因此不需要再锁其他词条。

## 8. 洗点体验定位

### 8.1 四段式目标

| 阶段 | 系统职责 | 主要消耗 |
|---|---|---|
| 起胚 | 快速刷新整套词条，成本低，随机性高 | 金币 |
| 保胚 | 锁住已满意词条，在不毁成果的前提下继续赌 | 金币 + 天机符 |
| 常规定向 | 针对单条词条做方向修补 | 灵髓 |
| 高阶定向 | 对关键槽位做高品质收尾 | 灵玉 |

一句话表达：

> 金币洗起胚，天机符锁胚，灵髓定向、灵玉高级收尾。

### 8.2 灵玉的价值

灵玉价值的核心不是“少一个选项”，而是“提供高级候选、排除已见组、保留关键控制”。

玩家付费感来源：

1. 当前 3 个结果不满意，可以继续抽下一组。
2. 下一组不会再出现当前组中全部候选。
3. 候选本身来自更高阶的高级池。
4. 这是一种用于最后一条或关键词条的高级收尾手段。

## 9. 前端交互建议

### 9.1 普通金币洗界面

- 5 个词条位展示，已解锁词条旁显示锁定入口。
- 直接点击锁定图标切换状态。
- 最多可锁 3 条，超出时其余锁定入口置灰。
- 未解锁词条位不展示锁定入口。
- 按钮实时显示总消耗。

### 9.2 灵髓界面

- 指定一个词条位。
- 显示若干候选结果，体验为“7选1”。
- 玩家选择一个结果后替换当前词条。
- 提示文案：

```text
灵髓可定向修补单个词条位，从常规候选池中选择一个结果。
```

### 9.3 灵玉界面

- 指定一个词条位。
- 显示当前 3 个高级候选。
- 提供“选中”与“换一组”按钮。
- “换一组”按钮消耗额外灵玉，下一组不会包含当前 3 个结果。
- 提示文案：

```text
灵玉高级指定洗，当前组不满意可换一组。新组不会出现已展示的这一组结果。
```

## 10. 后端与数据建议

### 10.1 请求字段

普通金币洗请求建议支持：

| 字段 | 说明 |
|---|---|
| `mode` | 洗点模式，普通金币洗为 `basic` 或现有等价枚举 |
| `slotIndex` | 灵宠槽位 |
| `lockedTraitSlotIndexes` | 本次锁定的词条位列表，最多 3 个 |
| `slotVersion` | 并发控制版本 |
| `requestIdempotencyKey` | 幂等键 |

灵髓请求建议支持：

| 字段 | 说明 |
|---|---|
| `mode` | 洗点模式，灵髓指定洗 |
| `slotIndex` | 灵宠槽位 |
| `material` | `lingsui` |
| `candidateCount` | 候选数量，体验上为 7 |

灵玉请求建议支持：

| 字段 | 说明 |
|---|---|
| `mode` | 洗点模式，灵玉高级指定洗 |
| `slotIndex` | 灵宠槽位 |
| `material` | `lingyu` |
| `candidateCount` | 候选数量，固定为 3 |
| `excludeCandidateIds` | 本次抽取前已展示的候选结果 ID 列表 |

### 10.2 校验规则

普通金币洗需要校验：

1. `lockedTraitSlotIndexes` 只能在普通金币洗中使用。
2. 锁定数量不能超过 3。
3. 锁定词条位必须已解锁。
4. 锁定词条位不能重复。
5. 锁定数量不能等于或超过当前已解锁词条数量。
6. 玩家天机符数量必须大于等于锁定数量。
7. 成功洗点后扣除金币和对应天机符。

灵髓指定洗需要校验：

1. 只能使用 `material=lingsui`。
2. `candidateCount` 与体验设定一致，候选来源为常规池。
3. 只改变目标词条位。

灵玉高级指定洗需要校验：

1. 只能使用 `material=lingyu`。
2. 当前组候选数量固定为 3。
3. 若传入 `excludeCandidateIds`，新组不得同时包含全部已排除候选。
4. 候选来源为高级池 / 传说池。
5. 只改变目标词条位。

## 11. 暂不进入本版的内容

本版先不加入以下内容，以保持规则核心清晰：

1. 天机符单独开一个第四洗法。
2. 指定洗支持天机符锁定。
3. 锁定 4 条只洗最后 1 条。
4. 使用天机符后新旧结果二选一。
5. 自动评分、自动保留最优词条、只升不降等隐藏规则。

这些方向以后仍可作为活动、进阶道具或高阶系统评估，但不进入当前定稿。

## 9. 待后续确认

后续设计和数值需要继续确认：

1. 普通金币洗是否默认提供新旧二选一。
2. 指定词条 7 选 1 与 3 选 1 的具体消耗差异。
3. 天机符主要产出来源、每日产出上限和商业点券兑换比例。
4. 锁 1-3 条时金币消耗是否保持不变，还是随锁定数量小幅递增。
5. 洗点是否计入每日阵营任务、活跃任务或赛季贡献。

当前建议先按最简规则实现：

> 普通金币洗可锁定最多 3 条，每锁 1 条每次额外消耗 1 枚天机符；指定洗不使用锁定。
