# Trinity War 匹配机制与目标池规则草案 v0.1

## 一、文档目标

这份文档只解决四个问题：

1. 日常匿名掠夺应如何匹配，才能保证可打、可懂、不过度霸凌。
2. 玩家在掠夺前可以看到哪些目标信息，哪些信息必须隐藏。
3. 当玩家规模变大后，目标列表如何展示，才能避免全服翻名单。
4. 好友、复仇、通缉等特殊关系，如何与常规匹配池并存。

这份草案的核心原则只有一句话：

系统要持续给玩家值得打、看得懂、差距不离谱的目标，而不是把全服用户名单摊开让玩家自己筛肥羊。

## 二、匹配机制设计底线

首发版本建议守住下面七条：

1. 常规掠夺优先走匿名匹配，不做全服公开点名掠夺。
2. 匹配优先看综合实力段，不只看单一等级。
3. 玩家能看到风险判断信息，但不能看到完整底牌。
4. 新手、连续受击玩家和极弱势玩家必须被隔离保护。
5. 同一目标不能被同一玩家高频反复命中。
6. 目标列表应由系统分发候选卡，而不是展示完整玩家名册。
7. 特殊关系目标独立成池，避免污染常规公平匹配。

## 三、目标池分层设计

### 1. 常规匿名目标池

这是首发阶段最重要的目标来源。

作用：

1. 让玩家上线就能打。
2. 保持匿名、公平和节奏稳定。
3. 避免新手和中层玩家长期被高段位定向追杀。

建议规则：

1. 系统按综合匹配分数筛选候选目标。
2. 每次向玩家展示 3 到 5 个匿名目标卡。
3. 玩家可以刷新列表，但刷新应有成本与上限。
4. 同一批目标不展示全部真实昵称。
5. 常规池中的目标默认不暴露完整履历和详细家底。

### 2. 复仇目标池

这是情绪回流池，不是常规刷资源池。

触发来源：

1. 最近 24 小时内成功掠夺过你的玩家。
2. 当前仍处于有效复仇窗口的目标。

建议规则：

1. 复仇目标从战报入口进入，不占常规匿名目标位。
2. 复仇可额外给少量荣誉、返还或积分加成。
3. 复仇失败不追加额外惩罚，避免情绪二次打击。
4. 复仇目标也应受连续受击保护和冷却限制约束。

### 3. 通缉与高价值目标池

这是高风险高收益池，用来制造上头感和榜单竞争。

触发来源：

1. 携带高额战利品。
2. 短时间内连续成功掠夺。
3. 进入赛季活动或阵营通缉状态。

建议规则：

1. 通缉目标单独展示，不进入普通匿名列表的主位。
2. 通缉目标奖励偏重荣誉、排行和故事性，不只看纯资源。
3. 这类目标允许略微放宽匹配范围，但不能无限放宽。

### 4. 新手保护池

这是保护生态的必要隔离层。

建议条件：

1. 开服前几天或赛季初前几天。
2. 主城阶段低于保护阈值。
3. 综合匹配分数低于保护线。
4. 当日受击次数达到保护阈值。

建议规则：

1. 新手保护池不进入高段位匿名池。
2. 被保护目标仅匹配相近新手或教学机器人目标。
3. 保护池收益偏低，避免成为刷分最优解。

## 四、综合匹配分数建议

首发不建议只按主城等级匹配，也不建议把所有明细直接暴露给玩家。

推荐用隐藏的综合匹配分数作为主筛选依据。

建议由下面几个维度组成：

1. 主城阶段。
2. 近期有效战力或可用兵力。
3. 外场位数量与外场阶段上限。
4. 余额容量与近期暴露能力。
5. 近几天发展值或成长速度。
6. 赛季进程中的位置，例如开服天数或赛季周次。

这套分数不需要完全公开给玩家，但系统要据此做三层筛选：

1. 第一层，优先投放同段位目标。
2. 第二层，若目标不足，则逐步扩圈。
3. 第三层，对高差距目标即便投放，也应收益递减或仅作为特殊目标出现。

## 五、玩家在掠夺前可见的信息

### 1. 建议可见信息

掠夺前建议让玩家看到下面这些信息：

1. 匿名代号或简化昵称。
2. 阵营。
3. 主城阶段或发展段位。
4. 外场成熟度预估。
5. 余额暴露等级。
6. 推荐风险标签。
7. 防守倾向标签。
8. 预计收益区间与预计胜率区间。

这里的重点是“区间”和“标签”，不是精确数值。

### 2. 建议默认隐藏的信息

首发不建议在常规匿名掠夺前直接展示：

1. 精确兵力数。
2. 精确战力。
3. 精确金币余额。
4. 每块外场的具体收益值。
5. 完整建筑等级列表。
6. 对方完整在线行为历史。

原因如下：

1. 如果信息太精确，玩家只是在做数学题，不是在做风险判断。
2. 如果底牌全开，防守配置的价值会明显下降。
3. 如果高价值弱目标过于清晰，生态会迅速退化成筛肥羊。

### 3. 侦察机制建议

如果要让玩家主动花成本换信息，建议做轻量侦察，而不是全量开图。

建议结构：

1. 默认状态只给模糊标签。
2. 花费一次侦察资源后，获得更精细的胜率区间和收益区间。
3. 侦察不应直接给出 100% 精确底表。
4. 侦察结果有时效，避免长期缓存旧数据。

## 六、目标展示方案

### 1. 不建议展示全服用户名单

当玩家规模变大后，全服玩家列表会同时带来四个问题：

1. 查询和刷新压力高。
2. 玩家会反复筛选最肥最弱目标。
3. 被打目标容易被长期定向骚扰。
4. 用户界面会被海量名字和筛选项拖复杂。

因此首发应明确不做全服公开名单式展示。

### 2. 建议使用候选目标卡流

推荐展示结构：

1. 常规匿名目标 3 到 5 张卡。
2. 额外 1 张高风险高收益卡。
3. 额外 1 个复仇或通缉入口。
4. 提供有限次刷新或付出成本换一批。

候选卡字段建议包括：

1. 目标代号。
2. 阵营。
3. 主城阶段。
4. 风险标签。
5. 收益标签。
6. 外场成熟度提示。
7. 防守倾向标签。

### 3. 目标列表的刷新原则

刷新机制不能让玩家无限刷到最优目标。

建议规则：

1. 每日给固定免费刷新次数。
2. 超出免费次数后，刷新成本逐步上升。
3. 同时应设置每日付费刷新上限。
4. 刷新只能换池，不保证更高收益。
5. 高价值目标出现概率不应被无限堆到接近必出。

## 七、好友与社交关系建议

### 1. 首发好友机制建议做轻

首发好友机制建议只承接社交，不承接资源流转。

建议保留：

1. 加好友。
2. 查看基础名片。
3. 留言或轻聊天。
4. 查看公开荣誉和部分战报摘要。
5. 好友在线或赛季表现提示。

### 2. 首发不建议开放的好友能力

首发不建议开放：

1. 好友直接送金币。
2. 好友之间无限互刷收益。
3. 好友之间自由指定刷榜。
4. 好友免除常规掠夺限制。

原因很直接：

1. 会被做成养号体系。
2. 会污染资源产出节奏。
3. 会破坏匿名匹配与公平掠夺的意义。

## 八、反霸凌与反刷子规则

为保证常规池长期健康，首发建议明确下面几条：

1. 同一玩家 24 小时内对同一目标成功掠夺次数应受限。
2. 同一目标每日被成功掠夺次数达到阈值后自动获得保护。
3. 高低段差距过大的目标即使进入列表，也应收益衰减。
4. 疑似小号、互刷号和资源转运号应进入风控池，不进入正常收益计算。
5. 新手池、保护池和风控池应与常规匿名池隔离。

## 九、首发推荐规则组合

如果首发要做最小可用版本，建议直接采用下面这套组合：

1. 常规匿名池作为主入口。
2. 复仇池和通缉池作为特殊入口。
3. 新手保护池独立隔离。
4. 目标展示采用候选卡流，不做全服名单。
5. 玩家只看到标签和区间，不看到完整精确底牌。
6. 刷新有成本、有上限、不保证刷出肥羊。
7. 匹配优先看综合分段，不写死为单一等级差。

只要这七条先守住，目标池就会同时满足三件事：

1. 玩家上线有目标打。
2. 目标差距不会太离谱。
3. 列表与系统压力都能被控制在首发可实现范围内。

## 十、与点券系统的边界说明

如果后续要把点券接进匹配系统，必须先写死一条底线：

点券可以购买便利和试错次数，但不能直接购买掠夺胜率、精确底牌和无限筛选肥羊的能力。

建议允许点券影响的部分：

1. 额外刷新次数，但要有每日上限。
2. 额外侦察次数，但只提高信息精度，不直接给精确底表。
3. 额外保存编队、快捷侦察或目标收藏位。

不建议点券直接影响的部分：

1. 直接购买高收益目标。
2. 直接查看精确兵力和精确金币。
3. 无限刷新直到刷出最优目标。
4. 直接突破匹配保护线攻击低段位目标。
