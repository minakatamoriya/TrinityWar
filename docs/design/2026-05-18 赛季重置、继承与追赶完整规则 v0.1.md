# 2026-05-18 赛季重置、继承与追赶完整规则 v0.1

## 一、文档目标

这份文档用于统一首发赛季制的完整执行规则，解决下面四类问题：

1. 四周赛季结束后，哪些内容必须重置，哪些内容必须保留。
2. 各类资产如何继承，继承比例建议是多少。
3. 赛季结算奖励应该如何发，既能承接老玩家投入感，又不破坏新赛季公平。
4. 新赛季与赛季中后段的新玩家追赶机制如何设计，以及怎样防止被小号和套利账号滥用。

这份规则面向首发最小可用版本，原则只有一句话：

赛季负责重置竞争秩序，永久系统负责承接投入感，任何跨赛季保留都不能直接变成新赛季开局碾压。

## 二、赛季基础框架

### 1. 赛季周期

首发建议固定采用 28 天赛季，也就是 4 个自然周。

四周节奏建议如下：

1. 第 1 周：开荒、建基础盘、解锁核心功能。
2. 第 2 周：种植分工、掠夺循环和阵营上缴开始拉开差距。
3. 第 3 周：榜单竞争、阵营对抗、复仇和高阶种子使用进入高峰。
4. 第 4 周：冲榜、结算预热、遗产结算和下赛季预告。

### 2. 赛季重置原则

首发不采用硬清档，而采用软重置。

软重置的处理逻辑固定分成四层：

1. 竞争资产：必须重置。
2. 经营资产：压缩保留。
3. 身份资产：永久保留。
4. 剩余资产：折算补偿，不原样带入。

### 3. 首发体验目标

首发赛季制必须同时做到下面四件事：

1. 新玩家进入新赛季时有追赶窗口。
2. 老玩家不会觉得一个月全部白打。
3. 高活跃玩家有比新玩家更强的身份感和轻度起步优势。
4. 任何上赛季积累都不能穿透赛季边界变成长期数值膨胀。

## 三、重置与保留总清单

### 1. 必须重置的内容

这些内容直接决定当赛季竞争秩序，赛季结束后必须清空或重建：

1. 赛季排行榜名次。
2. 赛季积分与赛季荣誉进度。
3. 赛季任务、赛季活动和赛季商店进度。
4. 阵营周结算、赛季冲榜状态和赛季限时规则。
5. 通缉、复仇、保护期、临时追杀状态。
6. 当前地图占领结果、争夺状态和临时据点归属。
7. 当前外场种植状态、成熟进度和收取窗口。
8. 当前暴露资产状态，包括余额金币、可掠夺临时待领取、战时临时槽位状态。
9. 临时战斗增益、限时 buff、限时掉落倍率和赛季法则效果。
10. 赛季专属兑换道具和只服务于当季竞争的临时票券。

### 2. 压缩保留的内容

这些内容需要承接经营成果，但不能完整继承：

1. 主城等级与主要功能建筑等级。
2. 金库上限。
3. 外场位数量。
4. 基础兵力与战备准备度。
5. 金库金币与部分基础材料。
6. 种子库存。
7. 阵营贡献中的长期历史部分。

### 3. 永久保留的内容

这些内容负责承接身份感、生涯感和收藏感，建议永久保留：

1. 账号身份和角色基础信息。
2. 历史赛季战绩与生涯统计。
3. 称号、头像框、皮肤、外观和纪念性展示物。
4. 成就、图鉴、种子解锁记录和收藏类解锁。
5. 阵营履历、历史职位、历史贡献勋章。
6. 付费购买的非战力型永久权益。

### 4. 折算继承的内容

这些内容不建议原样进入下赛季，但应提供遗产感补偿：

1. 赛季末剩余金币。
2. 赛季扩张出来的金库上限。
3. 赛季末剩余普通种子和稀有种子。
4. 赛季末剩余基础建设材料。
5. 赛季末剩余可折算的非战斗类消耗品。

## 四、各类资产继承比例建议

### 1. 主城与功能建筑

建议规则：

1. 主城等级不原样继承，按档位压缩回退。
2. 主城解锁的核心功能保留，不重复让老玩家走最枯燥的新手流程。
3. 主要功能建筑保留到基础成熟档，高阶效率回收。

首发建议比例：

1. 主城等级：继承到上赛季最高等级对应的 50% 到 60%，但至少保留核心功能解锁档。
2. 金库、兵营、防守、人口等功能建筑：继承到上赛季等级的 50%。
3. 高于基础成熟档的效率型加成全部清空，由新赛季重新拉升。

体验目标：

1. 老玩家开局不会回到完全新手。
2. 老玩家也不会带着完整成熟号开新季。

### 2. 外场位与农场能力

建议规则：

1. 外场位数量保留。
2. 外场当前作物、成熟度、成长计时和赛季临时加成全部重置。
3. 任何直接提高当季产量、缩短成熟时间的赛季强化全部清零。

首发建议比例：

1. 外场位数量：100% 保留。
2. 外场效率加成：0% 保留。
3. 当前播种状态：0% 保留。

这样做的原因很明确：

1. 外场位数量本身是经营成果，应该承接。
2. 当前外场产出状态是竞争资产，必须洗牌。

### 3. 金币与金库上限

建议规则：

1. 金币绝不原量继承。
2. 金库上限拆成永久基础值与赛季扩张值。
3. 赛季扩张值只转一小部分到永久基础值，其余折算为结算奖励。

首发建议比例：

1. 金库金币：仅保留赛季末余额的 5% 到 10%，并设置硬上限，建议不超过新赛季 1 到 2 天基础建设所需。
2. 余额金币与临时待领取：0% 保留。
3. 永久基础金库上限：保留已有永久值。
4. 赛季扩张金库上限：仅将其中 10% 到 15% 转为永久基础值，其余折算成结算奖励。

设计目的：

1. 给老玩家遗产感。
2. 阻止安全资产无限跨季堆叠。

### 4. 兵力与战备资产

建议规则：

1. 当前兵力不原样保留。
2. 兵力折成基础预备兵或开季训练资源。
3. 强战斗导向的即时消耗品不跨季继承。

首发建议比例：

1. 当前兵力：折算为 10% 到 20% 的开季预备兵额度。
2. 战备类即时消耗品：0% 保留。
3. 基础训练材料：保留 10% 到 20%，并设置硬上限。

### 5. 种子体系

种子必须单独看待，因为它不是金币替代物，而是农场循环的稀缺约束。

在当前规则下，种子具有三个特征：

1. 免费，不通过金币直接购买。
2. 主要来自主城等级赠送、每日首发包和掠夺获取。
3. 会随着赛季推进持续累积，天然存在越屯越多的问题。

因此，首发不建议完整保留种子库存。

建议规则：

1. 已解锁种子类型永久保留。
2. 种子图鉴、种子成就和历史解锁记录永久保留。
3. 种子库存数量赛季结束后不原样继承。
4. 普通种子可以按低比例折算为下赛季开局补给。
5. 稀有种子和高阶种子不直接继承，只折算遗产奖励。

首发建议比例：

1. 普通种子库存：保留 10% 到 20%，并设置总上限，建议不超过新赛季 1 到 2 天的基础播种量。
2. 稀有种子库存：保留 0% 到 10%，更建议直接折算为遗产点数或纪念补偿。
3. 传说种子库存：0% 保留，只折算荣誉型奖励或遗产货币。
4. 已解锁种子类型：100% 保留。
5. 当前地块中的播种进度：0% 保留。

这套规则的目标是：

1. 玩家不会觉得辛苦解锁过的种子白做了。
2. 老玩家不会带着大量种子库存穿透新赛季农场节奏。
3. 农场仍然保留赛季内的调度与掠夺意义。

### 6. 阵营贡献与阵营资产

建议把阵营贡献拆成两层：

1. 赛季贡献。
2. 生涯贡献。

首发建议比例：

1. 赛季贡献：0% 保留，全部重置。
2. 生涯贡献：100% 保留，作为历史履历和荣誉展示。
3. 历史职位和历史勋章：100% 保留。
4. 赛季阵营公库：0% 原样继承，改按赛季结算规则折算奖励。

### 7. 基础材料与非战斗消耗品

首发建议采用白名单保留策略。

建议比例：

1. 基础建设材料：保留 10% 到 20%，并设置硬上限。
2. 稀有建设材料：保留 0% 到 10%。
3. 限时活动材料：0% 保留。
4. 赛季专属兑换券：0% 保留。
5. 纪念型收藏道具：100% 保留。

## 五、赛季结算奖励结构

### 1. 结算奖励设计原则

赛季结算奖励不能只发直接强度，否则新赛季会变成头部玩家继续滚雪球。

建议把奖励拆成四层：

1. 荣誉层。
2. 遗产层。
3. 参与层。
4. 竞赛层。

### 2. 荣誉层奖励

用于承接身份感和展示欲，永久保留：

1. 赛季称号。
2. 头像框。
3. 皮肤或局内装饰。
4. 历史奖杯与赛季档案页。
5. 阵营荣誉展示徽章。

### 3. 遗产层奖励

用于把赛季经营成果转成下一季的轻度起步优势，但不形成碾压：

1. 遗产点数或遗产币。
2. 少量永久基础金库上限加成。
3. 少量开季基础种子包。
4. 少量开季基础建设包。
5. 少量开季训练补给。

建议规则：

1. 遗产奖励必须设置单赛季上限。
2. 遗产奖励不能直接换成大额金币。
3. 遗产奖励更适合换开季便利，而不是直接换开季战力。

### 4. 参与层奖励

用于避免普通玩家觉得白玩：

1. 只要达到最低参与门槛，均可获得基础结算奖励。
2. 基础奖励以外观碎片、基础遗产点和少量补给为主。
3. 不因为未进榜就让玩家赛季体验归零。

### 5. 竞赛层奖励

用于驱动头部竞争，但必须控制强度外溢。

建议内容：

1. 更高档位的称号与荣誉外观。
2. 更多遗产点数。
3. 少量点券或高级便利货币。
4. 额外档案展示内容。

不建议内容：

1. 大量金币直发。
2. 大量兵力直发。
3. 大量高阶种子直接带入下赛季。
4. 大量强战斗消耗品直接带入下赛季。

### 6. 结算奖励推荐结构

首发建议直接采用下面这套：

1. 所有达标玩家都拿参与奖励。
2. 赛季榜单按区间发荣誉层和竞赛层奖励。
3. 所有遗产资产先按折算规则换成遗产点数。
4. 遗产点数再兑换少量开季补给和永久轻成长。
5. 赛季结算页明确展示“本季保留了什么、折算了什么、失去了什么、下季带走了什么”。

## 六、新赛季追赶机制

### 1. 目标

追赶机制解决的是晚入场玩家的基础盘问题，不是让晚入场玩家跳过成本直接抢头部奖励。

### 2. 追赶对象

建议同时满足下面两类条件才开放完整追赶资格：

1. 进入时间晚于赛季开始若干天。
2. 主城、外场、兵力、贡献等核心进度明显低于当前赛季平均线。

### 3. 追赶内容

追赶内容只能补基础盘，不能直接给高阶战力。

首发建议包含：

1. 主城前几级升级时间缩短。
2. 前几档功能建筑升级成本下调。
3. 前几块外场的启动资源补贴。
4. 基础种子补给包。
5. 第一批造兵或训练补贴。
6. 晚入场追赶任务链。

### 4. 追赶任务链建议

任务链建议只覆盖主循环入门动作：

1. 完成第一次播种。
2. 完成第一次成熟收取。
3. 完成第一次造兵。
4. 完成第一次匿名掠夺。
5. 完成第一次金币上缴。

奖励方向应以功能补齐为主，不直接发大量赛季积分。

### 5. 新赛季统一开局保护

除了晚入场追赶，建议新赛季前 3 天统一给全体玩家一层轻保护：

1. 开季基础种子包统一发放。
2. 开季基础建设包统一发放。
3. 开季前若干次被掠收益保护略高。
4. 开季匹配优先同阶段对手。

这样可以减少老玩家遗产补给带来的体感差异。

## 七、反滥用边界

### 1. 追赶资格边界

建议明确写死下面几条：

1. 每个账号每赛季只可触发一次完整追赶资格。
2. 追赶 buff 只在前几天或前几档功能补齐前生效。
3. 追赶任务链完成或过期后自动关闭。
4. 同一赛季不能通过删号重建重复领取。

### 2. 完整结算奖励门槛

要区分“可以玩”和“可以拿完整高阶奖励”。

建议规则：

1. 所有晚入场玩家都可正常游玩，并获得基础参与奖励。
2. 完整榜单奖励资格需要满足最低参与门槛。

建议门槛至少选两到三类：

1. 最低参与天数。
2. 最低登录次数。
3. 最低主城阶段。
4. 最低任务链完成度。
5. 最低赛季活跃分。

### 3. 风控建议

首发不需要重审核，但必须有基础识别：

1. 同账号赛季内重复触发追赶资格。
2. 同设备或同环境短时间创建多个晚入场账号。
3. 账号创建后极短时间异常完成追赶任务。
4. 进度异常跳升但正常玩法行为很少。
5. 多个追赶账号集中冲某类结算奖励后快速弃坑。

命中风控后的处理建议从轻到重：

1. 冻结追赶 buff。
2. 冻结追赶任务奖励。
3. 只保留基础参与奖励。
4. 取消完整榜单结算资格。

## 八、首发推荐的最终执行版本

如果首发只做一版最小可用规则，建议直接采用下面这套：

1. 赛季固定 28 天。
2. 排行榜、赛季积分、赛季任务、赛季活动、通缉复仇、地图占领、当前外场状态全部重置。
3. 主城和功能建筑按 50% 左右压缩保留，但保留核心功能解锁。
4. 外场位数量 100% 保留，外场效率和当前播种状态全部清零。
5. 金币仅保留 5% 到 10%，并设置硬上限。
6. 金库扩张值仅 10% 到 15% 转成永久基础值。
7. 当前兵力折算为 10% 到 20% 的开季预备兵额度。
8. 种子已解锁类型永久保留，普通种子只保留 10% 到 20% 且有总上限，稀有与传说种子默认不原样继承。
9. 赛季贡献清零，生涯贡献与阵营履历永久保留。
10. 赛季结算以称号、外观、遗产点数、少量开季补给和少量便利货币为主，不发大额直接战力资源。
11. 晚入场追赶只补基础盘，不直通高阶榜单收益。
12. 完整榜单奖励必须满足最低参与门槛，并对异常多账号追赶做基础风控。

## 九、给玩家的外部表达建议

为了降低玩家对重置的抵触，外部文案建议不要强调“清档”，而强调下面三点：

1. 新赛季会重置竞争格局。
2. 你的荣誉、解锁与部分经营遗产会被保留。
3. 你不是从零开始，而是带着遗产重新开局。

最重要的一句外部表达建议是：

新赛季重置的是竞争，不是你过去一个月的价值。
