# 2026-06-23 首页精简 IA 与模块取舍 v0.1

## 一、文档目标

本文是对《2026-06-01 首页职责与轻引导方案 v0.1》的收口补充。

目标只有三件事：

1. 判断当前首页各模块是否仍有必要。
2. 把首页压缩到手机首屏内可理解。
3. 明确社交动态、推荐行动、农场灵宠卡等重复信息如何处理。

本方案不推翻“首页负责提醒、推荐和聚合入口”的方向，只进一步收紧信息密度。

## 二、核心结论

首页首屏不应该再承载五到六个完整模块，而应该收敛成：

1. 一行进度总览。
2. 一个当前最值机会。
3. 一组稳定入口。
4. 一个有真实待处理事项时才出现的轻提示。

一句话概括：

**首页只回答“我现在先点哪里”，其他信息交给对应系统页。**

## 三、当前首页的主要问题

当前首页的问题不是模块方向错，而是多个模块在反复解释同一批状态。

### 1. 机会区和推荐行动重复

“当前最值得处理”和“推荐行动”都在讲成熟田、灵宠养成、复仇、好友互动。

玩家读到第二遍时不会获得新的判断，只会感觉首页变长。

建议：

1. 保留“当前最值得处理”。
2. 删除独立的“推荐行动”模块。
3. 推荐文案并入机会卡的收益说明。

### 2. 农场灵宠卡和机会区重复

农场卡展示成熟田数量、培育中数量、下一块田倒计时。

灵宠卡展示主战灵宠、等级、可突破或可合成状态。

这些信息如果当前最重要，就应该进入主机会卡；如果当前不重要，就不该占完整卡片。

建议：

1. 农场和灵宠不再作为完整说明卡常驻首页。
2. 改成底部快捷入口，保留 1 个短状态。
3. 只有出现高优先级机会时，才进入主机会卡。

### 3. 社交动态不适合做首页核心提醒

社交动态目前更像记录流，不像真实待办。

当前数据模型里有 `isRead` 和 `feedUnread`，但页面没有清晰的“读过即消化”行为，也没有面向动态的已读入口。玩家点进社交页后，不一定能让首页提醒自然消失。

这会带来两个问题：

1. 首页显示“未读动态”会制造虚假的待处理感。
2. 玩家无法明确知道自己要做什么才能清掉这条提示。

建议：

1. 首页不展示“未读动态”。
2. 社交只展示可处理事项。
3. 可处理事项包括好友申请、可助力田地、组队邀请、可复仇关系。

### 4. 顶部总览信息过宽

顶部总览应保留轻目标感，但不应放太多资源和进度。

手机首屏里，顶部只需要告诉玩家今天有没有推进，以及离下一档还差多少。

建议：

1. 保留今日贡献。
2. 保留距下一档差值。
3. 可保留下一块田成熟倒计时。
4. 暂不展示本周贡献。
5. 暂不展开资源摘要。

## 四、模块取舍表

| 当前模块 | 处理方式 | 原因 |
| --- | --- | --- |
| 顶部总览区 | 压缩保留 | 提供轻目标感，但只保留最关键数字 |
| 当前最值得处理区 | 强化保留 | 首页核心职责，回答现在先点哪里 |
| 农场卡 | 合并为入口 | 状态与机会区重复 |
| 灵宠卡 | 合并为入口 | 状态与机会区重复 |
| 竞争与社交区 | 降级为条件提示 | 只有存在真实可处理事项时展示 |
| 推荐行动区 | 删除 | 与机会区重复，容易变回任务列表 |
| 新手引导卡 | 保留例外 | 新手期需要唯一下一步 |

## 五、精简首页首屏结构

### 1. 顶部一行总览

回答的问题：

1. 我今天推进了多少。
2. 我离下一档还有多远。
3. 下一次回来看什么。

建议样式：

一行或两行紧凑信息，不做大卡。

建议文案：

1. 今日贡献 82。
2. 距下一档 18。
3. 下一块田 23 分钟后成熟。

### 2. 当前最值机会

回答的问题：

1. 我现在先点哪里。

建议只展示 1 个主机会，最多再展示 1 个次机会。

主机会优先级：

1. 即将枯萎田地。
2. 已成熟田地。
3. 主战灵宠可突破。
4. 可复仇目标。
5. 好友申请或组队邀请。
6. 好友田地可助力。
7. 高价值探索目标。

建议文案：

1. 1 块田即将枯萎，处理后还能回收收益。
2. 2 块田已成熟，收取可推进贡献。
3. 主战灵宠可突破，战力还有提升空间。
4. 1 个目标可复仇，入口仍然有效。

### 3. 稳定入口栏

回答的问题：

1. 我想自己选玩法时去哪里。

建议保留 3 到 4 个入口：

1. 农场。
2. 灵宠。
3. 探索。
4. 阵营。

每个入口只放一个短状态，不放完整说明。

示例：

| 入口 | 状态 |
| --- | --- |
| 农场 | 可收 2 |
| 灵宠 | 可突破 |
| 探索 | 可复仇 1 |
| 阵营 | 差 18 |

### 4. 条件轻提示

回答的问题：

1. 是否有不适合压进主机会，但仍值得提醒的事。

这块不是常驻模块，只在有明确可处理事项时出现。

可出现内容：

1. 2 位好友可助力。
2. 1 条好友申请待处理。
3. 1 个组队邀请待确认。
4. 保护时间即将结束。

不可出现内容：

1. 未读动态 5。
2. 好友 12。
3. 关注 8。
4. 最新动态 3。

## 六、社交信息的新边界

首页不再展示“社交动态”，只展示“社交待处理”。

### 可以进首页的社交信息

1. 有按钮可处理。
2. 处理后状态会改变。
3. 处理后玩家能获得收益、关系变化或风险解除。
4. 玩家能理解为什么现在要点。

例如：

1. 好友申请。
2. 组队邀请。
3. 可助力好友田地。
4. 可复仇关系。

### 不建议进首页的社交信息

1. 普通动态。
2. 纯计数。
3. 只表示“有人发生过行为”的记录。
4. 没有清晰消化动作的未读数。

例如：

1. 某好友帮你回浇了灵田。
2. 某玩家关注了你。
3. 动态未读 6。
4. 好友数量 12。

这些内容更适合留在社交页动态标签中。

## 七、首屏线框草案

手机首屏建议如下：

```text
┌────────────────────────┐
│ 今日 82  距下一档 18   │
│ 下一块田 23 分钟后成熟 │
├────────────────────────┤
│ 现在最值               │
│ 2 块田已成熟           │
│ 收取可推进贡献         │
│ [去农场]               │
├────────────────────────┤
│ 农场 可收2  灵宠 稳定  │
│ 探索 可复仇1 阵营 差18 │
├────────────────────────┤
│ 2 位好友可助力 [查看]  │
└────────────────────────┘
```

如果没有条件轻提示，最后一行不展示。

如果新手引导存在，首页只展示新手引导卡，不展示正式首页结构。

## 八、落地规则

### 1. 首页只展示可压缩的信息

任何首页信息必须能在一行或一张小卡里表达。

如果需要解释两句以上，应该放回对应系统页。

### 2. 首页避免同时出现“原因”和“二次推荐”

主机会卡已经说明为什么值得做。

不再额外追加“推荐行动”列表。

### 3. 计数必须对应真实行动

首页可显示“可收 2”“可复仇 1”“待确认 1”。

首页不显示“未读 6”“好友 12”这类不能直接转化为动作的数字。

### 4. 社交提醒必须可消化

进入首页的社交提醒必须满足：

1. 有明确目标页。
2. 有明确处理动作。
3. 处理后提示会减少或消失。

在未补齐社交动态已读机制前，不把 feed unread 放进首页。

## 九、接口字段建议

如果后续从前端拼装改为后端聚合，首页可以只需要下面几组字段。

### 1. progress

```ts
interface HomeProgressSummary {
  todayContribution: number;
  nextContributionTarget: number;
  contributionRemaining: number;
  nextFarmReadySeconds: number | null;
}
```

### 2. primaryOpportunity

```ts
interface HomePrimaryOpportunity {
  type: 'farm' | 'spirit' | 'raid' | 'social' | 'faction' | 'calm';
  title: string;
  description: string;
  actionLabel: string;
  target: ClientSceneKey;
  priority: number;
}
```

### 3. entryBadges

```ts
interface HomeEntryBadge {
  target: ClientSceneKey;
  label: string;
  status: string;
  count?: number;
}
```

### 4. actionableSocial

```ts
interface HomeActionableSocialSummary {
  friendRequests: number;
  teamInvites: number;
  assistableFriends: number;
  revengeableTargets: number;
}
```

注意这里不包含 `feedUnread`。

## 十、建议后续落地顺序

1. 先调整前端首页结构，删除推荐行动区。
2. 把农场和灵宠完整卡压成入口状态。
3. 首页移除未读动态，只保留可处理社交事项。
4. 等社交动态补齐已读机制后，再考虑是否在社交页内展示未读。
5. 如果首页拼装逻辑继续变复杂，再把 `primaryOpportunity` 下沉到服务端聚合。

## 十一、最终建议

首页第一版精简目标应当是：

1. 玩家 3 秒内知道现在先点哪里。
2. 玩家不需要在首页阅读完整系统状态。
3. 首页不展示无法自然消失的提醒。
4. 首页不再用“推荐行动”重复包装机会。
5. 手机首屏内能看完核心内容。

这版首页会更像一个轻量驾驶台，而不是任务板。
