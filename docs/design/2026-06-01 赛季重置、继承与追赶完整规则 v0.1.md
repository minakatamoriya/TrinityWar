# 2026-05-18 赛季重置、继承与追赶完整规则 v0.1

> 2026-06-01 重写：本文件替代旧版规则。旧版存在编码损坏，并且仍沿用“种子库存”等过时概念；当前系统已经改为“可种植资格 + 植物精华”，因此赛季规则需要按现有开发进度重新核对后再开发。

## 一、文档目标

本文件用于定义首发阶段可执行的赛季重置、继承、结算奖励和追赶边界。

当前阶段的原则是：

1. 先明确资产边界，再开发自动重置。
2. 先做最小可靠闭环，再做奖励和追赶。
3. 不把旧赛季积累直接变成新赛季碾压优势。
4. 不再使用“种子库存”说法，统一改为“可种植资格”和“植物精华”。

一句话原则：

赛季重置的是竞争进度，不是玩家身份、图鉴、收藏和已经解锁的长期资格。

## 二、当前系统资产分类

### 1. 账号与身份资产

包含：

1. 账号、昵称、头像。
2. 当前阵营归属。
3. 历史身份记录。
4. 图鉴、发现记录、收藏记录。

处理原则：

永久保留。

### 2. 植物资产

当前植物系统不再有“种子库存”概念。

需要区分：

1. 可种植资格：玩家是否已经解锁某种植物的种植权限。
2. 植物精华：玩家通过收获、掠夺、奖励等获得的消耗型资源。
3. 当前田地状态：某块田当前是否正在成长、成熟、枯萎、可领取金币。
4. 植物研究/发现记录：玩家是否见过、研究过某种植物。

处理原则：

1. 可种植资格保留。
2. 植物研究/发现记录保留。
3. 当前田地状态重置。
4. 植物精华是否清零或折算，需要单独核对。P0 不先做折算奖励。

### 3. 灵宠资产

需要区分：

1. 灵宠本体：玩家已经拥有的灵宠。
2. 灵宠词条：已经洗练出的词条。
3. 灵宠图鉴和碎片。
4. 灵宠等级、经验、突破阶段。
5. 灵宠当前血量、赛季内战斗状态。
6. 灵宠培养资源：兽魂、天机符、灵根、灵髓、灵玉等。

处理原则：

1. 灵宠本体保留。
2. 灵宠词条保留。
3. 灵宠图鉴和碎片保留。
4. 灵宠等级、经验、突破阶段重置。
5. 灵宠血量回到 1 级基础状态。
6. 灵宠培养资源是否清零或折算，暂不在 P0 自动处理，需核对经济影响后再定。

### 4. 阵营资产

需要区分：

1. 个人赛季贡献。
2. 阵营赛季总贡献。
3. 贡献日志。
4. 每日俸禄领取状态。
5. 阵营归属。
6. 历史贡献、历史职位、历史荣誉。

处理原则：

1. 阵营归属保留。
2. 个人赛季贡献清零。
3. 阵营赛季总贡献清零。
4. 本赛季贡献日志清空或归档。
5. 每日俸禄状态清空。
6. 历史贡献和荣誉展示暂不实现，后续作为永久身份资产设计。

### 5. 战斗与掠夺资产

包含：

1. 当前掠夺目标池。
2. 未完成掠夺订单。
3. 当前资产锁。
4. 保护状态。
5. 战报。
6. 复仇、通缉、组队掠夺等非 MVP 状态。

处理原则：

1. 当前掠夺目标池重置。
2. 未完成掠夺订单取消。
3. 当前资产锁过期。
4. 保护状态清空或按新赛季保护规则重建。
5. 历史战报是否保留需要核对展示价值。P0 可以先保留历史记录，但不参与新赛季结算。
6. 通缉、组队掠夺等非 MVP 内容暂不进入赛季结算设计。

### 6. 经营资源

包含：

1. 金库金币。
2. 外露金币。
3. 待领取税收。
4. 待领取分红。
5. 掠夺溢出待领取金币。
6. 建筑等级与法术/科技等级。
7. 兵力、训练队列。

处理原则：

1. 金币类竞争资源重置或折算，不原样继承。
2. 兵力和训练队列重置。
3. 建筑等级、法术等级是否压缩继承，需要单独核对。P0 不先做复杂压缩。

### 7. 签到与每日状态

包含：

1. 赛季签到记录。
2. 每日任务状态。
3. 每日广告奖励次数。
4. 每日恢复次数、侦查次数等日限状态。

处理原则：

1. 赛季签到按赛季隔离，新赛季重新开始。
2. 每日任务和阵营任务状态清空。
3. 普通每日限制本来按日期自然刷新，不需要作为赛季资产继承。

## 三、P0 赛季重置范围

P0 只做“今天是新赛季时，玩家不会继续沿用旧赛季竞争状态”的闭环。

必须完成：

1. 服务端能计算当前赛季号、当前周、总周数。
2. 固定 28 天一个赛季。
3. 新赛季开始后，玩家首次进入时只执行一次赛季重置。
4. 赛季签到按赛季号隔离。
5. 阵营贡献清零。
6. 当前田地状态清空。
7. 金币、兵力、训练队列清空。
8. 灵宠本体和词条保留，但等级、经验、突破、血量重置。
9. 掠夺目标、资产锁、未完成订单等临时竞争状态清理。

P0 不做：

1. 赛季结算奖励。
2. 排名奖励。
3. 追赶机制。
4. 遗产点数。
5. 建筑压缩继承。
6. 植物精华折算。
7. 灵宠培养资源折算。

## 四、P1 赛季结算奖励方向

P1 需要先设计奖励结构，再开发。

推荐奖励分层：

1. 参与奖励：只要达到最低参与门槛即可获得。
2. 阵营贡献奖励：按个人赛季贡献档位发放。
3. 排名奖励：只给荣誉、外观、少量便利资源，不给大量战力资源。
4. 收藏奖励：称号、头像框、赛季纪念记录。

不建议：

1. 大量金币直发。
2. 大量兵力直发。
3. 大量植物精华直接带入新赛季。
4. 大量灵宠培养资源直接带入新赛季。
5. 让排名奖励变成新赛季开局碾压。

## 五、P1 追赶机制方向

追赶机制只服务晚入场玩家的基础盘，不服务冲榜套利。

可能包含：

1. 新赛季前几天基础保护。
2. 晚入场基础任务链。
3. 少量基础植物精华补给。
4. 前期建筑/田地节奏补齐。
5. 低贡献玩家的基础俸禄补齐。

限制：

1. 每账号每赛季只触发一次。
2. 不直接给高阶灵宠资源。
3. 不直接给排名积分。
4. 不绕过植物可种植资格。
5. 需要基础反滥用规则。

## 六、待核对问题

开发前需要确认：

1. 植物精华跨赛季是清零、保留、还是折算。
2. 天机符是否属于赛季资源，是否清零或保留。
3. 兽魂、灵根、灵髓、灵玉是否跨赛季保留。
4. 建筑等级是否本赛季先全部清零，还是等 P1 做压缩继承。
5. 战报是否需要按历史赛季归档。
6. 阵营历史贡献是否需要展示。
7. 赛季奖励是否在本阶段开发，还是先只做重置。

## 七、外部表达

面向玩家不强调“清档”，而强调：

1. 新赛季会重置竞争格局。
2. 已解锁资格、图鉴、灵宠和词条会保留。
3. 赛季内养成和竞争资源会重新开始。
4. 奖励和追赶机制会在确认经济边界后开放。

推荐表达：

新赛季重置的是竞争，不是你已经建立起来的身份与收藏。
