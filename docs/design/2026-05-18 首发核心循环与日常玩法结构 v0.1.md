# 2026-05-18 首发核心循环与日常玩法结构 v0.1

## 一、文档目标

这份文档只解决一件事：

把 Trinity War 首发版本的核心循环和日常玩法结构收敛成一套更上头、但仍然可开发落地的方案。

它重点回答三个问题：

1. 玩家为什么会反复上线。
2. 玩家每次上线应该先做什么、再做什么。
3. 首发版本应该优先强化哪些爽点，而不是盲目堆系统。

## 二、首发版本的一句话循环

玩家通过种植和主城建设积累资源，通过成熟窗口和暴露资产制造机会，通过单人掠夺、双人通缉和实名复仇制造情绪波峰，再把战利品继续投入主城成长、榜单竞争和荣誉收集。

用最短的话概括就是：

1. 养资源。
2. 抢时机。
3. 打别人或被别人盯上。
4. 复仇或摇人。
5. 升主城变强。
6. 上榜留名。

## 三、首发必须打透的爽点

首发不追求全，而是优先把最强的六类爽点做厚。

### 1. 养成爽

来源：种菜收菜、主城升级、金库变大、训练队列推进。

价值：让玩家始终觉得自己在持续变强，不会因为一次失败就完全失去积累感。

### 2. 临门一脚爽

来源：成熟时点击收取、黄金采收期、任务领奖、升级完成、榜单结算。

价值：强化每次上线第一分钟的正反馈，让玩家觉得这次上线立刻有收获。

### 3. 偷鸡爽

来源：抓住他人成熟窗口、资源暴露期、战备空虚期狠狠干一票。

价值：让掠夺不是纯数值对撞，而是机会判断。

### 4. 复仇爽

来源：被打后立刻看到攻击者、损失和可复仇入口，成功反打还能获得额外荣誉。

价值：把挫败感转成下一轮上线动机。

### 5. 社交协作爽

来源：好友接通缉、帮忙围剿高价值目标、领取协作奖励。

价值：制造共同记忆和传播性，不让对抗只停留在单人层面。

### 6. 身份爽

来源：发展榜、掠夺榜、复仇榜、协作榜、收藏榜、称号和奖章。

价值：让玩家争的不只是资源，而是地位和标签。

## 四、核心循环结构

首发建议采用三层循环叠加，而不是单一循环。

### 1. 底层循环：稳定养成

1. 种植或布置外场。
2. 等待成长。
3. 在成熟窗口收获。
4. 将收益投入主城、金库、外场位、训练队列。
5. 提升下一轮的产出效率和承载能力。

这一层负责提供稳定安全感，是整套玩法的地基。

### 2. 中层循环：风险博弈

1. 部分高收益资产进入可见暴露状态。
2. 玩家决定立刻收、继续养肥，还是拿来做诱饵。
3. 其他玩家观察到目标后发起单抢或通缉。
4. 结算后产生收益、损失、冷却、仇恨和新机会。

这一层负责制造“我现在要不要出手”的判断感。

### 3. 外层循环：对抗与身份

1. 玩家因成功收割、成功抢夺、成功复仇而获得榜单分数、奖章和称号进度。
2. 玩家为了更高排名和更稀有身份继续回到养成和博弈层。
3. 每日和每周结算放大身份差异，驱动下一轮竞争。

这一层负责让长期目标成立。

## 五、玩家单次上线的标准流程

首发版本建议强制把玩家上线后的心流控制在一个很清晰的顺序里。

### 1. 第一步：先领爽点

玩家上线后第一屏先看到领主简报，至少包含：

1. 已成熟资源和黄金采收提醒。
2. 已完成建筑或训练队列。
3. 待领取收益。
4. 被掠夺或可复仇提醒。
5. 可领取任务或成就奖励。

目标不是提供信息大全，而是先给玩家一个“我现在就有东西可拿”的感觉。

### 2. 第二步：处理成熟和升级

玩家先完成以下低阻力操作：

1. 收菜。
2. 补种。
3. 领取税收和分红。
4. 续上建筑或训练队列。

这一段必须足够顺滑，因为它承担首发版本的大部分保底体验。

### 3. 第三步：看机会和仇恨

玩家处理完基础收益后，再进入带判断的中段：

1. 看有没有高价值目标。
2. 看有没有刚打过自己的复仇对象。
3. 看有没有好友求援或可接通缉。
4. 判断这次上线是继续发育，还是狠狠干一票。

这一段是“上头”的主要来源。

### 4. 第四步：把结果投入成长

掠夺、复仇和收获所得，最终都要回流到成长系统：

1. 升主城。
2. 升金库。
3. 开外场位。
4. 训练兵力。
5. 完成图鉴、成就和榜单目标。

这样玩家每一次冒险都能转化成持续成长，而不是一次性刺激。

### 5. 第五步：埋下下次上线的钩子

玩家下线前，系统要明确制造下一次上线期待：

1. 某块地即将进入黄金采收期。
2. 某个建筑还差一点升级完成。
3. 某个复仇对象的保护即将结束。
4. 晚间活动即将开始。
5. 今日榜单还差一点就能上名次。

如果做不到这一步，玩家今天爽完了，明天未必还会来。

## 六、首发日常玩法结构

建议把日常拆成四条线，让不同玩家都能找到自己的主乐趣。

### 1. 保底线：稳定成长日常

这条线面向所有玩家，负责确保“上线不空手”。

内容建议包括：

1. 收成熟资源。
2. 补种和调度外场。
3. 主城和金库升级。
4. 训练和恢复兵力。
5. 领取税收、分红和日常任务奖励。

设计原则：短、顺、稳。

### 2. 机会线：时机博弈日常

这条线面向喜欢判断和偷鸡的玩家。

内容建议包括：

1. 黄金采收期收益加成。
2. 暴露资源窗口。
3. 高价值目标推荐。
4. 限时肥羊刷新。
5. 运输或成熟过程中的截击机会。

设计原则：有波峰、有窗口、有错过成本。

### 3. 对抗线：复仇与掠夺日常

这条线负责情绪波峰。

内容建议包括：

1. 今日可复仇名单。
2. 单人抢夺次数或机会。
3. 双人通缉目标。
4. 连续复仇奖励。
5. 今日战报摘要。

设计原则：被打不是结束，而是下一轮动作的起点。

### 4. 身份线：榜单与收藏日常

这条线面向重目标和重展示玩家。

内容建议包括：

1. 发展榜目标。
2. 掠夺榜目标。
3. 复仇榜目标。
4. 协作榜目标。
5. 称号、奖章、图鉴、事件收藏进度。

设计原则：让不同类型的玩家都有可追求的赢法。

## 七、首发版本的在线时段设计

为了适配异步玩法，首发不要求玩家长时间在线，而是给出几个固定波峰时段。

### 1. 早档：2 到 5 分钟

定位：保底上线。

推荐内容：

1. 收夜间成熟资源。
2. 领取待领取收益。
3. 续建造和训练队列。
4. 检查是否被打和是否能复仇。

目标：让玩家快速完成一次“我今天没掉队”的确认。

### 2. 中档：5 到 10 分钟

定位：机会处理。

推荐内容：

1. 处理黄金采收期。
2. 观察目标池。
3. 发起一到两次单人掠夺。
4. 对高价值目标发起双人通缉。

目标：制造当天第一次明显的刺激峰值。

### 3. 晚档：10 到 20 分钟

定位：主爆点时段。

推荐内容：

1. 集中开放高收益成熟窗口。
2. 刷新高价值通缉目标。
3. 集中结算好友求援和协作奖励。
4. 在榜单日结前制造最后冲刺。

目标：让玩家把最强记忆点集中留在晚上，形成讨论和回流。

## 八、种子系统建议

种子系统非常适合这款游戏，因为它天然能同时承接养成爽、收集爽、偷鸡爽和身份爽。

但前提是：

1. 它必须服务主循环，而不是变成独立小游戏。
2. 它必须鼓励玩家去看别人的田、抢别人的田。
3. 它不能让首发的时间系统复杂到失控。

### 1. 种子系统在首发里的定位

首发建议把种子定义为三层价值叠加：

1. 生产配方：不同种子决定种植周期、收益结构和风险暴露。
2. 收藏目标：玩家有自己的种子名录、图鉴和稀有度追求。
3. 对抗目标：玩家可通过掠夺、事件和成熟收取获取自己没有的稀有种子。

这样种子系统不是单纯“换皮作物”，而是让玩家多一个持续上头的目标。

### 2. 可以设置多种种子吗

可以，而且应该有多种，但首发不要做成每种都完全不同的复杂时间曲线。

更推荐的首发方式是：

1. 每块田一次只种一种种子。
2. 玩家可以在不同田种不同种子。
3. 种子很多，但时间模型只分少数几档。

也就是说：

1. 种子种类可以多。
2. 生长周期档位要少。

这样既保留选择感，又不会让玩家和程序都被复杂度拖死。

### 3. 周期不同会不会太复杂

如果每种种子都做独立周期，首发一定会复杂。

推荐收敛成三档时间带：

1. 短线种子：适合高频上线，收益低但回转快。
2. 标准种子：作为主流种植选择，收益和时间最平衡。
3. 长线种子：成熟慢，但峰值收益高，更适合做诱饵和争夺目标。

最重要的是：

1. 不同种子共享同一套状态机。
2. 只是成熟时间、丰熟时间和收益系数不同。

这样现有的田地时间状态机不用推翻，只是在配置层扩展。

### 4. 首发推荐的种子差异维度

首发不建议每种种子都有五六个属性。

建议只保留下面四个核心差异：

1. 生长周期。
2. 基础收益。
3. 黄金采收期长度。
4. 被掠夺时的种子掉落概率。

如果还想再加一个维度，建议优先加：

1. 丰熟阶段额外加成。

不建议首发就加太多特殊效果，例如：

1. 连锁增产。
2. 复杂混种加成。
3. 多田联动羁绊。
4. 大量天气克制。

这些都容易让系统变得不好懂。

### 5. 首发推荐的种子分类

可以按稀有度和用途双维度来做。

按稀有度：

1. 普通种子：商店稳定购买，承担基础生产。
2. 稀有种子：通过事件、掠夺、活动和高阶收取掉落。
3. 传说种子：首发建议极少量，主要作为身份目标和高价值诱饵。

按用途：

1. 金币型：直接提高收获金币。
2. 速熟型：缩短成熟周期，提高回转率。
3. 爆发型：丰熟时收益明显抬高，但更容易被盯上。
4. 收藏型：收益未必最高，但图鉴、称号和任务价值高。

### 6. 种子获取来源建议

你提的方向是对的，种子必须来自多条入口，这样才会制造外出掠夺动机。

首发建议保留五个来源：

1. 商店购买：普通种子稳定供应，作为保底入口。
2. 成熟收取：收菜时有小概率返还同类种子或掉落种子碎片。
3. 稀有事件：每日事件、限时事件、节日事件掉稀有种子。
4. 掠夺掉落：攻击他人田地时，有概率获得对方当前种植或名录中的稀有种子。
5. 图鉴阶段奖励：收集一定数量种子后领取额外种子箱或定向券。

这里最关键的是第四条。

掠夺不应只掉金币，还应让玩家觉得：

1. 这次我也许能抢到我没有的种子。
2. 那个大佬的田不只是肥，而且可能藏着我缺的图鉴。

这样掠夺的驱动力会比单纯金币高很多。

### 7. 掠夺中的种子掉落建议

掠夺掉种子可以做，但要避免过度随机和过度掠夺压力。

建议首发规则：

1. 只有目标当前种植的是稀有及以上种子时，才有种子掉落资格。
2. 掉落的不是整组产物，而是 1 份种子、碎片或解锁记录。
3. 首次获得新种子时，优先直接解锁图鉴。
4. 重复获得时，转为种子库存、升级材料或兑换碎片。

这样能保证两件事：

1. 新种子带来强烈“我抢到了新东西”的爽点。
2. 不会因为一次掠夺就把对方整个稀有生态打穿。

### 8. 每日事件如何和种子系统绑定

每日事件非常适合做种子系统的核心补给源。

建议事件主要围绕下面几类：

1. 流浪商人：限时出售稀有种子或折扣种子包。
2. 神秘风暴：某类种子今日成熟时间缩短。
3. 丰收节气：某类种子的返种率提升。
4. 野种侵入：击败指定目标或完成指定收取次数后获得特殊种子。
5. 黑市通缉：今日某类稀有种子在掠夺中掉率提升。

这样每日事件就不只是加数值，而是改变当天玩家的种植与掠夺选择。

### 9. 首发推荐的种子系统复杂度控制

这是最重要的收敛原则。

首发建议做到：

1. 种子很多，但时间档位只有三档。
2. 每块田一次只种一种，不做混种。
3. 不做复杂杂交。
4. 不做太多种子专属技能。
5. 稀有种子主要靠收益、速度、掉落价值和收集身份来体现差异。

也就是说，首发先把种子系统做成：

1. 易懂。
2. 好收集。
3. 能掠夺。
4. 会炫耀。

而不是先把它做成一套深度农业模拟。

### 10. 首发最值得做的爽点设计

如果种子系统首发只抓最强的爽点，建议优先抓下面五个：

1. 首次解锁新种子时的图鉴点亮和称号推进。
2. 稀有种子成熟时更强的视觉和音效反馈。
3. 掠夺成功后跳出“夺得新种子”的高亮提示。
4. 每日事件让某些种子当天突然变得很值钱。
5. 大佬田里种着稀有种子，天然变成全服关注目标。

### 11. 首发结论

种子系统非常值得做。

它最有价值的地方，不只是让玩家“多一种作物可选”，而是让玩家形成三种更强的动机：

1. 我想收集更多种子。
2. 我想种更值钱的种子。
3. 我想去抢别人没有被我解锁过的种子。

因此，首发最合理的落法不是“几十种完全独立作物模拟”，而是：

1. 少数时间档位。
2. 多种种子品类。
3. 稀有种子由事件和掠夺驱动获取。
4. 图鉴、称号和榜单承接长期目标。

这样它会明显增强你这款游戏的上头感，而且不会把系统做炸。

## 九、首发必做系统优先级

下面这些是最值得优先做厚的内容。

### P0：必须做扎实

1. 成熟状态与黄金采收期。
2. 资源分层，至少区分安全资金和暴露收益。
3. 单人掠夺。
4. 最近攻击者的实名复仇。
5. 主城、金库、外场位、训练队列。
6. 至少三条榜单中的发展榜、掠夺榜、复仇榜。

### P1：强烈建议首发带上

1. 双人通缉。
2. 好友求援。
3. 奖章、称号、图鉴中的至少一种身份系统。
4. 限时高价值目标刷新。
5. 战报回放摘要和仇恨入口。

### P2：可以后续增强

1. 更复杂的协作围猎。
2. 更复杂的赛季大事件。
3. 更重的阵营公共工程。
4. 复杂聊天和公会组织。

## 十、首发版本的数值和情绪原则

### 1. 爽点要前置

玩家每天第一次上线的前 30 秒内，至少要拿到一次明确奖励或完成一次明确进度推进。

### 2. 风险不能压垮成长

掠夺必须伤人，但不能把玩家前一天的努力全部清空，否则复仇会变成弃坑。

### 3. 失败后必须有动作

被打之后至少给玩家三种出路中的一种：

1. 立即复仇。
2. 摇人通缉。
3. 快速补损并继续发育。

### 4. 不让高活跃变成全时蹲守优势

波峰时段可以强化机会，但不要让长期在线变成唯一最优策略，核心收益仍应主要来自掐点而不是挂机盯盘。

### 5. 榜单要多维，不做单一总榜

只有总战力榜会让玩法收缩成一条路，多维榜单才能把不同爽点都留住。

## 十一、首发版本的推荐日常任务包

首发日常任务不要设计成简单签到，而应引导玩家走完核心循环。

推荐任务类型：

1. 收取 1 次成熟资源。
2. 完成 1 次补种。
3. 完成 1 次建筑或训练排队。
4. 发起 1 次单人掠夺。
5. 查看 1 份战报。
6. 完成 1 次复仇或发起 1 次通缉。
7. 领取 1 次待领取收益。

这样任务不是额外负担，而是把玩家自然推回核心循环里。

## 十二、首发版本的成功标准

这套结构是否成立，首发重点观察以下几个指标：

1. 玩家次日留存是否明显受成熟窗口和战报回流驱动。
2. 玩家每日平均上线次数是否高于纯静态经营游戏。
3. 被掠夺玩家中，有多少比例会触发复仇、通缉或补损行为。
4. 有多少活跃玩家至少进入一条榜单竞争。
5. 双人协作内容是否显著提高回流和分享意愿。

## 十三、结论

首发版本最重要的，不是做出一个“大而全”的策略战争游戏，而是做出一条清晰、重复、会让人惦记下一次上线的循环。

这条循环的核心不应该只是“种菜和打架”，而应该是：

1. 种菜提供积累。
2. 成熟窗口提供时机。
3. 掠夺和复仇提供情绪峰值。
4. 主城成长和榜单提供长期目标。
5. 协作和身份提供故事与传播。

只要这五层连起来，首发版本就已经具备“能上头”的基础。
