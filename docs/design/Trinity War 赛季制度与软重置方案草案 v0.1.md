# Trinity War 赛季制度与软重置方案草案 v0.1

## 一、文档目标

这份文档只解决五个问题：

1. 首发版本是否需要赛季制。
2. 赛季周期应该多长。
3. 赛季结束后哪些内容要重置，哪些内容要保留。
4. 赛季奖励如何与单一点券制度结合。
5. 如何限制点券对掠夺与经济系统的破坏。

这份草案的核心原则只有一句话：

赛季要负责重置竞争秩序，点券只负责提供有限便利，不能成为直接放大战斗与资源吞吐的万能货币。

## 二、为什么需要赛季制

这类阵营经营策略战争游戏如果长期不重置，通常会出现四个问题：

1. 老玩家长期滚雪球，新玩家很难追上。
2. 建筑、兵力、资源和榜单会逐步固化。
3. 匹配池越来越难保持同层对抗。
4. 经济系统会越来越依赖不断抬高数值上限。

因此，首发版本建议明确采用赛季制。

但赛季制不应做成“全员清档归零”的硬重置，而应采用软重置。

软重置的目标是同时做到三件事：

1. 让竞争秩序重新洗牌。
2. 让老玩家觉得自己没有白打。
3. 让新玩家有进入窗口。

## 三、赛季周期建议

结合你当前这版短周期成熟、短周期回流、频繁掠夺的设计，首发建议优先采用 28 天赛季，并按自然周推进。

原因如下：

1. 28 天足够让玩家经历开荒、拉开差距、争榜和结算。
2. 28 天不会长到让数值膨胀失控。
3. 28 天天然对应 4 个完整自然周，方便做周目标、周活动和周结算预热。

建议按下面四个自然周推进：

1. 第 1 周：开荒与建立基础盘。
2. 第 2 周：外场、掠夺、分红差距开始拉开。
3. 第 3 周：榜单竞争、通缉、复仇和阵营对抗进入高峰。
4. 第 4 周：赛季冲刺、结算和奖励预热。

如果后续实测发现玩家成长过慢，可以再考虑延长到自然月，但首发建议先从 28 天自然周版本起步。

## 四、赛季内新玩家追赶机制

如果一号就有老玩家入场，而新玩家在二十号才进服，即使不被高段位掠夺，也会面临两个问题：

1. 自己追不上当前赛季的核心进度。
2. 会觉得这赛季已经晚了，不如等下赛季再进。

因此，首发不能只做“保护不挨打”，还必须做“允许追赶”。

建议首发直接采用下面三类追赶机制：

### 1. 开服天数追赶增益

对赛季中后段入场玩家，系统应根据赛季已进行天数提供追赶 buff。

这类 buff 建议明确定位为成长追赶 buff，而不是直接战斗 buff。

建议作用范围：

1. 主城前几级升级时间缩短。
2. 前几档外场位解锁成本降低。
3. 新手阶段训练与恢复速度适度提高。
4. 新手前几天的分红保底略微抬高。

这类增益应只覆盖追赶基础盘，不应直接给高阶战力。

不建议在首发直接提供攻击、防守、掠夺成功率之类的正面对抗 buff。

### 2. 赛季阶段补给包

对赛季中途入场玩家，系统可按赛季周次发一份一次性的阶段补给。

建议内容：

1. 少量基础金币。
2. 少量基础兵力恢复资源。
3. 一次外场启动补贴。
4. 少量侦察或刷新资源。

补给的目标是让玩家更快进入主循环，而不是直接抬进头部竞争层。

### 3. 晚入场专属任务链

建议给中后期入场玩家一条短链追赶任务，用来快速补齐核心功能。

建议任务方向：

1. 完成第一次外场培育。
2. 完成第一次成熟收取。
3. 完成第一次造兵。
4. 完成第一次匿名掠夺。
5. 完成第一次金币上缴。

任务奖励要偏向功能补齐，而不是直接发大量赛季积分。

首发建议明确一条追赶底线：

新玩家可以更快补齐基础盘，但不能一步拿到老玩家已经承担过风险和时间成本换来的高阶位置。

更具体的每日任务承接方式，可见独立文档 Trinity War 玩家每日任务规则草案 v0.1.md。

### 4. 追赶机制的反滥用边界

首发建议明确一条原则：

追赶机制是帮助真晚入场玩家补基础盘，不是给老玩家开小号薅赛季奖励。

建议规则：

1. 每个账号每赛季只触发一次完整追赶资格。
2. 追赶 buff 与追赶任务只在前几天或前几档核心功能补齐前生效。
3. 晚入场玩家默认都能正常参与赛季内容，但完整赛季榜单奖励应满足最低结算门槛。
4. 异常多账号追赶、异常速通追赶任务的账号进入风控池。

更具体的反滥用边界，可见独立文档 Trinity War 晚入场追赶反滥用规则草案 v0.1.md。

## 五、软重置原则

软重置不是平均清空一切，而是分层处理。

### 1. 每赛季必须重置的内容

这些内容直接关系竞争秩序，建议赛季结束后清空或重建：

1. 赛季排行榜。
2. 赛季积分与赛季荣誉进度。
3. 赛季通缉、复仇和活动状态。
4. 当赛季的短期 buff、短期事件和临时法则。
5. 赛季限定任务与赛季限定商店进度。

### 2. 每赛季保留但压缩的内容

这些内容代表玩家成长积累，但如果原样继承，会继续拉大差距。

建议保留主体、压缩边际：

1. 主城与主要建筑保留基础阶段，但高段位增益进行压缩。
2. 外场位解锁数量保留，但高阶效率加成做回收或重置。
3. 基础人口和基础兵力保留一部分，赛季战力重建一部分。
4. 赛季末剩余资源不按原数继承，而按比例折算。
5. 阵营贡献保留历史荣誉，不完整保留当期竞争进度。
6. 金库上限不做完全硬重置，也不做完全继承，而是拆成永久基础值与赛季扩张值两部分处理。

### 2.1 金库上限的赛季处理建议

金库上限在玩家体感里，不只是一个仓储数值，而是经营能力和成长成果的证明。

如果赛季结束后直接把金库上限打回初始值，玩家会比失去部分兵力和资源更容易产生挫败感。

因此，首发不建议把金库上限做成纯赛季数值，也不建议让它完整跨赛季继承。

更稳的做法是：

金库上限 = 永久基础上限 + 本赛季扩张上限

建议规则如下：

1. 永久基础上限负责承接长期投入感。
2. 本赛季扩张上限负责当赛季的成长竞争。
3. 赛季结束后，本赛季扩张上限大部分回收。
4. 其中一小部分按压缩比例转入永久基础上限。
5. 赛季冲到的高档位，再额外折算成结算奖励。

这样做可以同时解决三个问题：

1. 玩家不会觉得一个月白扩仓。
2. 新赛季不会直接延续无限滚雪球。
3. 金库成长仍然保留长期目标感。

### 2.2 首发推荐的金库保留方式

如果首发要做最小可用版本，建议直接采用下面这套：

1. 每个玩家都有一个永久基础金库上限。
2. 赛季内通过升级获得的额外金库上限，计入赛季扩张部分。
3. 赛季结束时，赛季扩张部分只保留少量比例折算到永久基础值。
4. 其余未继承的部分，不直接清空为零价值，而是折算成赛季结算奖励。
5. 永久基础金库上限应设置长期硬上限，避免多赛季持续膨胀。

建议体验目标是：

1. 下赛季开局不会回到最初起点。
2. 但也绝不会带着上赛季完整的高仓储碾压开局。

### 2.3 金库赛季补偿建议

金库上限的赛季补偿更适合偏“遗产感”，而不是偏“直接强度”。

建议补偿方向：

1. 少量永久基础仓储加成。
2. 一次性赛季结算点券。
3. 开局补给包。
4. 纪念型称号、外观或成就记录。

不建议把金库成长补偿直接做成下赛季大量现成金币或高额战备优势。

原因如下：

1. 金库成长应该留下长期价值。
2. 但这种价值不应直接转化为下赛季初期压制力。
3. 否则赛季重开的意义会明显下降。

### 3. 长期永久保留的内容

这些内容负责承接玩家身份与投入感，建议长期保留：

1. 账号身份。
2. 赛季历史战绩。
3. 赛季称号、头像框、皮肤与荣誉展示。
4. 生涯成就与阵营履历。
5. 少量长期收藏型解锁内容。

## 六、首发推荐的软重置方案

如果首发要做最小可用版本，建议直接采用下面这套组合：

1. 排行榜、赛季积分、赛季任务全部重置。
2. 主城和主要功能建筑不清零，但高阶成长加成回收一部分。
3. 外场位数量保留，外场赛季增益清空。
4. 兵力不完整保留，按比例折算为基础可恢复兵力。
5. 战备槽金币和可掠夺暴露资产不完整继承。
6. 金库金币只允许按比例结转，不允许原量带入下赛季。
7. 金库上限采用“永久基础值 + 赛季扩张值”的拆分方式处理，不做完全硬重置。
8. 赛季外观、称号和历史档案永久保留。

这套方案的优点是：

1. 赛季结束后不会全员白干。
2. 下赛季不会直接延续无限滚雪球。
3. 老玩家仍有身份感和开局轻优势。
4. 新玩家仍有追赶空间。
5. 金库这类经营核心成长不会因为硬重置而产生过强挫败感。

## 七、赛季奖励与单一点券制度

### 1. 单一点券制度可以做

你希望只保留一种券，这个方向可以成立，而且对首发理解成本更低。

建议定义为：

点券是游戏内唯一的高级便利货币，可通过充值和赛季结算奖励获得。

这种设计的好处是：

1. 玩家理解成本低。
2. 赛季奖励感知强。
3. 商业化入口集中。

但前提是必须先写死一句底线：

点券是便利货币，不是战斗胜率货币，更不是无限资源吞吐货币。

### 2. 赛季奖励发点券的价值

赛季结算发点券是可行的，因为它能同时强化三件事：

1. 让赛季排名有更强的实际价值。
2. 让高排名玩家感到“这个月没白打”。
3. 让赛季奖励不仅停留在称号和头像框层面。

但它也有两个明显风险：

1. 高排名玩家下赛季会有更多便利资源，进一步拉开差距。
2. 充值玩家和高排名玩家的优势会叠加到同一货币上。

因此，赛季奖励点券的规模必须可控，用途也必须可控。

## 八、点券用途边界

### 1. 建议允许的用途

点券适合购买下面这些低破坏性的便利项：

1. 修改昵称。
2. 切换阵营，但必须有赛季期限制和冷却。
3. 额外保存编队位。
4. 额外刷新掠夺列表次数，但必须有限次。
5. 额外侦察次数，但只提升信息精度，不给精确底牌。
6. 战报皮肤、外观、头像框、称号特效等展示型内容。
7. 少量额外掠夺次数，但必须严格设置每日上限。

### 2. 谨慎允许的用途

下面这些可以做，但必须限次、限量、限边界：

1. 刷新掠夺列表。
2. 增加当日掠夺次数。
3. 轻量跳过非核心等待时间。

如果要开放，建议同时加三条限制：

1. 每日有明确上限。
2. 单次成本递增。
3. 不保证带来更高收益结果。

### 3. 不建议开放的用途

首发不建议点券直接作用于下面这些地方：

1. 直接加速外场成熟到破坏成熟窗口风险。
2. 直接购买高收益目标。
3. 无限刷新直到刷到最优肥羊。
4. 直接查看精确兵力和精确金币。
5. 直接增加战斗胜率或无视匹配保护线。

这里尤其要明确一条：

成长收获加速会直接碰你最核心的成熟窗口博弈，因此首发不建议开放。

## 九、点券与掠夺成本的关系

你提到“要限制用户掠夺的成本”，这个方向是对的。

首发推荐的理解不是“卖无限掠夺”，而是“允许适度补足尝试次数，但不能让掠夺成本失效”。

建议规则如下：

1. 每日基础掠夺次数免费且有限。
2. 点券可以购买额外次数，但每日有硬上限。
3. 额外次数的价格应递增，避免无限堆量。
4. 额外次数不能完全覆盖战损、体力、侦察等其他成本。
5. 高强度玩家和付费玩家能打得更多，但不能无限打。

这样做的结果是：

1. 点券能提升活跃空间。
2. 掠夺仍然有成本感。
3. 经济系统不会因为无限掠夺而快速崩盘。

## 十、赛季点券奖励的发放建议

首发建议不要把赛季点券奖励做得过重。

推荐分成三层：

1. 参与奖励：保证大部分活跃玩家都有基础点券回报。
2. 分段奖励：按榜单分段给差异化点券。
3. 头部奖励：榜单前列额外给点券加荣誉外观，但点券差距不要夸张。

核心原则：

1. 要让冲榜有意义。
2. 不能让头部因为点券奖励过大，下一赛季直接形成滚雪球碾压。
3. 荣誉型奖励与展示型奖励应占更大比重，点券只作为加成项。

## 十一、首发推荐的赛季与点券组合

如果首发要做最小可用版本，建议直接采用下面这套：

1. 赛季周期为 28 天，并按自然周推进。
2. 采用软重置，不做全清档。
3. 排行榜、赛季任务和赛季积分重置。
4. 建筑与长期身份保留，暴露资产与赛季加成压缩。
5. 保留单一点券制度，点券来源为充值加赛季奖励。
6. 点券只卖便利，不卖精确情报、不卖无限刷新、不卖收成硬加速。
7. 点券可有限增加掠夺尝试次数，但不能让掠夺成本失效。

只要先守住这七条，赛季、软重置和商业化就能同时满足三件事：

1. 控制数值膨胀。
2. 保住玩家投入感。
3. 避免点券把核心经济循环做坏。

## 十二、后续可继续细化的部分

后续如果继续展开，建议优先细化下面三项：

1. 赛季结算时各类资产的压缩比例。
2. 榜单分段与点券奖励梯度。
3. 点券商品表与每日限购规则。

更完整的首发商业化结构、月卡、VIP 与战令方案，可见独立文档 Trinity War 首发商业化与付费体系草案 v0.1.md。
