# 2026-05-18 完整游戏设计文档 v0.1

## 一、项目定义

Trinity War 是一款以天堂、恶魔、人类三阵营为核心的异步阵营经营策略游戏。

玩家不需要长时间在线操作，也不依赖即时战斗动画。游戏强调的是：

1. 离线期间持续生产、成长、积累风险。
2. 上线期间集中做经营、调度、掠夺、防守、上缴、决策。
3. 玩家个人行为会反馈到阵营强弱、区域归属、资源流向和世界格局。
4. 游戏主要体现策略和系统博弈，而不是战斗演出。

一句话定位：

这是一个以异步资源争夺为入口、以阵营经营和策略战争为核心的三阵营游戏。

## 二、核心设计目标

这份完整设计文档重点解决六个问题：

1. 玩家每天上线必须有事可做，而且这些事具有取舍。
2. 资源、人口、部队、掠夺、上缴和赛季要形成一个闭环。
3. 玩家对本阵营的贡献必须被持续记录，并真实影响世界格局。
4. 战斗表现以结果为主，不浪费制作成本在演出上。
5. 匿名掠夺、实名复仇、阵营对抗之间要兼顾公平与乐趣。
6. 氪金玩家和高活跃玩家可以更强，但不能破坏赛季公平和生态平衡。

## 三、产品核心卖点

### 1. 离线也运转，上线做关键决策

游戏不是实时在线拼操作，而是把价值集中在玩家上线后的判断上。

### 2. 个人经营会影响阵营与世界

玩家不是在玩一个孤立基地，而是在向阵营经济、战争和地区控制持续输血。

### 3. 掠夺不是纯骚扰，而是经济战争

掠夺会影响资源流向、部队损耗、排行榜走势和阵营强弱。

### 4. 战斗轻演出，重结果、重策略

战斗只需要清晰结算和高信息密度战报，不需要重动画。

### 5. 赛季制保证长期目标与新陈代谢

每个赛季都有竞争目标、结算奖励和一定程度的重置，让世界持续可玩。

## 四、核心循环

这款游戏的最小闭环应该是：

1. 领地产出资源。
2. 资源支撑人口、建筑、科技与部队。
3. 玩家调配人口和资源，生产不同配比的军队。
4. 部队执行掠夺、驻守、复仇、占点任务。
5. 战斗结果直接影响资源存量、部队存量、排行榜积分和阵营贡献。
6. 玩家将资源与战利品上缴阵营，换取贡献度和每日阵营分红。
7. 阵营整体强弱变化继续影响世界控制权、资源加成与赛季格局。

这条循环里，玩家每次上线都至少会处理以下五类事情中的三类：

1. 领取收益。
2. 处理损失。
3. 调整人口与产线。
4. 训练和编组部队。
5. 上缴资源、参与阵营竞争、争取更高分红。

## 五、玩家每日流程

### 1. 上线后的标准流程

建议玩家每次上线都经历这套顺序：

1. 查看领主简报。
2. 领取离线资源、分红和已完成研究。
3. 查看人口状态、仓库状态、部队状态和被掠夺战报。
4. 决定本轮资源优先投向造兵、发展、防守还是上缴。
5. 处理匿名掠夺、复仇或阵营任务。
6. 重新设置离线生产与防守策略。

### 2. 领主简报信息结构

领主简报建议固定展示以下内容：

1. 离线收益汇总。
2. 今日阵营分红结算。
3. 人口变化与异常事件。
4. 昨日被掠夺或掠夺战报摘要。
5. 当前阵营贡献排名变化。
6. 当前赛季目标推进。
7. 当前世界热点和阵营冲突区域。

这样玩家一上线就会明确知道：

1. 我拿到了什么。
2. 我失去了什么。
3. 我现在最该做什么。

### 3. 长时间在线玩家的应对机制

既然这是异步掠夺，就不能让长期在线玩家在被攻击时只能眼睁睁看着资源减少。

但同时也不能把系统做成实时手操防守，否则会破坏异步玩法的根基。

建议采用“在线应对窗口 + 离线托管策略”的双层方案。

当玩家在线且领地被攻击时，系统进入一次短暂应对窗口，建议为 60 到 120 秒。在这段时间内，防守方不能实时拖兵打仗，但可以做一次策略响应。

在线玩家可执行的应对动作建议只有少数几种：

1. 紧急封仓：立即把一部分暴露资源转入保护仓，但会消耗金币或政令点。
2. 民兵动员：临时提高守军强度，但会提升人口压力或次日生产惩罚。
3. 紧急撤离：保护部分高价值兵种，代价是放弃一部分资源。
4. 请求阵营支援：消耗一次求援次数，短时间内获得阵营驻军加成。
5. 启用预设防线：在多套防守模板中切换当前生效方案。

如果玩家不在线，则自动执行离线托管策略，例如：

1. 资源优先保护。
2. 守军优先存活。
3. 反击优先。
4. 降低损失优先。

这样做的价值是：

1. 在线玩家不会变成纯挨打。
2. 离线玩家也不会完全失去防守能力。
3. 游戏仍然保持异步，而不是转成实时攻防。

## 六、阵营上缴与分红系统

这是你这次新增内容里最值得单独做成核心系统的一块。

### 1. 设计目标

上缴系统必须同时实现四个目的：

1. 让玩家持续为阵营输血。
2. 让贡献行为成为每天稳定获得资源的另一种方式。
3. 让玩家清楚看到“我的贡献改变了世界”。
4. 避免所有玩家只顾自己发育，不管阵营死活。

### 2. 上缴内容

玩家可上缴的内容建议包括：

1. 基础资源：粮食、建材、金币。
2. 阵营资源：例如天堂的圣印、恶魔的血契、人类的商券。
3. 特殊材料：赛季材料、区域材料、奇观修建材料。
4. 部队支援：将部分部队投入阵营前线或公共驻军池。

后续如果采用你现在更明确的人口定义，还应允许上缴人口。

建议开放两种人口上缴方式：

1. 上缴农民：投入阵营生产和后勤池，提高阵营每日基础收益。
2. 上缴部队：投入阵营前线军备池，提高阵营大战役的出战规模与恢复速度。

### 3. 贡献度获取规则

玩家通过上缴行为获得贡献度，贡献度建议按以下维度计算：

1. 上缴资源价值。
2. 上缴资源的稀缺度。
3. 当前阵营最急需资源权重。
4. 上缴部队的战力与兵种价值。
5. 在阵营危机期间的额外加成。

贡献度不应只是排行榜数字，而应该成为一个长期积累身份值。

### 4. 阵营分红规则

阵营每天固定结算一次分红，建议在每日零点或固定服务器时间发放。

分红来源建议由四部分组成：

1. 阵营税收池。
2. 阵营控制区域收益池。
3. 阵营公共工程收益池。
4. 阵营战争胜利收益池。

玩家每日获得的分红由以下要素共同决定：

1. 当日个人贡献度。
2. 近七日累计贡献度。
3. 阵营职位或临时职责加成。
4. 阵营当前整体强弱状态。

### 5. 分红结构建议

为了兼顾公平和激励，分红建议拆成两部分：

1. 基础分红：所有活跃玩家都能拿到，保证阵营归属感。
2. 贡献分红：按贡献度和贡献排名加权分配，体现多劳多得。

这样既能鼓励上缴，也不会让低贡献玩家完全失去参与感。

### 6. 贡献如何影响世界格局

贡献系统必须真实改变世界，建议直接作用于以下项目：

1. 阵营边境防线等级。
2. 公共仓库容量与保护率。
3. 前线驻军恢复速度。
4. 区域奇观修建进度。
5. 阵营战争中的全员增益。
6. 阵营在赛季地图上的扩张速度。

玩家要能看到清晰反馈，例如：

1. 天堂本日全员上缴足够，圣壁提升一级，边境被掠夺损失下降 5%。
2. 恶魔连续三日高额上缴军备，深渊军团解锁额外掠夺加成。
3. 人类商会资金充足，跨区贸易收益提升，市场税率降低。

### 7. 阵营仓库与阵营大战役

阵营不应只是一个分红容器，而应是真实存在、可经营、可被攻击的公共体。

建议每个阵营拥有自己的公共仓库、公共人口池和公共军备池。

阵营公共资产来源包括：

1. 玩家上缴资源。
2. 玩家上缴农民。
3. 玩家上缴部队。
4. 控制区域的公共税收。
5. 阵营大战役胜利收益。

这些公共资产的用途包括：

1. 每日阵营分红。
2. 边境修缮。
3. 奇观修建。
4. 阵营大战役军费。
5. 弱势成员救济。

建议每个阵营每天开启一次“阵营大战役”。

阵营大战役的基本结构建议如下：

1. 每日固定时间结算一次。
2. 玩家可在结算前上缴部队、农民、资源。
3. 系统根据上缴总量、兵种配比、阵营科技、区域加成结算大战役结果。
4. 战役胜方夺取区域、公共资源或阵营荣誉。
5. 战役失败方损失一部分公共资源和边境稳定度。

进一步建议开放“阵营仓库可被掠夺”的机制，但必须控制强度。

合理做法不是让阵营仓库随时被个人打空，而是：

1. 只有在世界阵营掠夺或大战役失败后，阵营仓库才会损失部分资源。
2. 阵营仓库损失应有保护上限，避免一次性崩盘。
3. 公共仓库受损会降低次日分红和边境能力，形成真实压力。

## 七、人口系统

人口必须是底层资源，而不是边缘数字。

### 1. 人口作用

这里需要明确人口定义。

本设计中的人口，指的是玩家领地内所有可支配总人口，也就是：

1. 正在生产的农民。
2. 已征召为部队的人口。

换句话说，农民和士兵不是两套完全分开的数值，而是同一总人口池的不同分配方向。

人口直接决定：

1. 资源采集规模。
2. 建筑运转效率。
3. 研究效率。
4. 可编入部队的总规模。
5. 可承载城市规模。

### 2. 人口属性

建议人口至少有六个属性：

1. 总人口。
2. 空闲人口。
3. 农民人口。
4. 部队人口。
5. 忠诚。
6. 压力。

### 3. 人口来源

1. 自然增长。
2. 区域迁入。
3. 难民事件。
4. 战争俘虏转化。
5. 阵营政策加成。

人口应持续自然产出，但速度受住房、粮食、治安和主城等级共同影响。

### 4. 人口消耗

1. 征兵。
2. 饥荒。
3. 疫病。
4. 外流。
5. 暴乱。
6. 献祭或牺牲。

### 5. 人口上限与承载力

每个玩家领地都应有承载力。人口高于承载力后，会产生显著副作用：

1. 粮耗增加。
2. 治安下降。
3. 忠诚下滑。
4. 建筑效率降低。
5. 事件风险提高。

人口上限建议主要由两类东西提升：

1. 主城升级。
2. 住房、农庄、兵营、公共设施等建筑数量与等级。

你提出“上缴人口越多每天产生的阵营收益越好，但是太多又会有副作用”，这个方向是成立的。

建议阵营公共人口池也有自己的承载力：

1. 上缴人口越多，阵营基础产出、后勤效率、大战役征召能力越高。
2. 但如果阵营公共人口池超过承载上限，也会出现粮食紧张、骚乱、瘟疫、效率下降等副作用。
3. 这样玩家与阵营都要面对“人口越多越好，但不能失控”的同一道经营题。

### 6. 人口与氪金、高活跃成长的关系

你提到高活跃和氪金玩家每天可以获得更高人口上限，这可以做，但必须有限度。

建议允许通过以下方式提升人口上限：

1. 主城升级。
2. 住房和公共设施建设。
3. 赛季通行证或月卡提供小幅上限加成。
4. 高活跃奖励提供临时加成。

核心原则：

1. 允许更快成长。
2. 不允许无上限滚雪球。
3. 赛季内增幅可控，赛季间重新洗牌。

## 八、资源与造兵系统

### 1. 基础资源层

你提到资源不要设计太多种类，这个判断是对的。

建议第一版只保留少量高辨识度资源，避免玩家把精力浪费在记账上。

推荐的第一版资源结构如下：

1. 粮食：养人口、维持部队、支撑持续发展。
2. 建材：统一代表木材、矿石、基础工业材料，用于建筑和修复。
3. 金币：通用流通资源，用于训练、交易、加速、上缴。
4. 阵营资源：每个阵营单独一类，例如天堂的圣印、恶魔的血契、人类的商券。

这样第一版总资源数控制在 4 种左右，理解成本低，但仍有阵营差异。

关于“每个阵营要不要额外资源”，建议是：

1. 要，但只保留 1 种阵营专属资源。
2. 这类资源只服务阵营特色玩法、阵营科技和高阶兵种。
3. 不要给每个阵营设计两三种专属资源，否则维护成本和理解成本都会暴涨。

后续赛季玩法如果需要特殊材料，可以作为赛季资源临时加入，而不是常驻基础资源。

### 2. 造兵的基本消耗

每次训练部队都需要同时消耗：

1. 总人口中的可支配人口。
2. 粮食。
3. 金币。
4. 建材或阵营资源中的一种。

这保证了造兵不是单一按钮，而是经营结果的体现。

### 3. 造兵规则

建议部队训练遵循以下规则：

1. 兵种越强，资源消耗越高。
2. 兵种越强，占用人口越多。
3. 高级兵种需要科技前置。
4. 战损后不会自动满编，必须重新补兵。

### 4. 掠夺与部队损耗

每次出征掠夺都应产生真实损耗：

1. 进攻失败会损失更多部队。
2. 进攻成功也会有战斗损耗。
3. 长距离出征或高风险目标会增加额外损耗。

这能防止掠夺成为无成本点击行为。

## 九、兵种体系与配比

你提到部队配比必须影响战局结果，这一条必须成立，而且应该强成立。

### 1. 第一版兵种结构建议

MVP 建议先做四类兵种：

1. 前排兵：吸收伤害，保护后排。
2. 输出兵：主打杀伤。
3. 掠夺兵：提高运载和抢夺效率。
4. 特种兵：提供克制、破坏或指挥增益。

### 2. 兵种配比影响逻辑

战斗结果不只看总战力，而要看配比结构。

例如：

1. 前排不足，后排输出会迅速蒸发。
2. 掠夺兵过多，虽然拿得多，但正面战斗会偏弱。
3. 特种兵过少，面对高防守目标会打不穿。
4. 全输出阵容打脆皮快，但守城很差。

### 3. 推荐的结果计算维度

每次战斗至少读取以下内容：

1. 总兵力。
2. 兵种配比。
3. 指挥加成。
4. 防御建筑。
5. 地区加成。
6. 阵营科技。
7. 当前法令。

这样玩家真正比的是配置和经营，而不是单纯谁数值大。

### 4. 非歼灭式战斗与战损模型

你提到两个玩家竞争时，是否必须每次都把部队全部打光。结论是：不应该。

如果每场战斗都打到全灭，会带来几个明显问题：

1. 进攻成本过高，玩家不敢动。
2. 防守失败惩罚过重，容易退坑。
3. 每场战斗波动过大，缺少中间策略空间。

更合理的做法是采用“战死 + 伤兵 + 溃退 + 被俘”的混合损耗模型。

建议每场战斗结束后，部队分成四类结果：

1. 战死：永久损失，需要重新征兵。
2. 伤兵：进入恢复池，过一段时间可回归。
3. 溃退：保留下来，但短时间内不能再次出征。
4. 被俘：可通过赎回、交换或阵营事件回收。

只有在以下极端情况，才可能接近全灭：

1. 以极弱阵容强攻极强目标。
2. 被完全克制且没有撤退空间。
3. 在阵营大战役或高风险赛季战场中遭遇溃败。

常规掠夺更适合做成“双方都掉一部分兵，但结果有明显差距”的结构。

### 5. 撤退与止损机制

为避免战斗过度极端，建议加入撤退和止损规则：

1. 当主力前排损失达到某阈值时，系统自动判定撤退。
2. 掠夺目标达成后，进攻方倾向优先撤离而不是继续硬拼。
3. 防守方若撑过若干轮次，攻击方收益递减，逼迫其收兵。

这能让战斗更像一次资源争夺，而不是无脑拼到最后一个兵。

### 6. 阵营兵种差异

天堂：

1. 守军更强。
2. 防御建筑加成高。
3. 前排和治疗型辅助更有优势。

恶魔：

1. 掠夺兵与爆发输出强。
2. 掠夺成功收益加成更高。
3. 长线守城能力偏弱。

人类：

1. 兵种均衡。
2. 运输与编组灵活。
3. 更容易根据市场和局势快速换阵容。

## 十、战斗表现方式

你明确说不打算把精力放在动画演示上，这个方向是对的。

### 1. 设计原则

战斗系统应该重：

1. 战前判断。
2. 配比策略。
3. 战后结果。
4. 数据可读性。

战斗系统不应该重：

1. 实时操作。
2. 长时间播放动画。
3. 大量演出资源投入。

### 2. 推荐表现形式

每次掠夺和被掠夺，只输出结果页和战报页。

结果页展示：

1. 胜负。
2. 双方战损。
3. 掠夺资源数量。
4. 额外效果。
5. 榜单积分变化。
6. 是否触发复仇资格。

战报页展示：

1. 双方阵容。
2. 关键加成来源。
3. 克制关系触发。
4. 伤害与损失摘要。
5. 本次进攻成败原因分析。

### 3. 战报价值

战报不是装饰，而是玩家下一次调整策略的依据。

## 十一、掠夺模式设计

你说现在还没完全决定实名、匿名还是世界阵营掠夺，最核心要求是公平和平衡。基于这一点，建议先采用分层方案，而不是一刀切。

### 1. 第一层：匿名日常掠夺

用途：

1. 提供稳定资源获取。
2. 避免点名霸凌。
3. 给日常活跃玩家稳定目标。

规则建议：

1. 按综合战力段匹配目标。
2. 按近期被掠夺次数做保护筛选。
3. 只显示匿名信息，不显示真实昵称。
4. 每日免费次数有限。

### 2. 第二层：实名复仇

用途：

1. 形成私人故事线。
2. 给被打玩家补偿性目标。
3. 让战报有情绪价值。

规则建议：

1. 只有被攻击后才能实名锁定对方。
2. 每日免费复仇次数有限。
3. 复仇成功奖励高于普通掠夺。
4. 复仇失败惩罚轻。

### 3. 第三层：世界阵营掠夺

如果后续要加世界层面的阵营战争，建议不要一开始就把它做成玩家对玩家的常驻替代，而是做成赛季中后期的大型玩法。

建议形式：

1. 某些资源区属于世界争夺区。
2. 各阵营可对这些区域发起集体掠夺或封锁。
3. 结算的是阵营收益，不只是个人收益。
4. 玩家在其中的贡献也计入阵营贡献和赛季荣誉。

建议结论：

1. 前期主打匿名日常掠夺。
2. 中期加入实名复仇。
3. 后期赛季内容加入世界阵营掠夺。

这样公平性和玩法深度更容易平衡。

### 4. 在线状态与掠夺公平性

长期在线玩家不应该因为在线而额外吃亏，也不应该因为在线就完全无敌。

建议公平规则如下：

1. 在线玩家获得短暂应对窗口，但不能实时无限改配置。
2. 离线玩家依赖事先设置的托管方案。
3. 攻击发起后，防守方只能做一次有限响应，避免拖成实时拉扯。
4. 在线防守收益主要体现在“减少损失”，而不是“保证反杀”。

这样既承认在线投入的价值，也不至于让长期挂机玩家成为绝对防御者。

## 十二、公平性与平衡性原则

这是整套设计最不能放松的一部分。

### 1. 核心公平原则

1. 新手不会被连续霸凌。
2. 活跃玩家会更强，但不是无解地更强。
3. 氪金玩家成长更快，但不会买到绝对胜利。
4. 所有强势都必须伴随维持成本。

### 2. 保护机制

建议至少有以下保护：

1. 新手保护期。
2. 每日被成功掠夺次数上限。
3. 同一目标重复攻击冷却。
4. 同阵营默认禁止普通掠夺。
5. 护盾机制。

### 3. 数值平衡原则

1. 总战力不应直接决定胜负。
2. 配比、建筑、防守和科技应显著改变结果。
3. 高战力玩家更容易赢，但打错配比或强冲硬防也应付出代价。

## 十三、赛季制设计

你已经明确要做赛季，这非常适合这类游戏。

### 1. 赛季目的

1. 防止长期滚雪球。
2. 给玩家稳定竞争周期。
3. 给新玩家进入窗口。
4. 给氪金和高活跃玩家展示空间，但限制其无限累积。

### 2. 赛季周期建议

建议一个赛季持续 6 到 10 周。

### 3. 赛季中保留与重置

建议赛季结束后：

1. 清空区域控制权。
2. 重置排行榜。
3. 重置部分赛季资源和赛季建筑。
4. 保留外观、成就和部分长期成长。
5. 保留少量永久科技或账号级基础加成。

### 4. 赛季目标结构

1. 个人目标：发展、战争、防守、贡献。
2. 阵营目标：区域控制、奇观修建、总贡献。
3. 世界目标：击溃灾害、占领关键节点、完成赛季事件。

## 十四、活跃与付费成长设计

你提到每天常驻玩家和氪金玩家可以获得更强战力、部队更强、人口上限更高。这块可以成立，但必须做成“效率优势”，而不是“破坏规则的特权”。

### 1. 高活跃玩家优势

建议高活跃玩家获得：

1. 更高资源利用效率。
2. 更多日常掠夺机会。
3. 更快训练与研究速度。
4. 更高贡献获取效率。

### 2. 氪金玩家优势

建议付费提供：

1. 建设和训练加速。
2. 更方便的调度功能。
3. 小幅人口上限或仓库上限加成。
4. 更多外观、称号和个性化展示。

### 3. 必须禁止的付费设计

1. 直接售卖无上限属性。
2. 直接售卖不可反制的顶级兵种。
3. 直接售卖跳过全部经营逻辑的终局战力。

### 4. 推荐平衡原则

氪金玩家应该是：

1. 更快成型。
2. 更容易保持优势。
3. 但仍然需要正确经营、正确配兵和正确时机。

而不是：

1. 只要付费就能无脑碾压所有人。

## 十五、推荐的第一版玩法范围

为了尽快验证方向，第一版不需要把所有内容一次做满。

### 1. 第一阶段必须有

1. 三阵营基础差异。
2. 资源产出与仓库。
3. 人口与承载力系统。
4. 基础造兵系统。
5. 部队配比结算。
6. 匿名日常掠夺。
7. 被掠夺战报与实名复仇。
8. 上缴贡献与每日分红。
9. 两到三个基础排行榜。
10. 基础赛季目标。

### 2. 第二阶段再加入

1. 世界阵营掠夺。
2. 阵营奇观。
3. 复杂兵种分支。
4. 赛季大型事件。
5. 宿敌与挑战券系统。

## 十六、结论

这款游戏最重要的，不是让玩家“多收几次菜”，而是让玩家每天都面对有取舍的经营和战争问题：

1. 今天的资源，是继续发育，还是上缴阵营换分红。
2. 今天的人口，是投入生产，还是征召成军队。
3. 今天的军队，是拿去掠夺，还是留下防守。
4. 今天的贡献，是换个人收益，还是推动阵营赢下赛季。

如果这些问题每天都成立，而且结果会反馈到个人、阵营和赛季层面，这个系统就会有足够长的生命力。

## 十七、建议的后续深化方向

如果继续细化，建议下一步优先做这四块：

1. 兵种与配比克制表。
2. 上缴贡献与分红公式。
3. 掠夺结算公式与战报字段。
4. 赛季重置规则与长期保留规则。
