# Trinity War 简版游戏设计文档 v0.3

## 一、产品定位

Trinity War 是一款三阵营轻经营、轻对抗游戏。

玩家的核心体验不是长时间在线操作，而是在短时间上线后完成一轮清晰决策：

1. 看自己的田地有没有成熟或丰熟。
2. 决定现在收，还是继续赌更高收益与稀有掉落。
3. 决定把金币拿去扩张、造兵、防守还是上缴阵营。
4. 决定要不要去截获另外两个阵营玩家的成熟农作物。

一句话定位：

这是一款以成熟窗口争夺为核心、以农场经营和轻度异步掠夺为驱动、以阵营分红和贡献提供长期回流的三阵营游戏。

补充：

1. 客户端首发视觉方案采用“新东方极简 + 少量国风轻奢细节”。
2. 统一视觉规范见 docs/design/ui-visual-style-guide.md。
3. 首发不做完整好友系统，只做“关注列表 + 目标观察”。

## 二、首发核心目标

首发版本只验证六件事：

1. 玩家是否愿意为了成熟收取和丰熟窗口反复上线。
2. 玩家是否理解“早收更稳，晚收更赚，但更容易被截获”。
3. 玩家是否愿意为了更快成长，在经营、防守和掠夺之间做分配。
4. 玩家是否会因为阵营分红、贡献、排行榜和关注目标形成长期回流。
5. 玩家是否接受“轻战力损失，但一次失败不会被打废”的对抗强度。
6. 玩家是否会因为签到、每日小事件和稀有掉落保持日常活跃。

## 三、核心循环

首发版本的最小闭环如下：

1. 玩家获得主城税收、阵营分红、签到奖励和每日事件加成。
2. 玩家把金币投入田地，开始一轮农作物培育。
3. 田地经历播种、成长、成熟、丰熟、过熟几个阶段。
4. 玩家上线后决定提前收取、正常收取，还是继续等丰熟和稀有掉落。
5. 玩家把收获折算成金币，投入主城、金币容量、人口上限、防守建筑或造兵。
6. 玩家搜索另外两个阵营的目标，或顺着关注列表继续观察并发起掠夺。
7. 玩家掠夺成功后截获对方的农作物，并有概率获得稀有种子或特殊掉落。
8. 玩家查看战报，领取待领取收益、被掠返还、复仇机会与事件奖励。
9. 玩家用阵营上缴和分红继续扩张，进入下一轮成熟窗口。

这条循环必须稳定传达一个核心关系：

1. 金币保你活下去。
2. 田地决定你赚得快不快。
3. 人口决定你愿不愿意主动出手争成熟资源。
4. 关注、复仇和战报决定你会不会记住具体目标。

## 四、资金与收益系统

首发版本只保留一个核心货币：金币。

问题不再是“有几种货币”，而是“金币当前处于什么状态”。

### 1. 金币的三种状态

#### 1.1 当前金币

规则：

1. 当前金币是玩家日常经营的唯一资金池。
2. 主城建设、造兵、防守升级、田地培育、阵营上缴都直接消耗当前金币。
3. 当前金币有容量上限，容量由主城和金币建筑共同决定。
4. 当前金币不会作为掠夺主目标被直接抢走。

#### 1.2 待领取收益

规则：

1. 主城税收、阵营分红、掠夺战利品、被掠返还、签到奖励和事件奖励都可以进入待领取收益。
2. 待领取收益不会直接参与被掠夺。
3. 玩家需要主动领取，收益才会进入当前金币。
4. 待领取收益最多保留 24 小时，超过上限后暂停累积。
5. 领取时如果金币容量不足，先提示用户确认，再按可承接量入账。

设计目的：

1. 降低被掠即时痛感。
2. 减少“我明明赢了却因为容量浪费”的突兀感。
3. 把首页回访动作做得更明确。

#### 1.3 阵营公库金币

规则：

1. 玩家上缴后的金币进入阵营公库。
2. 阵营公库不被个人日常掠夺直接命中。
3. 阵营公库主要影响阵营小时分红、贡献榜和中后期玩法。
4. 首发阶段只做金币上缴，不做部队上缴。

### 2. 保底金币来源

首发建议只保留四种保底来源：

1. 主城基础税收。
2. 阵营小时分红。
3. 每日签到。
4. 每日小事件奖励。

其中主城基础税收建议明确为：

1. 按主城等级决定的固定小时收入。
2. 先进入待领取收益，玩家领取后再进入当前金币。
3. 只承担保底经营收入，不承担主要成长收入。
4. 如果领取时容量不足，系统先提示确认，不做自动溢出。

设计原则：

1. 保底收入足以继续玩。
2. 保底收入不足以吃满成长线。
3. 真正决定成长速度的，仍然是田地经营、成熟窗口判断和掠夺效率。

### 3. 主城与金币容量升级规则

主城和金币容量是首发阶段最重要的长期成长线。

建议规则如下：

1. 主城升级主要消耗当前金币。
2. 金币建筑升级提高金币容量。
3. 田地位升级提高可同时培育的项目数。
4. 人口升级提高可持有的人口上限。
5. 防守建筑升级降低单次被掠损失，并提升守家稳定性。

核心原则：

1. 金币负责经营和扩张。
2. 田地负责提供高收益窗口。
3. 主城负责解锁更高阶段的经营空间。
4. 防守负责减少被掠损失，而不是让玩家免疫对抗。

## 五、田地培育系统

这是首发版本最核心的经营与对抗结合点。

### 1. 农作物定义

首发版本的外场统一定义为田地，不再抽象成战备物资。

玩家理解上，它就是“种菜收菜”，但保留阵营世界观包装。

机制层面只处理一件事：

投入金币，等待成长，在合适时间收获更高收益。

### 2. 培育阶段

每块田地都按照固定阶段运行：

1. 播种期：刚投入，不能被掠夺，也不能高收益收取。
2. 成长期：可以提前收取，但收益偏低。
3. 成熟期：标准收益阶段，是主要可被掠夺窗口。
4. 丰熟期：超额收益阶段，回报更高，并带有稀有掉落概率。
5. 过熟期：收益开始衰减，防止玩家无限等待。

### 3. 收取规则

首发建议明确五条规则：

1. 玩家可以提前收取，但收益低于成熟收取。
2. 玩家在成熟期收取，可获得标准最佳性价比。
3. 玩家继续等待到丰熟期，可获得更高收益，并触发稀有掉落概率。
4. 玩家收取成功后，金币进入待领取收益，再由玩家主动领取到当前金币。
5. 稀有掉落优先表现为种子、样本或特殊材料，而不是直接给大量金币。

### 4. 明确的视觉反馈

首发必须把成熟窗口做得足够明显。

1. 成熟田有显著高亮和可收取按钮。
2. 丰熟田在视觉上更醒目，并带有稀有掉落提示。
3. 播种和成长阶段只做简洁进度反馈，不堆复杂数值。
4. 玩家一眼就要能看出“哪块田现在最值得处理”。

### 5. 首发推荐时间曲线

为保证回流频率，首发建议用短周期，不做太长链条。

建议第一版直接采用：

1. 播种后 1 小时进入成长期。
2. 再过 2 小时进入成熟期，收益为 100%。
3. 再过 1 小时进入丰熟期，收益为 130%，并带有稀有掉落概率。
4. 再过 2 小时进入过熟期，收益开始逐步衰减。
5. 成长期可提前收，收益约为标准收益的 70%。

这套曲线制造的回访理由是：

1. 我快成熟了，要不要回来看。
2. 我已经成熟了，要不要立刻收。
3. 我如果每 4 小时回来看一次，刚好会撞到丰熟窗口。
4. 我想再赌一小时更高收益和稀有掉落，但可能被抢。

### 6. 外场升级意义

外场系统必须具备长期成长空间，但首发不要做得太重。

首发只需要支持四种成长点：

1. 田地位数量。
2. 单次培育投入上限。
3. 成熟速度加成。
4. 被掠夺保护比例。

说明：

地块个性化首发不做。

## 六、人口与部队系统

首发版本不做复杂兵种，只保留一个基础兵种。

本文统一约定：

1. 人口 = 可用兵力。
2. 战力 = 人口的界面化表达。
3. 在玩法语义上，人口与战力是同一件事。

### 1. 人口结构

1. 人口上限由主城和人口建筑决定。
2. 当前人口同时承担进攻和防守。
3. 玩家只训练一种基础部队，不做多兵种克制。
4. 人口不承担生产职责，只负责轻对抗规模。

### 2. 单兵种规则

1. 训练 1 个单位，需要一笔当前金币。
2. 单位越多，主动进攻越强，守家也越稳。
3. 单位在战斗中会产生轻战力损失。
4. 首发不做伤病池、恢复建筑、伤兵加速等恢复系统。
5. 单次战力损失必须可控，不让玩家一次被打废。

### 3. 首发战斗字段

首发只保留三个核心字段：

1. 人口：决定能打多大规模。
2. 风险：决定这次出手值不值。
3. 损耗：决定这次掠夺带来的代价。

玩家不需要看到完整公式，只要看到：

1. 这个目标风险高不高。
2. 这次大概能拿多少。
3. 这次大概要掉多少人口。

## 七、掠夺、防守与复仇系统

### 1. 掠夺目标

首发版本只允许掠夺另外两个阵营的玩家。

系统展示的目标信息建议只有：

1. 匿名代号或简化昵称。
2. 阵营。
3. 主城阶段或发展段位。
4. 田地成熟度预估。
5. 推荐风险标签。
6. 预计农作物收益区间。
7. 防守倾向标签。
8. 是否已关注。

这里应明确一条原则：

首发只展示快速判断所需的标签和区间，不直接展示完整内部公式。

### 2. 掠夺资产边界

首发版本的掠夺目标只限定为两类：

1. 成熟或丰熟阶段的农作物收益。
2. 低概率稀有掉落，例如种子、样本或特殊材料。

首发不作为主要掠夺目标的资产有三类：

1. 当前金币。
2. 阵营公库金币。
3. 已成功领取入账的安全收益。

这样设计的原因是：

1. 玩家丢金币会明显更不爽。
2. 掠夺只截获农作物，更像“成熟窗口争夺”，而不是抄家。
3. 经营玩家的家底更稳，对抗玩家的收益也更聚焦。

### 3. 掠夺结算原则

掠夺必须满足下面五条：

1. 敌人不能看到成熟就自动拿走，必须打赢。
2. 打赢后也不应一次性清空目标全部收益。
3. 被攻击方必须保留继续经营的基础能力。
4. 战力损失必须真实存在，但不能重到需要伤病恢复系统。
5. 掠夺收益先进入待领取收益，不直接变成当前金币。

建议首发用下面这条结算顺序：

1. 校验目标是否处于可掠夺阶段。
2. 读取防守方当前防守配置和保护状态。
3. 黑盒结算攻防胜负和双方损耗。
4. 根据目标成熟度和保护比例计算实际可截获农作物。
5. 结算攻击方战利品，写入攻击方待领取收益。
6. 结算防守方保留收益、返还收益或安抚补贴，写入防守方待领取收益。
7. 写入双方战报，更新复仇资格与保护状态。

### 4. 被掠后的体验底线

首发必须守住下面四条：

1. 被掠后不会直接丢失当前金币。
2. 被掠的是田地收益，而不是家底。
3. 被掠后的保留部分和返还部分进入待领取收益，减少即时挫败感。
4. 玩家上线后能快速看到“我损失了什么，我还保住了什么，我能不能复仇”。

### 5. 防守的意义

防守系统在首发版本只解决一件事：

让玩家愿意投入资源减少自己在成熟窗口里的损失。

首发只需要三种防守收益：

1. 防守建筑提高成熟收益保留率。
2. 保护状态降低连续被掠的概率。
3. 驻守人口提高敌方损耗并压低对方收益。

### 6. 保护状态

首发版本建议明确为：

1. 玩家在被成功掠夺后，获得一段保护状态。
2. 保护状态持续期间，不再进入普通目标池。
3. 保护状态只解决连续受击问题，不覆盖赛季玩法和特殊活动。
4. 保护状态要在首页和目标详情中清晰展示。

### 7. 复仇规则

复仇保留，但只作为轻爽点，不作为主循环。

首发建议只在两种情况下开放实名复仇：

1. 对方最近 24 小时内成功掠夺过你。
2. 你还有当日免费复仇次数。

复仇的具体规则建议写死为：

1. 每天提供有限免费复仇次数。
2. 复仇收益略高于普通掠夺。
3. 复仇成功后目标进入保护状态。
4. 复仇不会再触发新的复仇链条。
5. 复仇主要提供情绪出口，而不是反复抄家入口。

## 八、关注与观察系统

首发不做完整好友系统，只做关注列表。

### 1. 关注功能的作用

1. 玩家可以把值得盯的目标加入关注列表。
2. 玩家可以顺着关注列表快速找到并观察这些目标。
3. 玩家可以记住“这个人常种高收益作物”“这个人防守偏弱”“这个人经常进入丰熟期”。
4. 关注只用于观察和快速回访，不要求双方确认。

### 2. 关注系统的价值

1. 让目标池不只是一次性卡片。
2. 让玩家形成“我知道谁家快熟了”的记忆点。
3. 在不做好友系统的情况下，保留轻社交感和轻追踪感。

## 九、阵营系统

首发版本的阵营系统只保留三件事：

1. 阵营身份。
2. 阵营小时分红。
3. 金币上缴换贡献。

### 1. 阵营小时分红

建议每小时结算一次。

入账规则建议写死为：

1. 每小时结算出的分红先进入待领取收益。
2. 玩家上线后看到提示，再主动领取进入当前金币。
3. 待领取收益最多保留 24 小时。
4. 如果金币容量不足，领取时先提示用户确认。

设计目的：

1. 提高用户回流频率。
2. 让阵营从背景设定变成实时收益来源。
3. 让不爱打架的玩家也愿意保持活跃。

建议结构：

1. 阵营基础分红。
2. 个人贡献加成分红。

### 2. 贡献系统

首发只保留一种贡献行为：

上缴金币。

首发明确去掉部队捐献。

原因如下：

1. 单一人口已经承担进攻、防守、战损和造兵成本。
2. 再要求捐兵，会明显放大重度感。
3. 金币上缴已经足够形成阵营贡献与分红循环。

贡献的作用：

1. 提高个人小时分红。
2. 提高贡献榜排名。
3. 强化“我在为阵营打资源”的感受。

## 十、三阵营首发特色

首发版本不做复杂阵营技能树，每个阵营只保留一个主优势、一个主弱点和一个明确定位。

### 1. 仙界

1. 主优势：被掠夺损失更低，防守端更稳。
2. 主弱点：主动掠夺收益偏低。
3. 适合玩家：稳收成熟资源、守家、靠分红滚成长。

### 2. 魔界

1. 主优势：主动掠夺收益更高。
2. 主弱点：战力损耗略高。
3. 适合玩家：主动进攻、快节奏、高风险高回报。

### 3. 人界

1. 主优势：上缴和分红效率更高，经营更灵活。
2. 主弱点：纯进攻和纯防守都不极致。
3. 适合玩家：平衡运营、控成熟节奏、靠调度拉开差距。

### 4. 阵营平衡原则

首发只需要守住下面三条：

1. 仙界最稳，但不该抢得最快。
2. 魔界最会抢，但不该又能抢又能守。
3. 人界最灵活，但不该在正面强度上全面压制另外两方。

首发数值差异优先只动四个字段：

1. 被掠夺损失比例。
2. 掠夺收益加成。
3. 战力损耗系数。
4. 分红或上缴效率。

## 十一、排行榜与日常活跃

### 1. 排行榜系统

首发只保留三张榜：

1. 掠夺榜。
2. 发展榜。
3. 贡献榜。

它们分别对应三种核心玩家目标：

1. 爱打架的想冲掠夺榜。
2. 爱经营的想冲发展榜。
3. 爱阵营和社交感的想冲贡献榜。

### 2. 每日签到

首发建议加入最轻量签到系统：

1. 每日首次登录即可签到。
2. 奖励以金币、种子、保护道具或事件增益为主。
3. 签到不做复杂连续断签惩罚。

### 3. 每日小事件

首发建议加入每日小事件，用来增加当天新鲜感。

事件可以很轻，重点是每天有变化。

示例方向：

1. 今日晴朗，某类作物成熟更快。
2. 今日商队到访，某类作物卖价更高。
3. 今日边境紧张，掠夺收益略高但风险略高。
4. 今日祥瑞降临，丰熟田稀有掉落概率提升。

设计原则：

1. 只改变当天节奏，不改变长期底层规则。
2. 让玩家每天上线时有一点不同。
3. 不把事件做成复杂活动系统。

## 十二、新手前 3 天开放表

### 第 1 天

目标：让玩家理解“成熟收取比放着不管更重要”。

开放内容：

1. 阵营选择。
2. 基础金币来源。
3. 金币容量。
4. 第一个田地培育位。
5. 提前收取与成熟收取演示。
6. 每日签到。
7. 阵营小时分红领取。

### 第 2 天

目标：让玩家理解“高收益田地会带来被截获风险，所以要防守和造兵”。

开放内容：

1. 真实异步目标。
2. 战报。
3. 防守建筑。
4. 保护状态说明。
5. 一次免费复仇机会。
6. 掠夺榜和发展榜。
7. 每日小事件。

### 第 3 天

目标：让玩家理解“我不是单机经营，我在和阵营一起吃分红，也能顺着关注继续盯目标”。

开放内容：

1. 金币上缴。
2. 贡献值。
3. 贡献榜。
4. 分红加成说明。
5. 关注列表。
6. 简单阵营任务。

## 十三、首发必须守住的设计底线

1. 玩家不能被一次掠夺打回原点。
2. 玩家不能只靠保底收入吃满主城成长线。
3. 玩家不能因为兵种太多而看不懂战报和战损。
4. 三阵营必须有差异，但差异不能复杂到影响新手理解。
5. 玩家被掠后不能直接因为丢金币而产生强烈退游情绪。
6. 每次上线都要能看到“收成熟、领收益、看事件、抢资源、拿分红”其中至少两件事。
7. 关注、复仇和战报必须增强记忆点，但不能把产品拖成重社交或重恩怨。

## 十四、结论

这份简版设计文档的核心不是做一个复杂世界，而是先做成一个成立的轻经营、轻对抗、轻回访循环。

只要下面这条关系成立，首发版本就有验证价值：

1. 金币保底。
2. 田地成熟窗口提供高收益与高风险。
3. 单兵种让对抗和损耗容易理解。
4. 被掠不直接丢金币，显著降低挫败感。
5. 关注、复仇、阵营分红和每日事件共同提供长期回流理由。
