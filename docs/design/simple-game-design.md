# Trinity War 简版游戏设计文档 v0.2

## 一、产品定位

Trinity War 是一款三阵营经营策略战争游戏。

玩家的核心体验不是长时间在线操作，而是在短时间上线后完成一轮明确决策：

1. 看自己的外场物资有没有成熟。
2. 决定现在收，还是继续赌更高收益。
3. 决定把金币拿去扩张、防守还是造兵。
4. 决定要不要去抢另外两个阵营玩家的成熟资源。

一句话定位：

这是一款以异步资源争夺为入口、以阵营经营和策略战争为核心的三阵营游戏。

补充：

1. 客户端首发视觉方案采用“新东方极简 + 少量国风轻奢细节”。
2. 统一视觉规范见 docs/design/ui-visual-style-guide.md。

## 二、首发核心目标

首发版本只验证四件事：

1. 玩家是否愿意为了成熟收取和掠夺窗口反复上线。
2. 玩家是否理解“早收更稳，晚收更赚，但更危险”。
3. 玩家是否会为了更快成长，主动把金币暴露到外场和余额。
4. 玩家是否会因为阵营分红、贡献和排行榜形成长期回流。

## 三、核心循环

首发版本的最小闭环如下：

1. 玩家获得小额保底金币收入和阵营小时分红。
2. 玩家把一部分金币投入外场培育位，开始培育高收益物资。
3. 外场物资经历成长、成熟、丰熟、过熟几个阶段。
4. 玩家上线后决定提前收、按时收，还是继续等更高收益。
5. 玩家把收获折算成金币，投入金库、余额、建筑、防守或造兵。
6. 玩家搜索另外两个阵营的成熟目标并发起掠夺。
7. 玩家用掠夺收益继续扩张、防守、上缴阵营并提高下一轮分红。

这条循环必须稳定传达一个核心关系：

1. 金库保你活下去。
2. 外场决定你赚得快不快。
3. 余额决定你愿不愿意把金币暴露出来承担成熟窗口带来的风险。

## 四、资产与经济系统

首发版本建议只保留一个核心货币：金币。

问题不再是“有几种货币”，而是“金币当前处于什么状态”。

### 1. 金币的四种状态

#### 1.1 金库金币

规则：

1. 玩家已领取入账的金币默认进入金库。
2. 金库金币不可被日常掠夺直接带走。
3. 金库容量有限，玩家必须升级金币上限承接更大收益。
4. 金库金币主要用于升级建筑、开启培育、上缴和主动转入余额。
5. 主城税收和阵营分红先累计为待领取收益，玩家领取后才进入金库。
6. 待领取收益最多保留 24 小时。
7. 当待领取收益达到 24 小时保留上限时，新的税收和分红暂停累积，直到玩家上线处理。

补充说明：

税收和分红在点击领取前，只是待入账记录，不算新的金币资金池，也不能直接拿来消费。

#### 1.2 外场投入金币

规则：

1. 玩家可把金币投入外场培育位，启动一次高收益培育。
2. 投入后，这笔钱不再视为安全金币，而是变成正在成长的外场物资。
3. 外场物资成熟前不能直接算作收益，必须等待收获。
4. 外场物资是首发日常掠夺的主要目标之一。

#### 1.3 余额金币

规则：

1. 玩家训练士兵、补兵和执行战备类操作前，需先把金币从金库转入余额。
2. 余额中的金币属于暴露资产，可被掠夺。
3. 余额有容量上限和保护比例。
4. 玩家可以把余额中的一部分回存金库，但应有冷却或冻结窗口限制，不建议额外收手续费。

这里应明确一条：

余额金币只应由玩家主动从金库转入，不应由税收或分红自动灌入。

#### 1.4 阵营公库金币

规则：

1. 玩家上缴后的金币进入阵营公库。
2. 阵营公库不被个人日常掠夺直接命中。
3. 阵营公库主要影响阵营分红、公共加成和中后期玩法。
4. 阵营公库只在阵营战或赛季玩法中承受损失，不进入首发个人掠夺主循环。

### 2. 保底金币来源

既然首发版本不做农民产出，就必须保留小额保底金币来源，避免玩家连续失误后无法继续经营。

首发建议只保留三种保底来源：

1. 主城基础税收。
2. 阵营小时分红。
3. 日常活跃奖励。

其中主城基础税收建议明确为：

1. 按主城等级决定的固定小时收入。
2. 先进入待领取收益，玩家领取后再进入金库。
3. 不与金库余额挂钩，不按余额计算利息。
4. 只承担保底经营收入，不承担主要成长收入。
5. 待领取收益最多保留 24 小时，超出后暂停累积，而不是自动转入余额。
6. 如果领取时金库空间不足，先提示用户确认。
7. 用户点击取消，则本次不领取，自己先去消费或扩仓。
8. 用户点击确认，则按可承接部分入库，超出金库上限的部分直接损失。

设计原则：

1. 保底收入足以继续玩。
2. 保底收入不足以吃满成长线。
3. 真正决定成长速度的，仍然是外场培育和掠夺效率。

### 3. 主城与金库升级规则

主城和金库是首发阶段最重要的长期成长线。

建议规则如下：

1. 主城升级主要消耗金库金币。
2. 金库升级提高安全资金上限。
3. 外场位升级提高可同时培育的项目数和单次收益上限。
4. 主城中后期升级需要玩家既有足够金库容量，也有足够外场收益效率。
5. 赛季结算时，金库上限不做完全硬重置，而应拆成永久基础值与赛季扩张值，保留少量长期成长并回收大部分赛季扩张。

默认花费顺序建议写死为：

1. 建设、外场投入、上缴等普通操作默认先从金库金币扣除。
2. 训练、补兵、强化等战备类操作必须先从金库主动转入余额，再从余额扣除。
3. 余额中的金币不会被普通建设自动优先消耗。
4. 待领取收益不会被自动用于任何花费，必须先领取入库。

这样设计更合理的原因是：

1. 金库负责安全存放和长期建设。
2. 余额负责主动暴露和战备消耗。
3. 普通建设如果自动优先吃余额，会破坏余额作为“主动暴露资金”的独立语义。

核心原则：

1. 金库负责存稳。
2. 外场负责赚快。
3. 主城负责解锁更高阶段的经营空间。
4. 金库成长要留下长期积累感，但不能跨赛季无限滚雪球。

更完整的金币来源、消耗、结算安全与通胀控制设计，可见独立文档 gold-economy-and-inflation-control-plan.md。

## 五、外场培育系统

这是首发版本最核心的经营与对抗结合点。

### 1. 外场物资定义

首发不必把它设计成“菜”，也不必一开始就做宠物。

建议把它定义成一类统一的外场战备物资。

它可以在美术和世界观上包装为：

1. 战备粮。
2. 魂晶。
3. 军械胚体。
4. 异界孵化物。

但首发机制层面只把它当成一类统一资源项目处理：

投入金币，等待成长，成熟后收获更高金币。

如果按玩家理解来讲，外场本质上就是“种植/培育收益位”。

只是为了避免题材过于局限，文档里统一用“外场物资”来描述，而不限定必须是传统种菜表现。

### 2. 培育阶段

每个外场培育位都按照固定阶段运行：

1. 播种期：刚投入，不能被掠夺，也不能高收益收取。
2. 成长期：可以提前收取，但收益偏低。
3. 成熟期：标准收益阶段，是主要可被掠夺窗口。
4. 丰熟期：超额收益阶段，回报更高，但暴露风险也更高。
5. 过熟期：收益开始衰减，防止玩家无限等待。

### 3. 收取规则

首发建议明确四条规则：

1. 玩家可以提前收取，但收益低于成熟收取。
2. 玩家在成熟期收取，可获得标准最佳性价比。
3. 玩家继续等待到丰熟期，可获得更高收益，但更容易被盯上。
4. 玩家收取成功后，收益直接折算成金币进入金库。

这条规则必须写死：

收取成功后的金币不进入日常掠夺目标。

### 3.1 外场被掠规则

首发不建议把一块外场物资设计成“被打赢后整块直接消失”。

更稳的做法是：

1. 外场当前收益由“当前可收价值”表示。
2. 攻击方打赢后，只能掠走当前可收价值中的一部分。
3. 剩余价值继续留在这块外场上，等待防守方后续收取。
4. 被掠夺不会重置成长阶段，只会降低剩余可收价值。

推荐理解如下：

1. 一块丰熟仙藤当前可收价值为 1000 金币。
2. 本次被掠夺后损失 200 金币。
3. 这块仙藤仍处于丰熟阶段，但剩余可收价值变为 800 金币。

这样设计的目的有三点：

1. 让被攻击方损失真实存在，但不至于一次被打空。
2. 让保护建筑、城防和保护比例有明确价值。
3. 让单块外场在首发阶段也能承受部分风险，而不是只剩全有或全无。

### 4. 首发推荐时间曲线

为保证回流频率，首发建议用短周期，不要做太长链条。

建议第一版直接采用：

1. 播种后 1 小时进入成长期。
2. 再过 2 小时进入成熟期，收益为 100%。
3. 再过 1 小时进入丰熟期，收益为 130%。
4. 再过 2 小时进入过熟期，收益开始逐步衰减。
5. 成长期可提前收，收益约为标准收益的 70%。

这套曲线的目的不是做复杂计算，而是制造明确的回访理由：

1. 我快成熟了，要不要回来看。
2. 我已经成熟了，要不要立刻收。
3. 我如果每 4 小时回来看一次，刚好会撞到丰熟窗口。
4. 我想再赌一小时更高收益，但可能被抢。

这里还应明确一条：

外场成长按玩家自己的播种时间独立推进，不按整点统一刷新。

### 5. 外场升级意义

外场系统必须具备长期成长空间，但首发不要做得太重。

首发只需要支持四种成长点：

1. 外场位数量。
2. 单次投入上限。
3. 成熟速度加成。
4. 被掠夺保护比例。

## 六、人口与部队系统

首发版本不做复杂兵种，只保留总人口和一个基础兵种。

这里的人口不再承担生产职责，只负责军事规模。

### 1. 人口结构

1. 总人口上限由主城和住房决定。
2. 总人口 = 掠夺兵 + 空闲人口。
3. 空闲人口可训练成掠夺兵。
4. 掠夺兵负责进攻和基础防守。

### 2. 单兵种规则

1. 训练 1 个掠夺兵，需要 1 点人口和一笔余额金币。
2. 掠夺兵越多，主动进攻越强，但维护成本也越高。
3. 掠夺兵在战斗中损失后，首发建议拆成“伤兵待恢复”和“永久阵亡”两部分，既保留真实战损，也保留恢复路径。
4. 人口上限不足时，玩家必须优先扩住房，不能无限爆兵。

### 2.1 人口恢复规则

首发建议明确四条恢复规则：

1. 战斗损失中的大部分兵力先进入伤兵池，按时间逐步恢复。
2. 玩家可以花金币执行伤兵恢复，缩短恢复时间。
3. 战斗损失中的一小部分视为永久阵亡，必须重新训练。
4. 系统应保留轻量保底补员或恢复加速，避免连续失败后直接失去继续参与的能力。

### 3. 首发战斗字段

首发只保留三个核心字段：

1. 战力：决定胜负倾向。
2. 运载：决定能带走多少成熟收益或战备金币。
3. 损耗：决定这次掠夺值不值。

## 七、掠夺与防守系统

### 1. 掠夺目标

首发版本只允许掠夺另外两个阵营的玩家。

系统展示的目标信息建议只有：

1. 匿名代号或简化昵称。
2. 阵营。
3. 主城阶段或发展段位。
4. 外场成熟度预估。
5. 余额暴露等级。
5. 余额暴露等级。
6. 推荐风险标签。
7. 防守倾向标签。
8. 预计收益区间与预计胜率区间。

这里应明确一条原则：

首发只展示风险判断所需的标签和区间，不直接展示精确兵力、精确战力、精确金币余额和完整建筑明细。

玩家不需要读大量数据，只要快速判断：

这个目标值不值得打。

### 2. 日常可掠夺资产边界

首发版本建议把可掠夺资产明确限定为三类：

1. 成熟中的外场物资。
2. 余额中的金币。
3. 出征与防守产生的军事损耗。

首发版本不建议作为主要掠夺目标的资产有三类：

1. 金库金币。
2. 已成功收取的收益。
3. 主城升级核心底料。

### 3. 掠夺结算原则

掠夺必须满足下面四条：

1. 敌人不能看到成熟就自动拿走，必须打赢。
2. 打赢后也不应一次性清空目标全部收益。
3. 被攻击方必须保留恢复和继续发育的基础能力。
4. 战损必须真实存在，否则掠夺只剩无脑最优解。

这里的“恢复能力”建议明确包括三件事：

1. 保底金币来源仍然可用。
2. 伤兵池提供可恢复兵力。
3. 玩家不会因为一次失败被完全清空可行动人口。

建议首发用下面这条结算顺序：

1. 校验目标是否处于可掠夺阶段。
2. 读取防守方当前防守配置。
3. 结算攻防胜负和双方战损。
4. 根据攻击方运载值计算最大可掠夺量。
5. 根据防守保护比例计算实际损失。
6. 先结算成熟外场收益，再结算余额金币。
7. 写入双方战报并更新复仇资格。

### 4. 防守的意义

防守系统在首发版本只解决一件事：

让玩家愿意投入资源减少自己在成熟窗口里的损失。

首发只需要三种防守收益：

1. 保护建筑提高成熟收益保留率。
2. 城防降低被掠夺比例。
3. 驻守兵提高敌方战损。

### 5. 复仇规则

实名掠夺不要作为日常主入口，而应绑定明确因果。

首发建议只在两种情况下开放实名复仇：

1. 对方最近 24 小时内成功掠夺过你。
2. 对方当前携带高额战利品并处于通缉状态。

复仇的设计目标是：

1. 给玩家情绪出口。
2. 形成个人恩怨和记忆点。
3. 不让实名机制退化成反复抄家。

## 八、阵营系统

首发版本的阵营系统只保留三件事：

1. 阵营身份。
2. 阵营小时分红。
3. 金币上缴换贡献。

### 1. 阵营小时分红

建议每小时结算一次。

入账规则建议写死为：

1. 每小时结算出的分红先进入待领取收益。
2. 玩家上线后看到提示，再主动领取进入金库。
3. 待领取收益最多保留 24 小时。
4. 如果金库空间不足，领取时先提示用户确认。
5. 用户取消则不领取，用户确认则按可承接部分入库，并损失超出部分。

设计目的：

1. 提高用户回流频率。
2. 让阵营从背景设定变成实时收益来源。
3. 让不爱打架的玩家也愿意保持活跃。

建议结构：

1. 阵营基础分红。
2. 个人贡献加成分红。

### 2. 贡献系统

首发只保留一种贡献行为：

上缴金币。

首发不建议把“日常部队上缴”直接纳入主循环。

原因如下：

1. 玩家会明显产生不舍情绪。
2. 部队已经承担进攻、防守和战损成本，再要求日常上缴，容易形成双重惩罚。
3. 这会显著抬高首发理解成本。

更稳的做法是：

1. 首发日常贡献只做金币上缴。
2. 如果后续做每周三界争霸，再开放“周度军备支援”作为第二阶段玩法。
3. 这类军备支援只影响周战公共军备或周积分，不进入日常必做循环。

贡献的作用：

1. 提高个人小时分红。
2. 提高贡献榜排名。
3. 强化“我在为阵营打资源”的感受。

## 九、三阵营首发特色

首发版本不做复杂阵营技能树，每个阵营只保留一个主优势、一个主弱点和一个明确定位。

### 1. 仙界

1. 主优势：被掠夺损失更低，防守端更稳。
2. 主弱点：主动掠夺收益偏低。
3. 适合玩家：稳收成熟资源、守家、靠分红滚成长。

### 2. 魔界

1. 主优势：掠夺收益最高，成熟资源争夺更强。
2. 主弱点：战损更高，守家更脆。
3. 适合玩家：主动进攻、快节奏、高风险高回报。

### 3. 人界

1. 主优势：上缴和分红效率更高，经济调度更灵活。
2. 主弱点：纯进攻和纯防守都不极致。
3. 适合玩家：平衡运营、控成熟节奏、靠调度拉开差距。

### 4. 阵营平衡原则

首发只需要守住下面三条：

1. 仙界最稳，但不该抢得最快。
2. 魔界最会抢，但不该又能抢又能守。
3. 人界最灵活，但不该在正面强度上全面压制另外两方。

首发数值差异优先只动四个字段：

1. 被掠夺损失比例。
2. 掠夺收益加成。
3. 掠夺战损系数。
4. 分红或上缴效率。

更完整的三阵营定位、福利差异与平衡原则，可见独立文档 faction-differentiation-and-balance-principles.md。

## 十、排行榜系统

首发只保留三张榜：

1. 掠夺榜。
2. 发展榜。
3. 贡献榜。

它们分别对应三种核心玩家目标：

1. 爱打架的想冲掠夺榜。
2. 爱经营的想冲发展榜。
3. 爱阵营和社交的想冲贡献榜。

## 十一、新手前 3 天开放表

### 第 1 天

目标：让玩家理解“成熟收取比放着不管更重要”。

开放内容：

1. 阵营选择。
2. 基础金币来源。
3. 金库。
4. 第一个外场培育位。
5. 提前收取与成熟收取演示。
6. 阵营小时分红领取。

### 第 2 天

目标：让玩家理解“高收益外场会带来被抢风险，所以要防守和造兵”。

开放内容：

1. 真实异步目标。
2. 战报。
3. 余额。
4. 保护建筑。
5. 城防。
6. 一次免费复仇机会。
7. 掠夺榜和发展榜。

### 第 3 天

目标：让玩家理解“我不是单机经营，我在和阵营一起吃分红”。

开放内容：

1. 金币上缴。
2. 贡献值。
3. 贡献榜。
4. 分红加成说明。
5. 简单阵营任务。

## 十二、首发必须守住的设计底线

1. 玩家不能被一次掠夺打回原点。
2. 玩家不能只靠保底收入吃满主城成长线。
3. 玩家不能因为兵种太多而看不懂战报和战损。
4. 三阵营必须有差异，但差异不能复杂到影响新手理解。
5. 每次上线都要能看到“收成熟、分金币、抢资源、拿分红”其中至少两件事。

## 十三、结论

这份简版设计文档的核心不是做一个复杂世界，而是先做成一个成立的异步成熟争夺循环。

只要下面这条关系成立，首发版本就有验证价值：

1. 金库保底。
2. 外场成熟窗口提供高收益与高风险。
3. 单兵种让战斗和损耗容易理解。
4. 阵营分红和贡献把个人行为接进长期循环。
