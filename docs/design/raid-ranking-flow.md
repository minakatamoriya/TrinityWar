# Trinity War 掠夺与排名系统运行设计 v0.1

## 一、文档目标

这份文档专门整理你新增的这批想法，重点解决四个问题：

1. 游戏核心循环如何围绕资源、部队、掠夺、反掠夺运转。
2. 排名系统如何提供长期目标，又不把普通玩家直接压死。
3. 掠夺系统如何同时满足公平、恩怨感、成长感。
4. 整套系统在玩家每天上线时，具体应该怎样流动起来。

这份文档和 [thinking.md](thinking.md) 的关系是：

1. [thinking.md](thinking.md) 负责整体世界观、阵营经营、人口和平衡的大框架。
2. 当前文档专门负责“资源产出 -> 生产部队 -> 发起掠夺 -> 遭受反击 -> 排名反馈”这一条主循环。

## 二、核心判断

### 1. 这套玩法像不像“复杂版偷菜”

从底层结构上看，确实相似。

共同点是：

1. 玩家离线期间持续产出资源。
2. 玩家上线后要收取资源。
3. 玩家可能被其他人掠夺。
4. 玩家也会寻找别人下手。

真正拉开差距的，不是“能不能抢”，而是以下四层深度：

1. 有军队损耗和兵种配置，不是单纯点一下就抢走。
2. 有阵营结构，掠夺不是个人骚扰，而是阵营竞争的一部分。
3. 有长期建设成本，抢到资源不一定立刻变强，还要考虑守得住。
4. 有复仇、排行榜、赛季目标和世界反馈，每次掠夺都会留下后续影响。

所以更准确的定位是：

这是一套以“偷菜式异步资源争夺”为底层入口、以“阵营经营和策略战争”为上层深度的游戏循环。

## 三、核心主循环

这套系统的主循环可以概括为：

1. 领地产出资源。
2. 资源投入建筑、科技、人口和军队。
3. 军队用于掠夺、复仇、驻守、争点。
4. 掠夺结果反过来影响个人发展、阵营局势和排行榜名次。
5. 排名与战报继续刺激玩家下一轮上线。

如果再拆成玩家视角的日常动作，就是下面这条闭环：

1. 上线领取离线收益。
2. 查看自己有没有被掠夺、损失了什么。
3. 决定是先修复领地，还是先进行复仇。
4. 根据当前资源与兵力，生产新部队或强化防守。
5. 发起匿名掠夺，争取稳定收益。
6. 在高风险时机发起实名挑战或针对性复仇。
7. 根据战果冲击某个排行榜目标。
8. 下线前重新安排防守与离线策略。

只要这个闭环保持通顺，玩家就会天然有“再上线一次”的动机。

## 四、系统定位

这套玩法建议分成四个互相咬合的系统：

### 1. 经济系统

负责生产、储存、消耗和转化资源。

### 2. 军事系统

负责训练部队、布置防守、发起掠夺、承担战损。

### 3. 对抗系统

负责匿名匹配、实名复仇、指名挑战和保护机制。

### 4. 荣誉系统

负责多维排行榜、阶段奖励、赛季目标和荣耀展示。

这四套系统里，真正驱动活跃的核心不是排名本身，而是：

排名让玩家的掠夺行为有了意义和方向。

## 五、资源与军队主循环设计

这是最基础、也最关键的一层。

### 1. 资源产出

资源建议分为三层：

1. 基础资源：粮食、木材、矿石、金币。
2. 军事资源：军粮、钢材、火药、灵魂残渣、圣印。
3. 稀有资源：赛季材料、世界掉落、挑战凭证。

产出来源建议包括：

1. 领地建筑自动产出。
2. 区域占点收益。
3. 阵营公共工程分红。
4. 掠夺所得。
5. 排名结算奖励。

### 2. 资源用途

玩家不能只把资源拿去升级建筑，必须面对用途冲突。

核心去向：

1. 扩建资源链。
2. 训练部队。
3. 维修防御。
4. 维持人口稳定。
5. 冲刺排行榜目标。

最关键的设计原则：

每一份资源都应该有机会成本。

也就是说，玩家把资源拿去造兵，就意味着短期更能抢；拿去建设，就意味着长期更强；拿去修城，就意味着不容易被吃。

### 3. 部队生产

部队建议分成三种常用定位，而不是一开始就做太多兵种：

1. 掠夺兵：运载高、进攻高、防御低。
2. 防守兵：防御高、驻守强、机动差。
3. 精锐兵：造价高、上限低、适合复仇和挑战。

阵营差异可以放在同一定位的不同特性上：

1. 天堂防守兵更强。
2. 恶魔掠夺兵收益更高。
3. 人类精锐兵和运输能力更灵活。

## 六、掠夺系统的三层结构

这里建议采用你提到的混合模式，但要明确层级，让玩家一眼知道每种玩法的目的。

### 1. 第一层：匿名日常掠夺

这是普通玩家最常用的掠夺来源，目标是提供稳定、可预期、相对公平的对抗收益。

规则建议：

1. 系统按照综合战力区间匹配匿名目标。
2. 玩家只能看到目标的阵营、等级段、资源预估、建筑简图、防御倾向。
3. 不显示目标昵称，不显示完整履历。
4. 每日提供 3 到 5 次免费掠夺机会。
5. 每次刷新目标需要消耗少量体力、金币或侦察令。

玩家体验目标：

1. 上线就有目标打。
2. 不容易被固定大佬盯死。
3. 战斗重点是判断“值不值得打”。

### 2. 第二层：实名复仇掠夺

这是制造私人故事和社交动力的核心机制。

触发方式：

1. 玩家被他人成功掠夺。
2. 战报中展示对方昵称、阵营、战斗摘要、资源损失。
3. 玩家获得一个复仇入口。

规则建议：

1. 每天至少提供 1 次免费复仇机会。
2. 复仇目标必须是最近 24 小时内攻击过你的玩家。
3. 复仇成功获得额外 10% 到 20% 荣誉收益或资源返还。
4. 复仇失败不追加惩罚，避免玩家因为情绪行为二次崩盘。

玩家体验目标：

1. 被打以后不只是吃亏，还会形成后续目标。
2. 战报不再只是通知，而是驱动下一次上线的理由。

### 3. 第三层：高阶实名挑战

这个层级不应该对所有人强开，而应该作为中后期玩法。

触发入口：

1. 排行榜界面。
2. 宿敌列表。
3. 阵营通缉令或赛季活动。

规则建议：

1. 需要消耗挑战券。
2. 同一目标在 24 小时内不能被同一玩家重复挑战。
3. 被挑战成功后，挑战者获得额外排行积分，不一定获得大量资源。
4. 高阶挑战更偏“荣誉争夺”，而不是纯资源压榨。

这样做的好处是：

1. 榜单高手之间有互动空间。
2. 不会把实名挑战变成低段位霸凌工具。
3. 荣耀型玩家和资源型玩家的目标被拆开。

## 七、掠夺完整运行流程

这一章是核心中的核心，要回答“系统到底怎么跑”。

### 1. 玩家上线后的对抗流程

建议流程如下：

1. 玩家打开游戏，进入领主简报。
2. 系统推送离线收益、战报摘要、当前护盾状态、今日剩余掠夺次数。
3. 玩家先处理受损建筑、补兵、收资源。
4. 系统根据当前战力给出 3 个匿名掠夺目标。
5. 玩家查看目标信息，选择一个发起侦察或直接掠夺。
6. 战斗结算后，生成资源收益、战损报告、排行积分变化。
7. 如果玩家被别人打过，则同步显示复仇入口。
8. 玩家决定继续掠夺、进行复仇，或转入建设与防守。

### 2. 匿名掠夺的结算流程

建议采用如下顺序：

1. 锁定攻击方当前出征部队。
2. 读取防守方的离线防守配置。
3. 读取防守方当前护盾、被掠夺次数、保护状态。
4. 校验是否满足攻击条件。
5. 结算攻防胜负。
6. 根据剩余运载量决定可带走资源上限。
7. 根据目标当前保护系数决定最终资源损失。
8. 写入双方战报。
9. 更新被攻击者当日被掠夺计数。
10. 若达到保护阈值，则自动挂上护盾。

### 3. 复仇流程

建议复仇比匿名掠夺更简单、更短链路。

流程如下：

1. 玩家从战报点击复仇。
2. 系统自动带入对方基础信息。
3. 玩家重新配置出征部队或使用快捷编队。
4. 系统执行战斗结算。
5. 若成功，则额外返还部分曾失去的资源，并附加复仇积分。
6. 若失败，仅正常扣除出征成本，不追加额外损失。

这样可以保证复仇是一种情绪出口，而不是新的挫败来源。

## 八、保护机制运行逻辑

保护机制不是可有可无，而是整套异步掠夺系统能否长期健康的关键。

### 1. 新手保护

建议条件：

1. 开服前 7 天。
2. 或主城等级低于某阈值。
3. 或综合战力低于匹配保护线。

保护内容：

1. 不进入全服公开排名。
2. 不能被高段位玩家随机掠夺。
3. 匿名匹配池单独隔离。
4. 被攻击损失有额外减免。

### 2. 连续受击保护

建议规则：

1. 单个玩家每天最多被成功掠夺 3 次。
2. 达到阈值后自动进入 8 小时护盾。
3. 护盾期间不可被匿名掠夺，也不可被排行榜挑战。
4. 若主动攻击他人，可选择立即取消护盾。

### 3. 同阵营保护

建议基础规则为：

1. 同阵营玩家之间默认禁止资源掠夺。
2. 允许切磋和演习，但无真实损失。
3. 特殊赛季玩法若要开放阵营内斗，必须明确作为特殊规则，不做常态内容。

### 4. 反霸凌冷却

建议增加针对单一目标的冷却机制：

1. 同一玩家在 24 小时内不能连续对同一目标成功掠夺超过 1 次。
2. 若为复仇关系，则可额外放宽 1 次，但收益递减。
3. 若目标已处于连续受击保护，所有实名挑战自动失效。

## 九、排行榜系统设计

结论不是“要不要排行榜”，而是“排行榜必须拆维度”。

如果只有一个总榜，游戏会迅速退化成：

1. 头部滚雪球。
2. 非头部玩家失去方向。
3. 掠夺行为只剩对弱者压榨。

所以建议采用多维榜单并且各自绑定不同玩法。

### 1. 战争之王榜

核心指标：

1. 掠夺总量。
2. 击破部队数量。
3. 主动进攻胜率。

作用：

1. 鼓励高频主动进攻。
2. 更适合恶魔或战争流玩家。

### 2. 钢铁壁垒榜

核心指标：

1. 防守成功次数。
2. 守城损失控制率。
3. 驻防强度评分。

作用：

1. 让防守型玩家也有荣誉目标。
2. 让“守得住”成为可炫耀能力。

### 3. 发展之星榜

核心指标：

1. 建筑总等级。
2. 科技完成度。
3. 单位时间资源效率。

作用：

1. 给经营玩家明确的长期方向。
2. 避免所有玩家都只盯着打架。

### 4. 阵营贡献榜

核心指标：

1. 阵营工程捐献。
2. 集结响应次数。
3. 前线驻守时长。
4. 公共任务完成度。

作用：

1. 强化阵营认同。
2. 让协作型玩家具备存在感。

### 5. 综合战力榜

核心指标：

1. 资产。
2. 军力。
3. 科技。
4. 主城发展。
5. 赛季加权荣誉。

作用：

1. 作为总体实力象征。
2. 用于赛季结算的主要展示位。

设计注意点：

综合榜必须是展示性榜单，不能变成唯一收益来源，否则其他玩法会被挤压掉。

## 十、排行榜奖励设计

奖励一定要分层，不能让榜单奖励直接变成永久数值碾压。

### 1. 日榜奖励

定位：即时反馈。

奖励建议：

1. 少量资源。
2. 侦察令。
3. 挑战券。
4. 加速卡。

### 2. 周榜奖励

定位：稳定追逐。

奖励建议：

1. 稀有材料。
2. 外观碎片。
3. 阵营贡献加成。
4. 赛季积分。

### 3. 赛季榜奖励

定位：荣耀沉淀。

奖励建议：

1. 限定称号。
2. 头像框。
3. 城堡外观。
4. 战报边框。
5. 不带强属性的纪念性皮肤。

核心原则：

高位玩家应该获得荣耀和识别度，而不是无限扩大的数值优势。

## 十一、排名与掠夺如何互相驱动

这两者不能并列摆着，必须互相喂养。

建议采用如下逻辑：

1. 匿名掠夺主要提供稳定资源和战争榜积分。
2. 实名复仇主要提供情绪反馈和额外荣誉积分。
3. 高阶挑战主要提供榜单冲刺积分和赛季故事性。
4. 防守成功则反向提供钢铁壁垒榜积分。
5. 被掠夺后的修复、补兵、升级，又会推动发展榜成长。

这样每个行为都不是孤立的：

1. 想冲战争榜，就要高效掠夺。
2. 想冲防守榜，就要降低被掠夺损失。
3. 想冲发展榜，就要平衡掠夺收益和建设投入。
4. 想冲综合榜，就必须四项都不太差。

## 十二、玩家每日运行流程示例

这里给出一版更贴近实际产品的日流程。

### 1. 轻度玩家

早上：

1. 收资源。
2. 看昨夜是否被打。
3. 若被打则复仇，否则打一场匿名掠夺。
4. 补少量兵。
5. 下线前重新挂防守。

晚上：

1. 领取第二轮资源。
2. 训练一批新兵。
3. 用免费次数再打 1 到 2 场匿名掠夺。
4. 查看自己在战争榜或发展榜的位置。
5. 决定把资源投建设还是留作明天冲榜。

### 2. 中度玩家

中午加一轮：

1. 看市场和阵营委托。
2. 看是否有新的复仇对象。
3. 若当前榜单接近晋级线，则消耗挑战券冲一次高阶目标。

### 3. 重度玩家

重度玩家的玩法不是多收几次菜，而是更高频率地：

1. 刷匿名目标。
2. 管理战报和宿敌列表。
3. 卡时间冲榜。
4. 在保护机制边界内追求收益最大化。

## 十三、推荐的 MVP 落地顺序

如果你想把这套东西做成最小可行产品，建议按下面顺序落地。

### 第一阶段

1. 资源自动产出。
2. 部队训练与驻防。
3. 匿名掠夺匹配。
4. 基础战报。
5. 新手保护与每日被掠夺上限。

目标：

验证最小循环是否成立，也就是“能不能收资源、能不能打别人、会不会被打、被打后会不会还想回来”。

### 第二阶段

1. 实名复仇。
2. 多维排行榜。
3. 日榜和周榜奖励。
4. 护盾与单目标冷却。

目标：

验证恩怨和荣誉是否足以提高留存和活跃。

### 第三阶段

1. 高阶挑战券。
2. 宿敌系统。
3. 阵营通缉或榜单挑战活动。
4. 赛季结算外观奖励。

目标：

验证中后期玩家是否有足够长期目标。

## 十四、风险点与修正建议

### 1. 风险：玩家觉得就是换皮偷菜

修正建议：

1. 强化兵种差异与战报反馈。
2. 强化复仇和阵营故事。
3. 让掠夺不是唯一收益来源，而是经营的一部分。

### 2. 风险：头部玩家霸凌中低层

修正建议：

1. 匿名匹配强绑定战力段。
2. 新手池隔离。
3. 每日成功被掠夺次数上限。
4. 单目标冷却。

### 3. 风险：排行榜变成数值碾压器

修正建议：

1. 多维榜替代单榜。
2. 赛季奖励以荣耀外观为主。
3. 不把综合榜做成唯一资源入口。

### 4. 风险：复仇没人用

修正建议：

1. 复仇入口必须显眼。
2. 复仇必须有额外收益。
3. 复仇失败惩罚必须轻。

## 十五、结论

这套系统最关键的不是“能不能抢人”，而是让玩家在每次上线时都面对一个清晰问题：

我现在这波资源，是拿去发育、拿去守家，还是拿去狠狠干一票。

只要这个问题每天都成立，这个异步循环就会有生命力。

如果用一句更产品化的话来总结，这套玩法可以定义为：

以异步资源争夺为入口，以复仇和排名为驱动，以阵营经营为长期目标的策略经营循环。
