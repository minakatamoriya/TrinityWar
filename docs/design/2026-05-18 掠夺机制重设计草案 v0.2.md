# 2026-05-18 掠夺机制重设计草案 v0.2

## 一、文档目的

这份文档只解决三件事：

1. 把当前掠夺玩法从“很多想法”整理成“首发可落地规则”。
2. 找出当前方案里不合理、容易失控或实现成本过高的部分。
3. 给出一套可以继续往数值和接口推进的统一框架。

这不是最终数值表。

这是一份首发规则整理稿，重点是先把玩法结构、边界、风险和推荐数值带确定下来。

## 二、当前方案的核心判断

当前方向是对的，但还存在四类问题：

1. 术语有混用，通缉、协力、雇佣兵有时在表达不同层，有时又混在一起。
2. 首发范围偏大，单人即时、双人异步、雇佣兵、助收、表情、传播、投资分成都想一起做，容易把实现面拉得过宽。
3. 数值边界缺少首批建议，导致很多规则看起来成立，但没有落地抓手。
4. 资产锁定、训练耗时、防守补偿、跨阵营协作这些点如果不先收紧，后面很容易失衡。

所以这版整理的核心原则是：

1. 术语统一。
2. 首发范围收紧。
3. 数值先给区间带，不急着给唯一答案。
4. 先保证玩法闭环成立，再逐步加社交层和传播层。

## 三、首发版本到底做什么

首发建议只保留两种掠夺模式：

1. 单人抢夺。
2. 双人通缉。

首发先不做下面几类复杂变体：

1. 三人以上围剿。
2. 阵营级集体掠夺。
3. 复杂的实时协防参战。
4. 自由留言和开放聊天。
5. 助收和协助防守玩法。

首发建议保留但做轻的内容：

1. 预设表情互动。
2. 微信好友助力雇佣兵。
3. 通缉分享卡片。
4. 复仇资格和结果战报。

一句话结论：

首发版本把掠夺收敛成“可见目标 + 单人即时 + 双人通缉 + 服务器结算 + 暴露资产锁定 + 训练耗时队列”。

## 四、术语统一

后续讨论尽量只用下面这套词。

### 1. 安全库存

定义：

1. 金库内的安全金币。
2. 不对外公开。
3. 不进入日常掠夺结算。

### 2. 暴露资产

定义：

1. 成熟外场果实。
2. 暴露金币。
3. 临时高价值掉落包。

规则：

1. 这是日常掠夺真正能命中的对象。
2. 这是目标列表里可展示的收益来源。
3. 这部分资产会进入锁定和结算流程。

### 3. 战力

定义：

1. 兵力、防守配置和相关修正折算后的对外展示值。
2. 用来支持“先判断能不能打，再判断值不值得打”。

规则：

1. 战力对外公开。
2. 总资产不公开。
3. 战斗公式不完全公开。

### 4. 单人抢夺

定义：

1. 玩家独自发起的即时掠夺。
2. 看到目标后直接打。
3. 几秒内出结果。

### 5. 通缉令

定义：

1. 玩家对明显高于自己的目标发起的双人协力掠夺。
2. 发起人和 1 名协力者组成进攻方。
3. 协力者确认后进入异步结算。

### 6. 雇佣兵

定义：

1. 通过通缉令加入战斗的协力者身份。
2. 可以是游戏外微信好友。
3. 也可以是游戏内玩家外放出的部队。

### 7. 锁定份额

定义：

1. 通缉令被接受后，目标暴露资产中被本次结算占用的那部分。
2. 从锁定到结算结束，这部分不能再被收走或转回安全区。

## 五、三阵营在掠夺玩法里的作用

既然世界观保留三阵营，掠夺规则层必须尽量轻阵营化。

首发建议：

1. 日常目标主要按战力和暴露收益筛选，不按阵营写复杂克制链。
2. 同阵营默认不开放普通掠夺。
3. 单人抢夺默认只打敌对阵营。
4. 通缉令中的雇佣兵可以无视阵营，因为它本质上是一次临时雇佣。
5. 阵营差异首发只保留轻量被动，不让它深度改写通缉规则。

这样处理的好处是：

1. 保住三阵营世界观。
2. 不把高频掠夺规则做得过重。
3. 给后续阵营大战和赛季内容留空间。

## 六、玩家看到什么

目标列表里建议固定展示下面信息：

1. 玩家昵称或简化昵称。
2. 阵营。
3. 当前战力。
4. 当前可抢果实值。
5. 当前可抢金币值。
6. 守备风险评级。
7. 热度标签。
8. 是否可发起通缉。

建议展示原则：

1. 暴露收益显示精确值。
2. 战力显示明确值或明确段位。
3. 总库存不显示。
4. 战斗公式和详细修正不显示。

这样玩家的决策顺序会很清楚：

1. 先看战力。
2. 再看收益。
3. 再决定单打、通缉还是放弃。

## 七、目标刷新池规则

这是目前方案里必须明确的一块，否则列表体验会很差。

建议刷新池按战力段分布：

1. 60% 同战力段目标。
2. 25% 略高于自己战力的目标。
3. 10% 略低于自己战力的目标。
4. 5% 高价值大佬目标。

这样设计的价值：

1. 大多数目标都可决策。
2. 经常会出现“我差一点能打，要不要发通缉”的张力。
3. 大佬目标是事件，而不是噪音。

建议再补两条保护：

1. 同一目标重复出现需要冷却。
2. 每次刷新优先过滤当天已被成功掠夺次数过高的玩家。
3. 玩家一旦对某目标成功发起单人抢夺或生成通缉令，该目标应立即从自己的当前列表移除。
4. 已被成功掠夺并进入保护期的玩家，不应继续出现在可攻击列表中。

建议首批数值带：

1. 免费刷新次数：5 到 8 次每天。
2. 付费刷新成本：低价递增，不要首发做太陡。
3. 同一目标再次出现冷却：15 到 30 分钟。
4. 已成功攻击过的目标再次回到本人列表的最短时间：不低于保护期结束时间。

## 八、单人抢夺规则

### 1. 定位

单人抢夺负责提供最快的偷菜爽感。

它必须满足三点：

1. 目标可见。
2. 发起简单。
3. 结果立刻反馈。

### 2. 流程

建议流程：

1. 玩家在列表中选择目标。
2. 服务器校验次数、兵力、冷却和目标状态。
3. 服务器锁定本次可抢份额。
4. 服务器同步结算胜负、收益和战损。
5. 接口直接返回结果。
6. 客户端立即播放入账和战损动效。

### 3. 推荐时长

1. 请求到结果：1 到 3 秒。
2. 动效展示：2 到 4 秒。

### 4. 单人抢夺的合理边界

单人抢夺不能做成：

1. 看到目标就百分百拿满。
2. 看见肥目标就零成本点击。
3. 防守方在线就能瞬间清空暴露资产逃脱。

所以建议保留：

1. 战损。
2. 掠夺次数限制。
3. 目标重复攻击冷却。
4. 防守修正。
5. 被成功掠夺方的保护期。

### 5. 首批数值带

1. 单人成功掠夺的收益比例：目标可抢份额的 20% 到 35%。
2. 单人失败惩罚：只承担战损，不获得收益。
3. 同一目标重复进攻冷却：30 到 60 分钟。
4. 每日单人发起次数：先做有限次数，再看活跃度调整。
5. 单人被成功掠夺后的保护期：20 到 40 分钟。

### 6. 失败后的处理口径

单人抢夺失败可以没有资源补偿，但不建议做成完全无反馈。

建议规则：

1. 失败方不获得金币或果实补偿。
2. 失败方承担已定义的直接战损。
3. 失败方至少获得战报、复仇资格或类似后续动作入口。

这样可以保持失败有代价，但不至于让体验过于生硬。

## 九、双人通缉规则

### 1. 定位

通缉令不是多人围殴。

首发应严格定义为：

1. 发起人。
2. 1 名协力者。
3. 接受后异步结算。

两人上限的价值很大：

1. 可控。
2. 易懂。
3. 好平衡。
4. 传播链够用。

### 2. 流程

建议流程：

1. 玩家发现明显高于自己的目标。
2. 玩家发起通缉令并投入首笔兵力。
3. 系统生成一张待接受通缉单。
4. 协力者点击同意后，进入异步结算状态。
5. 服务器锁定参战兵力和锁定份额。
6. 到达结算点后统一计算结果。
7. 双方按规则分配收益和战损。

### 3. 为什么通缉令必须异步

如果通缉令也做即时，问题会很大：

1. 协力者没有加入窗口。
2. 目标没有被锁定和冻结的意义。
3. 玩法会退化成“只是多带一个人瞬时结算”。

而异步后：

1. 协力邀请成立。
2. 锁定份额成立。
3. 防守准备和事前配置的价值成立。

### 4. 通缉令当前方案里最容易出问题的点

这里是最需要打磨的部分。

#### 4.1 问题一：发起时看到很肥，接受时已经不肥

这是当前最致命的体验风险之一。

如果不处理，会出现：

1. 我发给朋友时看起来值 5000。
2. 对方一同意，目标已经收走只剩 500。
3. 玩家会直接觉得系统在骗他。

推荐修正：

1. 通缉令发起时生成观察快照。
2. 通缉令待接受期间，对目标暴露资产做软锁。
3. 协力者接受的瞬间，把软锁升级为硬锁。
4. 最终结算只结算硬锁份额。

软锁的作用：

1. 防止目标在等待接受期间把核心暴露资产全部搬空。
2. 但不至于把目标整个经济面板冻结。

建议首发数值带：

1. 软锁上限：观察快照暴露资产的 30% 到 50%。
2. 硬锁上限：不高于软锁份额。
3. 待接受时间：20 到 45 秒。

#### 4.2 问题二：通缉被滥发去恶意占坑

如果通缉令可以免费发、无限挂，会出现：

1. 玩家恶意把高价值目标全部挂上待接受状态。
2. 别人既打不了，自己又不一定真打。

推荐修正：

1. 同一玩家同时最多挂 1 条待接受通缉令。
2. 同一目标同一时刻最多存在 1 条待接受通缉令。
3. 发起通缉需要消耗次数或体力。
4. 超时未接受自动失效并进入冷却。

这里建议再明确一点：

1. 通缉令不建议做成 1 小时级别的长时挂单。
2. 它应该是一种短事件，而不是长期占坑行为。

#### 4.3 问题三：协力者挂名蹭收益

如果协力者只点同意，不承担有效投入，就会变成白嫖分成。

推荐修正：

1. 协力者必须投入最低有效兵力。
2. 有效分成只按有效投入计算。
3. 挂名加入不满足门槛时，只记参与，不分主要收益。

### 5. 首批数值带

1. 通缉令待接受时间：20 到 45 秒。
2. 接受后异步结算窗口：60 到 180 秒。
3. 通缉成功收益比例：目标锁定份额的 35% 到 55%。
4. 发起人固定奖励：5% 到 10%。
5. 协力者最低有效投入：推荐总兵力的 20% 到 30%。
6. 通缉令自动失效时间：建议不超过 3 分钟，不建议做成 1 小时。

### 6. 成功与失败的收益约束

通缉令要成立，必须满足一个体验原则：

1. 打过大佬时，两名进攻方的合计收益应高于合计战损。
2. 打不过时，两名进攻方应明确亏损。

这意味着通缉令的目标不是稳定套利，而是：

1. 成功时有明显正收益。
2. 失败时有清晰代价。
3. 是否值得发起，仍然需要判断。

## 十、雇佣兵规则

### 1. 为什么要单独拆出来

雇佣兵本质上不是普通协力。

它承担的是：

1. 分享裂变。
2. 跨阵营合理化。
3. 外部好友低门槛参与。

### 2. 两类雇佣兵

#### 2.1 外部雇佣兵

定义：

1. 未进入游戏的微信好友。
2. 点击同意助力后，系统提供一支模板佣兵队。

这类雇佣兵的好处：

1. 极易传播。
2. 用户门槛低。

但也有风险：

1. 容易被刷。
2. 容易形成外部账号套利。

所以建议：

1. 首发只允许每条通缉令邀请 1 名外部好友。
2. 外部雇佣兵只提供模板战力，不吃完整浮动分成。
3. 外部雇佣兵奖励建议改成固定友情奖励或固定红包式奖励。

这是一个很关键的收紧点。

如果让外部好友直接按通缉收益比例分战利品，极易被薅。

#### 2.2 内部雇佣兵

定义：

1. 已在游戏中的玩家。
2. 选择外放自己的一部分兵力参战。

规则建议：

1. 内部雇佣兵承担真实战损。
2. 内部雇佣兵按有效投入正常参与分成。
3. 内部雇佣兵可以无视阵营参与，但身份显示为佣兵，不改阵营归属。

### 3. 当前方案里不合理的地方

最容易失衡的地方有两条：

1. 外部雇佣兵如果既低门槛又高收益，会被刷爆。
2. 内部雇佣兵如果无冷却、无机会成本，会让玩家把全部兵力长期外放套利。

推荐修正：

1. 外部雇佣兵只给固定奖励，不给浮动大头。
2. 内部雇佣兵参战后进入雇佣冷却。
3. 内部雇佣兵外放兵力在结算结束前不可回收。
4. 雇佣兵不参与防守协助，只参与通缉进攻。

## 十一、兵力训练规则

### 1. 结论先说

造兵不建议花钱瞬间完成。

如果造兵瞬发，会直接破坏四件事：

1. 战力展示的可信度。
2. 事前准备的价值。
3. 单人抢夺和通缉令的判断基础。
4. 防守和经营的资源张力。

### 2. 推荐规则

1. 训练必须走队列。
2. 数量越多，总时间越长。
3. 小批量可快速补，大批量不能瞬发。
4. 新训练完成的兵，不得加入已发起的战斗或已接受的通缉令。
5. 训练过程中如果玩家追加训练，允许直接并入当前队列并重新计算剩余时间。

### 3. 推荐实现方式

建议首发采用最简单可控的方式：

1. 单训练队列。
2. 每单位兵力都有基础训练时长。
3. 批量训练允许少量效率折扣，但不能折到接近瞬发。
4. 玩家后续追加训练时，不新开第二队列，而是直接叠加到当前队列中。

建议首批数值带：

1. 小批量训练：1 到 3 分钟。
2. 中批量训练：5 到 15 分钟。
3. 大批量训练：20 到 60 分钟。
4. 追加训练后的剩余时间应即时重算并明确展示给玩家。

### 4. 当前方案最要避免的不合理点

1. 看见肥目标时临时爆兵。
2. 被通缉后临时秒补兵。
3. 通过瞬发训练把通缉战力判断做废。

### 5. 关于伤兵的建议

首发版本不建议做独立伤兵系统。

推荐规则：

1. 战损直接扣除为已损失兵力。
2. 玩家通过重新训练补回战力。
3. 不单独维护伤兵池和伤兵恢复流程。

这样做的好处：

1. 计算简单。
2. 战力变化直观。
3. 不会把训练、恢复、战斗三套系统额外绑定。

如果后续要加伤兵，更适合作为中后期优化层，而不是首发基础规则。

## 十二、防守与补偿规则

### 1. 防守收益来源

首发建议只保留下面几种：

1. 驻守兵修正。
2. 防守建筑修正。
3. 阵营轻量被动。
4. 一次性轻度护符或防守技能。

不建议首发做：

1. 玩家实时手操防守。
2. 协防玩家直接加入战场。
3. 高频告警催上线守家。

### 2. 双边都有损失也可以，但要防通胀

“双方都有损失，但双方都有收获”这个方向可以保留。

但这里有一个数值大坑：

如果进攻方拿走资源，防守方又拿到大量系统补偿，而系统没有回收口，就会凭空造值。

所以建议把防守补偿定义成下面几类轻量收益：

1. 复仇值。
2. 护城积分。
3. 限额守城补贴。
4. 阵营英勇嘉奖。

不建议首发把防守补偿做成大额直接金币返还。

建议首批数值带：

1. 守城补贴应显著低于实际损失。
2. 守城补贴需要每日上限。
3. 复仇值和护城积分比直接金币更稳。

### 3. 成功防守与失败防守的推荐口径

建议拆开处理：

1. 成功防守：由阵营发放英勇嘉奖，形式以功勋、护城积分或轻量补贴为主。
2. 防守失败：不建议给大额资源补偿。
3. 防守失败后至少应给复仇资格、保护期和明确提示。

也就是说：

1. 守住了，可以被奖励。
2. 没守住，不一定给钱，但必须给后续动作和保护。

## 十三、保护机制

这部分必须先定，不然后面所有掠夺数值都不稳。

建议首发至少保留：

1. 新手保护期。
2. 每日被成功掠夺次数上限。
3. 同一目标重复攻击冷却。
4. 同阵营默认禁止普通掠夺。
5. 回流玩家保护。
6. 被成功掠夺后的显式保护期提示。

建议首批数值带：

1. 每日被成功单人抢夺次数上限：2 到 4 次。
2. 每日被成功通缉次数上限：1 到 2 次。
3. 同一攻击方对同一目标的重复成功间隔：至少 30 分钟。
4. 单人被成功掠夺后的保护期：20 到 40 分钟。
5. 被成功通缉后的保护期：40 到 90 分钟。

## 十四、当前方案里最不合理的地方与修正建议

### 1. 不合理点：想做的模式太多

表现：

1. 单人抢夺。
2. 双人通缉。
3. 雇佣兵。
4. 助收保护。
5. 表情互动。
6. 投资分成。

问题：

1. 首发实现面太宽。
2. 玩家第一次理解成本偏高。

修正：

1. 首发主打单人抢夺和双人通缉。
2. 助收保护放后置版本。

### 2. 不合理点：外部好友收益定义过重

问题：

1. 传播是强了，但可薅性也大。

修正：

1. 外部好友只拿固定奖励。
2. 真正浮动分成优先给内部参战者。

### 3. 不合理点：防守补偿容易做成通胀

问题：

1. 双方都赚钱很容易变成系统凭空造值。

修正：

1. 防守补偿尽量用积分、复仇值、限额补贴。
2. 直接金币补偿要非常克制。

### 4. 不合理点：资产锁定时机如果太晚，体验会崩

问题：

1. 玩家看到时很肥，接受时已经空了。

修正：

1. 发起时软锁。
2. 接受时硬锁。

### 5. 不合理点：训练如果瞬发，所有判断都会失真

修正：

1. 训练必须耗时。
2. 已发起事件不能吃到新训练兵。

## 十五、首发最小闭环建议

如果只保留最关键的闭环，建议首发版本按下面落：

1. 目标列表显示战力和暴露收益。
2. 单人抢夺即时结算。
3. 双人通缉待接受后异步结算。
4. 通缉支持一名好友或一名内部玩家协力。
5. 暴露资产走软锁加硬锁。
6. 兵力训练走耗时队列。
7. 战报只负责复盘和复仇资格。

这样已经足够形成一个完整体验：

1. 看目标。
2. 做判断。
3. 单抢或发通缉。
4. 承担战损。
5. 获得收益。
6. 回到训练、防守和下一轮掠夺。
