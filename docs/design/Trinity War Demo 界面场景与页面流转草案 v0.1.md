# Trinity War Demo 界面场景与页面流转草案 v0.1

## 一、文档目标

这份文档不产出视觉稿，只定义三件事：

1. Demo 要有哪些界面场景。
2. 每个场景里有哪些核心按钮和功能。
3. 每个场景之间如何跳转。

目标是让产品、程序和 UI 在开工前对页面结构有同一套理解。

## 二、全局导航原则

### 1. 首发导航结构

Demo 建议采用“1 个主城首页 + 5 个主入口”的结构：

1. 主城
2. 建筑
3. 农场
4. 掠夺
5. 战报
6. 阵营

排行榜放在阵营页内，不单独占一个底部标签。

### 2. 全局固定入口

下面这些入口建议几乎全局可见：

1. 返回主城按钮。
2. 待领取收益按钮。
3. 战报红点入口。
4. 设置入口。

## 三、页面总览

Demo 建议一共做 13 个场景：

1. 启动页
2. 登录授权页
3. 角色创建页
4. 阵营选择页
5. 新手引导弹层
6. 主城首页
7. 建筑页
8. 农场页
9. 部队页
10. 掠夺目标页
11. 掠夺结果页
12. 战报页
13. 阵营页

## 四、场景定义

### 1. 启动页

功能：

1. 展示 logo。
2. 检查登录态。
3. 拉取热更新配置。

按钮：

1. 开始游戏。
2. 公告。
3. 用户协议。

跳转：

1. 已有登录态且已建档，进入主城首页。
2. 无登录态，进入登录授权页。
3. 已登录但未建档，进入角色创建页。

### 2. 登录授权页

功能：

1. 获取微信登录授权。
2. 绑定游客态或正式态。

按钮：

1. 微信授权登录。
2. 返回。

跳转：

1. 登录成功后进入角色创建页或直接进入主城首页。
2. 点击返回则回到启动页。

### 3. 角色创建页

功能：

1. 输入昵称。
2. 选择头像。
3. 初始化角色。

按钮：

1. 随机昵称。
2. 确认创建。
3. 返回。

跳转：

1. 创建成功后进入阵营选择页。
2. 返回时回登录授权页。

### 4. 阵营选择页

功能：

1. 展示仙界、魔界、人界简介。
2. 展示每个阵营首发特点。
3. 选择阵营并确认。

按钮：

1. 选择仙界。
2. 选择魔界。
3. 选择人界。
4. 确认加入。
5. 返回。

跳转：

1. 选择并确认后进入新手引导弹层，再进入主城首页。
2. 返回时回角色创建页。

### 5. 新手引导弹层

功能：

1. 解释金库、外场、余额、待领取收益。
2. 引导第一次领取收益。
3. 引导第一次进入外场页或建设页。

按钮：

1. 下一步。
2. 跳过引导。

跳转：

1. 完成或跳过后进入主城首页。

### 6. 主城首页

功能：

1. 展示主城等级。
2. 展示金库金币、余额金币、待领取收益。
3. 展示外场位状态摘要。
4. 展示兵力摘要。
5. 展示战报红点与分红红点。
6. 展示今日免费掠夺次数。

按钮：

1. 领取收益。
2. 去收外场。
3. 去建设。
4. 去训练。
5. 去掠夺。
6. 看战报。
7. 去阵营。
8. 设置。

关键弹窗：

1. 领取收益确认弹窗。
2. 金库空间不足确认弹窗。
3. 今日战报摘要弹窗。

跳转：

1. 领取收益弹出领取流程，不离开当前页。
2. 去收外场或点击外场摘要，进入农场页。
3. 去建设进入建筑页。
4. 去训练进入部队页。
5. 去掠夺进入掠夺目标页。
6. 看战报进入战报页。
7. 去阵营进入阵营页。

### 7. 建筑页

功能：

1. 展示主城升级。
2. 展示金库升级。
3. 展示外场位升级。
4. 展示防守建筑升级。
5. 展示建筑线的成长说明和解锁条件。
6. 展示最近两档主城里程碑的未解锁能力预告，让用户同时看到“现在能升什么”和“再升几级会开什么”。

按钮：

1. 升级主城。
2. 升级金币上限。
3. 升级外场位。
4. 升级防守建筑。
5. 去农场。
6. 去训练。
7. 返回主城。

关键弹窗：

1. 升级条件不足提示。
2. 建筑升级确认弹窗。
3. 外场位解锁说明弹窗。

跳转：

1. 返回主城回主城首页。
2. 去农场进入农场页。
3. 如果主城升级缺少兵力或资源条件，则给出跳去部队页或农场页的引导按钮。

补充规范：

1. 建筑页详细展示规则、里程碑分组口径和配置归一建议，统一以 `docs/design/建筑页升级与未解锁能力展示规范.md` 为准。

### 8. 农场页

功能：

1. 展示外场培育状态。
2. 展示当前可收价值。
3. 展示成熟、丰熟、过熟倒计时。
4. 展示开始培育与收取入口。
5. 为后续农场扩展内容预留更多承载空间。

按钮：

1. 开始培育。
2. 提前收取。
3. 成熟收取。
4. 查看阶段说明。
5. 去建筑。
6. 返回主城。

关键弹窗：

1. 外场开始培育确认。
2. 提前收取收益预览。
3. 成熟或丰熟收取确认。
4. 农场阶段说明弹窗。

跳转：

1. 去建筑进入建筑页。
2. 返回主城回主城首页。

### 9. 部队页

功能：

1. 展示总兵力、空闲兵力、出征中兵力、伤兵。
2. 展示训练成本。
3. 展示恢复成本与恢复时间。
4. 展示余额与转入余额入口。

按钮：

1. 金库转入余额。
2. 余额回存金库。
3. 训练掠夺兵。
4. 恢复伤兵。
5. 快捷补满。
6. 去掠夺。
7. 返回主城。

关键弹窗：

1. 转入余额输入弹窗。
2. 回存余额限制提示。
3. 训练确认弹窗。
4. 恢复确认弹窗。

跳转：

1. 去掠夺进入掠夺目标页。
2. 返回主城回主城首页。

### 10. 掠夺目标页

功能：

1. 展示 3 个匿名目标。
2. 展示目标阵营、等级段、资源预估、风险倾向。
3. 展示剩余免费掠夺次数。
4. 展示刷新目标入口。

按钮：

1. 查看目标详情。
2. 发起掠夺。
3. 刷新目标。
4. 返回主城。
5. 查看战报。

目标详情内按钮：

1. 出兵。
2. 更换兵力。
3. 返回列表。

关键弹窗：

1. 兵力不足提示。
2. 当日次数不足提示。
3. 确认出兵弹窗。

跳转：

1. 发起掠夺后进入掠夺结果页。
2. 从次数不足提示可跳到阵营页或商城预留位，但 demo 先只提示不跳付费。
3. 返回主城回主城首页。

### 11. 掠夺结果页

功能：

1. 展示胜负结果。
2. 展示掠夺金币。
3. 展示我方战损。
4. 展示对方损失摘要。
5. 展示排行榜积分变化。

按钮：

1. 再找一个目标。
2. 回主城。
3. 查看完整战报。

跳转：

1. 再找一个目标回掠夺目标页。
2. 查看完整战报进入战报页并打开对应记录。
3. 回主城回主城首页。

### 12. 战报页

功能：

1. 展示进攻战报列表。
2. 展示防守战报列表。
3. 标记未读。
4. 标记是否可复仇。

按钮：

1. 查看详情。
2. 一键已读。
3. 复仇。
4. 返回主城。

战报详情内按钮：

1. 复仇。
2. 查看目标摘要。
3. 返回列表。

跳转：

1. 点击复仇进入掠夺目标页的复仇模式，自动带入目标。
2. 返回主城回主城首页。

### 13. 阵营页

功能：

1. 展示当前阵营。
2. 展示待领取分红。
3. 展示个人贡献值。
4. 展示上缴入口。
5. 展示三张榜单切页。

建议分栏：

1. 阵营总览。
2. 上缴与分红。
3. 排行榜。

按钮：

1. 领取分红。
2. 金币上缴。
3. 查看分红说明。
4. 切换掠夺榜。
5. 切换发展榜。
6. 切换贡献榜。
7. 返回主城。

关键弹窗：

1. 上缴输入弹窗。
2. 上缴收益说明弹窗。
3. 分红领取弹窗。

跳转：

1. 返回主城回主城首页。
2. 点击榜单玩家名字，demo 首发不跳实名挑战，只显示摘要即可。

## 五、关键弹窗清单

下面这些弹窗建议首发一定做：

1. 待领取收益领取弹窗。
2. 金库空间不足二次确认弹窗。
3. 升级条件不足弹窗。
4. 农场开始培育弹窗。
5. 农场收取确认弹窗。
6. 训练确认弹窗。
7. 出兵确认弹窗。
8. 上缴金币弹窗。
9. 复仇确认弹窗。

## 六、主链路页面流转

### 1. 新玩家主链路

1. 启动页。
2. 登录授权页。
3. 角色创建页。
4. 阵营选择页。
5. 新手引导弹层。
6. 主城首页。
7. 建筑页或农场页。
8. 部队页。
9. 掠夺目标页。
10. 掠夺结果页。
11. 战报页。

### 2. 老玩家日常主链路

1. 主城首页查看待领取收益和战报摘要。
2. 领取收益。
3. 去农场页收外场。
4. 去建筑页升级。
5. 去部队页补兵。
6. 去掠夺页打一轮。
7. 回主城或进战报页。
8. 去阵营页上缴和领分红。

### 3. 被攻击后的回流链路

1. 主城首页看到战报红点。
2. 进入战报页。
3. 查看损失详情。
4. 点击复仇。
5. 进入复仇模式发起掠夺。
6. 看结果后回主城或继续掠夺。

## 七、页面状态标记建议

为了让页面更容易做，建议统一下面几类状态标签：

1. 红点：有战报未读或有待领取收益。
2. 倒计时：外场成熟、恢复结束、保护结束。
3. 灰置：条件不足、次数不足、冷却中。
4. 警示：金库将满、待领取即将过期、兵力不足。

## 八、首发不建议做的交互

1. 多层嵌套页面树。
2. 长动画转场。
3. 大地图拖拽探索。
4. 复杂编队编辑器。
5. 多页签商城。

## 九、给 UI 的直接结论

如果要开始出图，先画下面 6 个高优先页面就够了：

1. 主城首页。
2. 建筑页。
3. 农场页。
4. 部队页。
5. 掠夺目标页。
6. 战报页。
7. 阵营页。

这 7 个页面决定了 demo 的 90% 用户体验，其余页面只需要轻量完成即可。
