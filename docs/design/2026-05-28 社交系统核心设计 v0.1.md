# 2026-05-28 社交系统核心设计 v0.1

## 1. 设计目标

社交系统是下一阶段的核心玩法入口，目标不是补一个好友列表，而是让玩家每天上线都能感受到：

1. 有人帮我。
2. 我可以帮别人。
3. 谁动了我的资源，我记得住。
4. 打不过的人可以找人一起打。
5. 阵营关系、好友关系、仇敌关系会持续影响游戏选择。

核心判断标准：

> 同阵营产生互助，敌对阵营产生冲突，好友关系产生反复互动。

## 2. 社交页定位

底部导航新增“社交”。

法术升级模块暂时屏蔽，避免把主注意力分散到纯数值加成。社交页承担新的中长期吸引力入口。

社交页不应该默认展示静态名单，而应该优先展示“今天可以做什么”。

推荐一级结构：

1. 动态
2. 好友
3. 关注
4. 阵营
5. 仇敌

默认进入“动态”。

动态页展示：

1. 谁帮了我。
2. 谁偷了我。
3. 谁攻击了我。
4. 谁可以复仇。
5. 哪个好友可以浇水。
6. 哪个同阵营成员快成熟了。
7. 哪个敌对目标适合挑战。
8. 哪个大佬可以发起组队挑战。

## 3. 关系类型

### 3.1 好友

来源：

1. 邀请进入。
2. 双方互相关注。
3. 战斗后主动添加。
4. 同阵营推荐添加。

主要行为：

1. 浇水催熟。
2. 帮忙守田。
3. 灵宠恢复助力。
4. 组队挑战邀请。
5. 查看有限状态。

好友关系重点是高频互助，不强调损失。

### 3.2 关注

关注是单向关系。

用途：

1. 标记强者。
2. 标记经常攻击自己的人。
3. 标记想以后挑战的人。
4. 标记同阵营活跃成员。

关注对象可以看到有限状态：

1. 是否在线或最近活跃。
2. 是否有可助力行为。
3. 是否有田地即将成熟。
4. 是否处于可挑战状态。

不建议展示过于精确的成熟时间和资源数量，否则会变成纯蹲点工具。

### 3.3 同阵营成员

同阵营关系偏协作。

主要行为：

1. 浇水催熟。
2. 帮忙守田。
3. 协助阵营任务。
4. 推荐好友。
5. 共同挑战敌对目标。

同阵营互助不能让被助力者损失收益。

### 3.4 仇敌

仇敌来源：

1. 偷过我。
2. 掠夺过我。
3. 我复仇过。
4. 多次挑战过我。

仇敌关系偏冲突。

主要行为：

1. 复仇。
2. 关注。
3. 组队挑战。
4. 查看战报。
5. 发送轻量互动消息。

仇敌列表不需要玩家手动维护，系统应自动沉淀。

## 4. 同阵营助力设计

### 4.1 浇水催熟

这是社交系统第一优先级功能。

效果：

1. 好友或同阵营成员可以对玩家田地进行浇水。
2. 每次浇水大幅缩短当前阶段剩余培育时间。
3. 被浇水玩家不减少任何收益。
4. 浇水者获得少量金币、阵营贡献或亲密度。

推荐数值：

1. 每次浇水缩短当前剩余时间的 20%-40%。
2. 设置最低缩短值，例如至少 10 分钟。
3. 设置最高缩短值，例如最多 60 分钟。
4. 教学田、新手田可以允许更高比例，用于制造爽感。

限制：

1. 每个玩家每天主动浇水 5 次。
2. 每个玩家每天最多被浇水 3 次。
3. 每块田每个生长周期最多被浇水 1 次。
4. 同一助力者对同一玩家每天最多助力 1 次。
5. 已成熟、枯萎、空地不能浇水。

设计理由：

大幅缩短时间可以制造爽感，但每日次数、被助力次数和单田限制可以避免刷号破坏经济。

### 4.2 帮忙守田

效果：

1. 对好友或同阵营成员施加一次守护。
2. 一定时间内被偷/被掠夺时，降低损失比例。
3. 守护不阻止战斗，只降低损失或增加防守收益。

推荐数值：

1. 守护持续 2-4 小时。
2. 降低被偷收益 20%-30%。
3. 每个玩家每天主动守护 2 次。
4. 每个玩家同时最多存在 1 个外部守护。

收益：

1. 守护者获得少量阵营贡献。
2. 如果守护期间真的发生攻击，守护者获得额外贡献或战报提示。

### 4.3 灵宠恢复助力

效果：

1. 好友可以帮助灵宠恢复少量生命。
2. 或者缩短恢复等待时间。

限制：

1. 每日次数少。
2. 只能恢复到一定比例，例如 80%，不能完全替代玩家自己的恢复资源。
3. 不影响付费/资源恢复的主要价值。

### 4.4 协助阵营任务

效果：

1. 同阵营成员可以对“阵营每日任务”提供少量协助进度。
2. 只协助阵营任务，不协助普通每日任务。

限制：

1. 每个阵营任务每天最多被协助 1 次。
2. 协助进度不能超过任务目标的 20%-30%。
3. 被协助者仍需完成主要行为。

## 5. 敌对阵营战斗设计

### 5.1 单人挑战

敌对阵营玩家可以被挑战。

目标来源：

1. 战报仇敌。
2. 关注对象。
3. 阵营推荐敌人。
4. 系统匹配目标池。

收益：

1. 金币。
2. 精华。
3. 阵营贡献。
4. 复仇额外奖励。

保护：

1. 连续被打后进入保护。
2. 被打损失不能过高。
3. 一部分收益来自系统奖励，避免对防守方体验过差。

### 5.2 组队挑战

组队挑战用于解决“打不过大佬”的问题，是可行的，但要控制好收益和参与条件。

核心定位：

> 组队挑战不是让弱者无限围殴强者，而是让玩家通过关系链获得一次“挑战强敌”的机会。

建议规则：

1. 最多二打一，首发只做 2 人组队。
2. 发起者选择目标，然后邀请一名好友或同阵营成员助战。
3. 助战者需要确认参与。
4. 双方都消耗一次今日组队挑战次数。
5. 战斗结算使用双方战力合并，但设置合并效率。

合并效率建议：

1. 发起者贡献 100% 战力。
2. 助战者贡献 50%-70% 战力。
3. 如果双方是好友或同阵营，可获得少量协同加成。
4. 如果双方亲密度较高，协同加成更高。

收益分配：

1. 发起者获得主要收益。
2. 助战者获得少量金币、阵营贡献和亲密度。
3. 目标如果防守成功，也获得防守奖励或声望。

限制：

1. 每个玩家每天最多发起 1 次组队挑战。
2. 每个玩家每天最多助战 2 次。
3. 同一目标每天最多被同一发起者组队挑战 1 次。
4. 高等级目标被组队挑战后进入更长保护。
5. 组队挑战不能绕过复仇窗口和保护规则。

反滥用：

1. 助战者不能是同设备/同登录来源高风险账号。
2. 新号需要完成一定新手进度后才能助战。
3. 组队收益有每日上限。
4. 组队挑战不产出高稀缺道具，主要产出金币、贡献、少量精华。

### 5.3 组队挑战的爽点

要让组队挑战有存在感，需要前端表现：

1. 邀请好友助战。
2. 好友确认加入。
3. 战斗前展示双方合力。
4. 战斗后展示“某某助你击败某某”。
5. 战报里保留双方名字。
6. 失败也给助战者少量友情/阵营贡献。

## 6. 邀请与助力绑定

邀请不应该只是一次性奖励。

建议：

1. 新玩家通过邀请进入后，默认与邀请者建立好友关系。
2. 邀请者获得一次“新友助力”额度。
3. 被邀请者完成新手流程后，双方都获得一次互助奖励。
4. 后续 3 天内，双方互助收益略高，帮助形成关系绑定。
5. 如果被邀请者加入同阵营，额外给阵营贡献奖励。

这样邀请带来的不是一次性资源，而是一个可继续互动的人。

## 7. 亲密度与关系沉淀

为了让用户关系更牢，需要亲密度。

亲密度来源：

1. 浇水。
2. 守田。
3. 灵宠恢复助力。
4. 组队挑战。
5. 邀请进入。
6. 连续互助。

亲密度用途：

1. 解锁更高协同加成。
2. 增加少量助力收益。
3. 组队挑战助战效率提高。
4. 好友列表排序靠前。
5. 后续可用于称号、头像框、纪念奖励。

限制：

1. 亲密度每天增长上限。
2. 高频刷同一个小号收益递减。
3. 亲密度不直接大量提高经济产出，避免变成硬性社交负担。

## 8. 社交动态

社交系统必须有动态流，否则玩家关系不容易被记住。

动态类型：

1. 好友帮你浇水。
2. 好友帮你守田。
3. 好友邀请你组队。
4. 你和好友组队挑战成功。
5. 敌人偷取了你的资源。
6. 敌人掠夺了你。
7. 你可以复仇。
8. 阵营成员请求协助。
9. 关注对象有新战报。

动态操作：

1. 回帮。
2. 复仇。
3. 关注。
4. 邀请组队。
5. 查看战报。
6. 删除/忽略。

## 9. MVP 范围

第一版只做最能验证社交价值的内容。

建议 MVP：

1. 社交页入口。
2. 好友/关注/仇敌关系。
3. 社交动态流。
4. 同阵营/好友浇水催熟。
5. 被浇水不减收益。
6. 每日助力次数限制。
7. 敌对目标挑战入口复用现有掠夺/战斗。
8. 组队挑战先做接口和基础结算，前端做简单邀请确认。
9. 邀请进入后自动建立好友关系。

暂缓：

1. 复杂亲密度等级奖励。
2. 多人组队。
3. 公会级团战。
4. 实时在线聊天。
5. 大型赛季社交榜。

## 12. 平衡参数建议

首版可配置参数：

1. 每日主动浇水次数：5
2. 每日被浇水次数：3
3. 单田单周期浇水次数：1
4. 浇水缩短比例：30%
5. 浇水最少缩短：10 分钟
6. 浇水最多缩短：60 分钟
7. 每日主动守田次数：2
8. 守田持续时间：3 小时
9. 守田降低被偷收益：25%
10. 每日组队挑战发起次数：1
11. 每日组队助战次数：2
12. 助战者战力效率：60%
13. 好友协同加成：5%
14. 同阵营协同加成：5%
15. 亲密度每日增长上限：100

## 13. 验收标准

MVP 完成标准：

1. 玩家能进入社交页。
2. 玩家能看到好友、关注、仇敌。
3. 玩家能看到可操作动态。
4. 玩家能给好友或同阵营成员浇水。
5. 浇水能显著缩短对方田地时间。
6. 浇水不扣对方收益。
7. 浇水次数限制生效。
8. 玩家能从战报沉淀仇敌。
9. 玩家能关注敌对目标。
10. 玩家能发起二打一组队挑战。
11. 助战者确认后进入结算。
12. 组队挑战战报展示双方参与者。
13. 邀请来的新玩家默认建立好友关系。

## 14. 设计风险

1. 助力过强会让不社交玩家落差过大。
2. 助力过弱会没有爽感。
3. 组队挑战如果收益过高，会变成刷大号工具。
4. 关注对象状态过精确，会变成蹲点偷菜工具。
5. 仇敌关系过重，可能让弱玩家感觉被针对。
6. 每日社交任务过多，会破坏轻量节奏。

解决方向：

1. 助力效果强，但每日次数少。
2. 组队收益受每日上限控制。
3. 被挑战方有保护和防守奖励。
4. 状态展示模糊化。
5. 社交行为尽量从动态页一键完成，减少操作负担。
