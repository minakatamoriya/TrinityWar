# 2026-06-01 首页职责与轻引导方案 v0.1

## 一、文档目标

这份文档只解决三件事：

1. 在弱化任务列表后，首页应该承担什么职责。
2. 首页应该如何引导玩家进入主循环，而不是制造上班感。
3. 首页首版建议放哪些模块，以及每个模块回答什么问题。

本文不处理具体 UI 视觉稿、接口字段和埋点实现，只先收敛产品职责和内容结构。

## 二、核心结论

弱化任务列表后，首页不再是“待办分发页”，而应该变成“状态驱动的引导页”。

首页的职责不是强制安排玩家今天必须完成什么，而是清楚告诉玩家：

1. 我现在有哪些可立即处理的机会。
2. 哪些收益已经成熟或快要错过。
3. 哪些养成和竞争动作现在做最值。
4. 我离下一段贡献和身份目标还有多远。

一句话概括：

**首页负责提醒、推荐和聚合入口，不负责把玩家变成按表打卡。**

## 三、首页的职责边界

首页建议承担五项职责。

### 1. 告诉玩家当前状态

首页第一职责是让玩家一眼看懂自己现在处于什么状态，而不是让玩家自己翻多个页面找信息。

至少需要让玩家快速知道：

1. 今天贡献推进到哪里。
2. 田地有没有成熟或即将枯萎。
3. 灵宠有没有可以处理的养成动作。
4. 有没有掠夺、复仇或社交机会。

### 2. 告诉玩家现在做什么最值

首页要回答“现在先点什么”这个问题，但回答方式不是任务清单，而是机会优先级。

例如：

1. 先收成熟田地。
2. 先处理快枯萎田地。
3. 先做灵宠加速或洗点。
4. 先看可复仇或高收益掠夺目标。

### 3. 给玩家一个轻量目标感

虽然不再用任务列表，但首页仍然要给轻度玩家一个“今天做一点也有推进”的感受。

这部分更适合用：

1. 今日贡献进度。
2. 本周贡献进度。
3. 距离下一档奖励还差多少。
4. 推荐行动带来的额外收益。

### 4. 把多系统压成一个可理解的入口层

首页不是信息大全，而是入口聚合层。

首页应该把农场、灵宠、掠夺、互助、排行榜这些分散系统，压成一眼能懂的状态卡和快捷入口，降低玩家决策成本。

### 5. 为下一次上线埋钩子

首页不仅要告诉玩家“现在能做什么”，还要暗示“下次回来会有什么值得处理”。

例如：

1. 下一块田将在 23 分钟后成熟。
2. 某只灵宠突破材料还差一点。
3. 某个复仇目标的保护快结束。
4. 本周贡献再推进一点就能进下一档。

## 四、首页不应该承担的职责

为了避免重新做成任务面板，首页首发不建议承担下面这些职责：

1. 列出 5 到 8 条固定日常任务。
2. 通过打勾清单强迫玩家逐项完成。
3. 用大量红点和未完成惩罚制造焦虑。
4. 把登录、浏览、点击等低价值动作包装成任务。
5. 同时展示过多细节数值，逼玩家在首页做复杂阅读。

首页应当引导玩家进入玩法，而不是在首页完成玩法。

## 五、首页结构建议

首发建议首页从上到下分成五块。

### 1. 顶部总览区

回答的问题是：

1. 我今天推进了多少。
2. 我本周推进了多少。
3. 我离下一档还有多远。

建议展示：

1. 今日贡献。
2. 本周贡献。
3. 距离下一档奖励的差值。
4. 当前资源摘要，例如金币、灵根、兽魂。

建议文案：

1. 今日贡献 82。
2. 距下一档还差 18。
3. 本周贡献 640。

这里强调“进度”和“还差多少”，不强调“你还有哪些任务没做”。

### 2. 当前最值得处理区

回答的问题是：

1. 我现在最该先点什么。

这块是首页真正的核心，建议按机会优先级展示 1 到 3 张卡片，而不是长列表。

优先级建议如下：

1. 已成熟田地。
2. 即将枯萎田地。
3. 灵宠可加速、可洗点、可突破。
4. 可复仇目标。
5. 高价值掠夺机会。
6. 好友互助请求。

建议文案：

1. 2 块田地已成熟，立即收取可获得贡献。
2. 1 块田地即将枯萎，建议优先处理。
3. 主战灵宠可洗点，战力还有提升空间。
4. 发现 1 个高收益掠夺目标。

这块的目标不是下指令，而是让玩家自然知道什么最值。

### 3. 农场与灵宠双核心区

回答的问题是：

1. 我的资源供给状态如何。
2. 我的战力出口状态如何。

建议做成两张并列卡片。

农场卡建议展示：

1. 当前成熟田地数量。
2. 下一块田成熟倒计时。
3. 今日已收取次数。
4. 快速进入农场入口。

灵宠卡建议展示：

1. 当前主战灵宠。
2. 当前战力或核心评分。
3. 可处理的加速、洗点、突破状态。
4. 快速进入灵宠页入口。

这样玩家能直观看懂：种田是供给，灵宠是消耗和展示。

### 4. 竞争与社交区

回答的问题是：

1. 别人现在和我有什么关系。
2. 我有什么值得出手的冲突或互动。

建议展示：

1. 最新被掠夺提醒。
2. 可复仇入口。
3. 今日可挑战的高价值目标。
4. 好友互助请求。
5. 好友或阵营中的最新强宠动态。

建议文案：

1. 玄虎道友刚刚掠夺了你，可立即复仇。
2. 3 位好友等待浇水互助。
3. 你的好友中已有 2 人战力超过当前主宠。

这块承担的是攀比、复仇和协作动机，能替代一部分旧任务系统的上线理由。

### 5. 推荐行动区

回答的问题是：

1. 如果我今天只做几件事，做哪些最划算。

这块不是任务列表，而是轻提示区。建议只列 2 到 4 条，不做打勾，不做完成率，不写“必须”。

推荐内容应直接绑定真实收益来源，例如：

1. 收取成熟田地可获得贡献。
2. 灵宠洗点可获得更高贡献。
3. 今日前 3 次好友浇水收益更高。
4. 掠夺高价值目标可获得额外贡献。

建议命名：

1. 推荐行动。
2. 今日可得贡献。
3. 当前更赚。

不建议命名为“今日任务”“待完成事项”“日常清单”。

## 六、首页模块与玩家问题对照表

| 首页模块 | 主要回答的问题 | 推荐展示方式 |
| --- | --- | --- |
| 顶部总览区 | 我今天和本周推进了多少 | 贡献条 + 差值提示 |
| 当前最值得处理区 | 我现在最该先做什么 | 1~3 张高优先级状态卡 |
| 农场卡 | 我的资源生产情况如何 | 状态摘要 + 快捷入口 |
| 灵宠卡 | 我的战力和养成状态如何 | 状态摘要 + 快捷入口 |
| 竞争与社交区 | 我现在有哪些冲突和互动机会 | 动态流 + 快捷入口 |
| 推荐行动区 | 今天顺手做什么更赚 | 2~4 条轻提示 |

## 七、推荐文案原则

首页文案建议统一遵守下面四条：

### 1. 用状态句，不用命令句

推荐写法：

1. 2 块田地已成熟。
2. 主战灵宠可突破。
3. 今日前 3 次浇水收益更高。

不推荐写法：

1. 立即完成 2 次收获。
2. 现在必须去突破。
3. 完成 3 次浇水任务。

### 2. 用机会提示，不用考勤提示

推荐强调：

1. 可获得什么。
2. 会错过什么。
3. 哪个动作当前更值。

不推荐强调：

1. 今天还有多少项没做。
2. 还差几项才算完成日常。
3. 若不完成会损失一整套奖励。

### 3. 用短句，少术语

首页文案要让玩家在几秒内看懂，因此更适合写成短句：

1. 1 块田地即将枯萎。
2. 灵宠洗点后战力可继续提升。
3. 可复仇目标已出现。

### 4. 直接写收益和风险

玩家最关心的是“做了有什么用”和“不做会发生什么”，所以首页文案应尽量直接对应收益和风险。

例如：

1. 立即收取可获得贡献。
2. 若继续放置，将进入枯萎期。
3. 前 3 次互助贡献更高。

## 八、新手期例外处理

虽然长期目标是不做任务列表，但新手期仍然可以保留单卡式强引导。

原因是新手期需要解决“下一步完全不知道去哪”的问题，而不是解决“任务太像上班”的问题。

因此建议：

1. 新手阶段首页只显示当前唯一引导卡。
2. 引导卡只指向一个下一步动作。
3. 完成该动作后立即刷新首页状态。
4. 完成新手流程后，切换到正式首页结构。

也就是说，强引导只保留在新手流程，正式日常阶段改为轻引导。

## 九、首发建议结论

首页首发建议直接定成下面这套方向：

1. 首页的职责是提醒、推荐、聚合入口，而不是发放任务清单。
2. 首页应围绕状态、机会、风险、收益来组织内容。
3. 首页中央应优先展示当前最值得处理的机会，而不是固定日常列表。
4. 首页必须同时承接种田供给、灵宠养成、竞争冲突和社交互助四条线。
5. 首页可以保留推荐行动，但推荐行动不是强制项。
6. 新手期允许单卡强引导，正式期切回轻引导首页。
