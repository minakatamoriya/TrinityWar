# TrinityWar UI 视觉方案 v0.1

## 一、视觉方向

当前客户端首版视觉方案统一采用：

1. 新东方极简。
2. 少量国风轻奢细节。

这套风格的目标不是做传统重装饰仙侠页游，而是保留仙侠世界观的气质，同时把按钮、菜单、弹框和信息面板统一成一套克制、清爽、可持续扩展的视觉语言。

方案代号：青玉仙府。

关键词：

1. 克制。
2. 空灵。
3. 简约。
4. 轻奢。
5. 统一。

## 二、总体原则

### 1. 结构现代，装饰收敛

页面结构采用现代移动端 UI 逻辑，不做大面积卷轴、木纹、厚金边。

仙侠感主要集中在以下位置：

1. 标题区。
2. 图标轮廓。
3. 面板角纹和边饰。
4. 少量淡金高光。

### 2. 功能优先于装饰

按钮必须一眼看懂主次关系，菜单必须一眼看懂当前层级，弹框必须一眼看懂标题、正文和操作区。

### 3. 全局统一，不按页面换风格

首发阶段禁止出现以下问题：

1. 首页是青玉极简，战报页变成厚重卷轴。
2. 主按钮是圆角玉色，弹框按钮又变成红金异形。
3. 菜单页签是一套语法，弹框页签又是另一套语法。

## 三、配色体系

### 1. 主色

主色用于主按钮、选中态、重点描边、可交互高亮。

1. Jade 500: #6FAE9B
2. Jade 400: #8BC2B0
3. Jade 300: #B8D8CB

### 2. 强调色

强调色只用于轻奢细节，不用于大面积铺色。

1. Gold 500: #C7A968
2. Gold 300: #E1CE9E

### 3. 背景色

背景以墨青、夜蓝、雾灰为主，避免纯黑。

1. Ink 900: #0D1416
2. Ink 800: #132024
3. Mist 700: #22353B

### 4. 文字色

1. 主文字: #F3EFE6
2. 次文字: #C9D0CB
3. 弱文字: #8E9D96
4. 禁用文字: #67746D

### 5. 反馈色

1. 成功: #78B99B
2. 警示: #C98F52
3. 危险: #A5645D
4. 信息: #85AFC0

## 四、材质与层次

### 1. 背景

背景不使用纯色平铺，统一使用：

1. 深墨渐变底。
2. 局部雾光团。
3. 极淡山影或流云噪点。

### 2. 面板

面板统一采用以下结构：

1. 深色半透明底。
2. 青玉描边。
3. 角部轻纹样。
4. 极弱高光，不做重阴影。

### 3. 分层

界面分为四层：

1. 场景背景层。
2. 信息面板层。
3. 菜单与按钮层。
4. 弹框与提示层。

弹框层必须明显高于面板层，但不允许采用夸张黑遮罩和刺眼发光。

## 五、按钮规范

### 1. 主按钮

用途：关键操作、确认、进入主功能。

样式：

1. 青玉渐变底。
2. 淡金或浅玉描边。
3. 20 到 24 圆角。
4. 白色或月白文字。
5. 轻微纵向高光。

状态：

1. 默认态有轻微光泽。
2. 按下态整体下沉 2 到 4 像素。
3. 选中态描边增强。
4. 禁用态降低对比度和透明度。

### 2. 次按钮

用途：取消、返回、次级入口。

样式：

1. 深墨半透明底。
2. 青灰描边。
3. 圆角与主按钮一致。
4. 不使用强渐变。

### 3. 危险按钮

用途：放弃、清空、删除、确认损失。

样式：

1. 暗赭红底。
2. 低饱和红棕描边。
3. 不使用纯红高亮。

## 六、菜单规范

### 1. 顶部页签

顶部菜单采用玉片式页签：

1. 默认态为低对比半透明片。
2. 选中态为青玉底 + 淡金细线。
3. 文案短促，保持 2 到 4 字。

### 2. 快捷入口卡片

首页功能入口统一为卡片按钮，而不是完全独立的异形按钮。

卡片结构：

1. 标题。
2. 一行状态或副标题。
3. 右侧箭头或角标。
4. 选中时只改变描边、底色和小范围高光。

### 3. 底部操作区

底部功能区若出现多个按钮，必须遵循：

1. 只有一个主按钮。
2. 其余为次按钮。
3. 主次按钮不能都做成高亮。

## 七、弹框规范

### 1. 基础结构

弹框必须统一包含：

1. 标题栏。
2. 主体内容区。
3. 底部操作区。

### 2. 视觉结构

1. 深墨半透明底。
2. 青玉描边。
3. 顶部标题区域允许使用淡金角纹。
4. 圆角比普通面板略大，建议 28 到 32。

### 3. 内容约束

弹框正文以信息清晰为主，禁止把整块正文做成纹样背景或过度书法化排版。

### 4. 交互约束

1. 主要确认按钮放右侧或视觉重心位。
2. 取消按钮始终保持次级样式。
3. 遮罩透明度不宜超过 55%。

## 八、字体与文案

### 1. 字体分工

1. 标题字：有东方气质，但必须清晰。
2. 正文字：现代黑体，优先可读性。
3. 数值字：与正文统一，不单独做夸张金属字。

### 2. 文案语气

文案应避免过度文绉绉，保持简洁、克制、有世界观味道即可。

例如：

1. 可用“外场待收”。
2. 不建议写成“灵田仙植已入丰熟之境，请君速取”。

## 九、动效语法

统一动效采用轻量规则：

1. 页面进入：淡入 + 上浮 12 到 20 像素。
2. 按钮点击：轻压 + 描边增强。
3. 菜单切换：横向滑移或淡切。
4. 弹框出现：淡入 + 缩放 0.96 到 1。

禁止事项：

1. 不做长时间粒子喷发。
2. 不做高频闪光。
3. 不做每个按钮都呼吸发光。

## 十、首页落地规则

首版首页应直接体现这套方案：

1. 背景为深墨渐变和雾化光斑。
2. 主城信息卡片采用青玉描边的深色面板。
3. 资源卡片保持统一圆角和细描边。
4. 快捷入口用卡片式菜单按钮，不做凌乱异形。
5. 首页必须至少有一个弹框示例，用来固化后续弹框语法。

## 十一、前端实现建议

针对 PixiJS 客户端，首版实现建议：

1. 先把颜色、圆角、描边、阴影做成统一 token。
2. 主按钮、次按钮、页签、弹框做成可复用绘制函数。
3. 页面级别先用纯代码绘制，等结构稳定后再替换贴图资源。
4. 非必要不引入大量图片，先把风格骨架立住。

## 十二、当前结论

TrinityWar 首发 UI 不追求传统重国风，而是采用：

1. 现代移动端信息结构。
2. 东方极简配色与材质。
3. 少量轻奢金线与角纹。
4. 按钮、菜单、弹框统一语法。

后续所有客户端页面都必须以本方案为基准展开，不再单页单风格设计。
