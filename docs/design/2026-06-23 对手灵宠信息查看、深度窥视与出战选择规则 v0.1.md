# 2026-06-23 对手灵宠信息查看、深度窥视与出战选择规则 v0.1

## 1. 文档目标

本文用于收口当前版本中与“查看自己/好友/敌方灵宠信息”“深度窥视”“确认出战前选宠”“后续宠物快照炫耀预留”有关的规则。

本文重点解决以下问题：

1. 玩家在不同关系和场景下，能看到哪些灵宠信息。
2. 五行和词条这两类关键战斗信息，何时公开，何时必须付费窥视。
3. 玩家在正式开战前，是否允许从自己的 5 只灵宠中临时选 1 只上场。
4. 如何在不增加太多复杂度的前提下，保留好友炫耀、敌方情报和深度窥视价值。
5. 如何为后续“宠物快照分享 / 炫耀”预留按钮和结构。

一句话总结：

> 好友看全，关注不看关键战斗信息；敌方守备主宠的五行和词条只通过一次性深度窥视获取；主动出战前允许玩家从自己 5 只灵宠中选 1 只上场。

## 2. 当前正式结论

### 2.1 五只可养，但每次只上一只

当前版本中：

1. 每名玩家有 5 个灵宠养育栏位。
2. 主动战斗时，每次只允许 1 只灵宠正式上场。
3. 被动防守时，每次也只使用 1 只守备主宠。

### 2.2 主动进攻时，允许在确认出战页临时选宠

本版正式结论：

1. 玩家在确认出战前，可以从自己当前可用的 5 只灵宠中选择 1 只作为本次进攻出战宠。
2. 这次选择只影响本次主动战斗。
3. 这次选择不自动修改首页展示宠。
4. 这次选择不自动修改守备主宠。

原因：

1. 玩家已经看完对手情报后，再退回灵宠页改主位再回来，会非常麻烦。
2. 既然灵宠战斗有五行和词条克制，就必须允许临场选宠。
3. 这样可以让 5 只灵宠的养成价值真正成立。

### 2.3 被动防守时，继续使用固定守备主宠

本版正式结论：

1. 防守方始终使用固定守备主宠。
2. 守备主宠切换继续沿用已有冷却和次数限制。
3. 不允许在被窥视后临时切守备宠来零成本反制。

原因：

1. 否则深度窥视价值会被削弱。
2. 否则战斗快照和情报失效问题会明显增多。
3. 否则玩家会产生“被耍了”的体验。

### 2.4 好友看全，关注不看关键战斗信息

本版正式结论：

1. 自己看自己：全开。
2. 好友查看你的灵宠：全开。
3. 关注对象或普通敌方查看你的灵宠：不开放五行和词条。
4. 深度窥视：只针对当前守备主宠，一次性公开五行和词条。

### 2.5 后续预留宠物快照分享 / 炫耀按钮

本版不直接开发完整分享功能，但要预留一个“快照分享 / 炫耀”按钮位。

当前口径：

1. 只针对自己的灵宠做快照分享。
2. 先不实现真正的图片生成、分享卡片、外链传播。
3. 前端和数据层先预留入口和字段。

## 3. 三类灵宠角色

为避免后续继续混淆，本版明确拆成三类角色：

### 3.1 展示宠

作用：

1. 首页、名片、参观页、炫耀场景的默认展示对象。

特点：

1. 主要承担外观和身份展示。
2. 不必等于守备主宠。
3. 不必等于本次主动出战宠。

### 3.2 守备主宠

作用：

1. 被别人挑战时，默认上场的灵宠。

特点：

1. 只能有 1 只。
2. 有切换冷却。
3. 有每日切换次数限制。
4. 深度窥视只针对它生效。

### 3.3 主动出战宠

作用：

1. 玩家本次主动发起战斗时，上场的灵宠。

特点：

1. 在确认出战页临时选择。
2. 只对本次战斗有效。
3. 不修改展示宠和守备主宠。

## 4. 不同场景下的信息可见性

## 4.1 可见性总表

| 场景 | 可见对象 | 是否看 5 只全部 | 是否看五行 | 是否看完整词条 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 自己看自己 | 自己 | 是 | 是 | 是 | 全开 |
| 好友参观 | 好友 | 是 | 是 | 是 | 全开，方便炫耀与讨论 |
| 关注对象参观 | 敌对关注对象 | 是 | 否 | 否 | 只看基础展示信息 |
| 普通敌方参观 | 敌对无关系对象 | 是 | 否 | 否 | 只看基础展示信息 |
| 免费目标详情 | 敌方守备主宠 | 否，只看当前守备主宠 | 否 | 否 | 只给模糊情报 |
| 深度窥视 | 敌方守备主宠 | 否，只看当前守备主宠 | 是 | 是 | 一次性付费情报 |

### 4.2 自己查看自己的灵宠

全部可看：

1. 5 只全部。
2. 真名。
3. 稀有度。
4. 等级。
5. 当前 HP。
6. 五行。
7. 固定特质。
8. 5 个词条。
9. 皮肤、边框、特效、快照分享入口。

### 4.3 好友查看你的灵宠

本版直接采用最简规则：好友全开。

好友可看：

1. 5 只全部。
2. 真名。
3. 稀有度。
4. 等级。
5. 当前 HP 状态。
6. 五行。
7. 固定特质。
8. 完整 5 词条。
9. 外观、皮肤、边框、特效。

原因：

1. 既然已经是好友，不再额外做复杂脱敏。
2. 好友之间更适合讨论配宠、洗点和打法。
3. 规则最简单，便于理解与开发。

### 4.4 关注对象或普通敌方查看你的灵宠

敌对关注对象和普通敌方，统一采用同一套展示口径。

允许看到：

1. 5 只全部。
2. 真名。
3. 稀有度。
4. 等级。
5. 外观、皮肤、边框、特效。

不允许看到：

1. 五行。
2. 固定特质详情。
3. 完整 5 词条。
4. 当前哪一只为守备主宠。
5. 当前守备主宠详细战斗底牌。

原因：

1. 关注的意义是找回目标，不是免费透视战斗底牌。
2. 五行和词条是关键战斗信息，必须保留给深度窥视。
3. 如果关注敌方就能看完五行和词条，深度窥视会失去价值。

## 5. 深度窥视规则

### 5.1 深度窥视的定位

深度窥视是：

1. 一次性。
2. 付费。
3. 只针对当前守备主宠。
4. 为正式出战决策服务。

### 5.2 深度窥视的消耗

本版直接收口为：

1. 查看敌方关键战斗信息必须消耗 1 次深度窥视。
2. 深度窥视需要消耗 1 次天机符。
3. 不再保留免费深度窥视次数。
4. 每次深度窥视只对当前这一名目标、当前这一次确认出战有效。
5. 再次挑战、再次复仇或重新查看其他目标时，需要重新消耗。

### 5.3 深度窥视后可见信息

深度窥视只公开当前守备主宠的关键信息：

1. 守备主宠真名。
2. 守备主宠等级。
3. 守备主宠当前血量状态。
4. 守备主宠五行。
5. 守备主宠固定特质。
6. 守备主宠完整 5 词条。

不公开：

1. 其他 4 只灵宠的五行。
2. 其他 4 只灵宠的词条。
3. 其他 4 只灵宠的详细战斗底牌。

### 5.4 深度窥视的价值

深度窥视的价值必须非常明确：

1. 不窥视：只能看外观、等级、稀有度和模糊战力感。
2. 窥视后：才能真正分析五行克制和词条打法。
3. 进攻方窥视后，再在确认出战页从自己的 5 只灵宠中选最合适的 1 只。

## 6. 主动出战前的选宠规则

### 6.1 入口位置

在“确认出战”页直接提供“本次出战灵宠选择区”。

### 6.2 展示方式

页面直接列出玩家当前可用的 5 只灵宠。

每只显示：

1. 灵宠头像 / 卡面。
2. 真名。
3. 等级。
4. 当前 HP 状态。
5. 五行。
6. 固定特质名称。
7. 5 个词条摘要。

### 6.3 对当前敌方守备主宠的提示

若玩家已经完成深度窥视，则在自己的 5 只灵宠上标提示：

1. `克制对方`
2. `被对方克制`
3. `推荐出战`
4. `当前血量偏低`

其中至少要提供：

1. 五行克制提示。
2. 当前血量不足提示。

### 6.4 默认选择规则

建议默认选中：

1. 上一次主动出战使用的灵宠；
2. 若上一次灵宠当前不可出战，则自动切到当前可出战且五行更优的一只；
3. 若无法给出明显更优解，则默认当前展示宠或当前主位宠。

### 6.5 确认后行为

玩家点击确认出战后：

1. 本次选择的 `selectedAttackerSpiritId` 进入确认出战请求。
2. 服务端在同一事务中锁定该灵宠快照。
3. 该选择只影响本次战斗，不回写玩家长期配置。

## 7. 参观与炫耀规则

### 7.1 参观别人时的灵宠展示

参观页主要用于展示，不是战斗详情页。

因此：

1. 好友参观页以炫耀和讨论为主，可全开。
2. 敌对关注对象和普通敌方参观页，只展示基础展示信息。

### 7.2 自己的快照分享 / 炫耀按钮预留

本版先不实现完整快照分享功能，但要预留一个按钮。

按钮文案建议：

1. `快照炫耀`
2. `分享灵宠`
3. `生成快照`

首版处理方式：

1. 先在自己的灵宠详情页或展示页放一个按钮。
2. 按钮先展示为 `敬请期待` 或灰态占位。
3. 不接真正图片生成和分享链路。

### 7.3 后续快照分享的目标

后续若开发正式快照分享，建议至少包含：

1. 灵宠立绘 / 卡面。
2. 皮肤、边框、特效。
3. 等级。
4. 五行。
5. 完整 5 词条。
6. 战绩或赛季徽章。

当前只预留，不实现。

## 8.2 推荐前端页面结构

### 参观页

分为两种布局：

1. 好友参观布局：5 只全量展开。
2. 敌方参观布局：5 只基础卡片展开。

### 确认出战页

页面结构建议：

1. 顶部：敌方当前守备主宠信息。
2. 中部：深度窥视结果区域。
3. 下部：我方 5 只灵宠可选卡列。
4. 底部：确认出战按钮。

### 自己灵宠详情页

需要预留：

1. `设置展示宠`
2. `设置守备主宠`
3. `快照炫耀`（预留）

## 8.3 推荐权限判断函数

```ts
type SpiritViewContext =
  | 'self'
  | 'friend-visit'
  | 'follow-visit'
  | 'enemy-visit'
  | 'deep-intel';

function canViewSpiritElement(context: SpiritViewContext): boolean {
  return context === 'self' || context === 'friend-visit' || context === 'deep-intel';
}

function canViewSpiritTraits(context: SpiritViewContext): boolean {
  return context === 'self' || context === 'friend-visit' || context === 'deep-intel';
}
```

## 9. 非目标

本版明确不做：

1. 防守方被窥视后临时换宠应对。
2. 关注对象免费看五行和词条。
3. 深度窥视后公开敌方全部 5 只的完整底牌。
4. 正式快照分享图片生成。
5. 复杂的词条质量标签体系。

## 10. 最终收口

当前版本推荐采用如下最终口径：

1. 玩家有 5 只可养灵宠，但每次战斗只上一只。
2. 主动战斗前，允许玩家在确认页从自己的 5 只灵宠中临时选 1 只上场。
3. 防守时继续使用固定守备主宠，不做被窥视后零成本反制。
4. 好友查看灵宠信息时，直接全开。
5. 关注对象和普通敌方查看灵宠时，只看基础展示信息，不看五行和词条。
6. 五行和完整词条只通过一次性付费深度窥视，对当前守备主宠公开。
7. 后续宠物快照分享 / 炫耀功能先预留按钮，不在本版实现。

一句话总结：

> 好友用来看全、看炫耀；敌方只能看展示；真正能分析打法的五行和词条，必须为当前守备主宠单次付费窥视后才能看到。
