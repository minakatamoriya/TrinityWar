# Trinity War 人口恢复与失败保护规则草案 v0.1

## 一、文档目标

这份文档只解决四个问题：

1. 掠夺兵在战斗损失后应如何结算。
2. 人口损失后如何恢复，避免玩家被连续失败打废。
3. 恢复过程里哪些损失是真损失，哪些损失可以逐步找回。
4. 首发阶段如何用最轻的规则保住失败成本与继续发育能力。

这份草案的核心原则只有一句话：

战损必须真实，但真实战损不等于全额永久清空，首发更适合采用“伤兵可恢复、部分永久阵亡、系统保底补员”的结构。

## 二、首发设计底线

首发版本建议守住下面六条：

1. 打输必须有真实损失。
2. 损失不能全部永久化。
3. 玩家连续失败后仍要能继续经营和补兵。
4. 恢复应同时包含时间成本和金币成本。
5. 不允许玩家完全免费瞬间回满兵力。
6. 不允许一次失败直接让玩家失去整轮参与资格。

## 三、首发推荐的战损结构

首发建议把每次战斗损失拆成三部分：

1. 永久阵亡。
2. 伤兵待恢复。
3. 保底留存兵力。

推荐理解如下：

1. 永久阵亡代表真实死亡，必须重新训练。
2. 伤兵待恢复代表重伤失能，可通过时间和金币恢复。
3. 保底留存兵力代表城内剩余可用基础战力，避免一战归零。

这样既能保住战损的情绪价值，也能给失败玩家留下重建空间。

## 四、单场战损的推荐结算方式

首发不建议把战斗损失全部判为永久死亡。

更稳的做法是：

1. 本次损失中的一小部分进入永久阵亡。
2. 本次损失中的大部分进入伤兵池。
3. 剩余兵力继续留在可用兵池中。

建议规则表达：

1. 打输以后，玩家会失去一部分可立即使用兵力。
2. 其中大头是可恢复伤兵，不是永久蒸发。
3. 只有较小比例需要重新花钱训练。

这条规则的目标是：

1. 让输掉战斗确实心疼。
2. 但不让玩家因为一次失败就被迫停服等待。

## 五、伤兵池规则

首发建议为玩家单独设置一个伤兵池。

伤兵池的作用：

1. 暂时承接战斗中损失的大部分兵力。
2. 通过时间自动恢复一部分。
3. 玩家也可以额外花金币加速恢复。

建议规则：

1. 战斗结束后，伤兵不会立刻回到可用兵池。
2. 伤兵按时间逐步恢复。
3. 玩家可以花费金库金币或战备槽金币执行伤兵恢复。
4. 恢复完成后，兵力重新回到空闲人口或可用兵池。

这条规则能自然回答“要不要花钱恢复兵力”这个问题：

要，但不是把全部死兵直接买回来，而是对伤兵执行恢复操作。

## 六、永久阵亡规则

首发仍然需要保留永久阵亡，否则战斗损失会太轻。

建议规则：

1. 伤兵之外的一部分损失进入永久阵亡。
2. 永久阵亡必须重新训练。
3. 重新训练仍然消耗人口位和金币。
4. 高风险或高强度战斗可以提高永久阵亡比例。

这样能保住两个核心感受：

1. 打输了不是等一会儿就什么都没发生。
2. 打高风险目标时，玩家会真的在意值不值。

## 七、保底补员与失败保护

为了满足“被攻击方必须保留恢复和继续发育的基础能力”，首发建议增加一层很轻的失败保护。

建议规则：

1. 玩家每日可获得少量基础补员恢复。
2. 连续失败后，系统可给极轻的保底补员或恢复加速。
3. 保底补员只保证玩家还能继续参与，不保证马上恢复到强势状态。
4. 保底补员不应替代正常训练和伤兵恢复。

这层保护的目标只有一个：

避免玩家因为连续两三次失败，直接失去继续玩的手感。

## 八、首发推荐的恢复路径

如果首发要做最小可用版本，建议直接采用下面这套恢复路径：

1. 战斗后损失兵力先拆成伤兵和永久阵亡两部分。
2. 伤兵进入伤兵池，按时间逐步恢复。
3. 玩家可花金币执行伤兵恢复，缩短等待时间。
4. 永久阵亡部分必须重新训练。
5. 系统每日提供少量保底补员或恢复加速，兜住最低可玩性。

这套路径比“全部永久死兵”更稳，也比“全部自动恢复”更有成本感。

## 九、与任务和追赶机制的关系

这套人口恢复规则还能自然接入现有系统：

1. 每日任务里可以出现“恢复一定数量伤兵”。
2. 晚入场追赶 buff 可以提高前几次伤兵恢复效率。
3. 阵营分红和上缴可以成为恢复资金来源之一。
4. 点券不建议直接买满兵力，但可以谨慎提供极轻的恢复便利。

## 十、首发推荐结论

如果要把规则收缩成一句话，建议直接定为：

首发采用“伤兵池 + 少量永久阵亡 + 轻保底补员”的人口恢复结构。玩家打输后会真实失去一部分兵力，但大部分损失会先进入可恢复伤兵池，玩家可用时间和金币逐步恢复，不会被一战打回无法继续参与的状态。
