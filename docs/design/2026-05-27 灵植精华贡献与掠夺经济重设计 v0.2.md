# 2026-05-27 灵植精华贡献与掠夺经济重设计 v0.2

## 一、文档目的

本文整理金币、灵植、精华、阵营贡献、首页阵营任务、偷菜与掠夺之间的新设计口径，并给出前后端开发落点。

本方案要解决的问题是：

如果阵营贡献可以由金币直接上缴获得，玩家后期会把金币全部洗入贡献，导致自己不留可掠夺金币，掠夺金币的长期意义下降。

因此本方案将金币和阵营贡献拆开：

```text
金币：个人经济资源，用于升级、经营、防守、掠夺成本等，可被掠夺。
灵植：种植资格和田地上的植物，解锁后可永久种植。
精华：灵植成熟后的收获物，可上缴、出售、被偷、被掠夺。
贡献：阵营荣誉资源，主要来自首页阵营任务和上缴精华，不再由金币直接兑换。
```

一句话总结：

```text
金币买能力，灵植产精华，精华换贡献，掠夺抢金币和部分精华。
```

## 二、术语调整

原先讨论中的“种子库存”不再作为对玩家展示的概念。

推荐统一术语：

```text
灵植：可种植品种，例如青灵麦、火椒、月莲。
灵植解锁：玩家是否永久拥有该灵植的播种资格。
精华：成熟后收获的材料，例如青灵麦精华、火椒精华、月莲精华。
精华库存：玩家当前拥有多少个某类精华。
```

精华命名规则：

```text
{灵植名}精华
```

示例：

```text
解锁青灵麦 -> 可以播种青灵麦
收获青灵麦 -> 获得青灵麦精华
上缴青灵麦精华 -> 获得阵营贡献
偷取青灵麦成熟地块 -> 获得青灵麦精华
掠夺库存 -> 抢走部分青灵麦精华
```

这样能避免“种子交光了还能不能种”的语义问题。

## 三、是否沿用之前种子字段

如果项目尚未上线，推荐现在就改字段语义，避免后续到处出现 `seed` 但实际代表精华。

推荐分成两组字段：

```text
plantType / cropType：灵植类型，用于播种、田地状态、解锁资格。
essenceType / materialType：精华类型，用于库存、上缴、偷取、掠夺。
```

如果当前代码已经大量使用 `seedType`，可以短期兼容：

```text
后端内部暂时保留 seedType 字段，但对外 API、前端文案、设计文档统一叫 plantType/essenceType。
后续做一次命名迁移，把 seedType 改为 plantType 或 essenceType。
```

不建议长期让 `seedType` 同时表示“可种植物”和“库存材料”，因为之后任务、背包、战报、日志都会变得难读。

## 四、是否需要背包系统

首发不需要做完整背包系统。

推荐做“轻量资源库存展示”，而不是完整背包：

1. 农场页展示已解锁灵植和对应精华库存。
2. 首页阵营任务展示任务需要的精华和当前库存。
3. 掠夺战报展示获得或损失的精华。
4. 可选在资源入口做一个“精华明细”弹窗。

完整背包可以暂缓，因为首发精华没有复杂装备、使用、堆叠品质、过期、绑定等逻辑。

首发最小展示即可：

```text
青灵麦
已解锁
库存：青灵麦精华 x28
用途：上缴 / 出售
```

## 五、核心规则

1. 阵营贡献不再通过上缴金币获得。
2. 阵营贡献主要来自首页阵营任务和上缴精华。
3. 玩家获得某种灵植后，永久解锁该灵植。
4. 播种只要求灵植已解锁，不消耗精华库存。
5. 灵植成熟后产出对应精华。
6. 精华可上缴、出售、被偷、被掠夺。
7. 被偷或被掠夺的精华必须从防守方原本应得或已有库存中扣除，不能凭空产出。

## 六、灵植解锁与播种

玩家第一次获得某种灵植时，永久解锁该种植项。

示例：

```text
玩家第一次获得火椒：
1. 火椒灵植永久解锁。
2. 火椒精华库存可增加一批初始数量，按奖励来源决定。
3. 即使火椒精华库存之后归零，玩家仍然可以继续播种火椒。
```

播种校验：

1. 玩家已解锁该灵植。
2. 田地空闲。
3. 满足等级、地块或任务阶段要求。

播种不扣除精华库存。

## 七、收获与被偷扣除

成熟收获后，玩家获得对应精华。

偷菜只针对成熟未收的地块，偷走的是本次成熟产出的部分精华。

扣除原则：

```text
被偷精华必须从防守方本次原本应得收成中扣除。
```

示例：

```text
火椒地块成熟，本次理论收成 10 个火椒精华。
敌方偷走 2 个。
防守方之后收获时只能获得 8 个。
攻击方获得 2 个。
系统总产出仍然是 10 个。
```

首发建议：

1. 每块成熟田最多被偷 1 次。
2. 偷取比例约 20%。
3. 最少偷 1 个，最多不超过固定上限。
4. 偷敌方阵营成熟精华可给少量贡献。

## 八、掠夺与被掠夺精华扣除

掠夺仍然以金币为主，精华为辅。

掠夺成功收益建议包含：

1. 目标暴露金币的一部分。
2. 目标库存精华的一小部分。
3. 少量阵营贡献。

扣除原则：

```text
掠夺获得的精华必须从防守方精华库存中扣除。
```

示例：

```text
防守方火椒精华库存 100。
本次掠夺成功，结算抢走 8 个火椒精华。
防守方火椒精华库存变为 92。
攻击方火椒精华库存增加 8。
如果攻击方此前未解锁火椒，则同时解锁火椒灵植资格。
```

首发建议：

1. 每次最多抢 1 到 2 类精华。
2. 每类最多抢库存的 5% 到 15%。
3. 设置最低保护库存，避免新手被抢空。
4. 优先抽取目标任务相关或高价值精华。

## 九、任务体系分层

当前任务体系建议拆成三类：

```text
新手任务：教会流程，只做一次。
日常任务：个人日活循环，每天刷新。
首页阵营任务：阵营身份和贡献主入口，每天刷新，放在首页重点展示。
```

### 1. 新手任务

目的：教会种植、收获、上缴、偷菜、掠夺的基础链路。

奖励以金币、初始灵植解锁、少量贡献为主。

建议贡献奖励：

```text
完成首次播种：5 贡献
完成首次收获：5 贡献
完成首次上缴精华：10 贡献
完成首次偷取敌方成熟精华：10 贡献
完成首次掠夺：15 贡献
```

新手任务贡献不应太高，主要负责引导。

### 2. 日常任务

目的：维持个人日活，不强制服务阵营目标。

示例：

```text
收获 3 次：10 贡献
完成 1 次掠夺：15 贡献
出售任意精华 20 个：10 贡献
```

日常任务可给少量贡献，但不要超过首页阵营任务。

建议每日总贡献：

```text
日常任务总贡献：30 到 50
```

### 3. 首页阵营任务

目的：作为贡献主入口，引导玩家在“种经济灵植”和“种阵营需求灵植”之间做选择。

首页阵营任务是玩家个人任务，不做全阵营共享大进度。

任务基于玩家已解锁灵植生成，而不是基于当前库存生成。

每天推荐 3 个首页阵营任务：

```text
阵营任务 A：基础上缴
阵营任务 B：重点需求
阵营任务 C：冲突对抗
```

建议首版贡献奖励：

```text
A 基础上缴：上缴初级精华 20 个，奖励 20 贡献。
B 重点需求：上缴中级或当日指定精华 10 个，奖励 35 贡献。
C 冲突对抗：偷取敌方成熟精华 3 次或成功掠夺 1 次，奖励 25 贡献。
```

首页阵营任务每日总贡献建议：

```text
普通玩家每日可完成：80 贡献左右。
活跃玩家含日常任务：110 到 130 贡献。
新手首日含新手任务：120 到 160 贡献。
```

贡献权重建议：

```text
首页阵营任务 > 日常任务 > 新手引导贡献
```

这样玩家会明确知道：想冲阵营贡献，主要看首页阵营任务。

## 十、首页展示方案

阵营任务应放在首页，而不是只藏在阵营页。

首页展示建议：

```text
今日阵营任务
1. 上缴青灵麦精华 20/20，奖励 20 贡献
2. 上缴火椒精华 4/10，奖励 35 贡献
3. 偷取敌方成熟精华 1/3，奖励 25 贡献
```

按钮规则：

1. 库存足够时显示“上缴”。
2. 库存不足但灵植已解锁时显示“去种植”。
3. 未解锁时显示“去获取”或不刷该任务，首发更推荐不刷。
4. 冲突任务显示“去偷取”或“去掠夺”。

首页只展示 3 个任务和进度，详细说明可以放到阵营页或任务弹窗。

## 十一、贡献数值首版

首发建议把贡献做成小整数，方便玩家理解。

### 1. 单项贡献

```text
上缴初级精华 1 个：约 1 贡献，但通过任务结算，不做无限自由兑换。
上缴中级精华 1 个：约 2 到 4 贡献。
上缴高级精华 1 个：约 5 到 10 贡献。
偷取敌方成熟精华 1 次：5 到 10 贡献。
成功掠夺 1 次：10 到 20 贡献。
防守成功 1 次：5 到 15 贡献，后续可做。
复仇成功 1 次：10 到 20 贡献，后续可做。
```

### 2. 每日贡献上限口径

首发不一定要硬上限，但需要有软上限。

建议：

```text
首页阵营任务：约 80 贡献/日。
日常任务：约 30 到 50 贡献/日。
偷菜和掠夺行为额外贡献：约 20 到 60 贡献/日，受次数限制。
普通活跃玩家：约 100 到 150 贡献/日。
高活跃玩家：约 180 到 250 贡献/日。
```

不要让自由上缴精华无限换贡献，否则囤积玩家会一次性冲爆排行榜。

### 2. 服务逻辑

播种：

1. 校验玩家拥有 plantType 解锁资格。
2. 校验田地空闲。
3. 不扣精华库存。

收获：

1. 读取 expectedEssenceYield。
2. 读取 stolenEssenceYield。
3. 发放 expectedEssenceYield - stolenEssenceYield。
4. 写入 PlayerEssenceInventory。
5. 写入 EssenceTransactionLedger。

偷菜：

1. 校验目标田地成熟且未收获。
2. 计算可偷精华数量。
3. 增加攻击方精华库存。
4. 增加目标地块 stolenEssenceYield。
5. 必要时解锁攻击方对应灵植。
6. 可选写入贡献流水。

上缴：

1. 校验首页阵营任务存在且未完成。
2. 校验玩家精华库存足够。
3. 扣除精华库存。
4. 增加任务进度。
5. 完成后发放贡献。
6. 写入 EssenceTransactionLedger 和 ContributionLedger。

掠夺：

1. 沿用现有 raid order / settlement 流程。
2. 成功时结算金币。
3. 额外计算可抢精华。
4. 从防守方精华库存扣除。
5. 给攻击方增加精华。
6. 必要时解锁攻击方对应灵植。
7. 写入战报、精华流水、贡献流水。

### 3. 接口建议

```text
GET  /client/farm/plants
返回已解锁灵植、精华库存、种植配置。

POST /client/farm/plant
使用已解锁灵植播种，不扣精华。

POST /client/farm/harvest
收获扣除被偷部分后的精华。

POST /client/farm/steal
偷取目标成熟地块的部分精华。

GET  /client/home/faction-tasks
返回首页阵营任务。

POST /client/faction/tasks/:taskId/submit
上缴任务指定精华。

POST /client/faction/tasks/:taskId/refresh
刷新未完成任务。
```

## 十三、前端开发落点

### 1. 首页

首页新增“今日阵营任务”模块。

展示内容：

1. 3 个阵营任务。
2. 每个任务的进度、奖励贡献、主操作按钮。
3. 今日已获得阵营贡献。
4. 可选展示阵营当日排名入口。

### 2. 农场页

展示内容：

1. 已解锁灵植列表。
2. 每种灵植对应精华库存。
3. 成熟时间、预计精华产量。
4. 播种按钮文案使用“已解锁可种植”。
5. 成熟地块展示理论产出、已被偷、可收获。

### 3. 阵营页

阵营页保留更完整的贡献记录和任务详情。

展示内容：

1. 今日阵营任务详情。
2. 个人今日贡献。
3. 累计贡献。
4. 贡献来源记录。
5. 阵营奖励或功勋商店入口。

### 4. 掠夺页和战报

战报展示：

```text
获得金币：120
获得精华：火椒精华 +8、月莲精华 +2
解锁灵植：月莲
获得贡献：15
```

目标列表可展示：

1. 可抢金币。
2. 可能掉落的精华类型或价值段。
3. 是否有首页阵营任务相关精华。

## 十四、MVP 开发顺序

1. 新增或重命名灵植解锁模型。
2. 新增或重命名精华库存模型。
3. 改播种逻辑：只校验灵植解锁，不扣库存。
4. 改收获逻辑：发放精华。
5. 新增首页阵营任务生成和展示。
6. 新增上缴精华获得贡献。
7. 贡献从金币上缴切换为任务奖励。
8. 偷菜结算扣除防守方本次应得精华。
9. 掠夺结算增加精华扣减与转移。
10. 前端首页展示阵营任务。
11. 前端农场页展示灵植与精华库存。
12. 前端掠夺战报展示精华收益。

## 十五、暂不做内容

首发先不做：

1. 完整背包系统。
2. 全阵营共享需求大进度。
3. 精华利息或商队投资。
4. 种子、作物、精华三层物品拆分。
5. 无限自由上缴精华换贡献。
6. 复杂阵营科技树。

## 十六、待确认问题

1. 精华是否允许出售换金币，若允许，需要确认每类精华出售价格。
2. 掠夺精华是优先抽任务相关精华，还是按库存价值随机抽取。
3. 首页阵营任务是否允许金币刷新，若允许，需要确认刷新价格。
4. 贡献排行按日、周、赛季三套都做，还是首发只做日榜和赛季累计。

## 十七、v0.3 补充：灵植发现、研究与解锁

### 1. 规则调整

精华首发只保留两个主要来源：

```text
1. 抢夺别人精华：包含掠夺胜利获得库存精华，也包含后续偷取成熟地块精华。
2. 阵营俸禄奖励：玩家达到一定贡献档位后，俸禄可奖励指定或随机精华。
```

获得某种精华不再直接永久解锁对应灵植。获得精华只代表玩家发现了该灵植线索，并开启研究进度。

```text
获得陌生精华 x1
-> 图鉴发现该灵植
-> 可查看名称、稀有度和部分描述
-> 不可播种
-> 满足精华消耗和贡献门槛后，永久解锁播种资格
```

### 2. 状态定义

灵植图鉴状态分为五类：

```text
未发现：玩家从未获得该灵植精华，图鉴显示？？？
已发现：玩家获得过至少 1 个该灵植精华，但还没有达到解锁条件。
研究中：已发现，且正在积累对应精华或贡献门槛。
可解锁：对应精华数量足够，贡献门槛也满足，可以点击解锁。
已解锁：永久拥有播种资格，播种不消耗精华。
```

### 3. 解锁条件

解锁条件由两部分构成：

```text
消耗：对应灵植精华
门槛：本赛季累计贡献，或当前贡献分
```

贡献只作为资格门槛，不消耗。原因是贡献已经影响阵营俸禄、排行和身份感，如果解锁灵植还扣贡献，会让玩家产生倒退感。

首版建议使用当前 `FactionMember.contributionScore` 作为门槛，后续如果做赛季贡献字段，再迁移为本赛季累计贡献。

### 4. 首版数值

启灵芽、青灵麦和风云稻属于基础路径，不走研究消耗。

```text
启灵芽：新手教程直接解锁，只用于引导，不进入日常阵营任务。
青灵麦：完成新手后直接解锁，作为正式日常慢节奏标准灵植。
风云稻：完成新手后直接解锁，作为正式日常快节奏碎片时间灵植。
```

其他灵植首版解锁门槛：

```text
普通灵植：消耗对应精华 x3，贡献 >= 50
进阶普通灵植：消耗对应精华 x6，贡献 >= 120
稀有灵植：消耗对应精华 x12，贡献 >= 300
传说灵植：消耗对应精华 x30，贡献 >= 800
```

具体分层：

```text
普通灵植第一档：凝露草、碎心花，精华 x3，贡献 50
普通灵植第二档：白玉莲、影月竹、牵机藤，精华 x6，贡献 120
稀有灵植：回春草、雪月花、劲道松、混沌果，精华 x12，贡献 300
传说灵植：斩情丝、忘川影、照幽冥，精华 x30，贡献 800
```

### 5. 数据模型

短期兼容现有 `PlayerSeedInventory`：

```text
PlayerSeedInventory.quantity：继续表示该类型精华库存。
PlayerSeedInventory.unlockedAt：继续表示该灵植是否永久可播种。
```

新增研究状态模型：

```text
PlayerPlantResearch
- id
- playerId
- seedDefinitionId
- discoveredAt
- researchVersion
- createdAt
- updatedAt
```

是否可解锁不需要单独落库，可由以下条件实时计算：

```text
已发现 + 未解锁 + 当前精华库存 >= 解锁消耗 + 当前贡献 >= 贡献门槛
```

### 6. 服务规则

获得精华时：

```text
1. 增加 PlayerSeedInventory.quantity。
2. 不设置 PlayerSeedInventory.unlockedAt。
3. upsert PlayerPlantResearch，记录 discoveredAt。
4. 写入 EssenceTransactionLog。
```

点击解锁时：

```text
1. 校验该灵植已发现。
2. 校验未解锁。
3. 校验精华库存足够。
4. 校验贡献门槛满足。
5. 扣除对应精华。
6. 设置 PlayerSeedInventory.unlockedAt。
7. 写入 EssenceTransactionLog，reason = plant-unlock。
```

播种时：

```text
只校验 PlayerSeedInventory.unlockedAt，不消耗精华。
```
