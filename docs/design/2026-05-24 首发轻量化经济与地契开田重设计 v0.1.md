# 2026-05-24 首发轻量化经济与地契开田重设计 v0.1

## 一、调整背景

当前首发版本已经包含种菜、收菜、掠夺、灵宠、阵营贡献等核心内容。作为微信小游戏，如果继续保留主城等级、主城税收、金库容量、主城开田等一整套城建逻辑，玩家会过早面对过多成长线，核心循环会被稀释。

本轮调整目标是：

1. 让金币主要来自农作物与掠夺结果，保持种田是主要产金口。
2. 让阵营成为余钱回收器和长期投资线，而不是另一个高额金币产出源。
3. 取消金库容量压力，让玩家专注收菜、花钱、培养、对抗。
4. 取消主城等级对核心玩法的硬绑定。
5. 用地契任务控制开田节奏，让玩家逐步体验完整流程。

## 二、核心结论

首发推荐采用以下新口径：

```text
金库：无上限安全钱包，不再通过容量卡玩家。
主城：保留为视觉、身份和展示壳，不再承担税收与核心功能解锁。
税收：删除主城每小时税收。
阵营：定位为金币回收 + 长期材料返还 + 身份竞争。
金币产出：主要来自农作物，掠夺作为风险收益补充。
开田：改为地契任务，不再绑定主城等级。
```

## 三、金库规则

### 1. 金库无上限

金库不再设置容量上限，也不再要求玩家升级金库来承接金币。

调整后：

1. 收菜金币直接进入金库。
2. 掠夺获得金币直接进入金库。
3. 阵营每日少量金币俸禄直接进入金库。
4. 取消因金库满而导致的溢出损失。
5. 取消临时待领取金币因金库不足而过期损失的设计。

### 2. 金库升级处理

首发建议直接移除金库升级。

如果仍希望保留金库作为场景建筑，可以改为展示型内容：

1. 显示累计获得金币。
2. 显示历史最高金库余额。
3. 显示玩家财富称号。
4. 后续可扩展外观，不影响容量和战斗。

不要在首发继续让金库承担数值门槛。

## 四、主城规则

### 1. 主城等级不再作为首发核心系统

主城等级暂时退出首发核心成长线。

以下绑定关系建议全部解除：

1. 主城等级不再决定税收。
2. 主城等级不再决定开田。
3. 主城等级不再决定灵宠培养上限。
4. 主城等级不再决定掠夺、阵营等核心功能入口。
5. 主城等级不再作为每日任务或成长任务的强制目标。

### 2. 主城保留什么

主城可以保留为：

1. 首页视觉场景。
2. 玩家领地身份展示。
3. 称号、装饰、赛季纪念展示入口。
4. 后续版本扩展入口。

首发不建议让主城继续承担金币产出和功能闸门。

## 五、金币产出口径

调整后，金币主要来源如下：

```text
主要来源：农作物收获
风险补充：掠夺结算
轻量补贴：每日任务、阵营金币俸禄
不再提供：主城每小时税收
```

### 1. 农作物是主产金口

农作物承担玩家日常金币增长的主体。

这能强化最核心的行为：

1. 种什么。
2. 什么时候收。
3. 要不要等丰收窗口。
4. 要不要把高价值作物暴露给别人抢。

### 2. 掠夺是风险收益

掠夺不应稳定替代种田，但应能提供明显波动收益。

掠夺收益建议包含：

1. 金币。
2. 兽魂。
3. 种子或种子碎片。
4. 少量地契相关进度。

### 3. 每日任务只做轻补贴

每日任务金币不应成为主产金来源，只用于鼓励玩家走完核心流程。

建议任务金币长期控制在普通玩家日均可支配金币的低比例区间。

## 六、阵营新定位

阵营不再是辅助收益线，而是：

```text
余钱回收器 + 长期投资收益 + 身份竞争线
```

玩家把金币捐给阵营，获得贡献。贡献越高，每日可领取的阵营俸禄越好，但俸禄重点返还材料，不返还大量金币。

### 1. 为什么不返大量金币

如果玩家捐金币后又每日返大量金币，阵营会变成存款利息系统。

这会带来三个问题：

1. 后期金币通胀。
2. 种田产金地位下降。
3. 高贡献玩家滚雪球过强。

所以阵营收益要从“返金币”改为“返资源包”。

### 2. 阵营每日俸禄结构

建议拆成两部分：

```text
固定俸禄：所有阵营成员每日可领，轻量。
贡献俸禄：按累计贡献档位每日可领，主要给材料。
```

固定俸禄可以给一点点金币，让玩家觉得加入阵营有日常存在感。

贡献俸禄则主要给：

1. 普通种子。
2. 稀有种子碎片。
3. 兽魂。
4. 灵宠培养材料。
5. 阵营声望或称号进度。
6. 少量金币。

### 3. 阵营俸禄档位草案

```text
贡献档位 | 每日俸禄建议
-------: | --------------------------------------------------
0        | 少量金币 + 普通种子
100      | 少量金币 + 普通种子包 + 少量兽魂
300      | 普通种子包 + 兽魂 + 小概率稀有种子碎片
600      | 兽魂 + 稀有种子碎片 + 阵营声望
1000     | 稳定稀有种子碎片 + 较多兽魂 + 阵营声望
2000+    | 高级种子碎片 + 高级兽魂材料 + 称号进度
```

金币只作为轻量补贴，不作为贡献俸禄的主体。

### 4. 每日发放方式

原先每小时收益改为每日领取。

推荐规则：

1. 每日可领取一次阵营俸禄。
2. 按领取时的累计贡献档位计算。
3. 当日未领取不建议无限累积。
4. 可以保留 1 天补领宽限，但不做长期堆积。

这样能降低后台持续结算复杂度，也更符合微信小游戏的每日节奏。

## 七、地契开田系统

### 1. 为什么用地契

地契用于替代主城等级开田。

它有四个优点：

1. 节奏可控。
2. 目标直观。
3. 能把收菜、阵营、掠夺串成任务链。
4. 不需要额外引入农场等级。

地契不是一个新的重系统，只是开田凭证。

### 2. 开田目标节奏

本轮目标节奏：

```text
第 1 天：默认 1 块田
第 2 天：大部分玩家开第 2 块田
第 5 天：大部分玩家开第 3 块田
第 2 周：大部分玩家开第 4 块田
```

这个节奏能避免第一天管理压力过大，同时保证前两周每天都有明确成长目标。

### 3. 推荐开田规则

```text
第 1 块田：默认拥有

第 2 块田：
条件：账号创建满 1 天，并累计收菜 3 次
预期：第 2 天开启

第 3 块田：
条件：累计收菜 10 次，并累计阵营贡献达到 100
预期：第 4~6 天开启

第 4 块田：
条件：累计收菜 30 次，并累计阵营贡献达到 500，并成功掠夺 3 次
替代条件：未完成掠夺时，累计收菜 45 次也可替代
预期：第 8~14 天开启
```

### 4. 为什么第 4 块田允许替代路径

第 4 块田需要鼓励玩家体验掠夺，但不应该强迫所有玩家都必须 PVP。

因此推荐双路径：

```text
进攻型路径：收菜 30 次 + 贡献 500 + 成功掠夺 3 次
经营型路径：收菜 45 次 + 贡献 500
```

进攻型玩家开得更快，经营型玩家也能开，只是慢一点。

### 5. 地契任务表现

前端可以表现为“地契任务”或“开垦任务”。

不建议表现为等级条。

推荐展示：

```text
二号地契
进度：
- 账号创建满 1 天：0/1
- 累计收菜：2/3

三号地契
进度：
- 累计收菜：7/10
- 阵营贡献：60/100

四号地契
进度：
- 累计收菜：18/30
- 阵营贡献：220/500
- 成功掠夺：1/3
或
- 经营替代收菜：18/45
```

这样玩家看到的是任务，而不是又一个成长系统。

## 八、新手前两周体验节奏

### 第 1 天

目标：理解种菜、收菜、金库、基础灵宠。

玩家拥有 1 块田，重点是：

1. 播种。
2. 等待成熟。
3. 收菜拿金币。
4. 第一次培养灵宠。

不要让玩家第一天管理太多田。

### 第 2 天

目标：开启第 2 块田，形成“我变强了”的反馈。

主要动作：

1. 登录。
2. 收菜达到累计 3 次。
3. 领取二号地契。
4. 开第 2 块田。

第 2 块田应明显提升玩家种植效率，但仍不复杂。

### 第 3~5 天

目标：接触阵营，并为了第 3 块田开始捐献。

主要动作：

1. 稳定收菜。
2. 加入阵营。
3. 第一次捐献金币。
4. 累计贡献到 100。
5. 开第 3 块田。

此时玩家会理解：多余金币可以投给阵营，阵营贡献能换长期材料和开田进度。

### 第 6~14 天

目标：体验掠夺，或通过经营路径慢慢开第 4 块田。

主要动作：

1. 持续收菜。
2. 培养灵宠。
3. 尝试掠夺。
4. 累计贡献 500。
5. 通过进攻型路径或经营型路径开第 4 块田。

第 4 块田是第二周的阶段性目标，不应第一周轻易普及。

## 九、任务与数据埋点建议

为了支持地契与阵营俸禄，需要记录以下玩家累计数据：

```text
累计收菜次数
累计成功掠夺次数
累计阵营贡献
账号创建时间
当日阵营俸禄领取状态
当前已解锁田地数量
各地契任务完成状态
```

这些数据也能服务后续新手漏斗分析：

1. 第 2 天开第 2 块田比例。
2. 第 5 天开第 3 块田比例。
3. 第 14 天开第 4 块田比例。
4. 玩家是否因阵营贡献要求卡住。
5. 玩家是否因掠夺要求卡住。
6. 经营替代路径是否被大量使用。

## 十、首批开发改动清单

### P0 必改

1. 金库容量上限失效或移除。
2. 收菜金币直接入金库，不做容量溢出判断。
3. 掠夺金币直接入金库，不做容量溢出判断。
4. 删除或停用主城税收。
5. 阵营收益从每小时结算改为每日领取。
6. 阵营每日俸禄改为材料为主、金币为辅。
7. 开田逻辑从主城等级改为地契任务。

### P1 建议同步

1. 前端隐藏主城升级入口或改为展示入口。
2. 前端隐藏金库升级入口或改为统计展示。
3. 新增地契任务面板。
4. 新增阵营俸禄领取面板。
5. 每日任务中加入收菜、捐献、掠夺相关引导。

### P2 后续扩展

1. 阵营贡献称号。
2. 阵营专属种子碎片。
3. 高贡献玩家的阵营外观。
4. 赛季末按贡献发放纪念奖励。

## 十一、风险与边界

### 1. 阵营不能返太多金币

阵营是回收器，不是印钞机。

如果贡献俸禄金币太高，会重新压过农作物产金，破坏本轮调整目标。

### 2. 地契不能太像新等级系统

地契只做开田凭证，不做长期等级成长。

不要新增“农场等级”“地契等级”“土地经验”等复杂概念。

### 3. 第 4 块田不能强制 PVP

掠夺可以加速开第 4 块田，但不能成为唯一通道。

否则低对抗玩家会在第二周被卡住。

### 4. 金币消耗必须跟上

金库无上限后，必须确保金币有稳定去处：

1. 灵宠培养。
2. 阵营捐献。
3. 掠夺战损和恢复。
4. 稀有种子相关消耗。
5. 赛季冲刺目标。

## 十二、最终推荐版本

首发轻量化后，核心循环应收敛为：

```text
种菜 -> 收菜得金币 -> 培养灵宠/阵营捐献 -> 掠夺或继续经营 -> 获得材料与种子 -> 开更多田 -> 更高价值作物与更强对抗
```

对应系统职责：

```text
农作物：主金币产出
金库：无限安全存储
灵宠：战斗与主要金币消耗
阵营：余钱回收、材料返还、长期身份
掠夺：风险收益与情绪波峰
地契：控制开田节奏和新手流程
主城：视觉与身份展示，不做首发闸门
```

本轮调整的关键不是减少内容数量，而是减少玩家必须同时理解的“等级闸门”。玩家前两周只需要理解一件事：认真种、适时收、把钱投到灵宠和阵营里，就能逐步开田、变强、去抢别人或保护自己。
