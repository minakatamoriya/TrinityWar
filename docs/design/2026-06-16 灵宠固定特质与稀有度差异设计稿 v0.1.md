# 2026-06-16 灵宠固定特质与稀有度差异设计稿 v0.1

## 1. 文档目的

本文用于收口灵宠固定特质系统的首版设计方向，目标是在保留当前“血量 + 攻击 + 词条 + 最多 10 回合互撞”的轻量框架下，为 18 只灵宠补出足够鲜明的身份差异、构筑深度和克制关系。

本稿重点解决以下问题：

1. 固定特质如何在微信小游戏场景下保持“有深度，但不复杂”。
2. 普通、稀有、传说三档灵宠如何做出差异，但不把普通宠做废。
3. 固定特质如何与现有 8 个词条联动，而不是变成另一套割裂系统。
4. 传说灵宠如何拥有强特质，但仍然存在被克制和被针对的空间。

## 2. 当前已知前提

### 2.1 当前灵宠战斗骨架

当前已明确保留：

1. 战斗形式为自动互撞。
2. 单场战斗最多 10 回合。
3. 基础战斗属性以攻击、血量为主。
4. 词条当前为 8 个，且每只宠物可装备 5 条，可重复。
5. 五行克制已存在，当前口径为“被克制时，攻击和词条效果翻倍”。

### 2.2 当前词条池

当前词条可按作用分为四类：

| 类别 | 词条 | 作用方向 |
|---|---|---|
| 基础输出 | `claw`、`crit`、`crit_damage` | 爆发、前压、斩杀 |
| 生存承伤 | `thick_skin`、`tenacity` | 厚血、抗暴、拖回合 |
| 反制博弈 | `dodge`、`counter` | 赌脸、反打、克爆发 |
| 续航滚雪球 | `lifesteal` | 持续作战、残局翻盘 |

固定特质不应另起炉灶，而应服务于这 8 个词条，让不同灵宠更适合不同词条构筑。

### 2.3 当前胜负判定口径

结合 [2026-05-30 灵宠互撞对战与数值模拟设计 v0.1.md](/d:/TrinityWar/docs/design/2026-05-30%20灵宠互撞对战与数值模拟设计%20v0.1.md)，当前最适合保留的胜负判定口径为：

1. 一方先死亡，另一方直接获胜。
2. 双方同回合死亡，防守方获胜，或在前台表现为“相持偏守成”。
3. 达到第 10 回合后双方都存活，则按“剩余血量比例”判定胜负，而不是按绝对剩余血量。

推荐判定公式：

```text
attackerHpRatio = attackerHp / attackerMaxHp
defenderHpRatio = defenderHp / defenderMaxHp
```

若 `attackerHpRatio > defenderHpRatio`，则进攻方胜。
若 `attackerHpRatio <= defenderHpRatio`，则防守方胜。

如需前台表现更柔和，也可增加一个“相持阈值”：

```text
if abs(attackerHpRatio - defenderHpRatio) <= 3%
  => 相持
```

### 2.4 为什么按剩余血量比例判定更合理

首版互撞战斗中，血量本身已经是最重要的核心资源，因此超时判定必须避免让“高血量宠物”天然吃满所有优势。

不推荐按绝对剩余血量判定的原因：

1. 厚血宠天然会在超时局拥有明显偏置。
2. 所有构筑会过度向堆血倾斜。
3. 爆发宠、残局宠、反击宠在未击杀时容易被系统性低估。

按“剩余血量比例”判定的好处：

1. 更公平，不会让高血模板天然统治超时局。
2. 能保留不同成长模板的价值。
3. 可以让“我虽然血少，但我把你打得更惨”被正确反映。
4. 更适合当前“单宠互撞 + 10 回合上限”的轻量模型。

一句话总结：

> 血量是胜负核心，但胜负比较的应是“有效保血能力”，而不是“谁的血条天生更长”。

### 2.5 血量是核心资源，但不是所有设计都只做加血

固定特质和词条都应围绕血量展开，但不应狭义理解为“都做生命、回血、减伤”。

更准确的说法是：

> 所有设计都应服务于“谁能在 10 回合结束时，留下更多有效血量比例”。

因此，词条和特质应围绕“血量交换关系”设计，而不是只围绕“血量数值”设计。

推荐分成四类理解：

1. `保住自己的血`
2. `更快压低对面的血`
3. `利用血线差做文章`
4. `围绕 10 回合节奏控血`

这四类一起构成当前灵宠系统真正的深度来源。

## 3. 首版设计原则

### 3.1 固定特质必须容易计算

首版固定特质只允许使用以下四类触发条件：

1. 回合条件：如 `1-2 回合`、`6-10 回合`、`奇数回合`、`第 3/6/9 回合`。
2. 自身血线条件：如 `自身血量 <= 50%`。
3. 对手血线条件：如 `目标血量 <= 50%`。
4. 五行判定条件：如 `不受金木水克制`、`被克制时仅攻击翻倍，词条不翻倍`。

首版固定特质不建议引入：

1. 护盾。
2. 复活。
3. 持续伤害。
4. 叠层。
5. 复杂状态机。
6. 需要记录过多中间状态的效果。

### 3.2 固定特质要服务 10 回合节奏

10 回合上限意味着特质最好围绕节奏位来设计：

1. 前压型：第 1-2 回合抢血。
2. 中盘型：第 3-5 回合开始体现价值。
3. 收官型：第 6-10 回合翻盘或终结。

这样战斗更容易出现“前期压制、中盘纠缠、后程翻盘”的观感，不会只是单纯比面板。

### 3.3 稀有度做成风格差异，不做绝对阶级

首版推荐口径：

1. 普通宠：特质更尖、更极端、更容易打出针对局。
2. 稀有宠：特质更稳、更泛用、容错更高。
3. 传说宠：拥有更强的复合特质或规则特质，但不能同时占满所有优点。

一句话总结：

> 普通宠赢在专精对局，稀有宠赢在泛用稳定，传说宠赢在高容错但仍可被拆解。

### 3.4 固定特质不应完全替代词条

固定特质最好做成“放大某些词条价值”，而不是直接包打一切。

例如：

1. 有反击型特质的宠物，`counter` 会更值钱。
2. 有闪避型特质的宠物，`dodge` 才会成为一条像样流派。
3. 有后程回血特质的宠物，`thick_skin`、`lifesteal` 会更有构筑意义。
4. 有残局增伤特质的宠物，`crit_damage`、`claw` 的收益会明显提升。

### 3.5 围绕血量关系设计，而不是只围绕生命数值设计

首版设计时，建议明确区分“血量数值设计”和“血量关系设计”。

不推荐的方向：

1. 大量词条都做成生命提升。
2. 大量特质都做成固定回血。
3. 所有强宠都靠拖回合取胜。

这样会导致：

1. 战斗节奏变慢。
2. 词条和特质差异感不足。
3. 爆发、反击、残局这些流派被边缘化。

更推荐的方向是让不同设计从不同角度影响血量交换关系：

1. `保血型`
说明：让自己在 10 回合内掉血更少。
示例：厚皮、韧性、前段减伤、后段减伤、固定回合回血。

2. `压血型`
说明：让对手更快掉血，争取在 10 回合前分胜负。
示例：开场增伤、奇数回合爆发、前 2 回合攻击强化、对高血目标压制。

3. `血线利用型`
说明：当自己或对手进入某个血量区间时，获得额外收益。
示例：自身低血暴击、低血吸血、目标低血斩杀、血量比例领先时增伤。

4. `超时判定型`
说明：不一定追求击杀，而是追求第 10 回合结束时血量比例更优。
示例：中后程回血、后 5 回合减伤、残局韧性、拖后程反击。

一句话总结：

> 不是所有特质和词条都该做成“加血”，而是所有特质和词条都该影响“最终血量结果”。

## 4. 固定特质系统结构建议

## 4.1 每个灵宠拥有 1 到 3 条固定特质

首版建议按稀有度区分固定特质数量：

| 稀有度 | 固定特质数 | 设计定位 |
|---|---:|---|
| 普通 | 1 条 | 单点尖锐，成型快，适合做针对 |
| 稀有 | 2 条 | 主特质 + 辅特质，形成稳定打法 |
| 传说 | 3 条 | 主特质 + 辅特质 + 规则特质，体现顶级身份 |

这里的关键不是“条数越多越强”，而是：

1. 普通宠单条特质系数可以更高，但触发条件更窄。
2. 稀有宠用两条中强度特质做稳定打法。
3. 传说宠的强点应体现在复合能力和规则改写上，而不是单条数值无脑碾压。

## 4.2 传说宠允许拥有规则特质

规则特质是传说宠最适合承接的设计空间，因为它容易制造身份感，而且实现仍然简单。

推荐可用的规则特质示例：

1. `三才护体`：不受 `金 / 木 / 水` 克制。
2. `逆克定元`：被克制时，仅攻击翻倍，词条不翻倍。
3. `真灵镇势`：被克制时，本场韧性额外 +20。

规则特质应谨慎使用，建议首发只给极少数传说宠，以保证其稀缺感和认知度。

## 4.3 固定特质的结算字段建议

为保证实现简单，固定特质只累加到少数几个统一字段中：

1. `attackBonusPercent`
2. `damageBonusPercent`
3. `damageReductionPercent`
4. `critBonusPercent`
5. `critDamageBonusPercent`
6. `dodgeBonusPercent`
7. `counterBonusPercent`
8. `counterDamageBonusPercent`
9. `lifestealBonusPercent`
10. `tenacityBonusPercent`
11. `healPercentOnRoundEnd`
12. `blockedElementAdvantage[]`
13. `elementAdvantageMode`

这样即使一只传说宠有三条特质，也只是把数值加到统一字段上，不需要写复杂状态逻辑。

## 4.4 固定特质配置表结构建议

建议把固定特质拆成两张配置表：

1. `spirit_innate_trait_definition`
2. `spirit_innate_trait_assignment`

这样做的好处：

1. 特质模板可复用。
2. 不同宠物可以共用同一特质模板，但数值可做轻微差分。
3. 传说宠的规则特质可以单独挂载，不污染普通模板。

### 4.4.1 特质定义表

建议字段如下：

| 字段 | 类型 | 说明 |
|---|---|---|
| `traitId` | string | 特质唯一 ID，如 `swift_hunt` |
| `name` | string | 特质名，如 `疾猎` |
| `category` | enum | `frontload` `backload` `bloodline` `execute` `counter` `recovery` `element_rule` |
| `targetSide` | enum | `self` `target` |
| `triggerType` | enum | `round_range` `round_parity` `round_list` `self_hp_ratio` `target_hp_ratio` `hp_ratio_compare` `on_crit_taken` `element_advantage_taken` `always` |
| `triggerParam1` | number | 触发参数 1，如起始回合、血线阈值 |
| `triggerParam2` | number | 触发参数 2，如结束回合 |
| `triggerListJson` | json | 用于 `3/6/9` 这种离散回合 |
| `effectType` | enum | `attack_bonus` `damage_bonus` `damage_reduction` `crit_bonus` `crit_damage_bonus` `dodge_bonus` `counter_bonus` `counter_damage_bonus` `lifesteal_bonus` `tenacity_bonus` `heal_percent_on_round_end` `block_element_advantage` `modify_element_advantage` |
| `effectValue` | number | 数值，如 `18` |
| `effectValue2` | number | 预留双值效果，如反击率和反击伤害 |
| `blockedElementsJson` | json | 被屏蔽的克制属性列表 |
| `elementAdvantageMode` | enum | `normal` `attack_only` `trait_only` `blocked` |
| `priority` | number | 结算优先级 |
| `rarityBudgetTag` | enum | `common_peak` `rare_stable` `legend_rule` |
| `description` | string | 面向前台描述 |

### 4.4.2 宠物绑定表

建议字段如下：

| 字段 | 类型 | 说明 |
|---|---|---|
| `spiritId` | string | 灵宠 ID |
| `slotIndex` | number | 固定特质槽位，从 1 开始 |
| `traitId` | string | 绑定的模板特质 ID |
| `overrideName` | string nullable | 如某只传说宠需要专属命名，可覆盖显示 |
| `overrideEffectValue` | number nullable | 单宠微调数值 |
| `overrideEffectValue2` | number nullable | 第二数值微调 |
| `isSignature` | boolean | 是否为该宠代表性特质 |
| `notes` | string nullable | 策划备注 |

### 4.4.3 推荐前台组装口径

前台显示时建议：

1. 只展示特质名称和一句话描述。
2. 若有多条特质，按“主特质 -> 辅特质 -> 规则特质”顺序展示。
3. 传说宠可额外展示一个“天命”或“真灵”样式标签，强调规则特质身份感。

## 4.5 宠物落表结构建议

为了方便后续出表、验算和调优，建议每只宠物最终都收成以下结构：

| 字段 | 说明 |
|---|---|
| `spiritId` | 宠物唯一 ID |
| `rarity` | 普通 / 稀有 / 传说 |
| `role` | 攻击 / 均衡 / 血量 |
| `battleIdentity` | 前压、续航、反击、残局、守成等一句话身份 |
| `trait1` | 主特质 |
| `trait2` | 辅特质，普通宠为空 |
| `trait3` | 规则特质，非传说多为空 |
| `recommendedTraits` | 推荐洗练词条方向 |
| `winsAgainst` | 主要克制对象 |
| `fears` | 主要被克制对象 |
| `notes` | 策划校验备注 |

## 5. 结算顺序建议

首版建议按如下顺序结算，便于后续实现：

1. 回合开始时，按当前回合数、自身血线、目标血线，计算双方固定特质加成。
2. 判断五行克制是否被规则特质拦截或改写。
3. 计算攻击、词条、固定特质带来的即时面板。
4. 结算闪避。
5. 结算暴击和韧性。
6. 结算伤害。
7. 结算吸血和反击。
8. 回合末结算固定回血特质。
9. 判断是否分出胜负；若未分出，进入下一回合；第 10 回合结束后按剩余血量比例判胜负。

### 5.1 胜负判定细化建议

推荐把判定进一步收成以下正式口径：

1. `即时击杀判定`
若一方在任意回合结束后生命值 `<= 0`，且另一方仍存活，则另一方立即获胜。

2. `同回合双死判定`
若双方在同一回合结束后都 `<= 0`，则防守方获胜。
原因：防守方在当前偷菜/守成语境下应享有最低守成收益，且该规则更容易解释。

3. `超时判定`
若 10 回合结束后双方都存活，则按剩余血量比例判定，不按绝对剩余血量判定。

4. `相持表现`
若双方剩余血量比例差距极小，则前台可展示为“相持”“鏖战不分”。
推荐阈值：`2% 到 3%`

### 5.2 为什么这套判定会让血量成为中心指标

因为在这套规则下，血量同时承担了四层意义：

1. `生存意义`
血量归零会直接失败。

2. `节奏意义`
血量领先的一方会更接近终结局或斩杀线。

3. `构筑意义`
很多词条和特质都应围绕“掉血速度”和“保血能力”展开。

4. `超时意义`
即使 10 回合内无人死亡，血量比例仍然决定最终输赢。

因此，血量应被视为本系统中的“第一战略资源”。

### 5.3 围绕血量关系的词条与特质设计框架

为了让后续设计更统一，建议把词条和特质都映射到同一个框架中：

| 设计类型 | 目的 | 适合的词条/特质方向 |
|---|---|---|
| 保血型 | 尽量在 10 回合结束时保住更高血量比例 | `thick_skin` `tenacity` 固定回血 回合减伤 |
| 压血型 | 尽量在前中期快速压低对方血量 | `claw` `crit` `crit_damage` 开场增伤 奇数回合爆发 |
| 血线利用型 | 把血量差转化为额外收益 | 低血暴击 低血吸血 低血斩杀 比例领先增伤 |
| 超时判定型 | 不一定击杀，但争取拖到判定时更优 | 后程回血 后程减伤 韧性反打 反击磨血 |

后续设计新词条或新特质时，都可以先问一个问题：

> 它是帮助玩家保血、压血、利用血线，还是赢下超时判定？

只要能回答这个问题，设计就不容易发散。

## 6. 可复用特质模板池

以下模板均符合“容易计算”的原则，可作为首发固定特质基础池。

### 6.1 前压型

1. `疾猎`：`1-2 回合` 攻击 +18%
2. `凶袭`：`1-3 回合` 攻击 +12%，暴伤 +25%
3. `先声`：`1-2 回合` 暴击 +15%

### 6.2 后程型

1. `战酣`：`6-10 回合` 攻击 +18%
2. `压阵`：`6-10 回合` 受到伤害 -10%
3. `终猎`：`6-10 回合` 对低血目标伤害 +18%

### 6.3 血线型

1. `逆鳞`：自身血量 `<= 50%` 时，暴击 +14%
2. `血噬`：自身血量 `<= 50%` 时，吸血 +12%
3. `坚心`：自身血量 `<= 50%` 时，韧性 +15%

### 6.4 目标血线型

1. `裂喉`：目标血量 `<= 50%` 时，伤害 +22%
2. `乘危`：自身血量比例高于对手时，伤害 +15%
3. `追命`：目标血量 `<= 35%` 时，暴伤 +35%

### 6.5 反制型

1. `幻行`：`1-4 回合` 闪避 +10%
2. `震骨`：反击率 +12%，反击伤害 +25%
3. `静御`：被暴击时，本场韧性额外 +10

### 6.6 恢复型

1. `沐露`：`3/6/9 回合` 回合末回复 7% 最大生命
2. `息潮`：`2/5/8 回合` 回合末回复 6% 最大生命
3. `涅羽`：`3/6/9 回合` 回合末回复 8% 最大生命

### 6.7 五行规则型

1. `三才护体`：不受 `金 / 木 / 水` 克制
2. `逆克定元`：被克制时，仅攻击翻倍，词条不翻倍

## 6.8 固定特质数值范围建议

为了避免固定特质失控，建议首发把不同类型的数值限制在清晰区间内。

### 6.8.1 普通宠数值区间

普通宠强调“尖”，但只在窄条件下生效。

| 类型 | 推荐区间 |
|---|---|
| 前 1-2 回合攻击提升 | `16% - 20%` |
| 前 1-3 回合攻击提升 | `10% - 14%` |
| 血线斩杀增伤 | `18% - 22%` |
| 低血暴击 / 吸血 / 韧性 | `12% - 16%` |
| 前段减伤 | `12% - 15%` |
| 闪避提升 | `8% - 12%` |
| 反击率提升 | `10% - 14%` |
| 反击伤害提升 | `20% - 30%` |
| 固定回合回血 | `6% - 8%` 最大生命 |

### 6.8.2 稀有宠数值区间

稀有宠强调“稳”，因此单条数值略低，但可以双特质叠加。

| 类型 | 推荐区间 |
|---|---|
| 前中后段攻击提升 | `10% - 16%` |
| 血线斩杀增伤 | `12% - 18%` |
| 低血暴击 / 吸血 / 韧性 | `10% - 15%` |
| 回合减伤 | `8% - 12%` |

| 反击伤害提升 | `18% - 25%` |
| 固定回合回血 | `5% - 7%` 最大生命 |

### 6.8.3 传说宠数值区间

传说宠强调“复合强度”和“规则身份”，不是每条都拉到最大。

| 类型 | 推荐区间 |
|---|---|
| 前中后段攻击提升 | `8% - 14%` |
| 血线斩杀增伤 | `12% - 18%` |
| 低血暴击 / 吸血 / 韧性 | `10% - 20%` |
| 回合减伤 | `8% - 12%` |
| 固定回合回血 | `6% - 8%` 最大生命 |
| 规则特质 | 只给 1 条，且必须保留拆解点 |

### 6.8.4 规则特质强度建议

规则特质是最容易出问题的部分，建议明确限制：

1. `完全免全部五行克制`：首发不开放。
2. `免 3 属性克制`：只允许 1 只传说宠拥有。
3. `被克制时词条不翻倍`：可以给厚血型传说。
4. `被克制时额外韧性`：可以给守成型传说。
5. 规则特质不要和最极端爆发模板叠在同一只宠上。

## 8. 普通、稀有、传说的强弱口径

为避免普通宠彻底退环境，建议按下述口径控制平衡：

### 8.1 普通宠

1. 单条固定特质系数可高于传说宠的单条系数。
2. 触发条件更窄，因此打错人会弱，打对人会很猛。
3. 应保留若干“抓传说”的专精位，例如前压爆发、反击反制、残局斩杀。

### 8.2 稀有宠

1. 整体表现更稳定，适合当主流泛用主宠。
2. 不一定在极端对局中强过普通宠，但更少出现完全废局。
3. 双特质组合应围绕“稳”和“完整打法”展开。

### 8.3 传说宠

1. 允许拥有强特质、复合特质、规则特质。
2. 不能同时具备最强爆发、最强承伤、最强续航。
3. 必须保留明确拆解点：五行克制、对症词条构筑、专精普通宠、特定稀有宠。

一句话约束：

> 普通宠打对局应能赢传说，传说宠多数局更稳，但不应在所有局都最优。

## 9. 五行与传说特质的关系建议

五行是当前灵宠系统中最重要的“强克制锚点”，不建议被传说特质完全抹掉。

推荐口径：

1. 大多数传说宠仍应正常吃五行克制。
2. 仅允许极少数传说宠拥有“部分抗克制”规则特质。
3. 规则特质最好是“免部分克制”或“弱化部分克制”，而不是“免全部克制”。
4. 即使有规则特质，也不建议同时给该宠最极端的爆发模板。

推荐的三种强规则表现：

1. `免三属性克制`：强身份感，但只能给一只。
2. `被克制时词条不翻倍`：强但可接受，适合厚血型传说。
3. `被克制时获得韧性`：偏防守向，适合守成型传说。

## 10. 首版实现建议

若要控制开发量，建议分两步上：

### 10.1 第一步

先实现以下 6 类固定特质效果：

1. 回合区间攻击提升
2. 回合区间减伤
3. 血线条件增伤
4. 血线条件暴击 / 韧性 / 吸血
5. 反击率 / 反击伤害提升
6. 固定回合回血

这样就已经足够把 18 只宠物差异立起来。

### 10.2 第二步

再补传说宠的规则特质：

1. 屏蔽部分五行克制
2. 弱化被克制时的词条翻倍
3. 被克制时附加韧性

这样可以把复杂度集中在极少数传说宠上，而不是让所有宠物都变复杂。

### 10.3 首版数值校验建议

正式实装前，建议至少验证以下几类对局：

1. `普通前压宠 vs 稀有续航宠`
目标：确认普通宠在打对局时真的有机会赢。

2. `普通反击宠 vs 传说爆发宠`
目标：确认传说宠不会无脑碾压。

3. `传说守护宠 vs 五行强克普通宠`
目标：确认规则特质不会抹掉强克制价值。

4. `厚血拖后宠 vs 斩杀宠`
目标：确认“血量比例判定”不会把所有局都导向堆血。

5. `同模板不同词条构筑`
目标：确认固定特质不会压过词条选择空间。

### 10.4 首版建议重点观察指标

1. 超时局占比是否过高。
建议目标：`20% - 35%`

2. 传说宠对普通宠胜率是否过高。
建议目标：平均不应长期高于 `65% - 70%`

3. 某一类特质是否明显统治环境。
如固定回血、前压爆发、反击流是否出现一边倒。

4. 五行克制是否仍然是决定性因素之一。
若规则特质让五行失去存在感，应立即回调。

## 11. 当前推荐结论

当前最推荐的整体方案如下：

1. 保留现有 8 词条和 5 词条位，不另起复杂系统。
2. 新增固定特质系统，并允许灵宠按稀有度拥有 1 到 3 条固定特质。
3. 普通宠做尖锐专精，稀有宠做稳定组合，传说宠做复合强特质和规则强特质。
4. 固定特质全部基于“回合、血线、五行规则”三类简单条件实现。
5. 传说宠允许很强，但必须仍有拆解点，尤其不能完全抹掉五行系统。

一句话总结：

> 灵宠的乐趣不再只是“谁面板高”，而是“这只宠天生擅长什么、我把它洗成什么、它在这局里能不能抓到对的人”。
