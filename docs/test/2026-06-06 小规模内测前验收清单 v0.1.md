# 2026-06-06 小规模内测前验收清单 v0.1

## 一、目标

本清单用于把“功能是否已经开发”转换成“真实玩家路径是否能跑通”。

通过标准：一名测试玩家可以从进入游戏开始，完成阵营、种田、灵宠、掠夺、通知附件、赛季展示和后台排障闭环，过程中不出现白屏、死链、资源异常、重复领取或后台无法定位问题。

## 二、验收前准备

本地环境：

- Docker Desktop 已启动。
- Redis 容器 `trinitywar-redis` 已运行。
- `docker exec trinitywar-redis redis-cli ping` 返回 `PONG`。
- 后端健康检查 `http://127.0.0.1:4000/api/health` 正常。
- 玩家端 `http://127.0.0.1:6001` 可以打开。
- 后台端 `http://127.0.0.1:6002` 可以打开。

建议同时启动：

```powershell
npm run dev
npm run worker:raid-settlement --workspace @trinitywar/game-server
```

如果没有常驻 worker，每次掠夺后需要执行：

```powershell
npm run worker:raid-settlement:sweep --workspace @trinitywar/game-server
```

## 三、自动化脚本先跑

先跑脚本，确认基础链路没有明显断裂：

```powershell
npm run build --workspace @trinitywar/shared
npm run build --workspace @trinitywar/game-client
npm run build --workspace @trinitywar/game-admin
npm run build --workspace @trinitywar/game-server
npm run verify:notifications --workspace @trinitywar/game-server
npm run verify:spirit-flow --workspace @trinitywar/game-server
npm run verify:raid-social --workspace @trinitywar/game-server
npm run verify:raid-worker --workspace @trinitywar/game-server
npm run verify:admin-boundary --workspace @trinitywar/game-server
npm run verify:admin-dangerous-audit --workspace @trinitywar/game-server
npm run verify:admin-resource-adjust --workspace @trinitywar/game-server
```

主循环脚本需要本地后端已启动：

```powershell
npm run verify:test-flow --workspace @trinitywar/game-server
```

## 四、手动验收路径

### 1. 新玩家进入

- 使用测试登录方式进入玩家端。
- 确认没有白屏、接口 500、明显 mock 占位或空数据卡死。
- 确认新玩家基础信息、钱包和初始任务正常出现。

### 2. 选择阵营

- 选择任一阵营。
- 刷新页面后阵营状态仍然保留。
- 后台玩家详情能看到阵营与贡献基础信息。

### 3. 种田收菜

- 查看田地和种子。
- 种下一颗可用种子。
- 到期后收获。
- 确认钱包、库存、田地状态刷新正确。
- 后台玩家详情能看到种田记录、订单或相关流水。

### 4. 灵宠链路

- 进入灵宠页面。
- 购买或获得灵宠资源。
- 执行灵宠升级。
- 执行恢复或设置主灵宠。
- 刷新后确认灵宠状态、资源余量和版本号一致。
- 后台玩家详情能看到灵宠资源变化。

### 5. 掠夺链路

- 打开目标列表。
- 查看目标详情和深度情报。
- 发起一次掠夺。
- 等待 worker 结算，或手动执行补扫。
- 查看战报、留言和战报附言。
- 确认奖励、损失、通知和战报内容一致。

### 6. 通知与附件

- 后台发送一条全服通知。
- 后台向指定玩家发送一条带附件通知。
- 玩家端查看通知列表。
- 领取附件。
- 刷新后确认附件不能重复领取。
- 后台审计能看到发送记录。

### 7. 赛季展示

- 查看当前赛季信息。
- 查看签到状态。
- 查看赛季奖励和奖励历史。
- 查看奖章陈列柜。
- 后台赛季只读页面能查到当前赛季、排行和奖励历史。

### 8. 后台排障闭环

- 后台按玩家 ID、昵称或登录标识定位玩家。
- 查看玩家概览、钱包、种子、灵宠、种田、订单、任务和日志。
- 对测试玩家执行一次小额资源修正，原因填写明确中文说明。
- 确认玩家端资源刷新正确。
- 系统状态页的后台审计记录能查到本次资源修正。

## 五、失败判定

出现以下任一问题，本轮内测前验收不通过：

- 玩家端白屏或核心页面无法进入。
- 任一核心接口返回 500 且无法通过刷新恢复。
- 前端出现未预期 mock 数据，导致无法判断真实链路。
- 金币、灵宠资源、天机符、阵营贡献等核心资源出现负数。
- 通知附件可重复领取。
- 掠夺订单长期卡住且补扫无法修复。
- 后台无法定位测试玩家。
- 危险写操作没有原因、确认或审计记录。
- 赛季奖励展示与后台奖励历史明显不一致。

## 七、暂不纳入本轮

- 7 天/14 天经济模拟。
- 机器人循环压测。
- 正式后台账号、角色和细粒度权限体系。
- 赛季奖励手动生成、补发缺失奖励、撤销未领取奖励。
