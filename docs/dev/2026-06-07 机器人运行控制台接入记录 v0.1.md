# 机器人运行控制台接入记录 v0.1

日期：2026-06-07

## 一、目标

让后台机器人循环任务运行时可观察：

- 页面展示类似控制台的滚动日志。
- 能看到每个机器人动作正在产生的结果。
- 能区分成功、成长卡点和硬错误。
- 不引入 WebSocket，先用 dashboard 轮询实现。

## 二、实现范围

后端：

- `GET /api/admin/robots/dashboard` 的 `recentActions` 从最近 20 条扩展到最近 100 条。
- 不新增数据表。
- 不改变机器人动作日志结构。

前端：

- `apps/game-admin/src/views/RobotTestView.tsx`
- `apps/game-admin/src/styles.css`

新增区域：

- 运行控制台。
- 自动刷新开关。
- 自动滚动开关。
- 滚到底部按钮。
- 清空控制台按钮。

## 三、显示规则

日志来源：

- `recentActions.items`
- `automation.loop`

颜色：

- 成功：绿色。
- 成长卡点：黄色。
- 硬错误：红色。
- 循环状态：蓝色。

刷新：

- 循环运行中默认每 2 秒刷新一次 dashboard。
- 自动刷新只在页面打开且循环运行中生效。
- 最新消息固定在控制台底部。
- 自动滚动默认开启；开启时每次 dashboard 刷新后都会滚到底部。
- 关闭自动滚动后可停留在旧日志位置排查。
