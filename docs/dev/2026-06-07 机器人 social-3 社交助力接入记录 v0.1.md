# 机器人 social-3 社交助力接入记录 v0.1

日期：2026-06-07

## 背景

在 `daily-3` 已经具备手动单轮、后台循环、持久化任务和开服自动启动配置后，继续补齐社交类机器人测试场景，用来覆盖好友关系、好友灵田拜访、好友采摘/浇水助力。

## 本次接入范围

- 新增 `social-3` 机器人测试模式。
- 复用三阵营日常机器人账号：
  - `robot-human-001`
  - `robot-immortal-001`
  - `robot-demon-001`
- 测试前准备三名机器人之间的环形好友关系。
- 每名机器人拜访下一名机器人的灵田，并通过真实 `SocialService` 执行一次好友田地助力。
- 后台新增 `POST /api/admin/robots/social-3` 手动触发入口。
- 后台新增 `social-3` 自动调度配置和循环启动入口：
  - `GET /api/admin/robots/social-3/automation-config`
  - `PATCH /api/admin/robots/social-3/automation-config`
  - `POST /api/admin/robots/social-3/loop/start`
- 管理后台机器人测试页新增“跑社交 1 轮”按钮。
- 管理后台全自动调度页新增 `social-3` 配置块和“开始社交循环”按钮。
- 后台和前端补充 `social`、`friend-link`、`friend-field-assist` 的展示标签。

## 关键实现

- `RobotService.runSocial3()`
  - 创建 `RobotTestRun`，模式为 `social-3`。
  - 登录并准备三名机器人玩家。
  - 写入好友关系准备动作 `friend-link`。
  - 执行三次 `friend-field-assist`。
- `prepareSocialRobotPlayer()`
  - 基于 daily 机器人资源准备。
  - 重置 1 号成熟田的周期起点，保证重复运行时不会被上一轮助力记录拦住。
  - 准备 2 号成长田，作为后续浇水动作的备用覆盖面。
- `upsertRobotFriendPair()`
  - 使用 Prisma 枚举写入双向 `FRIEND/ACTIVE` 关系。
- 自动调度
  - `RobotLoopState` 增加 `mode`。
  - 自动循环按 `daily-3` / `social-3` 分发运行。
  - 同一进程同一时间只允许一个自动循环运行，避免两条场景同时改同一批机器人账号。

## 边界说明

- 当前 `social-3` 已是可手动触发、可后台循环、可开服自动启动的真实服务链路。
- 当前同一时间只运行一个自动循环；这是为了保护机器人账号状态一致性。
- 下一步可以继续扩展 `raid-3` / `season-3` / `share-assist-3`，并在调度层增加串行任务队列，让多个场景按顺序自动跑。
