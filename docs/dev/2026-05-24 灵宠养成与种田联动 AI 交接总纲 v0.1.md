# TrinityWar 灵宠养成与种田联动 AI 交接总纲 v0.1

## 一、文档目的

这份文档是给 AI 执行灵宠与种田联动相关开发任务时使用的交接总纲。

它只做一件事：把已经拍板的玩法口径压成一组可直接执行的固定规则，避免 AI 在后续前后端实现时重新发散。

## 二、固定前提

以下内容视为已定，不再重新讨论：

1. 灵宠可以打架，是战斗系统的一部分。
2. 灵宠挂机获得经验，经验满自动升级。
3. 到达 10/20/30/40/50 级突破节点后，经验停止增长，不缓存溢出经验。
4. 突破必须手动点击，只消耗对应品质兽魂。
5. 兽魂只用于突破，不用于补经验、洗练或作物掉落。
6. 种田只产灵根、灵髓、灵玉。
7. 灵根用于喂食，灵髓用于基础洗练，灵玉用于锁定和定向洗练。
8. 每 10 级解锁一个词条槽，50 级最多 5 个词条。
9. 自然解锁词条不重复，洗练允许重复，重复不衰减。
10. 五行克制是强规则，本场战斗攻防血和普通战斗词条效果 x2。
11. 闪避和秒杀是前置判定，不吃五行翻倍。
12. 掠夺前默认不知道对方属性，需要通过窥视查看。
13. 掠夺失败不丢已有资源，主要惩罚是出战灵宠扣血。
14. 保护期内禁止被掠夺，保护期天赋继续保留。
15. 商业入口只保留天机符，不再额外叠第二套商业货币。

## 三、资源口径

### 1. 种田产出

| 作物品质 | 主要产出 | 额外概率掉落 |
| --- | --- | --- |
| 普通 | 灵根 x5~10 | 无 |
| 稀有 | 灵髓 x2~4 | 无 |
| 传说 | 灵玉 x1~2 | 无 |

植物不产兽魂。收菜只负责喂养和洗练材料，掠夺才是突破材料的主来源。

### 2. 灵根喂食

前端只保留两个按钮：

1. 投喂一次：最多消耗 10 灵根，饱食 +2 小时。
2. 补满：按当前剩余饱食时间折算消耗，把饱食补到 8 小时。

固定规则：

```text
每 1 小时饱食 = 5 灵根
每 1 灵根 = 当前等级经验条的 0.5%
饱食上限 = 8 小时
饱食期间挂机经验 +50%
倍率不叠加
```

如果灵宠卡在突破节点但尚未突破，不再获得挂机或喂食经验。

### 3. 洗练材料

1. 基础洗练：5 灵髓 + 金币，重置全部已解锁词条。
2. 高级洗练：1 灵玉 + 10 灵髓 + 金币，锁定 1 条后重置其余。
3. 终极洗练：5 灵玉 + 20 灵髓，指定某槽位变成目标词条。
4. 连续洗练：先扣材料，再连续执行 10 次基础洗练，最后选择一次结果。

## 四、成长和突破

1. 灵宠挂机经验持续可见。
2. 经验满后自动升级。
3. 到达突破节点后停止经验增长。
4. 不缓存溢出经验。
5. 玩家手动突破后才继续获得经验。

突破消耗：

| 突破节点 | 消耗 |
| --- | --- |
| 10 级突破 | 普通兽魂 x5 |
| 20 级突破 | 普通兽魂 x12 |
| 30 级突破 | 稀有兽魂 x10 |
| 40 级突破 | 稀有兽魂 x20 |
| 50 级突破 | 传说兽魂 x8 |

## 五、词条口径

首发 10 条基础词条：

| 词条 | 效果 |
| --- | --- |
| 利爪 | 攻击 +10% |
| 厚皮 | 生命 +10% |
| 硬甲 | 防御 +10% |
| 暴击 | 暴击率 +6% |
| 爆伤 | 暴击伤害 +20% |
| 闪避 | 闪避率 +5% |
| 反击 | 受击 +10% 概率反击，造成 50% 伤害 |
| 吸血 | 造成伤害的 10% 回复自身 |
| 破甲 | 攻击无视目标 10% 防御 |
| 韧性 | 受暴击时伤害降低 10% |

后续可扩展秒杀、护盾、反震、复苏、克制强化、克制抵抗等词条。秒杀规则为：命中后 1% 概率直接击败目标，可重复，5 条就是 5%，不受五行翻倍。

## 六、战斗和掠夺口径

战斗采用单次对撞，不做先后手和多回合。

触发顺序固定：

1. 闪避判定。
2. 秒杀判定。
3. 五行克制临时数值结算。
4. 普通伤害、暴击、爆伤、防御、破甲结算。
5. 吸血结算。
6. 反击结算。
7. 胜负判定。

五行克制：

```text
克制方本场战斗临时数值 x2
无克制关系时双方战力不变
```

掠夺规则：

1. 掠夺目标由系统算法匹配。
2. 玩家窥视前不知道对方灵宠属性和五行。
3. 掠夺失败不丢已有资源，只扣出战灵宠血量。
4. 血量不足的灵宠暂时不能参与战斗。
5. 保护期内禁止被掠夺。
6. 被克制的一方可以换上克制宠物复仇。

兽魂获取：

| 对方灵宠品质 | 掉落规则 |
| --- | --- |
| 普通灵宠 | 概率掉普通兽魂 |
| 稀有灵宠 | 概率掉普通兽魂，较低概率掉稀有兽魂 |
| 传说灵宠 | 概率掉普通兽魂，较低概率掉稀有兽魂，极低概率掉传说兽魂 |

## 七、最小执行顺序

后续 AI 继续干活时，建议严格按这个顺序：

1. 先做前后端共同认可的字段清单。
2. 再做后端数据模型与接口。
3. 再做前端页面与交互。
4. 最后补数值、文案和扩展词条。

## 八、AI 输出要求

后续让 AI 做任务时，必须让它输出：

1. 读过哪些本文档和配套文档。
2. 本轮只改了哪些范围。
3. 具体做了哪些接口或页面调整。
4. 验证了什么。
5. 还剩哪些未完成项。
