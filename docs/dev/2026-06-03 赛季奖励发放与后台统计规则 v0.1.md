# 2026-06-03 赛季奖励发放与后台统计规则 v0.1

## 一、文档目的

本文记录当前已落地的赛季奖励规则、通知发放链路、奖章/成就记录方式，以及后台只读统计入口。

这份文档用于策划、运营和开发核对当前实现。后续如果调整奖励数值，应同步修改代码和本文。

## 二、发放链路

当前赛季奖励不是直接打到玩家资源里，而是先发放到通知中，由玩家自行领取。

流程如下：

1. 赛季长度为 28 天。
2. 赛季切换前生成玩家赛季快照。
3. 系统根据上一个赛季的快照生成玩家赛季奖励单。
4. 每个奖励单会创建一条玩家通知。
5. 通知内包含可领取附件。
6. 玩家领取通知后，附件资源才实际入账。
7. 通知领取成功后，对应奖励单状态更新为 `claimed`。

赛季奖励通知有效期：赛季结束后 35 天。

玩家可见文案要求：

- 通知标题、通知正文、附件名称默认使用中文。
- 玩家可见内容全部使用中文。
- 旧接口中如存在 `nameEn`、`titleEn` 等英文兼容字段，只用于历史兼容或内部排障，不进入玩家端展示。

## 三、奖励结算原则

同一玩家、同一赛季、同一领域只发最高命中的一档奖励。

当前领域包括：

- 基础参与
- 种田
- 养宠
- 探索/战斗
- 贡献

复杂成就暂不实现，例如最高伤害、最快战斗、最稀有掉落、最佳防守记录等。这些需要更细的战斗过程统计，目前赛季快照中没有对应字段。

## 四、基础参与奖励

奖励类型：`participation`

满足以下任一条件即可获得：

- 赛季签到天数不少于 3 天
- 赛季登录/活跃天数不少于 3 天
- 阵营贡献不少于 100
- 收获次数不少于 3 次
- 已结算探索/掠夺次数不少于 1 次

| 档位 | 奖励 |
| --- | --- |
| `season-participation` | 天机符 x1，普通兽魂 x2 |

## 五、种田领域奖励

奖励类型：`domain_farming`

结算指标：`harvestCount`

| 档位 | 条件 | 奖励 | 是否记录奖章/成就 |
| --- | ---: | --- | --- |
| `season-farming-bronze` | 收获次数 >= 3 | 普通兽魂 x3 | 是 |
| `season-farming-silver` | 收获次数 >= 20 | 普通兽魂 x8 | 是 |
| `season-farming-gold` | 收获次数 >= 50 | 稀有兽魂 x3 | 是 |

## 六、养宠领域奖励

奖励类型：`domain_spirit`

结算指标来自赛季快照：

- `spiritOwnedCount`：赛季结算时持有的灵宠数量
- `maxSpiritLevel`：赛季结算时最高灵宠等级

| 档位 | 条件 | 奖励 | 是否记录奖章/成就 |
| --- | ---: | --- | --- |
| `season-spirit-bronze` | 持有灵宠数量 >= 1 | 普通兽魂 x3，苍狼精魄 x3 | 是 |
| `season-spirit-silver` | 最高灵宠等级 >= 10 | 普通兽魂 x8，苍狼精魄 x5 | 是 |
| `season-spirit-gold` | 最高灵宠等级 >= 20 | 稀有兽魂 x2，苍狼精魄 x8 | 是 |

## 九、通知附件资源类型

当前赛季奖励通知附件支持以下类型：

植物精华不再作为赛季奖励附件类型；如历史通知中存在旧记录，仅按历史兼容处理。

| 类型 | 含义 |
| --- | --- |
| `tianjiTalisman` | 天机符 |
| `spiritSoul` | 通用兽魂 |
| `ordinarySoul` | 普通兽魂 |
| `rareSoul` | 稀有兽魂 |
| `legendarySoul` | 传说兽魂 |
| `spiritShard` | 灵宠精魄/碎片 |
| `medal` | 赛季奖章展示附件 |

`medal` 是展示用附件，不单独增加资源数值。实际可追踪的成就记录写入 `player_season_achievement`。

## 十、奖章和成就记录

领域奖励会自动创建赛季成就记录。

当前记录领域：

- `farming`
- `spirit`
- `combat`
- `contribution`

每条成就记录包含：

- 玩家 ID
- 赛季编号
- 领域
- 成就 key
- 标题与描述
- 贡献快照
- 领域统计快照
- 关联的奖励单 ID

## 十一、后台只读接口

现有赛季后台接口：

- `GET /admin/seasons/current`
- `GET /admin/seasons`
- `GET /admin/seasons/:seasonNumber/player-snapshots`
- `GET /admin/seasons/:seasonNumber/faction-snapshots`
- `GET /admin/players/:playerId/season-state`
- `GET /admin/players/:playerId/season-history`

新增奖励与成就后台接口：

- `GET /admin/seasons/:seasonNumber/reward-summary`
- `GET /admin/seasons/:seasonNumber/reward-grants`
- `GET /admin/seasons/:seasonNumber/achievements`

奖励单列表支持筛选：

- `playerId`
- `rewardType`
- `status`
- `page`
- `pageSize`

成就列表支持筛选：

- `playerId`
- `domain`
- `page`
- `pageSize`

后台前端的赛季页面已经接入：

- 当前赛季奖励统计
- 奖励单列表
- 通知领取状态
- 通知过期时间
- 成就/奖章列表

客户端已接入：

- 顶部功能栏“奖章”入口。
- `第N赛季奖章陈列柜`，例如 `第2赛季奖章陈列柜`。
- 按赛季分组展示玩家已获得的奖章。
- 每枚奖章展示中文标题、领域和中文描述，不展示英文对照标题。

## 十二、当前实现边界

本版本已经实现：

- 按赛季快照自动生成奖励单。
- 奖励发放到通知附件。
- 玩家领取通知后资源入账。
- 奖励单状态随通知领取更新。
- 赛季奖章/成就记录。
- 后台只读统计与查看。

本版本暂未实现：

- 后台手动补发某个玩家的赛季奖励。
- 后台撤销未领取赛季奖励。
- 复杂战斗成就。
- 赛季奖励规则后台可配置化。
