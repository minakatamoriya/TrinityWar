# 2026-06-01 赛季后台功能清单 v0.1

## 目标

赛季后台先支持查询和排障，再支持修改和补发。

后台不应该提供任意改玩家资源的入口。高风险操作应通过“重算、重建奖励单、补发”完成，并记录审计日志。

## P0：只读查询

### 1. 赛季概览

接口：

- `GET /admin/seasons/current`
- `GET /admin/seasons`

能力：

- 查看当前赛季号。
- 查看开始/结束时间。
- 查看当前周。
- 查看历史赛季列表。
- 查看玩家重置进度统计。

### 2. 玩家赛季状态

接口：

- `GET /admin/players/:playerId/season-state`

能力：

- 查看玩家当前赛季号。
- 查看玩家最后重置赛季号。
- 查看玩家是否需要重置。
- 查看玩家当前个人贡献。
- 查看玩家赛季签到本地状态暂不可查，后续迁移到服务端后再补。

### 3. 赛季快照查询

接口预留：

- `GET /admin/seasons/:seasonNumber/player-snapshots`
- `GET /admin/seasons/:seasonNumber/faction-snapshots`

P0 可以先不实现，等快照表落地。

### 4. 奖励记录查询

接口预留：

- `GET /admin/seasons/:seasonNumber/reward-grants`
- `GET /admin/players/:playerId/season-rewards`

P0 可以先不实现，等奖励表落地。

## P1：受控操作

### 1. 赛季配置

能力：

- 创建下一个赛季。
- 调整未开始赛季的时间。
- 修改赛季状态。

限制：

- 进行中和已结束赛季不能随便改时间。
- 必须写审计日志。

### 2. 结算操作

能力：

- 手动触发赛季结算。
- 重跑单玩家结算。
- 查看结算失败列表。

限制：

- 已领奖的奖励单不能被静默覆盖。
- 重跑必须幂等。

### 3. 奖励操作

能力：

- 重建奖励单。
- 后台补发奖励。
- 标记奖励异常。

限制：

- 不允许直接编辑玩家背包。
- 必须基于奖励单入账。
- 必须记录操作者、原因、前后状态。

## P0 开发范围

本阶段只做：

1. 当前赛季查询。
2. 赛季列表查询。
3. 玩家赛季状态查询。
4. 后台 overview 增加 season 模块标识。
5. 验证脚本覆盖后台接口可用。

不做：

1. 修改赛季时间。
2. 手动触发结算。
3. 奖励生成。
4. 奖励补发。
5. 任意玩家资源编辑。
