# 机器人 daily-3 资源来源与养宠突破接入记录 v0.1

日期：2026-06-07

## 一、目标

让后台机器人测试从单点 smoke 推进到轻量日常循环：

- 只使用 3 个机器人账号：人界、仙界、魔界各 1 个。
- 每轮模拟玩家日常操作：收菜、领俸禄、互相掠夺、养宠成长。
- 页面只关注规则、运行状态和问题收集，不展开复杂日志表。
- 突破材料不足记为“成长卡点”，不当作硬错误。

## 二、资源规则

升级兽魂：

- 对应 `spiritSoul`。
- 机器人可用金币购买，避免测试卡在升级消耗。
- daily-3 准备层会补足机器人测试金币，但不直接补突破材料。

突破材料：

- 对应 `ordinarySoul`、`rareSoul`、`legendarySoul`。
- 来源只接入俸禄和掠夺。
- 若材料不足，记录为 `PROGRESSION_BLOCK`，页面展示为“成长卡点”。

成熟时间：

- 为了测试效率，daily-3 准备层每轮放置 1 块成熟田。
- 收菜动作仍走真实业务 service，不跳过业务逻辑。

## 三、接口与命令

后台接口：

- `POST /api/admin/robots/daily-3`：运行一轮 daily-3。
- `GET /api/admin/robots/dashboard`：查看规则、状态、问题收集箱。
- `DELETE /api/admin/robots/errors`：清空历史问题日志。

命令行：

```powershell
npm run robot:daily-3 --workspace @trinitywar/game-server
```

保留旧命令：

```powershell
npm run robot:smoke --workspace @trinitywar/game-server
```

## 六、后续观察点

- 连续多轮后，灵宠会推进到突破节点。
