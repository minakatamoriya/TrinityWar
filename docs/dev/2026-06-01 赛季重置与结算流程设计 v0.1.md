# 2026-06-01 赛季重置与结算流程设计 v0.1

## 目标

赛季结束不是删除数据，而是：

1. 生成历史快照。
2. 生成奖励单。
3. 重置当前赛季竞争状态。
4. 保留永久资产。

## 流程拆分

### 1. 赛季识别

服务端根据固定起点和 28 天周期计算当前赛季。

当前口径：

- S1：2026-05-04 00:00:00 Asia/Shanghai 开始。
- 每 28 天一个赛季。
- 2026-06-01 00:00:00 Asia/Shanghai 进入 S2。

### 2. 玩家懒重置

玩家进入游戏读取 `bootstrap` 时：

1. 计算当前赛季。
2. 读取 `player_season_state`。
3. 如果 `last_reset_season_number < current_season_number`，执行一次玩家赛季重置。
4. 更新 `last_reset_season_number`。

优点：

- 不需要一次性扫描所有玩家。
- 玩家不登录就不做高成本重置。
- 可以天然重试。

风险：

- 赛季结算快照不能依赖懒重置之后的数据。
- 所以结算快照必须在重置前生成，或从历史日志中生成。

### 3. 赛季结束结算

后台或定时任务触发：

1. 将赛季状态标记为 `settling`。
2. 生成阵营快照。
3. 生成玩家快照。
4. 生成奖励单。
5. 将赛季状态标记为 `ended` 或 `archived`。

结算必须幂等：

- 同一玩家同一赛季只能有一份快照。
- 同一玩家同一赛季同一奖励类型只能有一份奖励单。
- 重跑只能覆盖可重建字段，不能重复发放已领取奖励。

### 4. 玩家领奖

玩家登录后：

1. 查询未领取的 `season_reward_grant`。
2. 前端展示上赛季结算。
3. 玩家点击领取。
4. 后端按奖励单入账。
5. 更新奖励单为 `claimed`。

不建议赛季结束时直接自动入账，因为：

- 无法展示奖励来源。
- 失败重试复杂。
- 更容易重复发奖。

## 当前 P0 重置范围

已确认：

- 当前田地状态清空。
- 金币清空。
- 兵力清空。
- 训练队列取消。
- 个人阵营贡献清零。
- 阵营总贡献在新赛季创建时清零。
- 每日任务和阵营任务状态清空。
- 每日俸禄状态清空。
- 掠夺目标池、资产锁、未完成订单清理。
- 分享助力活动过期。
- 灵宠本体和词条保留。
- 灵宠等级、经验、突破、血量重置。

待确认：

- 植物精华是否清零或折算。
- 天机符是否清零或保留。
- 兽魂、灵根、灵髓、灵玉是否清零或保留。
- 建筑和法术等级是否压缩继承。

## 后台操作边界

P0 后台只做查询：

- 查询当前赛季。
- 查询赛季列表。
- 查询玩家赛季状态。
- 查询某赛季重置概况。

P1 再做操作：

- 手动触发结算。
- 重跑单玩家结算。
- 重建奖励单。
- 标记玩家需要重新重置。

所有写操作必须记录审计日志。
