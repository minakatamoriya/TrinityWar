# 2026-06-01 赛季奖励规则设计 v0.1

## 目标

赛季奖励要承接玩家投入感，但不能让上赛季优势直接碾压新赛季。

当前建议：

1. 奖励先生成记录，不直接入账。
2. 玩家登录后主动领取。
3. 首版只做参与奖和贡献档位奖。
4. 排名奖励和追赶机制放到 P1/P2。

## 奖励层级

### 1. 参与奖励

目标：避免普通玩家觉得一个赛季白玩。

建议门槛满足其一即可：

- 赛季签到 >= 3 天。
- 个人赛季贡献 >= 100。
- 完成过至少 3 次收获。
- 完成过至少 1 次掠夺或助力。

建议奖励：

- 少量天机符。
- 少量基础植物精华。
- 赛季纪念记录。

### 2. 贡献档位奖励

目标：奖励持续参与阵营建设的玩家。

建议档位：

- 100 贡献：基础供奉。
- 300 贡献：小有供奉。
- 800 贡献：阵营骨干。
- 1500 贡献：核心成员。
- 3000 贡献：赛季功臣。

奖励方向：

- 少量便利资源。
- 收藏型记录。
- 不直接发大量战力。

### 3. 排名奖励

P0 不做。

P1 可考虑：

- 称号。
- 头像框。
- 赛季徽章。
- 历史档案展示。
- 少量天机符。

不建议：

- 大量金币。
- 大量兵力。
- 大量高阶植物精华。
- 大量灵宠培养资源。
- 直接提升灵宠等级。

## 奖励发放方式

后端生成 `season_reward_grant`：

- `reward_type = participation`
- `reward_type = contribution_tier`
- 后续可扩展 `rank`、`faction_rank`、`compensation`

玩家领取时：

1. 校验奖励单存在。
2. 校验状态为 `generated`。
3. 按 `reward_json` 入账。
4. 更新状态为 `claimed`。

幂等要求：

- 同一玩家同一赛季同一奖励类型只能生成一次。
- 领取接口需要防重复请求。
- 后台补发必须产生审计记录。

## 暂不确定资源

以下资源是否可作为奖励，需要核对经济边界：

- 天机符。
- 植物精华。
- 兽魂。
- 灵根。
- 灵髓。
- 灵玉。

首版倾向只使用少量天机符和基础植物精华，不使用高阶灵宠培养资源。
