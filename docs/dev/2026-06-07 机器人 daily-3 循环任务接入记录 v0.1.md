# 机器人 daily-3 循环任务接入记录 v0.1

日期：2026-06-07

## 一、目标

把 daily-3 从“手动跑一轮”推进到“后台自动循环”：

- 后台可以开始循环。
- 后台可以停止循环。
- 可以配置间隔秒数和最大轮数。
- 每轮继续复用真实 daily-3 业务链路。
- 成长卡点只记录，不停止循环。
- 连续硬错误达到阈值后自动停止。

## 二、当前版本边界

本版本是进程内循环任务：

- 服务端运行期间有效。
- 服务端重启后循环状态会丢失。
- 不写数据库任务配置。
- 不做机器人池扩容。

这些能力留到下一阶段：

- 自动任务持久化。
- 服务重启后的任务恢复。
- 每阵营 N 个机器人池。
- 批量负载统计和资源曲线分析。

## 三、后台接口

- `POST /api/admin/robots/daily-3/loop/start`
- `POST /api/admin/robots/daily-3/loop/stop`
- `GET /api/admin/robots/dashboard`

启动参数：

```json
{
  "intervalSeconds": 10,
  "maxRounds": 20,
  "hardErrorLimit": 3
}
```

参数边界：

- `intervalSeconds`：3 到 3600 秒。
- `maxRounds`：1 到 1000 轮。
- `hardErrorLimit`：1 到 20 次。

## 四、后台页面

机器人测试控制台新增“循环任务”区域：

- 间隔秒数。
- 最大轮数。
- 开始循环。
- 停止循环。
- 已跑轮数。
- 下次运行时间。
- 最后结果。
- 停止原因。

页面仍保持简单，不展示复杂动作日志。
