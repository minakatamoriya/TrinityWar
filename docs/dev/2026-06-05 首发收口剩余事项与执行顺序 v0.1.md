# 2026-06-05 首发收口剩余事项与执行顺序 v0.1

## 一、文档目的

本文用于承接当前项目进入首发收口阶段后的剩余工作。

当前阶段不再继续堆新玩法，优先把已经落地的核心系统整理到“小规模可测、可排障、可运营”的状态。

本文同时用于后续逐项执行。每完成一项，应在对应条目后补充完成记录、验证命令和遗留风险。

## 二、当前判断

### 1. 赛季系统当前可视为阶段收口

已经完成并验证通过：

- 赛季周期识别、玩家赛季状态、赛季重置基础逻辑。
- 赛季快照、阵营快照、玩家快照。
- 赛季奖励单生成、防重复、通知领取、资源入账。
- 赛季奖章/成就记录。
- 奖章陈列柜按赛季分组展示，支持点击查看详情。
- `voided` 奖励对应奖章前台不展示。
- 后台可查看赛季概览、快照、奖励统计、奖励单、成就、玩家奖励历史。
- 后台可只读预览指定玩家赛季奖励，且预览不写入奖励、成就或通知。

已跑通验证：

- `npm run verify:season-rewards --workspace @trinitywar/game-server`
- `npm run verify:admin-season-rewards-readonly --workspace @trinitywar/game-server`

### 2. 赛季后台写操作暂缓

以下三项当前明确暂缓，不纳入本轮必做完成标准：

- 后台手动生成指定赛季奖励。
- 后台补发缺失赛季奖励。
- 后台撤销未领取赛季奖励。

相关模型状态和显示规则已保留，后续需要运营写操作时再单独补。

### 3. 首发收口剩余重点

接下来重点不是扩玩法，而是：

- 后端运行稳定性。
- 通知和奖励端到端回归。
- 掠夺异步结算链路验收。
- 前端真实 API 与 mock 边界收口。
- 后台权限、审计和命名边界收口。
- 小规模内测前主循环验收。

## 三、暂不处理范围

本轮暂不做：

- 复杂排行榜奖励。
- 阵营排名奖励。
- 大型活动系统。
- 完整 GM 系统。
- 大规模数值重构。
- 复杂赛季规则后台配置化。
- 真实商业化邀请裂变扩展。
- 正式多角色后台权限系统的完整产品化。

如后续需要启动，应另开专项文档。
