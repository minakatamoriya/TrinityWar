# 2026-05-28 新手流程脚本化说明 v0.1

## 1. 目标

新手流程不能散落在不同按钮、弹框和页面组件里。后续调整完整流程时，应优先修改集中脚本，再让页面代码消费脚本结果。

当前集中入口：

```text
apps/game-client/src/tutorial/tutorialFlow.ts
```

当前对话文案入口：

```text
apps/game-client/src/dialog/dialogScenes.json
```

## 2. 当前阶段

1. `home`：领取启灵芽。
2. `farm`：种植并收获启灵芽。
3. `spirit`：选择第一只灵宠，注入五行灵力，并获得灵宠。
4. `raid`：完成教程掠夺。
5. `faction`：领取第一次阵营俸禄。
6. `completed`：新手结束，开放后续功能。

已删除法术阶段。社交模块在 `completed` 后才开放，不在新手中介绍。

## 3. 账号与屏蔽

1. 已注册玩家永远是全功能，不走新手 UI 屏蔽。
2. 只有 `isNewUserInTutorial(session, stage)` 返回 `true` 时才启用新手限制。
3. 具体组件不能直接判断 `session.mode` 或 `tutorialStage`，只能消费 App 传入的 `uiRules`。
4. `getTutorialUiRules(stage, isTutorialActive)` 是新手期 UI 显隐和按钮可用性的唯一规则入口。

## 4. 流程动作

所有确认弹框、奖励弹框、解锁弹框、leader 对话和跳页都应通过 `TutorialFlowAction` 编排。

当前事件：

1. `starterSeedsClaimed`
2. `fieldCultivationStarted`
3. `farmRewardConfirmed`
4. `spiritAwardConfirmed`
5. `raidSettled`
6. `factionStipendClaimed`

后续如果要调整为“先对话再弹奖励”或“领奖后先弹解锁再跳页”，优先改 `getTutorialFlowActions()`，不要在组件 JSX 里临时写判断。

## 5. Leader 对话分类

leader 对话分为两类：

1. 必须读：关键规则、关键下一步、会影响玩家理解或操作的说明。必须读不能被设置屏蔽。
2. 非必读：氛围话、补充世界观、非关键提示。后期设置中可以允许玩家屏蔽非必读对话。

`可快进` 不是分类，只是播放属性。必须读和非必读都可以配置为可快进或不可快进。

推荐规则：

1. 新手主线关键节点使用必须读。
2. 必须读默认不允许点遮罩关闭，除非该节点只是轻提示。
3. 非必读允许自动消失、点击遮罩关闭，并且未来可以被设置屏蔽。
4. 不要把“是否可快进”和“是否必须读”混为一类。

## 6. 目标反馈

每个关键动作完成后，都必须给玩家一个明确反馈。

推荐统一反馈链路：

1. 真实操作成功。
2. 刷新客户端数据。
3. 展示奖励、获得或解锁弹框。
4. 玩家确认弹框。
5. 推进新手阶段或播放 leader 对话。

当前已使用的反馈类型：

1. 种子/金币/俸禄：奖励弹框。
2. 第一只灵宠：获得弹框。
3. 新灵植资格：解锁弹框。
4. 阻断操作：toast。

后续新增模块时，不要只用 toast 承载关键奖励或关键解锁。

## 7. 新手资源极简

新手期资源展示继续保持极简：

1. 顶部只保留金币。
2. 隐藏天机符、图鉴、资源背包。
3. 隐藏灵宠碎片数量。
4. 隐藏灵宠养成资源面板。
5. 隐藏掠夺精魄集合。
6. 隐藏会引出长期系统的入口，例如排行、社交、地契、图鉴、商店、灵宠养成 tab。

所有新模块默认 `completed` 后开放，除非明确写入新手流程。

## 8. 教程目标

教程掠夺目标必须由后端固定标记。

当前字段：

```text
ClientRaidTarget.tutorialTarget
ClientRaidTargetDetailResponse.tutorialTarget
```

新手期前端应优先展示 `tutorialTarget === true` 的目标，而不是依赖列表排序。

## 9. 程序改动顺序

后续调整新手流程时，优先按这个顺序改：

1. 修改 `tutorialFlow.ts` 中的阶段、任务文案、场景解锁、阻断提示、UI 规则和流程动作。
2. 修改 `dialogScenes.json` 中对应阶段的 leader 对话。
3. 在 `App.tsx` 里只保留真实操作成功后的事件分发。
4. 如果涉及真实经济或解锁，必须同步检查服务端建档、奖励发放和解锁写入。
5. 如果涉及新手固定目标，必须同步检查后端目标标记和 shared 类型。

不要把新手流程判断直接写进具体场景组件里，除非该组件只负责展示并且通过 props 接收集中规则结果。
