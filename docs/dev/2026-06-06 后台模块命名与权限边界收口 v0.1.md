# 2026-06-06 后台模块命名与权限边界收口 v0.1

## 当前结论

当前后台对外路径继续保持 `/api/admin/*`，暂不迁移已有前端和验证脚本依赖的接口路径。

历史代码目录和类名仍为 `admin-readonly` / `AdminReadonlyModule` / `AdminReadonlyGuard`，但当前后台已经不再是纯只读模块。现阶段把它视为“开发/内测后台入口”，能力边界由 `/api/admin/overview` 的 `adminCapabilities` 字段声明。

## 能力分组

### 只读能力

- 系统状态。
- 玩家搜索与玩家聚合详情。
- 钱包、建筑、田地、订单日志。
- raid order 详情。
- 城堡等级配置读取。
- 分享助力记录读取。
- 赛季概览、快照、奖励、成就和玩家赛季历史读取。

### 配置写能力

- 种子定义创建、修改、删除。
- 灵宠定义创建、修改、删除。
- 任务配置覆盖修改。

### 通知写能力

- 全服通知发送。
- 单玩家通知发送。
- 通知历史和玩家通知历史读取。

### 危险写能力

- 删除玩家。
- 删除种子定义。
- 删除灵宠定义。

这些操作后续进入 P1-5 时需要补充更明确的前端二次确认、审计记录和生产权限策略。

## 鉴权边界

当前后台守卫仍为 `AdminReadonlyGuard`，但行为已经按读写分层：

- 读请求使用 `x-admin-debug-key`。
- 写请求可额外使用 `x-admin-write-debug-key`。
- `NODE_ENV=production` 时写请求必须提供 `ADMIN_WRITE_DEBUG_KEY` 对应的 `x-admin-write-debug-key`。
- 非 production 且未配置写 key 时，写请求仍允许通过，用于本地开发。

后台前端已支持：

- `VITE_ADMIN_DEBUG_KEY`
- `VITE_ADMIN_WRITE_DEBUG_KEY`

## 兼容与迁移

本轮不重命名目录和类名，原因：

- 当前前端、验收脚本和文档已经依赖 `/api/admin/*`。
- TypeScript/Nest 模块大规模改名会扩大变更面。
- 首发收口阶段优先保证边界清晰和可验证。

后续可单独迁移：

- `admin-readonly` 目录改为 `admin`。
- `AdminReadonlyModule` 改为 `AdminModule`。
- `AdminReadonlyGuard` 改为 `AdminDebugGuard` 或正式 `AdminAuthGuard`。
- 保留 `/api/admin/*` 路径不变。

## 验证命令

```powershell
npm run verify:admin-boundary --workspace @trinitywar/game-server
```

验证内容：

- `/api/admin/overview` 服务层返回 `adminCapabilities`。
- 只读、配置写、通知写、危险写分组存在。
- 生产环境写请求必须具备读 key 和写 key。
