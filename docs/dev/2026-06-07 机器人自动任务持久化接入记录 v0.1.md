# 机器人自动任务持久化接入记录 v0.1

日期：2026-06-07

## 一、目标

把 daily-3 后台循环从纯进程内状态推进到可复盘：

- 后台启动循环时创建自动任务记录。
- 每轮运行后更新完成轮数、最近结果和连续硬错误次数。
- 停止循环时记录停止原因。
- 服务重启后不自动恢复任务，只把数据库中残留的 RUNNING 任务标记为中断。
- 后台展示最近自动任务历史。

## 二、实现范围

新增 Prisma 表：

- `robot_automation_job`

字段覆盖：

- 任务名称、模式、状态。
- 间隔秒数、最大轮数、硬错误阈值。
- 已完成轮数、连续硬错误数。
- 最近 run、最近状态、最近错误。
- 停止原因、开始/停止/最近运行/下次运行时间。

后端接入：

- `RobotService.startDaily3Loop` 创建自动任务记录。
- `RobotService.runLoopTick` 每轮后持久化 loop 状态。
- `RobotService.stopDaily3Loop` 写入停止原因。
- `RobotService.getDashboard` 返回 `automation.jobs.items`。
- dashboard 查询时会把残留 `RUNNING` 自动任务标记为 `INTERRUPTED`，原因是“服务重启，进程内循环状态已丢失”。

后台接入：

- 循环任务区新增“硬错误阈值”输入。
- 状态行展示连续硬错误计数。
- 新增“自动任务历史”区域，展示最近任务状态、轮数、硬错误和停止原因。

## 三、当前边界

本版本仍不做：

- 服务重启后自动恢复循环。
- 任务分页和复杂筛选。
- 多任务并行。
- 每阵营机器人池扩容。

服务重启后的处理策略是显式中断，便于测试人员知道上一次自动循环没有自然结束。
