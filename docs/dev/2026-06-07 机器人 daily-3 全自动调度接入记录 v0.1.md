# 机器人 daily-3 全自动调度接入记录 v0.1

日期：2026-06-07

## 一、目标

把 daily-3 从“后台可启动循环”推进到“服务启动后可按配置自动运行”。

本阶段只接 daily-3 单场景，不扩机器人池，不做多任务并行。

## 二、实现范围

新增 Prisma 表：

- `robot_automation_config`

默认写入一条配置：

- `mode=daily-3`
- `enabled=false`
- `autoStartOnBoot=false`
- `intervalSeconds=10`
- `maxRounds=20`
- `hardErrorLimit=3`

后端接入：

- `RobotService` 实现 `OnApplicationBootstrap`。
- 服务启动时会标记历史残留 `RUNNING` 自动任务为 `INTERRUPTED`。
- 如果配置 `enabled=true` 且 `autoStartOnBoot=true`，服务启动后自动调用 daily-3 loop。
- 支持 `ROBOT_AUTOMATION_BOOTSTRAP=disabled` 禁用启动自动调度，便于命令行验证脚本和临时服务使用。

后台接口：

- `GET /api/admin/robots/daily-3/automation-config`
- `PATCH /api/admin/robots/daily-3/automation-config`
- `GET /api/admin/robots/dashboard` 返回 `automation.config`。

后台页面：

- 机器人测试页新增“全自动调度”区。
- 可配置是否启用 daily-3 自动调度。
- 可配置服务启动后自动开始。
- 可配置默认间隔秒数、最大轮数、硬错误阈值。
- 关闭调度时会停止当前循环。

## 三、当前边界

默认配置保持关闭，避免开发服务启动后意外跑机器人。

本阶段不做：

- 自动恢复未完成任务。
- 多场景统一调度。
- 定时 cron。
- 告警通知。
- 每阵营多机器人池。
