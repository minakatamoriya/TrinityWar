# 机器人长期仿真与平衡数据记录接入记录 v0.1

日期：2026-06-07

## 目标

把 `player-sim-v1` 从单轮手动验证推进到可长期循环的平衡性测试入口：

- 机器人持续执行完整勤奋玩家清单。
- 每轮记录玩家状态快照，后续可做趋势分析。
- 支持 `maxRounds=0` 表示不按轮数停止，只由手动停止或硬错误阈值保护。

## 接入内容

### 后端

- `player-sim-v1` 接入自动循环调度：
  - `POST /api/admin/robots/player-sim-v1/loop/start`
  - `GET /api/admin/robots/player-sim-v1/automation-config`
  - `PATCH /api/admin/robots/player-sim-v1/automation-config`
- 调度模式支持：
  - `daily-3`
  - `social-3`
  - `player-sim-v1`
- 自动配置默认值：
  - `player-sim-v1`: `intervalSeconds=5`, `maxRounds=0`, `hardErrorLimit=3`, 默认不启用、不自启。
- 循环停止规则：
  - `maxRounds > 0` 时达到轮数后停止。
  - `maxRounds = 0` 时不因轮数停止。
  - 连续硬错误达到 `hardErrorLimit` 后停止。

### 数据记录

新增表：`robot_sim_snapshot`

每轮为 9 个勤奋机器人各记录一条快照：

- 玩家身份：`runId`, `mode`, `robotKey`, `playerId`, `factionCode`
- 资源：`vaultGold`, `walletGold`, `contributionScore`
- 灵宠：`spiritSoul`, `ordinarySoul`, `rareSoul`, `legendarySoul`, `mainSpiritLevel`, `mainSpiritStage`
- 军队：`armyTotal`, `armyAvailable`, `armyCapacity`, `queuedArmy`
- 农田：`matureFields`, `growingFields`, `emptyFields`
- 本轮动作结果：`successActionCount`, `blockedActionCount`, `failedActionCount`, `actionSummaryJson`

### 管理台

- 机器人测试页增加 `player-sim-v1` 场景卡。
- 支持单轮、开始循环、停止循环、保存调度配置。
- 最大轮数允许填 `0`，语义为无限轮。
- 场景卡网格改为自适应布局，避免三张卡挤乱。

## 验证记录

已执行：

- `npm run prisma:validate --workspace @trinitywar/game-server`
- `npm run prisma:generate --workspace @trinitywar/game-server`
- `npm run build:shared`
- `npm run build --workspace @trinitywar/game-server`
- `npm run build --workspace @trinitywar/game-admin`
- `npx prisma migrate deploy --schema prisma\schema.prisma`

接口验证：

- 手动 `POST /api/admin/robots/player-sim-v1`
  - 生成 1 个 run。
  - `robot_sim_snapshot` 写入 9 条。
- 循环 `POST /api/admin/robots/player-sim-v1/loop/start`
  - `intervalSeconds=3`, `maxRounds=2`, `hardErrorLimit=20`
  - 完成 2 轮后按最大轮数停止。
  - 最后一轮 `robot_sim_snapshot` 写入 9 条。
- 无限轮语义验证：
  - `intervalSeconds=3`, `maxRounds=0`, `hardErrorLimit=1`
  - 不因轮数停止，因连续硬错误阈值停止。

## 当前观察

长测已经能暴露真实卡点。当前验证中可见：

- 农田收获常被阻塞：没有成熟或枯萎地块可领取。
- 灵宠成长/突破链路会产生持续卡点：升级耗时、突破材料不足。
- 部分动作在累积状态后会变成硬错误，需要下一轮继续拆解具体 error group。
