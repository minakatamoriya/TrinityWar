# 2026-05-24 首发轻量化旧口径落点盘点 v0.1

## 一、任务范围

本文件对应 `TW-LITE-001`。本轮只做盘点，不改业务代码。

目标是找出当前代码中仍然依赖旧规则的落点，为后续 `TW-LITE-002` 到 `TW-LITE-012` 逐项改造提供清单。

旧规则包括：

1. 主城税收。
2. 金库容量。
3. 金币溢出。
4. 临时待领取金币。
5. 主城等级开田。
6. 每小时阵营金币分红。
7. 主城、金库、田地位作为升级入口。
8. 待领取金币保留时间相关科技。

## 二、结论摘要

当前旧口径不是单点逻辑，而是贯穿了数据库字段、shared DTO、后端读模型、后端写链路、掠夺结算、玩家初始化、游戏前端 mock、游戏前端交互、管理后台和只读后台接口。

后续不建议直接物理删除字段。第一阶段应先做到：

1. 新增地契与阵营每日俸禄数据结构。
2. shared 契约停止向前端暴露旧语义。
3. 后端读写链路不再使用旧字段驱动业务。
4. 前端和后台不再展示旧规则。
5. 稳定后再单独做数据库瘦身 migration。

## 三、数据库旧字段盘点

### 1. `PlayerWallet`

文件：

1. `services/game-server/prisma/schema.prisma`
2. `services/game-server/prisma/migrations/003_core_current_state/migration.sql`

字段：

```text
vaultCapacity
walletCapacity
walletProtectedRatio
pendingTaxGold
pendingDividendGold
pendingRaidOverflowGold
pendingRaidOverflowExpiresAt
```

处理建议：

1. `vaultCapacity`、`walletCapacity`：本轮业务停用，后续物理删除。
2. `pendingTaxGold`：税收取消，本轮业务停用，后续物理删除。
3. `pendingDividendGold`：每小时分红取消，本轮业务停用；每日俸禄改用新表。
4. `pendingRaidOverflowGold`、`pendingRaidOverflowExpiresAt`：金币无上限后停用，后续物理删除。
5. `walletProtectedRatio`：当前与新设计关系弱，本轮不主动改，后续清理时复核。

### 2. `PlayerBuilding`

文件：

1. `services/game-server/prisma/schema.prisma`
2. `services/game-server/prisma/migrations/003_core_current_state/migration.sql`

字段：

```text
castleLevel
vaultLevel
fieldSlotLevel
protectionTechLevel
farmYieldTechLevel
ripeWindowTechLevel
pendingClaimTechLevel
```

处理建议：

1. `castleLevel`：主城等级退出首发闸门，本轮停用核心规则；可暂留作兼容展示。
2. `vaultLevel`：金库容量升级取消，本轮停用。
3. `fieldSlotLevel`：田地位升级取消，本轮停用。
4. `protectionTechLevel`：保留，改为领地科技。
5. `farmYieldTechLevel`：保留，改为领地科技。
6. `ripeWindowTechLevel`：保留，改为领地科技。
7. `pendingClaimTechLevel`：与待领取金币保留时间相关，本轮停用，后续删除或迁移。
