# TrinityWar 灵宠养成与种田联动 前端开发文档 v0.1

## 一、文档目的

这份文档是给 AI 和前端开发执行用的灵宠与种田联动前端开发文档。

目标是把首发灵宠页、种田产出展示、灵根投喂、突破、洗练、词条、兽魂和掠夺相关交互一次讲清楚，让前端实现时不用自己猜逻辑。

## 三、前端首发页面范围

首发阶段建议把灵宠与种田联动收进这些界面：

1. 灵宠成长页。
2. 种田产出页。
3. 突破弹窗。
4. 洗练弹窗。
5. 灵宠商店页。
6. 掠夺目标和窥视界面。

如果当前客户端工程已经存在对应页面，就在现有页面上改，不要额外发明一套全新导航结构。

## 四、前端必须展示的核心状态

### 1. 灵宠成长卡

至少展示：

1. 当前等级。
2. 当前经验条。
3. 是否卡在突破节点。
4. 当前饱食倒计时和效率 +50%。
5. 当前 HP / maxHp。
6. 当前五行。
7. 当前灵宠品质。
8. 当前兽魂库存。
9. 当前词条列表。
10. 当前基础属性摘要。

到达突破节点后，经验条应明确显示“待突破”，不要继续模拟增长。

### 2. 灵根投喂区

只展示两个按钮：

1. 投喂一次。
2. 补满。

展示规则：

```text
投喂一次：最多消耗 10 灵根，饱食 +2 小时
补满：按当前剩余饱食时间折算消耗，补到 8 小时
每 1 小时饱食 = 5 灵根
每 1 灵根 = 当前等级经验条 0.5%
```

前端需要显示：

1. 当前灵根库存。
2. 当前饱食剩余时间。
3. 投喂一次预计消耗和经验收益。
4. 补满预计消耗和经验收益。
5. 饱食已满时按钮置灰。
6. 卡突破节点时提示“突破后继续获得经验”。

### 3. 种田产出页

收菜展示必须保持简单：

| 作物品质 | 主要产出 | 额外概率掉落 |
| --- | --- | --- |
| 普通 | 灵根 x5~10 | 无 |
| 稀有 | 灵髓 x2~4 | 无 |
| 传说 | 灵玉 x1~2 | 无 |

前端不要展示作物掉兽魂，避免和设计冲突。

### 4. 突破区

至少展示：

1. 当前等级和下一个突破节点。
2. 所需普通 / 稀有 / 传说兽魂数量。
3. 当前兽魂库存。
4. 突破按钮状态。
5. 突破成功后的属性变化和新词条槽提示。

突破区不要展示灵根、灵髓、灵玉作为突破材料。

### 5. 词条区

至少展示：

1. 当前已解锁词条槽。
2. 未解锁槽位的等级提示。
3. 当前词条名称和效果。
4. 洗练按钮和消耗。
5. 洗练结果对比。

首发词条表：

| 词条 | 效果 |
| --- | --- |
| 利爪 | 攻击 +10% |
| 厚皮 | 生命 +10% |
| 硬甲 | 防御 +10% |
| 暴击 | 暴击率 +6% |
| 爆伤 | 暴击伤害 +20% |
| 闪避 | 闪避率 +5% |
| 反击 | 受击 +10% 概率反击，造成 50% 伤害 |
| 吸血 | 造成伤害的 10% 回复自身 |
| 破甲 | 攻击无视目标 10% 防御 |
| 韧性 | 受暴击时伤害降低 10% |

文案要写给普通玩家看，不要只显示内部 code。

### 6. 洗练弹窗

必须展示三类主动洗练：

1. 基础洗练：5 灵髓 + 金币，重置全部已解锁词条。
2. 高级洗练：1 灵玉 + 10 灵髓 + 金币，锁定 1 条后重置其余。
3. 终极洗练：5 灵玉 + 20 灵髓，指定某槽位变成目标词条。

连续洗练：

1. 入口可以放在基础洗练旁边。
2. 必须提示会先扣 10 次基础洗练材料。
3. 展示 10 次结果记录。
4. 玩家最后选择其中一次替换当前词条。

基础洗练和连续洗练都需要二次确认。

### 7. 掠夺和窥视区

掠夺目标由系统算法提供。

窥视前不要展示对方灵宠五行、属性和关键词条。窥视后展示：

1. 对方五行。
2. 对方灵宠品质。
3. 对方关键属性摘要。
4. 是否处于保护期。
5. 五行克制提示。

文案建议：

1. `窥视后可查看对方五行和灵宠品质`
2. `金克木，本场攻防血和普通战斗词条效果翻倍`
3. `保护期内不可掠夺`

掠夺失败提示：

1. 不丢已有资源。
2. 出战灵宠扣血。
3. 血量不足时暂时不能继续战斗。
4. 可等待回血或使用恢复手段。

### 8. 商店区

至少展示：

1. 当前金币。
2. 当前天机符。
3. 当前可购买的材料。
4. 每件商品的用途说明。

天机符商店商品：

| 商品 | 说明 |
| --- | --- |
| 灵根 x100 | 日常补粮 |
| 灵髓 x5 | 洗练基础材料 |
| 灵玉 x1 | 高级洗练材料 |
| 普通兽魂 x10 | 低段突破 |
| 稀有兽魂 x1 | 中段突破 |
| 传说兽魂 x1 | 高层突破 |

## 五、前端交互规则

### 1. 挂机进度表现

1. 经验条必须连续增长，不能只在结算后跳变。
2. 到突破节点后经验条停止，并展示“待突破”。
3. 不要在前端显示或模拟溢出经验。
4. 页面上应明确区分“当前升级进度”和“突破节点状态”。

### 2. 灵根投喂表现

1. 不做数量输入框。
2. 不做三个或更多投喂按钮。
3. 只保留“投喂一次”和“补满”。
4. 补满需要实时显示预计消耗灵根。
5. 投喂后刷新经验、饱食倒计时、灵根库存。

### 3. 突破表现

1. 突破按钮必须手动点击。
2. 未满足兽魂条件时按钮置灰并显示缺口。
3. 突破成功后要弹出明显反馈。
4. 如果玩家不突破，就停在当前节点。

### 4. 词条和洗练表现

1. 自然解锁词条不重复。
2. 洗练允许重复词条。
3. 重复词条效果直接相加。
4. 洗练结果要清楚展示替换前后变化。
5. 高级洗练要先选择锁定词条。
6. 终极洗练要先选择槽位和目标词条。
7. 连续洗练要明确“先扣材料，再展示 10 次结果”。

### 5. 战斗结果表现

战斗报告需要展示触发顺序中的关键事件：

1. 是否闪避。
2. 是否触发秒杀。
3. 是否发生五行克制。
4. 暴击和爆伤。
5. 吸血。
6. 反击。
7. 胜负。
8. 宠物扣血结果。
9. 兽魂掉落和装备奖励。

## 六、前端 API 对接建议

前端页面至少应对接以下几类数据：

1. 主灵宠状态。
2. 灵根、灵髓、灵玉库存。
3. 兽魂库存。
4. 突破节点需求。
5. 当前词条和洗练结果。
6. 商店商品列表。
7. 掠夺目标列表。
8. 窥视结果。
9. 掠夺战斗报告。

建议调用顺序：

1. 页面初始化先拉主灵宠状态和库存。
2. 再拉突破需求、词条定义和商店静态定义。
3. 掠夺页先拉目标列表。
4. 用户窥视时再拉目标详情。
5. 用户操作时再调用 feed / breakthrough / roll-traits / buy / peek / attack 接口。

## 七、前端状态管理建议

1. 主灵宠状态必须以服务端返回为准。
2. 投喂后要刷新经验、饱食时间、库存和突破提示。
3. 突破后要立即刷新等级、属性、词条槽和可突破条件。
4. 洗练后要刷新词条列表和材料库存。
5. 掠夺后要刷新宠物血量、兽魂库存、装备奖励和保护状态。
6. 不要在前端自行计算最终战斗结果。

## 八、非目标

首发前端不要做这些：

1. 复杂天赋树。
2. 多卡组上阵。
3. 繁殖系统。
4. 太多主动技能按钮。
5. 速度作为基础属性。
6. 作物掉兽魂展示。
7. 兽魂补经验入口。
8. 灵髓、灵玉作为突破材料展示。

## 九、最小验收标准

1. 灵宠页可以清楚看到成长条在变。
2. 到突破节点后经验停止并提示手动突破。
3. 灵根喂食只有“投喂一次 / 补满”两个按钮。
4. 种田产出只展示灵根、灵髓、灵玉。
5. 突破只展示兽魂消耗。
6. 词条能展示 10 条首发词条和重复叠加结果。
7. 洗练能完成基础、高级、终极、连续洗练的交互。
8. 掠夺前需要窥视才能知道对方五行。
9. 掠夺失败不显示资源损失，只显示宠物扣血。
10. 战斗报告能展示闪避、秒杀、五行克制、暴击、吸血、反击等关键事件。
