# TrinityWar 灵宠养成与种田联动 前端开发文档 v0.1

## 一、文档目的

这份文档是给 AI 和前端开发执行用的灵宠与种田联动前端开发文档。

目标是把首发灵宠页、种田产出展示、灵粮投喂、突破、洗练、词条与兽魂展示一次讲清楚，让前端实现时不用自己猜交互逻辑。

## 三、前端首发页面范围

首发阶段建议把灵宠与种田联动收进 4 个主要界面：

1. 灵宠成长页。
2. 种田产出页。
3. 突破与洗练弹窗。
4. 灵宠商店页。

如果当前客户端工程已经存在对应页面，就在现有页面上改，不要额外发明一套全新导航结构。

## 四、前端必须展示的核心状态

### 1. 灵宠成长卡

至少展示：

1. 当前等级。
2. 当前经验条。
3. 当前饱灵状态。
4. 当前兽魂库存。
5. 当前突破阶段。
6. 当前可突破条件。
7. 当前词条列表。
8. 当前基础属性摘要。

### 2. 灵粮投喂区

至少展示：

1. 可投喂灵粮数量。
2. 批量投喂输入框或加减器。
3. 单次投喂后经验变化。
4. 单次投喂后饱灵值变化。
5. 预计挂机效率提升提示。

### 3. 突破区

至少展示：

1. 当前等级和下一个突破节点。
2. 所需灵粮 / 灵髓 / 仙髓数量。
3. 所需普通 / 稀有 / 传说兽魂数量。
4. 突破按钮状态。
5. 突破成功后的属性变化提示。

### 4. 词条区

至少展示：

1. 当前词条数量。
2. 当前词条名称。
3. 当前词条品质或稀有度。
4. 是否锁定。
5. 洗练消耗材料。
6. 洗练后的结果对比。

### 5. 商店区

至少展示：

1. 当前金币。
2. 当前天机符。
3. 当前可购买的材料。
4. 每件商品的用途说明。

## 五、前端交互规则

### 1. 挂机进度表现

1. 经验条必须连续增长，不能只在结算后跳变。
2. 前 1 级到 2 级要明显快，让玩家第一眼看到成长。
3. 后续等级逐步放慢，但进度条必须持续变化。
4. 页面上应明确区分“当前挂机经验”和“突破后等级进度”。

### 2. 灵粮投喂表现

1. 支持批量选择投喂数量。
2. 点击一次后展示两个结果：立即经验变化和饱灵值变化。
3. 文案不要写成复杂系统名，直接写“喂养后更高效修炼”即可。
4. 如果饱灵状态生效，页面应展示明确的效率提升文案。

### 3. 突破表现

1. 突破按钮必须是手动点击。
2. 未满足条件时按钮置灰并显示缺口。
3. 突破成功后要弹出明显反馈。
4. 如果玩家不突破，就留在当前节点，不要伪装成自动升阶。

### 4. 兽魂表现

1. 兽魂只显示三档：普通、稀有、传说。
2. 解释文案要直接写出“基础加速 / 中段冲刺 / 高层冲刺”。
3. 不要把兽魂做成多个互相独立的货币标签。

### 5. 词条表现

1. 数值词条和机制词条都可以展示。
2. 基础池先展示暴击、爆伤、闪避、破甲、格挡、坚韧等直观项。
3. 阵营相关词条要放在更醒目的机制区。
4. 词条文案必须让普通玩家看得懂，不要只显示内部 code。

## 六、前端 API 对接建议

前端页面至少应对接以下几类数据：

1. 主灵宠状态。
2. 灵粮库存。
3. 兽魂库存。
4. 突破节点需求。
5. 当前词条和可洗练结果。
6. 商店商品列表。

### 建议调用顺序

1. 页面初始化先拉主灵宠状态和库存。
2. 再拉突破需求和商店静态定义。
3. 用户操作时再调用 feed / breakthrough / roll-traits / buy 接口。

## 七、前端状态管理建议

1. 主灵宠状态必须以服务端返回为准。
2. 批量投喂后要刷新经验、饱灵值、库存和突破提示。
3. 突破后要立即刷新等级、属性、词条槽和可突破条件。
4. 词条洗练后要刷新词条列表和材料库存。

## 八、非目标

首发前端不要做这些：

1. 复杂天赋树。
2. 多卡组上阵。
3. 繁殖系统。
4. 太多主动技能按钮。
5. 速度作为基础属性。

## 九、最小验收标准

1. 灵宠页可以清楚看到成长条在变。
2. 灵粮批量投喂后有立即反馈。
3. 突破必须手动点，且不满足条件时会明确提示。
4. 兽魂三档展示清晰。
5. 词条里能看见暴击、爆伤、闪避和阵营类特殊效果。
