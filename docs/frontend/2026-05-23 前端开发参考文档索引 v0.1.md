# TrinityWar 前端开发参考文档索引 v0.1

## 一、目标

这份文档只回答两件事：

1. 当前前端开发必须依赖哪些文档。
2. 这些文档现在是否已经齐全。

当前前端默认指微信小游戏客户端，也就是 Cocos Creator 客户端工程。

## 二、前端开发必须具备的参考文档

### 1. 产品最小闭环说明

用途：

1. 确认首页应该突出哪些状态。
2. 确认按钮优先级和核心回流路径。
3. 确认前端不该为了次要功能过度建模。

当前来源：

1. docs/design/2026-05-18 简版游戏设计文档 v0.3.md

状态：已具备

### 2. 页面与跳转流转说明

用途：

1. 拆页面模块。
2. 设计页面栈、弹窗层、全局入口。
3. 明确首屏进入逻辑。

当前来源：

1. docs/design/2026-05-18 Demo 界面场景与页面流转草案 v0.1.md

状态：已具备

### 3. 客户端与服务器的权责边界

用途：

1. 明确哪些数字只展示，哪些必须以后端为准。
2. 避免前端本地计算关键资产结果。
3. 明确遇到并发或过期数据时的交互处理。

当前来源：

1. docs/design/2026-05-18 异步结算与并发规则草案 v0.1.md
2. docs/design/2026-05-18 微信小游戏首发技术方案 v0.1.md

状态：已具备

### 4. 页面状态与展示字段清单

用途：

1. 明确每个页面展示哪些字段。
2. 明确哪些字段是摘要态，哪些字段是明细态。
3. 明确哪些字段需要红点、倒计时、风险提示。

当前来源：

1. docs/design/2026-05-18 简版游戏设计文档 v0.3.md
2. docs/design/2026-05-18 Demo 界面场景与页面流转草案 v0.1.md
3. docs/design/2026-05-18 掠夺机制重设计草案 v0.2.md

状态：部分具备

当前缺口：

1. 还没有单独的页面字段清单文档。
2. 还没有把页面字段和接口字段一一对应起来。
3. 掠夺页的多人协作、表情互动和即时结算字段还需要继续落成接口契约。

### 5. 客户端 API 契约文档

用途：

1. 确认页面初始化接口。
2. 确认每个按钮对应的请求与返回。
3. 确认错误码、幂等键、版本号的使用方式。

当前来源：

1. services/game-server 的 Swagger 文档

状态：部分具备

当前缺口：

1. 目前只有基础接口样例，没有覆盖主城、外场、部队、掠夺、战报等核心页面。

### 6. 资源与 UI 规范文档

用途：

1. 统一按钮、标题、卡片、弹窗、颜色和字号。
2. 方便后续反复改样式。
3. 降低前端页面拼装时的返工。

当前来源：

1. docs/frontend/2026-05-18 Frontend Validation Client.md
2. docs/frontend/2026-05-18 Cocos 页面拆分方案 v0.1.md

状态：已具备基础版本

### 7. 首屏加载与资源分包策略

用途：

1. 保证首屏加载流畅。
2. 避免把次级页面资源塞进首包。
3. 明确哪些资源常驻、哪些按需加载。

当前来源：

1. docs/design/2026-05-18 微信小游戏首发技术方案 v0.1.md

状态：部分具备

当前缺口：

1. 当前技术方案仍偏总览，还没有落到 Cocos Creator 客户端的资源目录和加载分组细节。

### 8. 灵宠与种田联动前端开发文档

用途：

1. 确认灵宠页、种田产出、突破弹窗、洗练弹窗和商店页的展示字段。
2. 确认灵粮批量投喂、饱灵值、突破手动点击、词条展示和兽魂三档展示方式。
3. 确认前端在首发阶段应如何从服务端读状态并刷新页面。

当前来源：

1. docs/frontend/2026-05-23 灵宠养成与种田联动 前端开发文档 v0.1.md
2. docs/dev/2026-05-23 灵宠养成与种田联动 AI 交接总纲 v0.1.md
3. docs/design/2026-05-23 灵宠养成与种田联动首发方案 v0.1.md

状态：已具备

## 三、当前结论

前端开发所需文档不算齐全，但已经足够开始第一阶段开发。

目前已具备的核心文档有：

1. 玩法闭环。
2. 页面流转。
3. 后端权威与并发规则。
4. Cocos 客户端迁移骨架。
5. Cocos 页面拆分方案。

目前最缺的三份前端实施文档是：

1. 页面字段清单。
2. 客户端 API 契约清单。
3. UI 组件与主题规范。
4. 灵宠与种田联动专项前端文档。

## 四、前端下一步建议补齐顺序

1. 页面字段清单。
2. UI 组件分层与主题规范。
3. 资源目录与加载策略。
4. 客户端 API 契约清单。
