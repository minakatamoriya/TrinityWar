# TrinityWar 灵宠养成与种田联动 后端开发文档 v0.1

## 一、文档目的

这份文档是给 AI 和后端开发执行用的灵宠与种田联动后端开发文档。

目标不是把系统一次写完，而是把首发阶段需要落库、落接口、落规则的内容一次定清楚，避免后续实现时在表结构、接口契约、幂等语义和数值口径上反复返工。

## 三、后端首发范围

后端首发只做四件事：

1. 提供灵宠成长读写状态。
2. 提供灵粮投喂、突破、洗练的命令接口。
3. 提供兽魂、材料、商店的后端权威校验。
4. 让首页、农场页、灵宠页可以从同一套状态读取。

## 四、后端核心领域模型

### 1. SpiritProfile

建议作为主灵宠的核心状态对象，至少包含：

1. playerId
2. mainSpiritId
3. level
4. exp
5. breakthroughStage
6. growthProgress
7. breathState 或饱灵状态
8. equippedTraitSlots
9. lockedTraitSlots
10. spiritVersion

### 2. SpiritTrait

词条对象至少包含：

1. spiritId
2. traitCode
3. traitType
4. traitValue 或 traitRank
5. slotIndex
6. lockedAt
7. createdAt

### 3. SpiritFeedLog

用于记录灵粮投喂行为，至少包含：

1. playerId
2. spiritId
3. feedItemType
4. feedCount
5. immediateExpGain
6. breatherGain
7. createdAt

### 4. SpiritBreakthroughLog

用于记录突破行为，至少包含：

1. playerId
2. spiritId
3. fromStage
4. toStage
5. consumedMilkCount 或 consumedFeedCount
6. consumedSoulCount
7. consumedMaterialCount
8. createdAt

### 5. SpiritSoulInventory

用于记录兽魂库存，建议按品质分层：

1. ordinarySoulCount
2. rareSoulCount
3. legendarySoulCount

## 五、首批接口建议

### 1. 读接口

1. `GET /api/client/spirit`
2. `GET /api/client/spirit/definitions`
3. `GET /api/client/spirit/shop`

### 2. 命令接口

1. `POST /api/client/spirit/feed`
2. `POST /api/client/spirit/breakthrough`
3. `POST /api/client/spirit/roll-traits`
4. `POST /api/client/spirit/select-main`
5. `POST /api/client/spirit/shop/buy`

### 3. 接口语义

#### `GET /api/client/spirit`

返回：

1. 主灵宠当前等级和经验。
2. 当前饱灵值或饱灵状态。
3. 当前已装备词条。
4. 当前兽魂库存。
5. 当前可突破节点和所需材料。

#### `POST /api/client/spirit/feed`

输入：

1. feedItemType
2. feedCount
3. requestIdempotencyKey

输出：

1. immediateExpGain
2. breatherGain
3. updated exp / breather state
4. 更新后的材料库存

#### `POST /api/client/spirit/breakthrough`

输入：

1. targetStage
2. requestIdempotencyKey

约束：

1. 等级必须达标。
2. 材料必须足够。
3. 必须消耗对应档位兽魂。
4. 不能自动突破。

#### `POST /api/client/spirit/roll-traits`

输入：

1. spiritId
2. useMaterialCount
3. requestIdempotencyKey

约束：

1. 洗练消耗灵髓。
2. 高层洗练可消耗仙髓保底。
3. 结果必须落日志。

## 六、后端验证规则

### 1. 挂机成长

1. 经验增长必须是连续可见的。
2. 前期等级应明显快于后期。
3. 不活跃玩家不会自动跨越突破节点。

### 2. 灵粮投喂

1. 必须支持批量投喂。
2. 投喂后要同时更新经验和饱灵值。
3. 饱灵值必须能影响后续挂机效率。

### 3. 突破

1. 只允许手动触发。
2. 不允许未达成条件直接进入下一阶段。
3. 必须产生突破日志。

### 4. 兽魂

1. 兽魂只负责经验加速和冲层燃料。
2. 兽魂必须按品质区分单次加成。
3. 兽魂不能变成唯一成长来源。

### 5. 词条

1. 暴击、爆伤、闪避可作为基础词条。
2. 阵营相关词条要保留扩展空间。
3. 基础属性里不要放速度。

## 七、错误与并发要求

1. 所有写接口都必须带幂等键。
2. 同一操作重复提交不能重复扣材料。
3. 突破和洗练都必须遵守版本号或等价并发控制。
4. 如果当前灵宠状态已变化，必须返回明确冲突错误，而不是静默回写。

## 八、最小实现顺序建议

1. 先做 SpiritProfile 读接口。
2. 再做 feed 接口。
3. 再做 breakthrough 接口。
4. 再做 trait roll 接口。
5. 最后补商店与静态定义读接口。

## 九、人工验收关注点

1. 灵粮投喂是否批量且有反馈。
2. 突破是否会停留在当前节点，不自动跳层。
3. 兽魂是否按品质给不同的加速感。
4. 词条是否能扩展到阵营效果。
5. 基础属性池是否没有速度。
