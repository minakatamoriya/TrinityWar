## 一、文档目的

这份文档是给 AI 和后端开发执行用的灵宠与种田联动后端开发文档。

## 三、后端首发范围

后端首发只做这些事：

1. 提供灵宠成长状态和实时经验结算。
2. 提供灵根投喂、手动突破、洗练命令接口。
3. 提供作物收获接入灵根、灵髓、灵玉产出。
4. 提供兽魂库存、突破消耗、商店购买的后端权威校验。
5. 支持掠夺战斗中的灵宠属性、五行、词条、扣血、回血和兽魂掉落。
6. 让首页、农场页、灵宠页、掠夺页可以从同一套状态读取。

## 四、核心领域模型

### 1. SpiritProfile

建议作为主灵宠核心状态对象，至少包含：

1. playerId
2. mainSpiritId
3. level
4. exp
5. currentLevelExpRequired
6. breakthroughStage
7. isAtBreakthroughNode
8. satiatedUntil
9. hp
10. maxHp
11. elementType
12. rarity
13. equippedTraitSlots
14. spiritVersion
15. updatedAt

关键约束：

1. 到达突破节点后 exp 停止增长。
2. 不缓存溢出经验。
3. satiatedUntil 最高只能补到当前时间后 8 小时。

### 2. SpiritTrait

词条对象至少包含：

1. spiritId
2. slotIndex
3. traitCode
4. traitValue
5. sourceType
6. lockedAt
7. createdAt

规则：

1. 自然解锁词条不重复。
2. 洗练允许重复。
3. 重复词条直接相加，不衰减。

### 3. SpiritFeedLog

用于记录灵根投喂行为，至少包含：

1. playerId
2. spiritId
3. actionType: feed_once / fill_full
4. feedCount
5. satiatedSecondsAdded
6. immediateExpGain
7. beforeSatiatedUntil
8. afterSatiatedUntil
9. createdAt

经验公式：

```text
每 1 灵根 = 当前等级经验条的 0.5%
投喂一次最多消耗 10 灵根 = 当前等级经验条的 5%
补满按实际消耗灵根给经验
```

### 4. SpiritBreakthroughLog

用于记录突破行为，至少包含：

1. playerId
2. spiritId
3. fromStage
4. toStage
5. consumedSoulQuality
6. consumedSoulCount
7. createdAt

突破只消耗兽魂，不消耗灵根、灵髓或灵玉。

### 5. SpiritSoulInventory

兽魂库存按品质分层：

1. ordinarySoulCount
2. rareSoulCount
3. legendarySoulCount

兽魂只用于突破，不用于经验、洗练或作物掉落。

### 6. SpiritRaidState / BattleState

掠夺战斗至少需要支持：

1. attackerSpiritId
2. defenderSpiritId
3. attackerHpBefore / attackerHpAfter
4. defenderHpBefore / defenderHpAfter
5. attackerElement / defenderElement
6. attackerTraits / defenderTraits
7. battleResult
8. soulDropResult
9. equipmentRewardResult
10. protectionState

掠夺失败不扣已有资源，只扣出战灵宠血量。血量不足的灵宠不能继续参与战斗。

## 五、首批接口建议

### 1. 读接口

1. `GET /api/client/spirit`
2. `GET /api/client/spirit/definitions`
3. `GET /api/client/spirit/shop`
4. `GET /api/client/raid/spirit-targets`

### 2. 命令接口

1. `POST /api/client/spirit/feed`
2. `POST /api/client/spirit/breakthrough`
3. `POST /api/client/spirit/roll-traits`
4. `POST /api/client/spirit/select-main`
5. `POST /api/client/spirit/shop/buy`
6. `POST /api/client/raid/peek`
7. `POST /api/client/raid/attack`

## 六、接口语义

### `GET /api/client/spirit`

返回：

1. 主灵宠当前等级和经验。
2. 当前等级经验需求。
3. 是否卡在突破节点。
4. 当前饱食剩余时间和效率倍率。
5. 当前 HP / maxHp。
6. 当前已解锁词条槽和词条。
7. 当前兽魂库存。
8. 当前可突破节点和所需兽魂。
9. 当前灵根、灵髓、灵玉库存。

### `POST /api/client/spirit/feed`

输入：

1. actionType: feed_once / fill_full
2. requestIdempotencyKey

约束：

1. feed_once 最多消耗 10 灵根，加 2 小时饱食。
2. fill_full 按剩余饱食时间折算消耗，补到 8 小时。
3. 每 1 小时饱食 = 5 灵根。
4. 每 1 灵根给当前等级经验条 0.5%。
5. 卡在突破节点时，不再增加经验。
6. 饱食倍率固定 +50%，不叠加。

输出：

1. consumedSpiritFood
2. satiatedSecondsAdded
3. immediateExpGain
4. updatedExp
5. updatedSatiatedUntil
6. updatedInventory

### `POST /api/client/spirit/breakthrough`

输入：

1. targetStage
2. requestIdempotencyKey

约束：

1. 等级必须达突破节点。
2. 只消耗对应品质兽魂。
3. 不消耗灵根、灵髓、灵玉。
4. 不允许自动突破。
5. 突破后才允许继续获得经验。

消耗表：

| 突破节点 | 消耗 |
| --- | --- |
| 10 级突破 | 普通兽魂 x5 |
| 20 级突破 | 普通兽魂 x12 |
| 30 级突破 | 稀有兽魂 x10 |
| 40 级突破 | 稀有兽魂 x20 |
| 50 级突破 | 传说兽魂 x8 |

### `POST /api/client/spirit/roll-traits`

输入：

1. mode: basic / advanced / ultimate / batch_basic
2. lockedSlotIndex
3. targetSlotIndex
4. targetTraitCode
5. requestIdempotencyKey

约束：

1. 基础洗练消耗 5 灵髓 + 金币，重置所有已解锁词条。
2. 高级洗练消耗 1 灵玉 + 10 灵髓 + 金币，锁定 1 条后重置其余。
3. 终极洗练消耗 5 灵玉 + 20 灵髓，指定某槽位变成目标词条。
4. 连续洗练必须先扣材料，再生成 10 次结果。
5. 洗练允许重复词条。
6. 结果必须落日志。

### `POST /api/client/raid/peek`

输入：

1. targetPlayerId 或 targetId
2. requestIdempotencyKey

输出：

1. 对方灵宠五行。
2. 对方灵宠品质。
3. 对方关键属性摘要。
4. 对方是否处于保护期。

### `POST /api/client/raid/attack`

约束：

1. 目标处于保护期时禁止攻击。
2. 攻击方出战灵宠血量不足时禁止攻击。
3. 战斗采用服务端权威结算。
4. 失败不扣已有资源，只扣出战灵宠血量。
5. 每次掠夺仍发放装备或其他正向奖励。
6. 胜利后按对方灵宠品质判定兽魂掉落。

## 七、后端战斗规则

触发顺序固定：

1. 闪避判定。
2. 秒杀判定。
3. 五行克制临时数值结算。
4. 普通伤害、暴击、爆伤、防御、破甲结算。
5. 吸血结算。
6. 反击结算。
7. 胜负判定。

五行克制：

```text
克制方本场战斗攻击、生命、防御和普通战斗词条效果 x2
闪避和秒杀不受五行翻倍
```

基础伤害：

```text
伤害 = 战斗攻击 x (1 - 战斗防御 / (战斗防御 + 200)) x 随机系数
随机系数 = 0.9 ~ 1.1
```

首发词条表：

| 词条 | 效果 |
| --- | --- |
| 利爪 | 攻击 +10% |
| 厚皮 | 生命 +10% |
| 硬甲 | 防御 +10% |
| 暴击 | 暴击率 +6% |
| 爆伤 | 暴击伤害 +20% |
| 闪避 | 闪避率 +5% |
| 反击 | 受击 +10% 概率反击，造成 50% 伤害 |
| 吸血 | 造成伤害的 10% 回复自身 |
| 破甲 | 攻击无视目标 10% 防御 |
| 韧性 | 受暴击时伤害降低 10% |

## 八、后端验证规则

1. 经验到突破节点后停止，不缓存溢出。
2. 兽魂不提供经验。
3. 作物不产兽魂。
4. 突破只消耗兽魂。
5. 灵髓和灵玉只用于洗练。
6. 投喂和补满都按实际消耗灵根给经验。
7. 洗练必须先扣材料再出结果。
8. 掠夺目标保护期内不可被攻击。
9. 掠夺失败不扣已有资源。
10. 战斗结算必须由后端权威执行。

## 九、错误与并发要求

1. 所有写接口都必须带幂等键。
2. 同一操作重复提交不能重复扣材料。
3. 突破、投喂、洗练、掠夺都必须遵守版本号或等价并发控制。
4. 如果当前灵宠状态已变化，必须返回明确冲突错误，而不是静默回写。
5. 连续洗练和掠夺结算必须有完整日志，方便追查。

## 十、最小实现顺序建议

1. 先做 SpiritProfile 读接口和经验结算。
2. 再做 feed 接口。
3. 再做 breakthrough 接口。
4. 再做 trait roll 接口。
5. 再做 crop harvest 产出接入。
6. 再做 raid peek / attack。
7. 最后补商店与静态定义读接口。

## 十一、人工验收关注点

1. 灵根投喂是否只有“投喂一次 / 补满”两个动作。
2. 到突破节点后是否停止经验且不缓存溢出。
3. 突破是否只消耗兽魂。
4. 作物是否不再掉兽魂。
5. 洗练是否允许重复词条且不衰减。
6. 连续洗练是否先扣材料。
7. 掠夺前是否需要窥视才能知道对方五行。
8. 掠夺失败是否只扣宠物血，不扣已有资源。
9. 保护期是否禁止被掠夺。
10. 战斗触发顺序是否符合设计。
