# TrinityWar 农场卡槽化主界面 UI 设计草案 v0.1

## 一、文档目的

这份文档用于整理农场主界面的卡槽化视觉方向，重点解决以下问题：

1. 农场主界面如何保留场景感。
2. 土地如何表现为可以插入作物卡的卡槽。
3. 未解锁、空闲、种植中、成熟、收获后的土地状态如何区分。
4. 后续土地升级和灵宠联动如何预留表现空间。

本方案的核心方向是：

> 主界面仍然是农场，但每块土地以卡槽形式存在；每个地块插入一张作物卡；成熟后作物卡从地块中浮起。

## 二、设计定位

### 1. 不做传统全景农田复制

市面上已有大量以整片农田为主视觉的种田游戏，例如多块田地平铺、作物直接长在地里、收取时点击作物完成收获。

TrinityWar 的种田界面建议避免只做传统农田格子，而是把农场、作物、灵宠养成和后续战斗统一到“卡片”这一表达载体上。

### 2. 卡片是成长与资产的核心表达

作物不只是地里的植物，也是一张可成长、可展示、可收获、可参与联动的作物卡。

卡片可以承载：

1. 作物类型。
2. 成长阶段。
3. 成熟度。
4. 稀有度。
5. 品质边框。
6. 成熟特效。
7. 与灵宠、五行、材料产出的关系。

### 3. 农场是卡片生长的场景容器

卡片化不能让农场变成纯背包界面。主界面仍然要有农场空间感，让玩家感受到自己在经营一块土地，而不是只在管理卡牌列表。

推荐结构：

1. 农场背景提供空间、氛围和世界观。
2. 土地提供卡槽承载关系。
3. 作物卡提供成长状态和收获反馈。
4. 灵宠可以在农场中巡逻、守护或触发加成。

## 三、主界面布局原则

### 1. 农场作为第一视觉层

进入种田主界面时，第一眼应看到一个完整的农场区域，而不是纯 UI 面板。

推荐包含：

1. 农场地面。
2. 分区路径。
3. 土地卡槽阵列。
4. 少量背景物件，例如篱笆、树、井、仓库、灵宠小屋。
5. 顶部资源栏和底部功能入口。

### 2. 土地以卡槽阵列呈现

土地不建议画成完全真实的田垄，也不建议画成纯矩形 UI 格子。

推荐做成“真实土地 + 卡片凹槽”的混合形态：

1. 外层是农田土壤、木边、石边或藤蔓边。
2. 内层有一块接近卡牌比例的浅凹槽。
3. 卡槽角落可以有木桩、灵纹、泥土压痕或小叶片。
4. 空槽中心显示轻量的加号、种子图标或插卡提示。

### 3. 卡槽需要稳定尺寸

同一农场区域内，土地卡槽尺寸应稳定，不随作物状态、倒计时、特效、收益数字变化而挤压布局。

推荐规则：

1. 卡槽宽高比接近作物卡比例，但略宽于卡牌。
2. 作物卡插入后，卡片底部进入土地 10% 到 20%。
3. 成熟浮起时，卡片上浮但不改变土地占位。
4. 倒计时、收益和状态图标放在卡槽固定附属区域，不影响卡片主体位置。

## 四、土地状态设计

### 1. 未解锁土地

未解锁土地建议表现为“未来可以开垦的荒地”，而不是简单灰色锁格。

视觉组成：

1. 荒草、碎石或未整理土壤。
2. 中央或后方插一个木牌。
3. 木牌上显示锁图标、解锁等级或消耗。
4. 地块轮廓隐约可见，但还没有完整卡槽。
5. 色彩饱和度低于已解锁土地。

推荐文案：

1. Lv.5 解锁。
2. 1000 灵石。
3. 待开垦。

交互反馈：

1. 点击未解锁土地时打开解锁弹窗。
2. 条件不足时展示缺少等级或资源。
3. 条件满足时木牌可以轻微发光或抖动，提示可解锁。

### 2. 已解锁空闲土地

空闲土地需要明确表达“这里可以插入作物卡”。

视觉组成：

1. 整理过的土壤。
2. 接近卡牌比例的浅凹槽。
3. 凹槽内有淡淡的卡片投影。
4. 中心显示加号或种子图标。
5. 边缘有轻量木边、石边或藤蔓。

交互反馈：

1. 点击后打开选择作物卡界面。
2. 可种植时卡槽边缘有轻微呼吸光。
3. 无可用作物卡时显示空状态提示，不跳转到无意义列表。

### 3. 种植中土地

作物种下后，土地应表现为卡片已经插入土地。

视觉组成：

1. 作物卡底部插入土壤 10% 到 20%。
2. 卡片略微倾斜，形成立体感。
3. 土地和卡片之间有根须、嫩芽、灵气或细小光点连接。
4. 卡面展示当前作物阶段，例如种子、发芽、成长、成熟前。
5. 底部或侧边展示成长倒计时。

成长阶段建议：

1. 种子期：卡面偏暗，土壤无明显光效。
2. 发芽期：卡面出现嫩芽图案，凹槽边缘有绿色微光。
3. 成长期：卡面更完整，边框开始出现品质色。
4. 成熟前：卡片轻微呼吸，提示即将可收。

### 4. 成熟土地

成熟状态是种田界面的主要正反馈，需要明显但不过度。

视觉组成：

1. 作物卡从土地中浮起，底部离开土壤。
2. 土地中心出现柔和光环。
3. 卡片边框发光，按品质显示不同强度。
4. 卡面作物图案完整展示。
5. 土壤中可以冒出叶片、粒子或灵气。

交互反馈：

1. 点击成熟卡执行收获。
2. 收获时卡片轻微翻转。
3. 收益飞向资源栏或仓库入口。
4. 土地恢复为空闲卡槽。

### 5. 枯萎或枯竭土地

如果后续存在枯萎、收益衰减或枯竭状态，应避免和成熟状态混淆。

视觉组成：

1. 卡面亮度下降。
2. 边框光效变弱或断续闪烁。
3. 土地颜色变干、出现裂纹或落叶。
4. 倒计时区域显示收益衰减提示。

交互原则：

1. 仍可收获时保留收获入口。
2. 已枯竭时点击进入清理或重新播种流程。
3. 不建议首发阶段做过多负反馈动画，避免让种田体验显得惩罚过重。

### 6. 收获后土地

收获后需要快速回到可播种状态。

视觉组成：

1. 作物卡飞走或淡出。
2. 土壤光效收束。
3. 卡槽恢复为空闲凹槽。
4. 中心重新出现加号或种子图标。

## 五、作物卡表现规则

### 1. 作物状态用卡面变化表达

同一种作物可以根据成长阶段使用不同卡面。

建议阶段：

1. 种子。
2. 发芽。
3. 成长。
4. 成熟。
5. 枯萎或特殊变异。

### 2. 首发美术资源减负原则

首发阶段不建议为每一种作物都准备完整的多阶段专属图片。

如果十多种作物每种都准备 4 个成长状态，很快会膨胀为 40 到 60 张以上作物图，增加美术成本、资源管理成本和包体压力。

推荐采用“通用成长阶段 + 成熟专属图 + 外层特效”的组合方式。

建议规则：

1. 种子期：使用通用种子图，搭配作物名称、作物小图标或五行颜色。
2. 发芽期：使用通用嫩芽图，搭配作物小图标或属性色。
3. 成长期：使用通用幼苗、藤蔓、花苞或药草底图，按作物类型区分。
4. 成熟期：每种作物准备一张成熟专属图。
5. 成熟期：使用成熟专属图，通过边框、光效和可收标识突出。

这样首发需要重点准备的是：

1. 每种作物 1 张成熟图。
2. 3 到 5 张通用成长阶段图。
3. 一套卡牌底图、品质边框和状态光效。
4. 少量成熟、枯萎、收获类图标。

### 3. 成熟可收的表现方式

首发阶段只保留成熟可收状态，不再拆分额外收益档位。

成熟状态使用同一张成熟作物图，通过外层表现强调可点击收获。

| 状态 | 作物图 | 外层表现 | 交互暗示 |
| --- | --- | --- | --- |
| 成熟 | 使用成熟作物图 | 普通发光边框、柔和光环、少量粒子 | 可收获 |
| 成熟可收 | 成熟作物图 | 更强边框、青金或金色流光、额外粒子、收获角标 | 点击收获 |

成熟状态建议：

1. 作物卡浮起。
2. 边框轻微发光。
3. 土地中心出现柔和光环。
4. 粒子数量较少。
5. 呼吸动效较慢。

成熟可收状态建议：

1. 复用成熟作物图。
2. 边框亮度更高。
3. 边框增加流光循环。
4. 卡片角落增加“丰”或高收益图标。
5. 土地边缘灵纹亮起。
6. 粒子从土地持续上升。
7. 收获时收益图标更多，飞向资源栏的轨迹更亮。

只有在以下情况下，才建议单独绘制特殊成熟图：

1. 特殊成熟会明显改变作物外观，例如果实变大、开花、结晶或异色。
2. 该作物是传说作物、活动作物或核心付费展示作物。
3. 后续已有稳定美术产能，可以为高价值作物补充完整阶段图。

首发推荐资源组合：

```text
成熟 = 成熟作物图 + 普通成熟边框 + 柔和光环
成熟可收 = 成熟作物图 + 可收边框 + 青金/金色粒子 + 收获角标
```

### 4. 等级和品质用边框表达

卡片边框用于承载稀有度、等级和特殊状态。

推荐层级：

1. 普通：简洁木质或土色边框。
2. 稀有：绿色、蓝色或青玉色描边。
3. 史诗：更明显的流光边框。
4. 传说：动态光边、背景纹路和轻量粒子。

成熟度可以通过：

1. 卡片底部成长槽。
2. 卡槽下方进度条。
3. 卡面亮度变化。
4. 边框呼吸频率。

### 5. 收获价值用角标或底栏表达

不建议在卡面中央堆叠大量数字。

推荐位置：

1. 左上角：作物品质或五行属性。
2. 右上角：成熟状态图标。
3. 底部：倒计时或预计产出摘要。
4. 点击详情后：展示完整产出、加成和风险。

## 六、土地与卡片的动效建议

### 1. 插入作物卡

流程建议：

1. 玩家选择作物卡。
2. 作物卡从选择面板飞向土地。
3. 卡片底部插入土壤。
4. 土地出现一圈轻微尘土或灵气。
5. 倒计时开始。

### 2. 成熟浮起

流程建议：

1. 成熟瞬间卡片轻微震动。
2. 卡片从土壤中上浮。
3. 边框亮起。
4. 土地光环展开。
5. 卡片进入可收获待机状态。

### 3. 收获

流程建议：

1. 玩家点击成熟卡。
2. 卡片向前放大 5% 到 8%。
3. 卡片翻转或闪光。
4. 收益图标飞向资源栏。
5. 卡片淡出，土地恢复空槽。

## 七、灵宠联动预留

灵宠系统后续可以围绕卡片化农场继续扩展。

### 1. 灵宠巡逻

灵宠可以在农场中小范围巡逻，增强场景生命力。

表现方式：

1. 灵宠在土地之间移动。
2. 停在某块作物卡旁边时触发小表情或光效。
3. 守护中的土地显示灵宠标记。

### 2. 灵宠加成

灵宠对作物卡的加成可以通过卡槽附属效果表达。

示例：

1. 成长加速：卡槽边缘出现流动光点。
2. 产出提升：卡片底部出现增益角标。
3. 防掠夺：卡槽旁出现护盾图标。
4. 五行契合：土地和卡片之间出现对应元素纹路。

### 3. 契合卡片

如果后续设计宠物与作物的契合卡，可以采用以下表达：

1. 作物卡上显示宠物契合标识。
2. 宠物卡详情中展示推荐作物卡。
3. 契合成功时，土地卡槽边缘出现联动特效。
4. 高契合状态下成熟卡浮起时增加专属光效。

## 八、土地升级预留

当前土地尚未开发升级系统，但 UI 需要预留扩展空间。

### 1. 首发阶段不展示复杂土地等级

首发阶段建议只区分：

1. 未解锁。
2. 已解锁空闲。
3. 种植中。
4. 成熟。
5. 收获后。

不要提前在主界面堆叠土地等级、土地经验、升级按钮和多种土壤属性，避免界面过早复杂。

### 2. 后续土地品质方向

后续可以扩展为：

1. 普通土地：木边、普通土壤。
2. 灵壤：绿色灵纹和柔和荧光。
3. 水田：浅水反光和水波卡槽。
4. 火山土：红色裂纹和热气。
5. 月光田：银色边缘和夜光植物。

### 3. 升级入口建议

土地升级不建议一直占用主界面。

推荐方式：

1. 长按土地打开土地详情。
2. 土地详情中展示升级入口。
3. 可升级时在土地角落显示小型上箭头角标。
4. 不可升级时不显示强提示，避免干扰种植和收获。

## 九、界面信息优先级

### 1. 主界面必须一眼看懂

玩家进入农场后，必须能快速判断：

1. 哪些地未解锁。
2. 哪些地空闲。
3. 哪些地正在成长。
4. 哪些地可以收获。
5. 当前资源和入口在哪里。

### 2. 主界面不展示过多详情

主界面只显示必要状态。

详细信息放到点击后的详情层：

1. 作物完整产出。
2. 成长曲线。
3. 灵宠加成来源。
4. 被掠夺风险。
5. 土地升级信息。

### 3. 成熟状态优先级最高

成熟可收的作物卡应该是农场中最突出的状态。

视觉优先级建议：

1. 可收获成熟卡。
2. 可解锁土地。
3. 空闲可播种土地。
4. 成长中土地。
5. 未解锁且条件不足土地。

## 十、首发落地建议

首发原型建议只做一个小闭环：

1. 农场背景。
2. 6 到 9 块土地卡槽。
3. 未解锁土地使用荒地 + 木牌 + 锁。
4. 空闲土地使用土壤 + 卡片凹槽 + 加号。
5. 种植中土地使用半插入作物卡。
6. 成熟土地使用浮起作物卡 + 边框光效。
7. 点击成熟卡后完成收获并恢复空槽。

这套闭环能验证卡槽化农场是否成立，同时为后续灵宠契合、土地升级和宠物对战卡牌化保留足够空间。

## 十一、微信小游戏轻量动画架构建议

本项目如果以微信小游戏和轻量前端为目标，首发阶段不建议引入 Cocos、Phaser、Three.js 等较重方案。

推荐采用：

```text
Canvas 2D：负责绘制
Tween.js：负责属性动画
ParticleSystem：自写粒子
EffectManager：管理雷电、灵气、光环
AnimationManager：管理卡片、按钮、收获动画
```

### 1. Canvas 2D

Canvas 2D 负责所有画面绘制。

主要绘制对象：

1. 农场背景。
2. 土地卡槽。
3. 木牌、锁、加号、状态图标。
4. 作物卡底图、卡面、边框。
5. 成熟光环、可收流光、雷电折线。
6. 灵气粒子、收益飞行动效。
7. 按钮、资源栏和轻量 UI 元素。

Canvas 只负责“画出来”，不直接承担复杂业务状态。

### 2. Tween.js

Tween.js 负责属性随时间变化，不直接生成特效。

适合控制：

1. 卡片浮起。
2. 卡片插入土地。
3. 卡片飞向资源栏。
4. 卡片缩放、旋转、伪 3D 翻转。
5. 按钮按下和回弹。
6. 光环半径、透明度、发光强度。
7. 收益图标飞行路径。
8. 战斗卡牌对撞和回弹。

推荐理解方式：

```text
Tween.js = 动画参数控制器
Canvas 2D = 动画和特效的最终绘制层
```

### 3. ParticleSystem

ParticleSystem 建议自写，不引入大型粒子库。

首发需要支持：

1. 灵气升腾：粒子从土地或卡片底部向上飘，逐渐淡出。
2. 成熟光点：少量粒子围绕成熟卡上升。
3. 可收粒子：粒子数量更多，颜色偏青金或金色。
4. 收获飞屑：点击收获时从卡片向资源栏方向飞出。
5. 战斗碎光：卡牌对撞时产生短暂爆点。

基础粒子字段建议：

```text
x, y
vx, vy
life, maxLife
size
alpha
color
rotation
textureKey
```

### 4. EffectManager

EffectManager 负责管理持续性或短时特效。

建议管理：

1. 雷电环绕。
2. 灵气升腾。
3. 成熟光环。
4. 可收流光。
5. 卡片边框呼吸光。
6. 土地可种植提示光。
7. 灵宠守护加成光效。

雷电环绕建议用程序生成，不依赖逐帧图片：

1. 围绕卡片边缘生成路径点。
2. 点与点之间绘制随机抖动折线。
3. 使用青白、紫白或金白描边。
4. 使用阴影模糊制造发光。
5. 每隔数帧重算一次随机偏移，形成闪烁感。

灵气升腾建议用粒子实现：

1. 粒子从卡槽底部生成。
2. 缓慢上升。
3. 横向轻微摆动。
4. 透明度逐渐降低。
5. 可叠加发光贴图或浅色圆点。

### 5. AnimationManager

AnimationManager 负责业务动作对应的动画编排。

建议封装为明确方法：

```text
playPlantCard(card, plot)
playCardMature(card, plot)
playCardOverripe(card, plot)
playHarvest(card, targetResourceIcon)
playButtonTap(button)
playCardSelect(card)
playBattleClash(attackerCard, defenderCard)
```

这些方法内部可以组合 Tween.js、ParticleSystem 和 EffectManager。

例如收获动画：

1. 卡片放大。
2. 卡片闪光或轻微翻转。
3. 收益图标飞向资源栏。
4. 粒子散开。
5. 卡片淡出。
6. 土地恢复空槽。

### 6. 首发不建议使用的方案

首发阶段建议暂不使用：

1. Three.js，除非需要真实 3D 灵宠或 3D 卡牌。
2. Phaser，避免微信小游戏适配和框架体积风险。
3. 大型粒子库，当前特效复杂度不需要。
4. DOM / CSS 动画，微信小游戏不是完整浏览器环境。

| 状态 | 土地表现 | 卡片表现 | 主交互 |
| --- | --- | --- | --- |
