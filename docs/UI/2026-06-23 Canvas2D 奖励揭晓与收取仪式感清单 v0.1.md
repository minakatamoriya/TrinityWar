# 2026-06-23 Canvas2D 奖励揭晓与收取仪式感清单 v0.1

## 一、文档目标

本文用于收口 Trinity War 后续转 `canvas2d` 时，和“领奖、收取、揭晓、翻开、掉落、开箱感”相关的演出方向。

目标不是新增重系统，而是：

1. 把现有收取动作做出仪式感。
2. 把已有奖励入口做出揭晓感。
3. 让玩家感觉自己不是“点了一下拿数值”，而是“打开了一次值得期待的结果”。
4. 优先做低规则成本、高爽感收益的演出层增强。

一句话总结：

> 先把“领取”做爽，再决定要不要做“抽奖”。

## 二、核心结论

- 当前版本不是没有奖励点，而是奖励点还没有被包装成“揭晓时刻”。
- 首发和中期最值得做的，不是完整抽卡系统，而是把已有入口做成“小开箱”。
- 这类工作最适合放在转 `canvas2d` 后统一打磨，因为更容易统一动效、粒子、翻面、飞入和音效节奏。
- 目标不是增加更多规则，而是提高每次领奖时的情绪密度。

## 三、适合做成“揭晓时刻”的现有入口

### 1. 灵田成熟收取

- 普通成熟收取。
- 丰熟收取。
- 过熟回收或止损收取。

建议：

- 收取后不要直接加资源。
- 先展示主奖励卡，再展示额外奖励。
- 丰熟应有更强的光效、卡片放大和额外掉落提示。
- 如果掉出稀有种子、材料或图鉴相关内容，应单独高亮。

### 2. 掠夺结算奖励

- 胜利结算。
- 平局结算。
- 完败安慰奖励。
- 复仇结算。

建议：

- 先展示战斗结果。
- 再翻开奖励内容。
- 奖励一项项揭晓，而不是一行数字直接入账。
- 完败也要有很轻的“安慰奖励揭晓”，避免完全空掉。

### 3. 洗练结果揭晓

- 基础洗练。
- 高级洗练。
- 终极洗练。
- 连续洗练 10 次。

建议：

- 五个词条逐条翻开。
- 金词条、目标词条、重复极品词条要有额外表现。
- 连续洗练建议做成“结果列展开 + 最终选定”。
- 洗练是最适合做出翻牌感和开奖感的入口之一。

### 4. 图鉴解锁

- 新灵植首次解锁。
- 新灵宠首次解锁。
- 图鉴阶段奖励领取。

建议：

- 新内容首次解锁时做大卡面展开。
- 不要只做 toast。
- 图鉴点亮应成为“值得停一下看一眼”的时刻。

### 5. 签到与阶段奖励

- 每日签到。
- 多日阶段奖励。
- 新手阶段奖励。
- 赛季参与奖与贡献档奖励。

建议：

- 奖励不要直接平铺数字。
- 可以做成“补给包打开”的形式。
- 稀有奖励与普通奖励区分层级表现。

### 6. 通知附件与补偿领取

- 后台补偿附件。
- 赛季补偿。
- 活动邮件奖励。

建议：

- 这类奖励虽然来源偏系统，但领取时也应有统一揭晓面板。
- 不要只是列表里点领取然后数字变化。

## 四、最值得优先打磨的五个入口

如果只做最关键的一批，建议优先顺序如下：

1. 丰熟收取揭晓。
2. 洗练结果翻牌。
3. 掠夺结算奖励揭晓。
4. 图鉴首次解锁展开。
5. 签到 / 阶段奖励补给包。

原因：

- 这五个入口出现频率高。
- 都天然适合“翻开、爆光、飞入、追加惊喜”。
- 能在不增加系统复杂度的前提下，显著提高玩家爽感。

## 五、演出设计原则

### 1. 先压一下，再揭晓

- 先给半秒左右的停顿、聚焦或聚能感。
- 再翻开、弹出或爆出结果。
- 爽点来自“短延迟后的释放”。

### 2. 先主奖励，再副惊喜

- 先给稳定主奖励。
- 再给额外掉落、稀有副产物、图鉴推进等奖励。
- 这样既有确定感，也有惊喜感。

### 3. 稀有物单独站出来

- 普通奖励可以快过。
- 稀有奖励必须单独高亮。
- 不能所有奖励都用一个层级表现。

### 4. 高频入口要能跳过

- 每天会重复很多次的入口必须可快速跳过。
- 第一次解锁、稀有掉落、重大奖励则允许强展示。
- 演出再好也不能拖慢日常高频操作。

### 5. 奖励要有飞入感

- 奖励揭晓后，应能看到资源飞入背包、资源条或图鉴。
- 玩家会更直觉感到“我获得了东西”，而不是只看数字变化。

### 6. 重复结果也要有价值感

- 重复词条、重复材料、重复图鉴进度都不应做成纯失落。
- 应明确告诉玩家它转化成了什么积累。

## 六、推荐的演出层级

建议把奖励揭晓演出拆成三层，避免所有入口都做得一样重。

### 1. 轻演出

适用：

- 普通成熟收取。
- 普通签到。
- 常规附件领取。

表现：

- 轻微缩放。
- 一次翻开。
- 小粒子。
- 奖励飞入。

时长建议：

- 0.4 到 0.8 秒。

### 2. 中演出

适用：

- 丰熟收取。
- 普通掠夺胜利。
- 基础洗练出现好词条。
- 图鉴普通解锁。

表现：

- 明显聚焦。
- 逐项揭晓。
- 稍强粒子与音效。
- 可看到“还有额外奖励”。

时长建议：

- 0.8 到 1.5 秒。

### 3. 重演出

适用：

- 赛季奖励。
- 首次传说图鉴解锁。
- 稀有掉落。
- 终极洗练命中目标词条。
- 重大活动奖励。

表现：

- 大卡面展开。
- 背景变暗或聚光。
- 多段音效。
- 明显的奖品定格。

时长建议：

- 1.5 到 3 秒。

## 七、首发不建议直接做的重系统

### 1. 完整抽卡 / 卡包系统

不建议一上来就做完整抽卡系统，因为它会引入：

- 奖池配置。
- 权重管理。
- 保底机制。
- 重复转化。
- 额外货币或票券。
- 抽卡记录、历史、审计。
- 付费与免费平衡。

这不是一个小演出问题，而是一整条新经济线。

### 2. 重度随机宝箱体系

不建议一开始就把大量核心资源改成随机宝箱产出，否则：

- 玩家难以理解稳定成长线。
- 经济边界会变糊。
- 你当前“轻经营、轻掠夺、轻养宠”的清晰结构会被稀释。

## 八、推荐的中间路线

如果后续想比“单纯领奖演出”再多走一步，建议优先做轻量补给包，而不是完整抽卡。

适合的轻箱子方向：

- 丰熟秘匣。
- 掠夺战利品包。
- 赛季补给箱。
- 灵宠材料匣。
- 图鉴奖励包。

设计原则：

- 奖池窄。
- 结果可预期。
- 重点是揭晓感，不是赌博感。

## 九、Canvas2D 阶段建议优先级

### P0：必须做

- 统一奖励揭晓面板。
- 统一奖励飞入效果。
- 普通 / 稀有 / 重大奖励三层表现规范。
- 丰熟收取揭晓。
- 掠夺结算揭晓。
- 洗练结果翻牌。

### P1：强烈建议做

- 图鉴首次解锁展开。
- 签到 / 阶段奖励补给包。
- 稀有掉落特殊停顿与高亮。
- 可跳过与快速结算模式。

### P2：后续增强

- 轻量补给箱系统。
- 更复杂的多段开奖表现。
- 奖励历史回看。
- 稀有奖励收藏陈列。

## 十、统一视觉与交互建议

为避免后续演出风格分裂，建议尽早统一以下规范：

- 普通奖励色。
- 稀有奖励色。
- 传说奖励色。
- 卡背样式。
- 翻开方向与速度。
- 奖励飞入路径。
- 粒子数量级。
- 稀有停顿时长。
- 是否自动关闭。
- 是否允许点击跳过。

## 十一、最关键的设计判断

- 玩家不一定需要完整抽奖系统，但一定需要“揭晓感”。
- 奖励爽感未必要来自随机池，也可以来自“翻开已有结果”的演出包装。
- Trinity War 当前最有潜力做成“开箱感”的，不是额外新系统，而是：
  - 丰熟收取。
  - 洗练翻牌。
  - 掠夺结算。
  - 图鉴点亮。
  - 签到与阶段奖励。

## 十二、最终结论

- 现阶段优先做“奖励揭晓演出”，不优先做完整抽卡系统。
- 现有规则已足够提供大量值得包装的领奖入口。
- 转 `canvas2d` 后，奖励揭晓与收取仪式感应成为一条独立打磨主线。
- 这类工作能显著提升爽感密度，但不会显著增加规则复杂度，投入产出很高。

一句话总结：

> Trinity War 后续最值得打磨的，不是再发明更多奖励来源，而是让现有每一次“拿到东西”都更像一次小小的开箱时刻。
