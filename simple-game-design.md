# Trinity War 简版游戏设计文档 v0.2

## 一、产品定位

Trinity War 是一款三阵营异步经营掠夺游戏。

玩家的核心体验不是长时间在线操作，而是在短时间上线后完成一轮明确决策：

1. 看自己的外场物资有没有成熟。
2. 决定现在收，还是继续赌更高收益。
3. 决定把金币拿去扩张、防守还是造兵。
4. 决定要不要去抢另外两个阵营玩家的成熟资源。

一句话定位：

安全金币保底，外场培育拉收益，成熟窗口触发掠夺。

## 二、首发核心目标

首发版本只验证四件事：

1. 玩家是否愿意为了成熟收取和掠夺窗口反复上线。
2. 玩家是否理解“早收更稳，晚收更赚，但更危险”。
3. 玩家是否会为了更快成长，主动把金币暴露到外场和战备槽。
4. 玩家是否会因为阵营分红、贡献和排行榜形成长期回流。

## 三、核心循环

首发版本的最小闭环如下：

1. 玩家获得小额保底金币收入和阵营小时分红。
2. 玩家把一部分金币投入外场培育位，开始培育高收益物资。
3. 外场物资经历成长、成熟、丰熟、过熟几个阶段。
4. 玩家上线后决定提前收、按时收，还是继续等更高收益。
5. 玩家把收获折算成金币，投入金库、战备槽、建筑、防守或造兵。
6. 玩家搜索另外两个阵营的成熟目标并发起掠夺。
7. 玩家用掠夺收益继续扩张、防守、上缴阵营并提高下一轮分红。

这条循环必须稳定传达一个核心关系：

1. 金库保你活下去。
2. 外场决定你赚得快不快。
3. 想跨过关键成长门槛，就要承担成熟窗口带来的风险。

## 四、资产与经济系统

首发版本建议只保留一个核心货币：金币。

问题不再是“有几种货币”，而是“金币当前处于什么状态”。

### 1. 金币的四种状态

#### 1.1 金库金币

规则：

1. 玩家已收取的金币默认进入金库。
2. 金库金币不可被日常掠夺直接带走。
3. 金库容量有限，玩家必须升级金库承接更大收益。
4. 金库金币主要用于升级建筑、开启培育、补兵和转入战备槽。

#### 1.2 外场投入金币

规则：

1. 玩家可把金币投入外场培育位，启动一次高收益培育。
2. 投入后，这笔钱不再视为安全金币，而是变成正在成长的外场物资。
3. 外场物资成熟前不能直接算作收益，必须等待收获。
4. 外场物资是首发日常掠夺的主要目标之一。

#### 1.3 战备槽金币

规则：

1. 玩家训练士兵、开启强化、执行高阶操作前，需先把金币从金库转入战备槽。
2. 战备槽中的金币属于暴露资产，可被掠夺。
3. 战备槽有容量上限和保护比例。
4. 玩家可以把战备槽中的一部分金币回存金库，但应有冷却或手续费。

#### 1.4 阵营公库金币

规则：

1. 玩家上缴后的金币进入阵营公库。
2. 阵营公库不被个人日常掠夺直接命中。
3. 阵营公库主要影响阵营分红、公共加成和中后期玩法。
4. 阵营公库只在阵营战或赛季玩法中承受损失，不进入首发个人掠夺主循环。

### 2. 保底金币来源

既然首发版本不做农民产出，就必须保留小额保底金币来源，避免玩家连续失误后无法继续经营。

首发建议只保留三种保底来源：

1. 主城基础税收。
2. 阵营小时分红。
3. 日常活跃奖励。

设计原则：

1. 保底收入足以继续玩。
2. 保底收入不足以吃满成长线。
3. 真正决定成长速度的，仍然是外场培育和掠夺效率。

### 3. 主城与金库升级规则

主城和金库是首发阶段最重要的长期成长线。

建议规则如下：

1. 主城升级主要消耗金库金币。
2. 金库升级提高安全资金上限。
3. 外场位升级提高可同时培育的项目数和单次收益上限。
4. 主城中后期升级需要玩家既有足够金库容量，也有足够外场收益效率。

核心原则：

1. 金库负责存稳。
2. 外场负责赚快。
3. 主城负责解锁更高阶段的经营空间。

## 五、外场培育系统

这是首发版本最核心的经营与对抗结合点。

### 1. 外场物资定义

首发不必把它设计成“菜”，也不必一开始就做宠物。

建议把它定义成一类统一的外场战备物资。

它可以在美术和世界观上包装为：

1. 战备粮。
2. 魂晶。
3. 军械胚体。
4. 异界孵化物。

但首发机制层面只把它当成一类统一资源项目处理：

投入金币，等待成长，成熟后收获更高金币。

### 2. 培育阶段

每个外场培育位都按照固定阶段运行：

1. 播种期：刚投入，不能被掠夺，也不能高收益收取。
2. 成长期：可以提前收取，但收益偏低。
3. 成熟期：标准收益阶段，是主要可被掠夺窗口。
4. 丰熟期：超额收益阶段，回报更高，但暴露风险也更高。
5. 过熟期：收益开始衰减，防止玩家无限等待。

### 3. 收取规则

首发建议明确四条规则：

1. 玩家可以提前收取，但收益低于成熟收取。
2. 玩家在成熟期收取，可获得标准最佳性价比。
3. 玩家继续等待到丰熟期，可获得更高收益，但更容易被盯上。
4. 玩家收取成功后，收益直接折算成金币进入金库。

这条规则必须写死：

收取成功后的金币不进入日常掠夺目标。

### 4. 首发推荐时间曲线

为保证回流频率，首发建议用短周期，不要做太长链条。

建议第一版直接采用：

1. 播种后 2 小时进入成长期。
2. 成长期可提前收，收益约为标准收益的 70%。
3. 再过 2 小时进入成熟期，收益为 100%。
4. 再过 1 小时进入丰熟期，收益为 130%。
5. 再过 2 小时进入过熟期，收益开始逐步衰减。

这套曲线的目的不是做复杂计算，而是制造明确的回访理由：

1. 我快成熟了，要不要回来看。
2. 我已经成熟了，要不要立刻收。
3. 我想再赌一小时更高收益，但可能被抢。

### 5. 外场升级意义

外场系统必须具备长期成长空间，但首发不要做得太重。

首发只需要支持四种成长点：

1. 外场位数量。
2. 单次投入上限。
3. 成熟速度加成。
4. 被掠夺保护比例。

## 六、人口与部队系统

首发版本不做复杂兵种，只保留总人口和一个基础兵种。

这里的人口不再承担生产职责，只负责军事规模。

### 1. 人口结构

1. 总人口上限由主城和住房决定。
2. 总人口 = 掠夺兵 + 空闲人口。
3. 空闲人口可训练成掠夺兵。
4. 掠夺兵负责进攻和基础防守。

### 2. 单兵种规则

1. 训练 1 个掠夺兵，需要 1 点人口和一笔战备槽金币。
2. 掠夺兵越多，主动进攻越强，但维护成本也越高。
3. 掠夺兵在战斗中死亡后，玩家要承受人口与金币的双重损失。
4. 人口上限不足时，玩家必须优先扩住房，不能无限爆兵。

### 3. 首发战斗字段

首发只保留三个核心字段：

1. 战力：决定胜负倾向。
2. 运载：决定能带走多少成熟收益或战备金币。
3. 损耗：决定这次掠夺值不值。

## 七、掠夺与防守系统

### 1. 掠夺目标

首发版本只允许掠夺另外两个阵营的玩家。

系统展示的目标信息建议只有：

1. 阵营。
2. 推荐风险。
3. 外场成熟度预估。
4. 战备槽暴露等级。
5. 防守倾向。

玩家不需要读大量数据，只要快速判断：

这个目标值不值得打。

### 2. 日常可掠夺资产边界

首发版本建议把可掠夺资产明确限定为三类：

1. 成熟中的外场物资。
2. 战备槽中的金币。
3. 出征与防守产生的军事损耗。

首发版本不建议作为主要掠夺目标的资产有三类：

1. 金库金币。
2. 已成功收取的收益。
3. 主城升级核心底料。

### 3. 掠夺结算原则

掠夺必须满足下面四条：

1. 敌人不能看到成熟就自动拿走，必须打赢。
2. 打赢后也不应一次性清空目标全部收益。
3. 被攻击方必须保留恢复和继续发育的基础能力。
4. 战损必须真实存在，否则掠夺只剩无脑最优解。

建议首发用下面这条结算顺序：

1. 校验目标是否处于可掠夺阶段。
2. 读取防守方当前防守配置。
3. 结算攻防胜负和双方战损。
4. 根据攻击方运载值计算最大可掠夺量。
5. 根据防守保护比例计算实际损失。
6. 先结算成熟外场收益，再结算战备槽金币。
7. 写入双方战报并更新复仇资格。

### 4. 防守的意义

防守系统在首发版本只解决一件事：

让玩家愿意投入资源减少自己在成熟窗口里的损失。

首发只需要三种防守收益：

1. 保护建筑提高成熟收益保留率。
2. 城防降低被掠夺比例。
3. 驻守兵提高敌方战损。

### 5. 复仇规则

实名掠夺不要作为日常主入口，而应绑定明确因果。

首发建议只在两种情况下开放实名复仇：

1. 对方最近 24 小时内成功掠夺过你。
2. 对方当前携带高额战利品并处于通缉状态。

复仇的设计目标是：

1. 给玩家情绪出口。
2. 形成个人恩怨和记忆点。
3. 不让实名机制退化成反复抄家。

## 八、阵营系统

首发版本的阵营系统只保留三件事：

1. 阵营身份。
2. 阵营小时分红。
3. 金币上缴换贡献。

### 1. 阵营小时分红

建议每小时结算一次。

设计目的：

1. 提高用户回流频率。
2. 让阵营从背景设定变成实时收益来源。
3. 让不爱打架的玩家也愿意保持活跃。

建议结构：

1. 阵营基础分红。
2. 个人贡献加成分红。

### 2. 贡献系统

首发只保留一种贡献行为：

上缴金币。

贡献的作用：

1. 提高个人小时分红。
2. 提高贡献榜排名。
3. 强化“我在为阵营打资源”的感受。

## 九、三阵营首发特色

首发版本不做复杂阵营技能树，每个阵营只保留一个主优势、一个主弱点和一个明确定位。

### 1. 天堂

1. 主优势：被掠夺损失更低，防守端更稳。
2. 主弱点：主动掠夺收益偏低。
3. 适合玩家：稳收成熟资源、守家、靠分红滚成长。

### 2. 恶魔

1. 主优势：掠夺收益最高，成熟资源争夺更强。
2. 主弱点：战损更高，守家更脆。
3. 适合玩家：主动进攻、快节奏、高风险高回报。

### 3. 人类

1. 主优势：上缴和分红效率更高，经济调度更灵活。
2. 主弱点：纯进攻和纯防守都不极致。
3. 适合玩家：平衡运营、控成熟节奏、靠调度拉开差距。

### 4. 阵营平衡原则

首发只需要守住下面三条：

1. 天堂最稳，但不该抢得最快。
2. 恶魔最会抢，但不该又能抢又能守。
3. 人类最灵活，但不该在正面强度上全面压制另外两方。

首发数值差异优先只动四个字段：

1. 被掠夺损失比例。
2. 掠夺收益加成。
3. 掠夺战损系数。
4. 分红或上缴效率。

## 十、排行榜系统

首发只保留三张榜：

1. 掠夺榜。
2. 发展榜。
3. 贡献榜。

它们分别对应三种核心玩家目标：

1. 爱打架的想冲掠夺榜。
2. 爱经营的想冲发展榜。
3. 爱阵营和社交的想冲贡献榜。

## 十一、新手前 3 天开放表

### 第 1 天

目标：让玩家理解“成熟收取比放着不管更重要”。

开放内容：

1. 阵营选择。
2. 基础金币来源。
3. 金库。
4. 第一个外场培育位。
5. 提前收取与成熟收取演示。
6. 阵营小时分红领取。

### 第 2 天

目标：让玩家理解“高收益外场会带来被抢风险，所以要防守和造兵”。

开放内容：

1. 真实异步目标。
2. 战报。
3. 战备槽。
4. 保护建筑。
5. 城防。
6. 一次免费复仇机会。
7. 掠夺榜和发展榜。

### 第 3 天

目标：让玩家理解“我不是单机经营，我在和阵营一起吃分红”。

开放内容：

1. 金币上缴。
2. 贡献值。
3. 贡献榜。
4. 分红加成说明。
5. 简单阵营任务。

## 十二、首发必须守住的设计底线

1. 玩家不能被一次掠夺打回原点。
2. 玩家不能只靠保底收入吃满主城成长线。
3. 玩家不能因为兵种太多而看不懂战报和战损。
4. 三阵营必须有差异，但差异不能复杂到影响新手理解。
5. 每次上线都要能看到“收成熟、分金币、抢资源、拿分红”其中至少两件事。

## 十三、结论

这份简版设计文档的核心不是做一个复杂世界，而是先做成一个成立的异步成熟争夺循环。

只要下面这条关系成立，首发版本就有验证价值：

1. 金库保底。
2. 外场成熟窗口提供高收益与高风险。
3. 单兵种让战斗和损耗容易理解。
4. 阵营分红和贡献把个人行为接进长期循环。
