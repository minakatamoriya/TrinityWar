# Trinity War 简版游戏设计文档 v0.1

## 一、产品定位

Trinity War 是一款三阵营异步经营掠夺游戏。

玩家的核心体验不是长时间在线操作，而是在短时间上线后做一次明确决策：

1. 先收资源。
2. 再分配资源。
3. 再决定要不要去抢另外两个阵营的玩家。

一句话定位：

自己生产保底，掠夺敌营加速，贡献阵营换分红。

## 二、首发核心目标

首发版本只需要验证四件事：

1. 玩家是否愿意为了掠夺和分红反复上线。
2. 玩家是否理解“生产慢但稳，掠夺快但有风险”。
3. 玩家是否会因为主城后期需要外露资源而主动进入对抗循环。
4. 玩家是否会因为贡献和排行榜形成阵营归属感。

## 三、核心循环

首发版本的最小闭环如下：

1. 农民离线持续产出稳定资源。
2. 玩家上线领取稳定资源和阵营小时分红。
3. 玩家决定把这轮资源投入发育、防守还是造兵。
4. 玩家搜索另外两个阵营的异步目标并发起掠夺。
5. 玩家抢到外露资源后，用于训练兵力、推动主城升级或上缴阵营。
6. 玩家因贡献提高下一轮阵营分红。
7. 玩家下线，等待下一轮生产、分红、战报和复仇机会。

这条循环必须稳定传达一个核心关系：

1. 自己种也能活。
2. 想快就得去抢。
3. 想跨过关键成长门槛，就不能只埋头发展。

## 四、资源系统

首发版本建议只保留两类资源。

### 1. 稳定资源

也可以叫民生资源。

规则：

1. 主要由农民持续产出。
2. 用于资源建筑升级、住房扩建、基础恢复。
3. 原则上不可被直接掠夺，或只损失极低比例。
4. 作用是保证玩家不会被一次掠夺直接打废。

### 2. 外露资源

也可以叫军资。

规则：

1. 主要来自掠夺、阵营小时分红、活动奖励和战报返还。
2. 用于训练掠夺兵、主城中后期升级、上缴阵营和短期强化。
3. 这是玩家之间主要争夺的资源。
4. 被掠夺时，主要损失也应该落在这部分资源上。

### 3. 主城升级规则

主城是首发阶段最重要的反种田装置。

建议分三段：

1. 前期主城升级主要消耗稳定资源，让玩家能顺利入门。
2. 中期主城升级需要稳定资源加外露资源。
3. 后期主城升级必须以外露资源为硬门槛，逼玩家进入掠夺、分红和上缴循环。

核心原则：

稳定资源保证能玩下去，外露资源决定能不能跨阶段成长。

## 五、人口与部队系统

首发版本不做复杂兵种，只保留总人口和一个兵种。

### 1. 人口结构

1. 总人口上限由主城和住房决定。
2. 总人口 = 农民 + 掠夺兵 + 空闲人口。
3. 农民负责稳定资源产出。
4. 掠夺兵负责进攻、运输和基础防守。

### 2. 单兵种规则

1. 训练 1 个掠夺兵，需要 1 点人口和一笔外露资源。
2. 掠夺兵越多，主动进攻越强，但农民越少，稳定资源产出越低。
3. 掠夺兵在战斗中死亡后，玩家要承受人口与军资的双重损失。
4. 人口上限不足时，玩家必须优先扩住房，不能无限爆兵。

### 3. 首发战斗字段

首发只保留三个核心字段：

1. 战力：决定胜负倾向。
2. 运载：决定能抢走多少外露资源。
3. 损耗：决定这次掠夺值不值。

## 六、掠夺与防守系统

### 1. 掠夺目标

1. 只允许掠夺另外两个阵营的玩家。
2. 目标信息只展示资源预估、防守倾向、阵营和推荐风险。
3. 玩家上线后应能快速看到少量可攻击目标。

### 2. 掠夺结算原则

1. 掠夺主要带走对方外露资源。
2. 稳定资源不作为主要掠夺目标。
3. 战损必须真实存在，否则掠夺只剩无脑最优解。
4. 被攻击方必须保留恢复和继续发育的基础能力。

### 3. 防守的意义

防守系统在首发版本只解决一件事：

让玩家愿意花资源减少自己被抢时的损失。

首发只需要三种防守收益：

1. 保护仓提高外露资源保留率。
2. 城防降低被掠夺比例。
3. 驻守兵提高敌方战损。

## 七、阵营系统

首发版本的阵营系统只保留三件事：

1. 阵营身份。
2. 阵营小时分红。
3. 资源上缴换贡献。

### 1. 阵营小时分红

建议每小时结算一次。

设计目的：

1. 提高用户回流频率。
2. 让阵营从背景设定变成实时收益来源。
3. 让不爱打架的玩家也愿意维持活跃。

建议结构：

1. 阵营基础分红。
2. 个人贡献加成分红。

### 2. 贡献系统

首发只保留一种贡献行为：

上缴外露资源。

贡献的作用：

1. 提高个人小时分红。
2. 提高贡献榜排名。
3. 强化“我在为阵营打资源”的感受。

## 八、三阵营首发特色

首发版本不做复杂阵营技能树，每个阵营只保留一个主优势、一个主弱点和一个明确定位。

### 1. 天堂

1. 主优势：被掠夺损失更低，防守端更稳。
2. 主弱点：主动掠夺收益偏低。
3. 适合玩家：稳发育、守家、靠分红滚成长。

### 2. 恶魔

1. 主优势：掠夺收益最高，外露资源来得最快。
2. 主弱点：战损更高，守家更脆。
3. 适合玩家：主动进攻、快节奏、高风险高回报。

### 3. 人类

1. 主优势：上缴和分红效率更高，资源使用更灵活。
2. 主弱点：纯进攻和纯防守都不极致。
3. 适合玩家：平衡运营、靠调度和分红拉开差距。

### 4. 阵营平衡原则

首发只需要守住下面三条：

1. 天堂最稳，但不该抢得最快。
2. 恶魔最会抢，但不该又能抢又能守。
3. 人类最灵活，但不该在正面强度上全面压制另外两方。

首发数值差异优先只动四个字段：

1. 被掠夺损失比例。
2. 掠夺收益加成。
3. 掠夺战损系数。
4. 分红或上缴效率。

## 九、排行榜系统

首发只保留三张榜：

1. 掠夺榜。
2. 发展榜。
3. 贡献榜。

它们分别对应三种核心玩家目标：

1. 爱打架的想冲掠夺榜。
2. 爱经营的想冲发展榜。
3. 爱阵营和社交的想冲贡献榜。

## 十、新手前 3 天开放表

### 第 1 天

目标：让玩家理解“抢比等快”。

开放内容：

1. 阵营选择。
2. 稳定资源产出。
3. 基础建筑升级。
4. 掠夺兵训练。
5. 新手保护下的演示掠夺。
6. 阵营小时分红领取。

### 第 2 天

目标：让玩家理解“抢有风险，所以要防守和分配人口”。

开放内容：

1. 真实异步目标。
2. 战报。
3. 保护仓。
4. 城防。
5. 一次免费复仇机会。
6. 掠夺榜和发展榜。

### 第 3 天

目标：让玩家理解“我不是单机发育，我在和阵营一起吃分红”。

开放内容：

1. 上缴外露资源。
2. 贡献值。
3. 贡献榜。
4. 分红加成说明。
5. 简单阵营任务。

## 十一、首发必须守住的设计底线

1. 玩家不能被一次掠夺打回原点。
2. 玩家不能只靠埋头种田吃满主城成长线。
3. 玩家不能因为兵种太多而看不懂战报和战损。
4. 三阵营必须有差异，但差异不能复杂到影响新手理解。
5. 每次上线都要能看到“收资源、分资源、抢资源、拿分红”其中至少两件事。

## 十二、结论

这份简版设计文档的核心不是做一个复杂世界，而是先做成一个成立的异步掠夺循环。

只要下面这条关系成立，首发版本就有验证价值：

1. 农民保底生产。
2. 主城后期逼玩家争夺外露资源。
3. 单兵种让战斗和损耗容易理解。
4. 阵营分红和贡献把个人行为接进社交循环。
